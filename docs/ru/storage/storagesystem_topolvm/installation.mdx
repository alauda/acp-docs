# Установка

Локальное хранилище — это программно-определенное серверное локальное хранилище, которое предоставляет простую, легкую в обслуживании и высокопроизводительную услугу локального хранилища. На основе решения TopoLVM от сообщества оно достигает управления оркестрацией постоянных томов локального хранилища с помощью подхода LVM системы.

## Предварительные требования

Пакет lvm2 должен быть установлен на каждом узле кластера хранилища. Если он не установлен, пожалуйста, выполните команду `yum install -y lvm2` на узле.

## Шаги

1. Перейдите в **Управление платформой**.

2. В левой навигационной панели нажмите **Управление хранилищем** > **Локальное хранилище**.

3. Нажмите **Настроить сейчас**.

4. На странице мастера **Установить оператор** нажмите **Начать развертывание**.

   - Когда страница автоматически перейдет к следующему шагу, это указывает на успешное развертывание оператора.

   - Если развертывание не удалось, пожалуйста, обратитесь к подсказкам интерфейса для решения. Затем нажмите **Очистить** и заново разверните оператора.

5. На странице мастера **Создать кластер** добавьте устройства.

   | Параметр            | Описание                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
   |----------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
   | **Выбрать узел**     | Узел с как минимум 1 голым диском.                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
   | **Класс устройства** | Каждый класс устройства соответствует набору устройств хранения с одинаковыми характеристиками. Рекомендуется заполнить название в зависимости от природы диска, например *hdd*, *ssd*.                                                                                                                                                                                                                                                                                                             |
   | **Тип устройства**   | Поддерживаются только типы дисков.                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
   | **Хранилище устройства** | Например, */dev/sda*. Если дисков несколько, их можно добавлять по одному.                                                                                                                                                                                                                                                                                                                                                                                                        |
   | **Снимок**          | При включении поддерживает создание снимков PVC и использование снимков для настройки новых PVC для быстрого резервного копирования и восстановления данных бизнеса. <br />Если снимок не был включен при создании хранилища, вы все равно можете включить его по мере необходимости в разделе **Операции** на странице деталей кластера хранилища. <br />**Примечание**: Перед использованием убедитесь, что [Плагин снимков томов](/configure/storage/functions/snapshot_con.mdx) был развернут для текущего кластера. |

   - Нажмите «Далее». Когда страница автоматически перейдет к следующему шагу, это указывает на успешное развертывание кластера.

   - Если создание не удалось, пожалуйста, обратитесь к подсказкам интерфейса и своевременно очистите ресурсы.

6. На странице мастера **Создать класс хранения** настройте соответствующие параметры.

   | Параметр              | Описание                                                                                                                                                                                                                                                                                                                                                                                          |
   |----------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
   | **Имя**               | Имя класса хранения. Оно должно быть уникальным в текущем кластере.                                                                                                                                                                                                                                                                                                                         |
   | **Отображаемое имя**  | Имя, которое помогает вам идентифицировать или фильтровать, например, китайское описание класса хранения.                                                                                                                                                                                                                                                                                                        |
   | **Класс устройства**   | Класс устройства — это способ категоризации устройств хранения в TopoLVM. Каждый класс устройства соответствует набору устройств хранения с одинаковыми характеристиками. Если нет специальных требований, вы можете использовать класс устройства **Авто-выделенный** из кластера.                                                                                                                                 |
   | **Файловая система**   | - **XFS** — это высокопроизводительная журналируемая файловая система, хорошо справляющаяся с параллельными рабочими нагрузками ввода-вывода, поддерживающая обработку больших файлов и обеспечивающая плавный перенос данных. <br />- **EXT4** — это журналируемая файловая система в Linux, предлагающая методы хранения файлов с использованием экстентов и поддерживающая обработку больших файлов. Файловая система может достигать емкости 1 EiB, с максимальным поддерживаемым размером файла 16 TiB. |
   | **Политика утилизации**| Политика утилизации для постоянных томов. <br />- Удалить: Когда требование постоянного тома удаляется, привязанный постоянный том также удаляется. <br />- Сохранить: Даже если требование постоянного тома удалено, привязанный постоянный том все равно будет сохранен.                                                                                                                           |
   | **Режим доступа**      | ReadWriteOnce (RWO): Может быть смонтирован одним узлом в режиме чтения-записи.                                                                                                                                                                                                                                                                                                                             |
   | **Проект выделения**  | Этот тип запроса постоянного тома может быть создан только в определенных проектах. <br />Если проект временно не назначен, проект можно также **Обновить** позже.                                                                                                                                                                                                                               |

7. Нажмите **Далее** и дождитесь завершения создания ресурсов.
