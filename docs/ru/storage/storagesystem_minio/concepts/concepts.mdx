# Основные концепции

- **Кодирование с коррекцией ошибок (EC)**: MinIO использует кодирование с коррекцией ошибок Рида-Соломона для разбивки объектов на данные и четные части, распределяя их по нескольким дискам для обеспечения отказоустойчивости. Например, в конфигурации из 16 дисков данные могут быть разделены на 12 частей данных и 4 четные части, что позволяет системе восстанавливать данные даже в случае отказа до 4 дисков.

- **Пулы серверов и наборы кодирования с коррекцией ошибок**: Пулы серверов MinIO представляют собой логические группы хранилищ, где каждый пул состоит из нескольких узлов, делящих вычислительные и хранилищные ресурсы. Внутри пула диски автоматически организуются в один или несколько **наборов кодирования с коррекцией ошибок**.

  - **Распределение данных**: Когда объект сохраняется, он разбивается на части данных и четные части и распределяется по различным дискам внутри набора кодирования с коррекцией ошибок. 
  - **Модель избыточности**: Наборы кодирования с коррекцией ошибок формируют основную единицу избыточности данных, обеспечивая устойчивость на основе настроенных соотношений частей данных к четным частям.
  - **Масштабируемость**: Один пул хранения MinIO может содержать несколько наборов кодирования с коррекцией ошибок, а новые данные всегда записываются в набор кодирования с коррекцией ошибок с наибольшей доступной емкостью.
