# Основные концепции

- **Кодирование с удалением (EC)**: MinIO использует кодирование с удалением Рида-Соломона для разбиения объектов на данные и паритетные фрагменты, распределяя их по нескольким дискам для обеспечения отказоустойчивости. Например, в конфигурации из 16 дисков данные могут быть разделены на 12 фрагментов данных и 4 паритетных фрагмента, что позволяет системе восстанавливать данные даже в случае выхода из строя до 4 дисков.

- **Пулы серверов и наборы удалений**: Пулы серверов MinIO представляют собой логические группы ресурсами хранения, где каждый пул состоит из нескольких узлов, делящих возможности хранения и обработки. Внутри пула диски автоматически организуются в один или несколько **наборов удалений**.

  - **Распределение данных**: Когда объект сохраняется, он разделяется на фрагменты данных и паритета и распределяется по различным дискам в наборе удалений.
  - **Модель избыточности**: Наборы удалений образуют основную единицу избыточности данных, обеспечивая устойчивость на основе заданных соотношений фрагментов данных к паритетным.
  - **Масштабируемость**: Один пул хранения MinIO может содержать несколько наборов удалений, и новые данные всегда записываются в набор удалений с наибольшей доступной емкостью.
