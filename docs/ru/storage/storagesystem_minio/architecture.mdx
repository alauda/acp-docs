# Архитектура

Платформа облачных контейнеров Alauda (ACP) с объектным хранилищем MinIO представляет собой высокопроизводительную, распределённую систему объектного хранения, разработанную для облачно-ориентированных сред. Она использует кодирование с исправлением ошибок, распределённые хранилищные пулы и механизмы высокой доступности для обеспечения долговечности данных и масштабируемости в Kubernetes.

### Основные компоненты:

- **MinIO Оператор**: Управляет развертыванием и обновлением кластеров MinIO.
- **MinIO Партнёр**: Конфигурирует и управляет функциональностью репликации сайта MinIO.
- **MinIO Пул**: Основной компонент MinIO, отвечающий за обработку запросов к объектному хранилищу. Каждый пул соответствует StatefulSet и предоставляет ресурсы хранения.

### Архитектура развертывания:

Развертывание MinIO в Kubernetes требует определения арендатора MinIO, указания количества серверных экземпляров (подов) и количества томов (дисков) на экземпляр. Каждый сервер MinIO управляется через StatefulSet, что обеспечивает стабильные идентификаторы и постоянное хранилище. MinIO агрегирует все диски в один или несколько наборов с исправлением ошибок и применяет кодирование с исправлением ошибок для обеспечения отказоустойчивости.

### Множественное расширение пулов:

Кластеры MinIO могут масштабироваться путём добавления дополнительных серверных пулов, каждый из которых имеет свой набор с исправлением ошибок. Хотя это обеспечивает большую ёмкость хранения, это также приводит к усложнению обслуживания кластера и снижению общей надёжности кластера. Отказ в любом серверном пуле может сделать недоступным весь кластер MinIO, даже если другие пулы продолжают работать.

### Заключение:

MinIO является высокомасштабируемым решением объектного хранения, ориентированным на облачные технологии, которое балансирует между производительностью и надёжностью. При проектировании кластера MinIO важно тщательно проектировать пул хранения, настраивать параметры кодирования с исправлением ошибок и реализовывать стратегии высокой доступности для обеспечения целостности данных и стабильности работы в средах Kubernetes.
