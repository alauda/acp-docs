# Добавление пула хранения

Пул хранения — это логическая партиция, используемая для хранения данных. В одном и том же кластер хранения можно одновременно использовать различные типы базового хранилища для удовлетворения различных бизнес-требований.

Помимо пулов хранения, созданных в процессе конфигурации объектного хранилища, вы также можете добавлять дополнительные пулы хранения.

## Примечания

Добавление пула хранения приведет к кратковременному прерыванию работы службы MinIO, но затем она автоматически восстановится до нормального состояния.

## Процедура

1. Перейдите в **Управление платформой**.

2. В левой панели навигации нажмите **Управление хранилищем** > **Объектное хранилище**.

3. На вкладке **Информация о кластере** прокрутите вниз до секции **Пул хранения** и нажмите **Добавить пул хранения**.

4. Настройте соответствующие параметры в соответствии с инструкциями ниже.

   | Параметр                  | Описание                                                                                                                                                                                                                                                                                                |
   | ------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | **Базовое хранилище**     | Базовое хранилище, используемое кластером MinIO. Пожалуйста, выберите существующий класс хранилища, созданный в текущем кластере, рекомендуется использовать TopoLVM.                                                                                                                                     |
   | **Узлы хранения**         | Выберите узлы хранения, необходимые для кластера MinIO. Рекомендуется использовать 4-16 узлов хранения; платформа развернет 1 сервис хранения для каждого выбранного узла хранения.<br />**Примечание:** При использовании 3 узлов хранения для обеспечения надежности для каждого узла будет развернуто 2 сервиса хранения.      |
   | **Единственный объем хранения** | Емкость одного PVC объема хранения. Каждый сервис хранения управляет 1 объемом хранения, и как только емкость единственного объема хранения будет введена, платформа автоматически рассчитает емкость пула хранения и другую информацию, которую можно просмотреть в **Обзор пула хранения**. |

5. Нажмите **Подтвердить**.
