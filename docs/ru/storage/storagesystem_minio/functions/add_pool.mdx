# Добавление пула хранения

Пул хранения относится к логическому разделению, используемому для хранения данных. В рамках одного хранилища могут одновременно использоваться различные типы базового хранилища для удовлетворения различных бизнес-потребностей.

В дополнение к пулам хранения, созданным во время конфигурации объектного хранилища, вы также можете добавить дополнительные пулы хранения.

## Примечания

Добавление пула хранения вызовет кратковременное прерывание в работе службы MinIO, но затем она автоматически восстановится до нормального состояния.

## Процедура

1. Перейдите к **Управление платформой**.

2. Нажмите на **Управление хранилищем** > **Объектное хранилище** в левой боковой панели.

3. Вкладке **Информация о кластере** прокрутите вниз до раздела **Пул хранения** и нажмите **Добавить пул хранения**.

4. Настройте соответствующие параметры в соответствии с инструкциями ниже.

   | Параметр                 | Описание                                                                                                                                                                                                                                                                                               |
   | ------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | **Базовое хранилище**    | Базовое хранилище, используемое кластером MinIO. Пожалуйста, выберите существующий класс хранения, созданный в текущем кластере, с рекомендуется использовать TopoLVM.                                                                                                                                                       |
   | **Узел хранения**        | Выберите узлы хранения, необходимые для кластера MinIO. Рекомендуется использовать от 4 до 16 узлов хранения; платформа развернет 1 сервис хранения для каждого выбранного узла хранения.<br />**Примечание**: При использовании 3 узлов хранения для обеспечения надежности, 2 сервиса хранения будут развернуты для каждого узла.     |
   | **Единый объем хранения** | Ёмкость одного PVC для объема хранения. Каждый сервис хранения управляет 1 объемом хранения, и после ввода ёмкости одного объема хранения платформа автоматически вычислит ёмкость пула хранения и другую информацию, которую можно просмотреть в разделе **Обзор пула хранения**. |

5. Нажмите **Подтвердить**.
