# Введение

**Платформа хранения Alauda Container Platform (ACP) с Ceph** является гиперконвергентным решением для хранения, предоставляемым платформой в пределах кластера. Основанное на решении с открытым исходным кодом Rook + Ceph, распределенное хранилище обеспечивает автоматическое управление, автоматическое масштабирование и автоматическое восстановление, удовлетворяя потребности в блочном, файловом и объектном хранилище для малых и средних приложений.

:::note
В этом документе **распределенное хранилище** относится к хранилищу Ceph внутри этого кластера, в то время как **внешнее хранилище** относится к хранилищу Ceph вне этого кластера.
:::

## Обзор функций

- **Легкое развертывание**: Предоставляет графические автоматизированные услуги развертывания и управления для кластеров хранения; поддерживает как интегрированные, так и раздельные режимы развертывания для вычислений и хранения.

- **Профессиональные операции**: Предлагает функциональность резервного копирования снимков постоянных томов и клонирования новых томов; визуальное мониторинг емкости, производительности и уровня компонентов; оборудовано встроенными политиками оповещения для удовлетворения потребностей большинства сценариев операций по хранению.

- **Безопасность и надежность**: Распределенные и многореплицированные механизмы обеспечивают безопасность и надежность данных; простое и надежное автоматизированное управление поддерживает онлайн-расширение ресурсов хранения.

- **Отличная производительность**: Предоставляет эластичные и высокопроизводительные услуги хранения; поддерживает развертывание гибридных дисковых устройств для повышения производительности и эффективности системы хранения.

## Сравнение решений для хранения

Платформа поддерживает следующие два типа решений для хранения; вы можете выбрать один из них.

### Создание кластера хранения

| Требование                                                                                                                                                                   | Преимущества                                                                                                                  |
| ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------- |
| **Вы можете выбрать создание либо [кластера стандартного типа](./installation/create_service_stand.mdx), либо [кластера расширенного типа](./installation/create_service_extend.mdx)** | Не требуется дополнительная подготовка решения для хранения; конфигурация может быть завершена в бизнес-кластере, что сокращает затраты. |

### [Доступ к внешнему хранилищу](./functions/access_storage_service.mdx)

Вариант 1: Доступ к распределенным ресурсам хранения других бизнес-кластеров в рамках платформы для обеспечения изоляции хранения и бизнеса для более легкого управления и обслуживания.

Вариант 2: Интеграция внешних ресурсов хранения Ceph в качестве распределенного хранилища.

| Требование (выберите один)                                                       | Преимущества                                                                                                                                                                                                                                                                                                                                                                                                   |
| ------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Вариант 1: Распределенное хранилище уже развернуто в других бизнес-кластерах.** | Можно полностью использовать ресурсы хранения между кластерами и избежать вмешательства со стороны изменений в бизнесе. Обеспечивает безопасность и стабильность данных при снижении операционной сложности.<br /><br /> **Примечание**: Если хранилище, к которому нужно получить доступ, является распределенным хранилищем с других платформ, таких как основная/резервная платформа в среде аварийного восстановления, используйте метод интеграции внешнего Ceph. |
| **Вариант 2: Внешнее хранилище Ceph вне платформы, версия ≥ 14.2.3.**             | По сравнению с прямым созданием класса хранения, этот метод более удобен для использования интерфейса платформы для снимков томов, масштабирования и других функций.                                                                                                                                                                                                                                         |

**Примечание**: Если вам необходимо поддерживать пул хранения, устройства хранения и другие конфигурации внешнего хранилища, операции должны выполняться в интерфейсе управления кластером хранения.
