# Введение

**Платформа контейнеров Alauda (ACP) с хранилищем Ceph** является гиперконвергентным решением для хранения, предоставляемым платформой внутри кластера. Основанное на открытом решении для хранения Rook + Ceph, распределенное хранилище обеспечивает автоматическое управление, автоматическое масштабирование и возможности автоматического восстановления, удовлетворяя потребности в блочном хранилище, файловом хранилище и объектном хранилище для малых и средних приложений.

:::note
В данном документе **распределенное хранилище** относится к хранилищу Ceph внутри этого кластера, в то время как **внешнее хранилище** относится к хранилищу Ceph за пределами этого кластера.
:::

## Обзор функций

- **Простота развертывания**: Обеспечивает графическое автоматическое развертывание и управление для кластеров хранения; поддерживает как интегрированные, так и декомпилированные режимы развертывания для вычислений и хранения.

- **Профессиональные операции**: Предлагает функции резервного копирования снимков постоянного тома и клонирования новых томов; визуальный мониторинг емкости, производительности и уровней компонентов; снабжен встроенными политиками оповещения для удовлетворения потребностей большинства сценариев операций по хранению.

- **Безопасность и надежность**: Распределенные и многократные механизмы репликации обеспечивают безопасность и надежность данных; простое и надежное автоматическое управление поддерживает онлайн-расширение ресурсов хранения.

- **Отличная производительность**: Обеспечивает эластичные и высокопроизводительные услуги хранения; поддерживает развертывание гибридных дисковых устройств для повышения производительности и эффективности системы хранения.

## Сравнение решений по хранению

Платформа поддерживает следующие два типа решений по хранению; вы можете выбрать одно из них.

### Создание кластера хранения

| Требование                                                                                                                                                                        | Преимущества                                                                                                               |
| ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------- |
| **Вы можете выбрать создание либо [кластера стандартного типа](./installation/create_service_stand.mdx), либо [кластера расширенного типа](./installation/create_service_extend.mdx)** | Не требуется дополнительная подготовка к решению для хранения; конфигурация может быть завершена в бизнес-кластере, что позволяет сэкономить деньги. |

### [Доступ к внешнему хранилищу](./functions/access_storage_service.mdx)

Вариант 1: Доступ к распределенным ресурсам хранения других бизнес-кластеров на платформе, чтобы обеспечить изоляцию хранения и бизнеса для более легкого управления и обслуживания.

Вариант 2: Интеграция внешних ресурсов хранения Ceph в качестве распределенного хранилища.

| Требование (выберите один)                                             | Преимущества                                                                                                                                                                                                                                                                                                                                                                                                  |
| ---------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Вариант 1: Распределенное хранилище уже развернуто в других бизнес-кластерах.** | Можно в полной мере использовать ресурсы хранения между кластерами и избегать вмешательства со стороны изменений в бизнесе. Обеспечивает безопасность и стабильность данных, снижая операционную сложность.<br /><br /> **Примечание**: Если хранилище, к которому требуется доступ, является распределенным хранилищем с других платформ, например, основная/резервная платформа в среде восстановления после бедствий, пожалуйста, используйте метод интеграции внешнего Ceph. |
| **Вариант 2: Внешнее хранилище Ceph за пределами платформы, версия ≥ 14.2.3.**    | По сравнению с непосредственным созданием класса хранения, этот метод более удобен для использования интерфейса платформы для снимков томов, масштабирования и других функций.                                                                                                                                                                                                                                        |

**Примечание**: Если требуется поддерживать пул хранения, устройство хранения и другие конфигурации внешнего хранилища, операции должны выполняться в интерфейсе управления кластером хранения.
