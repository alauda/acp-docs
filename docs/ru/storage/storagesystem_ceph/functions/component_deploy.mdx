# Развертывание компонентов, специфичных для узлов

После создания распределенного хранилища вы можете просматривать и изменять местоположение развертывания компонентов, что упрощает расширение и обслуживание хранилища.

## Обновление конфигурации развертывания компонентов

### Предостережения

- Обновление конфигурации вызовет автоматическое воссоздание экземпляров компонентов, что может повлиять на доступ к сервису хранилища. Рекомендуется выполнять обновление в нерабочие часы.

- Когда тип кластера **Расширяемый**, функция фиксированного развертывания компонентов не поддерживается.

### Процедура

<Steps>
  1. Перейдите в **Управление платформой**.

  2. В левой панели навигации нажмите на **Управление хранилищем** > **Распространенное хранилище**.

  3. На вкладке **Компоненты хранилища** нажмите на **Конфигурация развертывания компонентов**.

  4. Включите/выключите переключатель **Фиксированное развертывание** в соответствии с бизнес-требованиями и разверните компоненты на указанных узлах. Количество узлов должно быть не менее трех для обеспечения минимальной доступности. Компоненты, применимые для конфигурации фиксированного развертывания, включают MON, MGR, MDS, RGW.

  5. Нажмите **Обновить**, и компоненты начнут планироваться на назначенные узлы.
</Steps>

## Перезапуск компонентов хранилища

Когда вы удаляете развернутые компоненты хранилища, система автоматически переназначает и заново разворачивает компоненты на узлах в соответствии с текущей стратегией развертывания компонентов.

### Процедура

<Steps>
  1. Перейдите в **Управление платформой**.

  2. В левой панели навигации нажмите на **Управление хранилищем** > **Распространенное хранилище**.

  3. На вкладке **Компоненты хранилища** нажмите ⋮ рядом с названием компонента > **Удалить**.
</Steps>
