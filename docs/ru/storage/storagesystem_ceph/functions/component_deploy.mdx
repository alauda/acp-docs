# Внедрение компонентов, специфичных для узлов

После создания распределенного хранилища вы по-прежнему можете просматривать и изменять расположение развертывания компонентов, что облегчает расширение и обслуживание хранилища.

## Обновление конфигурации развертывания компонентов

### Предостережения

- Обновление конфигурации вызовет автоматическую перезапись экземпляров компонентов, что может повлиять на доступ к сервису хранилища. Рекомендуется выполнять обновление в часы низкой нагрузки.

- Когда тип кластера **Расширить**, функция фиксированного развертывания для компонентов не поддерживается.

### Процедура

<Steps>
  1. Перейдите в раздел **Управление платформой**.

  2. В левой навигационной панели нажмите на **Управление хранилищем** > **Распределенное хранилище**.

  3. На вкладке **Компоненты хранилища** нажмите на **Конфигурация развертывания компонентов**.

  4. Включите/выключите переключатель **Фиксированное развертывание** в зависимости от бизнес-требований и разверните компоненты на указанные узлы. Количество узлов должно быть не менее трех для обеспечения минимальной доступности. Компоненты, применимые для конфигурации фиксированного развертывания, включают MON, MGR, MDS, RGW.

  5. Нажмите **Обновить**, и компоненты начнут распределяться на назначенные узлы.
</Steps>

## Перезапуск компонентов хранилища

Когда вы удаляете развернутые компоненты хранилища, система автоматически перераспределит и заново развернет компоненты на узлах в соответствии с текущей стратегией развертывания компонентов.

### Процедура

<Steps>
  1. Перейдите в раздел **Управление платформой**.

  2. В левой навигационной панели нажмите на **Управление хранилищем** > **Распределенное хранилище**.

  3. На вкладке **Компоненты хранилища** нажмите ⋮ рядом с именем компонента > **Удалить**.
</Steps>
