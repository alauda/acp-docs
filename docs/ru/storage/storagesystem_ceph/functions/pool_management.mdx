# Управление пулами хранилищ

Пул хранилищ представляет собой логическую партию, используемую для хранения данных. Один кластер хранения поддерживает одновременное использование различных типов пулов хранилищ, таких как файловое хранилище и блочное хранилище, чтобы соответствовать различным бизнес-требованиям.

## Создание пула хранилищ

В дополнение к пулам хранилищ, созданным во время настройки распределенного хранения, вы можете также создать дополнительные типы пулов хранилищ.

**Совет**: В рамках одного кластера хранилищ допустимы только один файловый и один объектный пул хранилищ, в то время как можно создать до восьми блочных пулов хранилищ.

### Процедура

<Steps>
  1. Перейдите в **Управление платформой**.

  2. В левой навигационной панели нажмите **Управление хранилищем** > **Распределенное хранилище**.

  3. В вкладке **Информация о кластере** прокрутите вниз до области **Пул хранилищ** и нажмите **Создать пул хранилищ**.

  4. Настройте соответствующие параметры в соответствии со следующими инструкциями.

     | Параметр          | Описание                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
     | ----------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
     | **Тип хранилища** | Выберите в данный момент неразвернутый тип хранилища.<br />- Файловое хранилище: обеспечивает безопасные, надежные и масштабируемые услуги совместного доступа к файлам. Подходит для обмена файлами, резервного копирования данных и т. д.<br />- Блочное хранилище: предоставляет высокие IOPS и услуги хранения с низкой задержкой. Подходит для баз данных, виртуализации и т. д.<br />- Объектное хранилище: предлагает услуги хранения с стандартным интерфейсом S3, подходит для больших данных, резервного архивирования, облачных услуг хранения и т. д. |
     | **Количество реплик** | <ul><li>**Когда тип кластера - Стандарт**: более высокое количество реплик увеличивает избыточность и безопасность данных, но также снижает использование хранилища. Обычно настройки 3 достаточно для большинства потребностей.</li><li>**Когда тип кластера - Расширенный**: количество реплик по умолчанию равно 4 и не может быть изменено.</li></ul>                                                                                                                                     |
     | **Класс устройства**  | <ul><li>**Когда тип кластера - Стандарт**: выберите уже добавленный класс устройства в созданном пуле хранилищ.<ul><li>При выборе класса устройства данные будут храниться в выбранном классе устройства.</li><li>Если класс устройства не выбран, данные будут случайно храниться на всех устройствах в пуле хранилищ.</li></ul></li><li>**Когда тип кластера - Стандарт**: добавление класса устройства не поддерживается.</li></ul>                          |

     Если это тип объектного хранения, вы также можете настроить следующие параметры:

     | Параметр            | Описание                                                                     |
     | ------------------- | ------------------------------------------------------------------------------- |
     | **Регион**          | Укажите регион, в котором расположен пул хранилищ.                           |
     | **Тип шлюза**       | По умолчанию S3, изменению не подлежит.                                      |
     | **Внутренний порт** | Укажите порт для внутреннего доступа к кластеру.                            |
     | **Внешний доступ**  | Включение/выключение внешнего доступа создаст/уничтожит сервис типа NodePort. |
     | **Количество экземпляров** | Количество экземляров ресурсов для объектного хранения.                                |

  5. Нажмите **Создать**.
</Steps>

## Удаление пула хранилищ

Если какой-либо тип хранения больше не требуется, пул хранилищ можно удалить после его отвязки от класса хранения.

### Процедура

<Steps>
  1. Перейдите в **Управление платформой**.

  2. В левой навигационной панели нажмите **Управление хранилищем** > **Распределенное хранилище**.

  3. В вкладке **Информация о кластере** прокрутите вниз до области **Пул хранилищ**, нажмите на ⋮ рядом с пулом хранилищ, который вы хотите удалить > **Удалить**.

  4. Прочитайте информацию на экране и введите имя пула хранилищ.

  5. Нажмите **Удалить**.
</Steps>

## Просмотр адресов объектного пула хранения \{#storepool}

После создания объектного пула хранилищ вы можете просмотреть внутренние и внешние адреса доступа к пулу хранилищ.

### Процедура

<Steps>
  1. Перейдите в **Управление платформой**.

  2. В левой навигационной панели нажмите **Управление хранилищем** > **Распределенное хранилище**.

  3. В вкладке **Информация о кластере** прокрутите вниз до области **Пул хранилищ**, нажмите на ⋮ рядом с объектным пулом хранилищ и выберите **Просмотреть адрес**.
</Steps>
