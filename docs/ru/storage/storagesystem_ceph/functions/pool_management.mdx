# Управление пулами хранения

Пул хранения относится к логическому разделу, используемому для хранения данных. Один кластер хранения поддерживает одновременное использование различных типов пулов хранения, таких как файловое и блочное хранилище, чтобы соответствовать различным бизнес-требованиям.

## Создание пула хранения

В дополнение к пулам хранения, созданным во время настройки распределенного хранения, вы также можете создать дополнительные типы пулов хранения.

**Совет**: Внутри одного кластера хранения допускается только один файловый и один объектный пул хранения, в то время как можно создать до восьми блочных пулов хранения.

### Процедура

<Steps>
  1. Перейдите в **Управление платформой**.

  2. В левой навигационной панели щелкните **Управление хранением** > **Распределенное хранилище**.

  3. На вкладке **Информация о кластере** прокрутите вниз до области **Пул хранения** и щелкните **Создать пул хранения**.

  4. Настройте соответствующие параметры в соответствии с приведенными ниже инструкциями.

     | Параметр         | Описание                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
     | ----------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
     | **Тип хранения**  | Выберите в настоящее время неразвернутый тип хранения.<br />- Файловое хранилище: Обеспечивает безопасные, надежные и масштабируемые услуги совместного использования файлов. Подходит для совместного использования файлов, резервного копирования данных и т. д.<br />- Блочное хранилище: Обеспечивает высокую производительность ввода-вывода и низкую задержку. Подходит для баз данных, виртуализации и т. д.<br />- Объектное хранилище: Обеспечивает услуги хранения с интерфейсом S3, подходит для больших данных, резервного архивания, облачных хранилищ и т. д. |
     | **Количество реплик** | <ul><li>**Когда тип кластера стандартный**: Более высокое количество реплик увеличивает избыточность и безопасность данных, но также снижает коэффициент использования хранилища. Обычно достаточно настройки 3 для большинства нужд.</li><li>**Когда тип кластера расширенный**: Значение по умолчанию для количества реплик равно 4 и не может быть изменено.</li></ul>                                                                                                                                                                   |
     | **Класс устройства**  | <ul><li>**Когда тип кластера стандартный**: Выберите уже добавленный класс устройства в созданном пуле хранения.<ul><li>При выборе класса устройства данные будут храниться в выбранном классе устройства.</li><li>Если класс устройства не выбран, данные будут случайным образом распределены по всем устройствам в пуле хранения.</li></ul></li><li>**Когда тип кластера стандартный**: Добавление класса устройства не поддерживается.</li></ul>                          |

     Если это объектный тип хранения, вы также можете настроить следующие параметры:

     | Параметр           | Описание                                                                     |
     | ------------------- | ------------------------------------------------------------------------------- |
     | **Регион**          | Укажите регион, в котором расположен пул хранения.                           |
     | **Тип шлюза**       | По умолчанию устанавливается значение S3 и не может быть изменено.          |
     | **Внутренний порт** | Укажите порт для внутреннего доступа к кластеру.                            |
     | **Внешний доступ**  | Включение/выключение внешнего доступа создаст/уничтожит службу типа NodePort. |
     | **Количество экземпляров** | Количество экземпляров ресурсов для объектного хранилища.                     |

  5. Щелкните **Создать**.
</Steps>

## Удаление пула хранения

Если определенный тип хранения больше не требуется, пул хранения можно удалить после его отключения от класса хранения.

### Процедура

<Steps>
  1. Перейдите в **Управление платформой**.

  2. В левой навигационной панели щелкните **Управление хранением** > **Распределенное хранилище**.

  3. На вкладке **Информация о кластере** прокрутите вниз до области **Пул хранения**, щелкните ⋮ рядом с пулом хранения, который вы хотите удалить > **Удалить**.

  4. Прочтите информацию подсказки и введите имя пула хранения.

  5. Щелкните **Удалить**.
</Steps>

## Просмотр адресов пула объектного хранения \{#storepool}

После создания пула объектного хранения вы можете просмотреть внутренние и внешние адреса доступа к пулу хранения.

### Процедура

<Steps>
  1. Перейдите в **Управление платформой**.

  2. В левой навигационной панели щелкните **Управление хранением** > **Распределенное хранилище**.

  3. На вкладке **Информация о кластере** прокрутите вниз до области **Пул хранения**, щелкните ⋮ рядом с пулом объектного хранения и выберите **Просмотреть адрес**.
</Steps>
