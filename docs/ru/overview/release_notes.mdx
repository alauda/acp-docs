## 4.0.0

### Новые функции и улучшения

#### Установка и обновление: Модульная архитектура

Мы полностью переработали архитектуру нашей платформы, чтобы обеспечить беспрецедентную гибкость, более быстрые обновления и снижение операционных затрат.

**Упрощенная установка**
Наша платформа теперь развертывается через компактный основной пакет, содержащий только основные компоненты. После установки основы клиенты могут выбирать, какие Операторы или плагины кластера им нужны — будь то DevOps, Service Mesh или другие специализированные функции — и загружать, устанавливать их по отдельности.

**Целевые патчи**

- Выпуски патчей включают только те компоненты, которые действительно требуют исправлений.
- Компоненты без исправлений остаются без изменений, что гарантирует, что остальная часть платформы остается нетронутой.
- Клиенты применяют патчи через встроенный, стандартизированный механизм обновления платформы, а не вручную обновляя отдельные компоненты, что упрощает обслуживание и отслеживание.

**Интеллектуальные обновления**

- Во время обновления заменяются и перезапускаются только компоненты с новым кодом.
- Неизмененные компоненты сохраняют свои существующие версии и время работы.
- Это минимизирует время простоя и сокращает окно обслуживания для более плавного процесса обновления.

**Независимое версионирование компонентов**

- Большинство Операторов следуют своим собственным графикам выпуска, отдельно от основной платформы.
- Новые функции и исправления становятся доступными сразу после их готовности — нет необходимости ждать полного обновления платформы.
- Этот подход ускоряет доставку и позволяет клиентам быстрее получать преимущества от улучшений.

#### Кластеры: Декларативное управление жизненным циклом кластера с помощью Cluster API

Локальные кластеры теперь используют Kubernetes Cluster API для полностью декларативных операций, включая:

- Создание кластера
- Масштабирование узлов и присоединение

Эта бесшовная интеграция Cluster API напрямую вписывается в ваши IaC-пайплайны, обеспечивая программный контроль над жизненным циклом вашего кластера.

#### Оператор и расширение: Полная видимость возможностей

**Полный каталог Операторов**

OperatorHub теперь отображает всех поддерживаемых Операторов, независимо от того, были ли их пакеты загружены на платформу. Это улучшение:

- Обеспечивает полную видимость возможностей платформы даже в изолированных средах
- Устраняет информационные пробелы между тем, что доступно, и тем, что известно пользователям
- Уменьшает трение при исследовании возможностей платформы

**Гибкость версий**

Пользователи теперь могут выбирать конкретные версии Операторов во время установки, а не ограничиваться только последней версией, что обеспечивает больший контроль над совместимостью компонентов и путями обновления.

**Расширения веб-консоли**

Операторы теперь поддерживают расширения веб-консоли на основе якорей, позволяя включать функционально-специфичные фронтенд-изображения в Операторы и бесшовно интегрировать их в веб-консоль платформы.

**Улучшения плагинов кластера**

Все улучшения видимости Операторов, выбора версий и возможностей расширения веб-консоли также применимы к плагинам кластера, обеспечивая единообразный пользовательский опыт во всех расширениях платформы.

#### Оптимизация логики запросов журналов

Страница запросов журналов была оптимизирована для решения проблем с опытом и производительностью, с которыми сталкиваются пользователи при использовании функции запроса журналов:

- Оригинальный радиобокс был заменен на компонент расширенного поиска. Теперь вы можете использовать поиск журналов так же, как используете поиск GIT.
- Независимые условия запроса для содержимого журналов
- Местоположение критериев временного запроса было изменено. Теперь вы не сбросите свои критерии фильтрации журналов, когда измените диапазон времени.
- Оптимизирован API запроса журналов для улучшения общей производительности запросов

#### Обновление ElasticSearch до 8.17

Мы обновили версию ElasticSearch до 8.17, чтобы следовать функциям и улучшениям сообщества.

#### Аутентификация ALB

ALB теперь поддерживает различные механизмы аутентификации, что позволяет пользователю обрабатывать аутентификацию на уровне Ingress, а не реализовывать ее в каждом бэкенд-приложении.

#### ALB поддерживает аннотации ingress-nginx

Этот выпуск добавляет поддержку общих аннотаций ingress-nginx в ALB, включая настройки keepalive, конфигурации таймаутов и HTTP перенаправления, улучшая совместимость с сообществом ingress-nginx.

#### Оптимизация живой миграции Kubevirt

Во время процесса живой миграции время сетевого прерывания было сокращено до менее чем 0.5 секунд, и существующие TCP-соединения не будут разрываться. Эта оптимизация значительно улучшает стабильность и надежность миграций виртуальных машин в производственных средах.

#### Оптимизация интеграции LDAP/OIDC

Поля формы интеграции LDAP/OIDC были скорректированы, в основном включают удаление ненужных/дублирующих полей и оптимизацию описаний полей. Интеграция LDAP/OIDC теперь поддерживает конфигурацию через YAML, позволяя сопоставление атрибутов пользователя в файле YAML.

#### Поддержка Source to Image (S2I)

- Добавлен оператор **Alauda Container Platform Builds** для автоматизированного создания образов из исходного кода
- Поддерживает языковые стеки Java/Go/Node.js/Python
- Упрощает развертывание приложений через репозитории исходного кода

#### Решение для локального реестра

- **ACP Registry** предоставляет легковесный Docker Registry с функциями, готовыми к предприятию
- Обеспечивает готовые возможности управления образами
- Упрощает доставку приложений

#### Рефакторинг модуля GitOps

- Разделен **ACP GitOps** на архитектуру отдельного плагина кластера
- Обновлен Argo CD до версии v2.14.x
- Улучшено управление жизненным циклом приложений на основе GitOps

#### Мониторинг на уровне пространства имен

- Введены динамические панели мониторинга на уровне пространства имен
- Обеспечивает визуализацию метрик Приложений/Нагруз/Подов

#### Интеграция Crossplane

- Выпущена дистрибуция **Alauda Build of Crossplane**
- Реализует централизованное предоставление приложений через составления XRD

#### Обновления виртуализации

- Обновлено до KubeVirt 1.4 для улучшенных возможностей виртуализации
- Оптимизировано управление образами для более быстрого развертывания ВМ
- Оптимизирована живая миграция ВМ, теперь инициируемая непосредственно из интерфейса с видимым статусом миграции
- Улучшено связывание сетей с поддержкой двойного стека (IPv4/IPv6)
- Добавлена поддержка vTPM для повышения безопасности ВМ

#### Обновления Ceph Storage

- Metro-DR с растянутым кластером обеспечивает синхронизацию данных в реальном времени между зонами доступности
- Региональный-DR с зеркалированием на основе пулов улучшает защиту данных

#### Обновления TopoLVM

- Добавлена поддержка развертывания устройств с несколькими путями, улучшая гибкость и стабильность
