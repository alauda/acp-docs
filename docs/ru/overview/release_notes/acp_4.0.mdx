# ACP 4.0

## Небольшая версия: 4.0.0

### Возможности и улучшения

#### Установка и обновление: Модульная архитектура

Мы полностью переработали архитектуру нашей платформы, чтобы обеспечить беспрецедентную гибкость, более быстрые обновления и снижение операционных затрат.

**Установка основного пакета**

Наша платформа теперь поддерживает установку через оптимизированный основной пакет, который содержит только основные компоненты. После установки этой основы пользователи могут выборочно загружать, выгружать и устанавливать только операторы или плагины кластера, которые им необходимы - будь то DevOps, Service Mesh или другие специализированные функции.

**Целевые исправления**

Исправления платформы теперь доставляются с значительно повышенной эффективностью:

- Сокращено время доставки критических исправлений с одного месяца до двух недель
- Обновления платформы теперь включают только компоненты, требующие исправлений
- Стабильные компоненты остаются нетронутыми во время обновлений, что минимизирует воздействие на систему

**Интеллектуальные обновления**

Обновления платформы были революционизированы для минимизации операционного воздействия:

- Компоненты без изменений в коде сохраняют стабильность версии во время обновлений
- Только изменённые компоненты обновляются во время миграций версий
- Перезапуски системы ограничены только теми компонентами, которые требуют изменений

**Независимое версионирование компонентов**

Многие операторы платформы теперь следуют независимым циклам выпуска, что позволяет нам быстрее предоставлять новые возможности, не дожидаясь завершения полных циклов выпуска платформы.

#### Кластеры: Улучшенная инфраструктура как код

**Управление кластерами на основе YAML**

На локальных кластерах теперь полностью поддерживаются операции на основе YAML, включая:

- Создание кластера
- Добавление узлов
- Управление конфигурацией

Это улучшение бесшовно интегрируется в рабочие процессы инфраструктуры как кода (IaC), позволяя программное управление жизненным циклом кластеров.

#### Расширение: Всесторонняя видимость возможностей

**Полный каталог операторов**

OperatorHub теперь отображает всех поддерживаемых операторов, независимо от того, загружены ли их пакеты на платформу. Это улучшение:

- Обеспечивает полную видимость возможностей платформы даже в изолированных средах
- Устраняет пробелы в информации между тем, что доступно, и тем, что известно пользователям
- Упрощает поиск возможностей платформы

**Гибкость версий**

Теперь пользователи могут выбирать конкретные версии операторов во время установки, а не ограничиваться только последней версией, что обеспечивает большую контроль над совместимостью компонентов и путями обновления.

**Расширения веб-консоли**

Операторы теперь поддерживают веб-консоль с расширениями на основе якорей, позволяя включать функционально-специфические изображения интерфейса в операторов и бесшовно интегрироваться в веб-консоль платформы.

**Улучшения плагинов кластера**

Все улучшения видимости операторов, выбора версий и возможностей расширений веб-консоли также применяются к плагинам кластера, обеспечивая последовательный пользовательский опыт во всех расширениях платформы.

#### Оптимизация логического запроса логов

Страница запроса логов была оптимизирована для решения проблем с опытом работы и производительностью, с которыми пользователи сталкиваются при использовании функции запроса логов:

- Оригинальная радиокнопка была заменена на компонент расширенного поиска. Теперь вы можете использовать поиск по логам так же, как и поиск в GIT.
- Независимые условия запроса для содержания логов
- Местоположение критериев временного запроса было скорректировано. Теперь вы не сбросите критерии фильтрации логов, когда измените диапазон времени.
- Оптимизирован API запроса логов для улучшения общей производительности запросов.

#### Обновление ElasticSearch до 8.17

Мы обновили версию ElasticSearch до 8.17, чтобы продолжать следить за функциями и улучшениями сообщества.

#### Аутентификация ALB

ALB теперь поддерживает различные механизмы аутентификации, что позволяет пользователю обрабатывать аутентификацию на уровне Ingress вместо реализации её в каждом бэкенд-приложении.

#### ALB поддерживает аннотации ingress-nginx

Этот релиз добавляет поддержку общих аннотаций ingress-nginx в ALB, включая настройки keepalive, конфигурации таймаутов и HTTP-перенаправления, улучшая совместимость с ingress-nginx сообщества.

#### Оптимизация живой миграции Kubevirt

Во время процесса живой миграции время прерывания сети было снижено до менее чем 0.5 секунд, а существующие TCP соединения не будут разорваны. Эта оптимизация значительно улучшает стабильность и надежность миграции виртуальных машин в производственных средах.

#### Оптимизация интеграции LDAP/OIDC

Поля формы интеграции LDAP/OIDC были скорректированы, в основном включив удаление ненужных/дублирующихся полей и оптимизацию описаний полей. Интеграция LDAP/OIDC теперь поддерживает конфигурацию через YAML, позволяя сопоставление атрибутов пользователя в YAML файле.

#### Поддержка Source to Image (S2I)

- Добавлен оператор **ACP Builds** для автоматического сборки образов из исходного кода
- Поддерживает языковые стеки Java/Go/Node.js/Python
- Упрощает развертывание приложений через репозитории исходного кода

#### Решение реестра на месте

- **ACP Registry** предоставляет легкий Docker реестр с функциями, готовыми для предприятий
- Обеспечивает встроенные возможности управления образами
- Упрощает доставку приложений

#### Рефакторинг модуля GitOps

- Расконструирован **ACP GitOps** в самостоятельную архитектуру плагина кластера
- Обновлён Argo CD до версии v2.14.x
- Улучшено управление жизненным циклом приложений на основе GitOps

#### Мониторинг на уровне пространств имен

- Внедрены динамические панели мониторинга на уровне пространств имен
- Обеспечивает визуализацию метрик приложений/нагрузок/pods

#### Интеграция Crossplane

- Выпущена **Alauda Build of Crossplane** дистрибуция
- Реализует управление провизией, ориентированное на приложение, через композиции XRD

#### Обновления виртуализации

- Обновлено до KubeVirt 1.4 для улучшенных возможностей виртуализации
- Оптимизировано управление образами для более быстрого развертывания ВМ
- Оптимизирована живая миграция ВМ, теперь инициируемая непосредственно из UI с видимым статусом миграции
- Улучшение сетевой привязки с поддержкой двойного стека (IPv4/IPv6)
- Добавлена поддержка vTPM для повышения безопасности ВМ

#### Обновления хранения Ceph

- Metro-DR с растянутым кластером обеспечивает синхронизацию данных в реальном времени между зонами доступности
- Региональный-DR с зеркалированием на основе пулов улучшает защиту данных

#### Обновления TopoLVM

- Добавлена поддержка развертывания многопадовых устройств, улучшая гибкость и стабильность
