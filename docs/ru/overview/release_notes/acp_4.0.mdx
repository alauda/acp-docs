# ACP 4.0

## Незначительная версия: 4.0.0

### Функции и улучшения

#### Установка и обновление: Модульная архитектура

Мы полностью переработали архитектуру нашей платформы для обеспечения беспрецедентной гибкости, более быстрой обновляемости и снижения эксплуатационных затрат.

**Установка основного пакета**

Наша платформа теперь поддерживает установку через упрощенный основной пакет, который содержит только необходимые компоненты. После установки этой основы пользователи могут выборочно загружать и устанавливать только те операторы или плагины для кластера, которые им нужны — будь то DevOps, Service Mesh или другие специализированные функции.

**Целевые исправления**

Исправления платформы теперь предоставляются с значительным повышением эффективности:

- Снижено время доставки критических исправлений с одного месяца до двух недель
- Обновления платформы теперь включают только те компоненты, которые требуют исправлений
- Стабильные компоненты остаются нетронутыми во время обновлений, минимизируя воздействие на систему

**Интеллектуальные обновления**

Обновления платформы были революционизированы для минимизации операционного воздействия:

- Компоненты без изменений в коде сохраняют стабильность версии во время обновлений
- Только измененные компоненты обновляются во время миграции версий
- Перезагрузки системы ограничены только теми компонентами, которые требуют изменений

**Независимое версионирование компонентов**

Многие операторы платформы теперь следуют независимым циклам выпуска, что позволяет нам быстрее предоставлять новые возможности, не дожидаясь полного цикла выпуска платформы.

#### Кластеры: Улучшенная инфраструктура как код

**Управление кластерами на основе YAML**

Локальные кластеры теперь полностью поддерживают операции на основе YAML, включая:

- Создание кластера
- Добавление узлов
- Управление конфигурацией

Это улучшение бесшовно интегрируется с рабочими процессами инфраструктуры как кода (IaC), позволяя управлять жизненным циклом кластера программным способом.

#### Расширение: Полная видимость возможностей

**Полный каталог операторов**

ОператорHub теперь отображает всех поддерживаемых операторов, независимо от того, были ли их пакеты загружены на платформу. Это улучшение:

- Обеспечивает полную видимость возможностей платформы даже в изолированных средах
- Устраняет информационные разрывы между тем, что доступно, и тем, что известно пользователям
- Уменьшает трение в процессе открытия возможностей платформы

**Гибкость версии**

Пользователи теперь могут выбирать конкретные версии операторов во время установки, а не ограничиваться только последней версией, что обеспечивает больший контроль над совместимостью компонентов и путями обновления.

**Расширения веб-консоли**

Операторы теперь поддерживают расширения веб-консоли на основе якорей, позволяя включать изображения на основе конкретной функциональности внутри операторов и плавно интегрировать их в веб-консоль платформы.

**Улучшения плагинов кластера**

Все улучшения видимости операторов, выбора версии и возможностей расширения веб-консоли также применяются к плагинам кластера, обеспечивая согласованный пользовательский опыт на всех расширениях платформы.

#### Оптимизация логики запросов логов

Страница запросов логов была оптимизирована для решения проблем с опытом и производительностью, с которыми пользователи сталкиваются при использовании функции запросов логов:

- Оригинальный радиокнопка была заменена на компонент расширенного поиска. Теперь вы можете использовать поиск логов так же, как поиск GIT.
- Независимые условия запроса для содержимого логов
- Местоположение критериев временного запроса было скорректировано. Теперь вы не сбросите ваши критерии фильтрации логов, когда будете настраивать временной диапазон.
- Оптимизирован API запросов логов для повышения общей производительности запросов

#### Обновление ElasticSearch до 8.17

Мы обновили версию ElasticSearch до 8.17, чтобы следовать функциям и улучшениям сообщества.

#### Аутентификация ALB

ALB теперь поддерживает различные механизмы аутентификации, что позволяет пользователю обрабатывать аутентификацию на уровне Ingress, а не реализовывать её в каждом бэкенд-приложении.

#### ALB поддерживает аннотации ingress-nginx

Это обновление добавляет поддержку общих аннотаций ingress-nginx в ALB, включая настройки keepalive, конфигурации таймаута и HTTP перенаправления, что повышает совместимость с ingress-nginx сообщества.

#### Оптимизация живой миграции Kubevirt

Во время процесса живой миграции время прерывания сети было сокращено до менее чем 0.5 секунд, и существующие TCP соединения не будут разорваны.
Эта оптимизация значительно улучшает стабильность и надежность миграции виртуальных машин в производственных средах.

#### Оптимизация интеграции LDAP/OIDC

Поля формы интеграции LDAP/OIDC были скорректированы, в основном включая удаление ненужных/дублирующих полей и оптимизацию описаний полей.
Интеграция LDAP/OIDC теперь поддерживает конфигурацию через YAML, позволяя сопоставление атрибутов пользователя в файле YAML.

#### Поддержка Source to Image (S2I)

- Добавлен оператор **Alauda Container Platform Builds** для автоматизированного создания образов из исходного кода
- Поддерживает языковые стеки Java/Go/Node.js/Python
- Упрощает развертывание приложений через репозитории исходного кода

#### Решение для реестра на месте

- **ACP Registry** предоставляет легковесный Docker Registry с готовыми к использованию функциями
- Обеспечивает возможности управления образами прямо из коробки
- Упрощает доставку приложений

#### Рефакторинг модуля GitOps

- Результаты **ACP GitOps** выделены в отдельную архитектуру плагина кластера
- Обновлен Argo CD до версии v2.14.x
- Улучшено управление жизненным циклом приложений на основе GitOps

#### Мониторинг на уровне пространств имен

- Введены динамические панели мониторинга на уровне пространств имен
- Обеспечивает визуализацию метрик Приложений/Работ/Подов

#### Интеграция Crossplane

- Выпущена дистрибуция **Alauda Build of Crossplane**
- Реализует центрию приложений через составления XRD

#### Обновления виртуализации

- Обновлено до KubeVirt 1.4 для улучшенных возможностей виртуализации
- Оптимизировано управление образами для более быстрого предоставления ВМ
- Оптимизирована живая миграция ВМ, теперь инициируемая напрямую из интерфейса с видимым статусом миграции
- Улучшено связывание сети с поддержкой двойного стека (IPv4/IPv6)
- Добавлена поддержка vTPM для повышения безопасности ВМ

#### Обновления Ceph Storage

- Metro-DR с расширенным кластером позволяет синхронизировать данные в режиме реального времени между зонами доступности
- Regional-DR с зеркалированием на основе пула усиливает защиту данных

#### Обновления TopoLVM

- Добавлена поддержка развертывания устройств с многопутевым подключением, что улучшает гибкость и стабильность
