# Установка ACP Builds

## Предварительные требования

> ACP Builds - это инструмент контейнеризации, предлагаемый <Term name="product" textCase="capitalize" /> и позволяющий интегрировать сборку (возможна функция Source to Image) и создание приложения.

1. Скачайте последний пакет **ACP Builds**, подходящий для вашей платформы. Если оператор **Tekton** еще не установлен в кластере Kubernetes, рекомендуется скачать его вместе.
2. Используйте инструмент командной строки `violet` для загрузки пакетов **ACP Builds** и **Tekton** на ваш целевой кластер. Для получения подробных инструкций по использованию `violet`, пожалуйста, обратитесь к [CLI](/ui/cli_tools/index.mdx).

## Процедура

### Установите оператор ACP Builds

1. Войдите в систему и перейдите на страницу **Управление платформой**.
2. Нажмите **Маркетплейс** > **OperatorHub**.
3. Найдите оператор **ACP Builds**, нажмите **Установить** и перейдите на страницу **Установка**.

Параметры конфигурации:

| **Параметр**         | **Рекомендуемая конфигурация**                                                                                                                                                    |
| :-------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Канал**            | `Alpha`: Канал по умолчанию установлен на **alpha**.                                                                                                                                     |
| **Версия**           | Пожалуйста, выберите последнюю версию.                                                                                                                                          |
| **Режим установки**   | `Cluster`: Один оператор используется во всех пространствах имен кластера для создания и управления экземплярами, что приводит к меньшему потреблению ресурсов.                             |
| **Пространство имен** | `Рекомендуется`: Рекомендуется использовать пространство имен **shipyard-operator**; оно будет создано автоматически, если не существует.                                               |
| **Стратегия обновления** | Пожалуйста, выберите `Manual`. <ul><li>`Manual`: При наличии новой версии в OperatorHub</li><li>действие **Обновить** не будет выполнено автоматически.</li></ul> |

4. На странице **Установка** выберите конфигурацию по умолчанию, нажмите **Установить** и завершите установку оператора **ACP Builds**.

### Установка экземпляра Shipyard

1. Нажмите на **Маркетплейс** > **OperatorHub**.

2. Найдите установленный оператор **ACP Builds**, перейдите в **Все экземпляры**.

3. Нажмите кнопку **Создать экземпляр** и нажмите карточку **Shipyard** в области ресурсов.

4. На странице конфигурации параметров для экземпляра вы можете использовать конфигурацию по умолчанию, если нет специфических требований.

5. Нажмите **Создать**.

### Проверка

- После успешного создания экземпляра подождите примерно "20 минут", прежде чем перейти к **Платформе контейнеров** > **Приложения** > **Приложения** и нажмите **Создать**.
- Вы должны увидеть запись для **Создать из кода**. В это время установка ACP Builds завершена, и вы можете начать ваше S2I путешествие с помощью [Создания приложения из кода](../how_to/s2i_howto.mdx).
