# Установка ACP Builds

## Предварительные условия

> ACP Builds — это инструмент контейнеров, предлагаемый <Term name="product" textCase="capitalize" />, который объединяет сборку (способную выполнять “Source to Image”) и создание приложений.

1. Загрузите пакет последней версии **ACP Builds**, соответствующий вашей платформе. Если **Tekton** оператор не был установлен в кластере Kubernetes, рекомендуется загрузить его вместе.
2. Используйте инструмент командной строки `violet` для загрузки пакетов **ACP Builds** и **Tekton** в целевой кластер. Для получения подробных инструкций по использованию `violet` обратитесь к [CLI](/ui/cli_tools/index.mdx).

## Процедура

### Установка оператора ACP Builds

1. Войдите в систему и перейдите на страницу **Управление платформой**.
2. Нажмите **Marketplace** > **OperatorHub**.
3. Найдите оператора **ACP Builds**, нажмите **Install** и перейдите на страницу **Установка**.

Параметры конфигурации:

| **Параметр**         | **Рекомендуемая конфигурация**                                                                                                                                               |
| :-------------------- | :---------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Канал**            | `Alpha`: Канал по умолчанию установлен на **alpha**.                                                                                                                         |
| **Версия**           | Пожалуйста, выберите последнюю версию.                                                                                                                                       |
| **Режим установки**   | `Cluster`: Один оператор используется во всех пространствах имен в кластере для создания и управления экземплярами, что приводит к более низкому использованию ресурсов.         |
| **Пространство имен** | `Рекомендуется`: Рекомендуется использовать пространство имен **shipyard-operator**; оно будет создано автоматически, если его нет.                                        |
| **Стратегия обновления** | Пожалуйста, выберите `Manual`. <ul><li>`Manual`: Когда новая версия доступна в OperatorHub</li><li>действие **Обновить** не будет выполнено автоматически.</li></ul> |

4. На странице **Установка** выберите конфигурацию по умолчанию, нажмите **Install** и завершите установку оператора **ACP Builds**.

### Установка экземпляра Shipyard

1. Нажмите на **Marketplace** > **OperatorHub**.

2. Найдите установленный оператор **ACP Builds**, перейдите в **Все экземпляры**.

3. Нажмите кнопку **Создать экземпляр**, а затем нажмите на карточку **Shipyard** в области ресурсов.

4. На странице конфигурации параметров для экземпляра вы можете использовать конфигурацию по умолчанию, если нет специфических требований.

5. Нажмите **Создать**.

### Проверка

- После того, как экземпляр успешно создан, подождите примерно "20 минут", прежде чем переключиться на **Платформа контейнеров** > **Приложения** > **Приложения** и нажмите на **Создать**.
- Вы должны увидеть запись для **Создать из кода**. На этом этапе установка ACP Builds завершена, и вы можете начать свое путешествие S2I с [Создания приложения из кода](../how_to/s2i_howto.mdx).
