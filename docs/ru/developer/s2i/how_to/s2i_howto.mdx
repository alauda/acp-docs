# Создание приложения на основе кода

> Используя мощные возможности установки ACP Builds для реализации всего процесса от исходного кода Java до создания приложения и, в конечном итоге, позволяя приложению эффективно работать в контейнеризованном режиме на Kubernetes.

## Предварительные требования

Перед использованием этой функциональности убедитесь, что:

- [Установка ACP Builds](../install_builds/install_builds_opeator.mdx)
- На платформе есть доступный репозиторий образов. Если нет, пожалуйста, свяжитесь с администратором для [Установки ACP Registry](/developer/registry/install_registry.mdx)

## Процедура

1. **Платформа контейнеров**, нажмите **Приложения** > **Приложения**.

2. Нажмите **Создать**.

3. Выберите **Создать из кода**.

4. Завершите настройку в соответствии с приведенными ниже параметрами:
   | **Параметр**        | **Рекомендуемая настройка**                                                                                                          |
   | :------------------- | :------------------------------------------------------------------------------------------------------------------------------------- |
   | **Репозиторий кода** | **Тип**: `Input`<br /> **URL репозитория**: `https://github.com/alauda/spring-boot-hello-world`                                        |
   | **Метод сборки**     | `Build`                                                                                                                                |
   | **Репозиторий образов** | Свяжитесь с администратором.                                                                                                          |
   | **Приложение**      | **Приложение**: `spring-boot-hello-world`<br /> **Имя**: `spring-boot-hello-world` <br />**Ограничения ресурсов**: Используйте значение по умолчанию. |
   | **Сеть**            | **Целевой порт**: `8080`                                                                                                                |

5. После заполнения параметров нажмите **Создать**.

6. Вы можете проверить соответствующий статус приложения на странице **Детали**.
