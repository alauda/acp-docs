# Создание приложения из кода

> Используя мощные возможности установки Alauda Container Platform Builds, можно выполнить весь процесс от исходного Java-кода до создания приложения, а в конечном итоге обеспечить его эффективную работу в контейнерном режиме на Kubernetes.

## Предварительные условия

Перед использованием этой функции убедитесь, что:

- [Установка Alauda Container Platform Builds](../install_builds/install_builds_operator.mdx)
- На платформе доступен репозиторий изображений. Если нет, свяжитесь с администратором для [установки Alauda Container Platform Registry](/developer/registry/install/index.mdx)

## Процедура

1. **Платформа контейнеров**, нажмите **Приложения** > **Приложения**.

2. Нажмите **Создать**.

3. Выберите **Создать из кода**.

4. Заполните конфигурацию в соответствии с указанными параметрами:
   | **Параметр**        | **Рекомендуемая конфигурация**                                                                                                          |
   | :------------------- | :------------------------------------------------------------------------------------------------------------------------------------- |
   | **Репозиторий кода** | **Тип**: `Input`<br /> **URL репозитория**: `https://github.com/alauda/spring-boot-hello-world`                                        |
   | **Метод сборки**     | `Build`                                                                                                                                |
   | **Репозиторий изображений** | Свяжитесь с администратором.                                                                                                             |
   | **Приложение**       | **Приложение**: `spring-boot-hello-world`<br /> **Имя**: `spring-boot-hello-world` <br />**Ограничения ресурсов**: Используйте значение по умолчанию. |
   | **Сеть**            | **Целевой порт**: `8080`                                                                                                                |

5. После заполнения параметров нажмите **Создать**.

6. Вы можете проверить соответствующий статус приложения на странице **Детали**.
