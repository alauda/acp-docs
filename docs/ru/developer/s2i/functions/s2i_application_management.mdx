# Управление приложениями, созданными на основе кода

## Ключевые особенности

- Введите URL-адрес репозитория кода для запуска процесса S2I, который преобразует исходный код в образ и публикует его как приложение.
- При обновлении исходного кода инициируйте действие **Пересобрать** через визуальный интерфейс, чтобы обновить версию приложения одним щелчком.

## Преимущества

- Упрощает процесс создания и обновления приложений на основе кода.
- Снижает порог для разработчиков, устраняя необходимость понимать детали контейнеризации.
- Обеспечивает визуальный процесс строительства и управления эксплуатацией, облегчая локализацию, анализ и устранение проблем.

## Предварительные требования

- [Установка ACP Builds](../install_builds/install_builds_opeator.mdx) завершена.
- Необходим доступ к репозиторию образов; если он недоступен, свяжитесь с Администратором для [Установки ACP Registry](/developer/registry/install_registry.mdx)

## Процедура

1. **Платформа контейнеров**, перейдите в **Приложение** > **Приложение**.

2. Нажмите **Создать**.

3. Выберите **Создать на основе кода**.

## Связанные операции

### Сборка

После создания приложения соответствующая информация может быть просмотрена на странице деталей.

| Параметр       | Описание                                                                                                |
| :-------------- | :--------------------------------------------------------------------------------------------------------- |
| **Сборка**       | Нажмите на ссылку, чтобы просмотреть конкретную информацию об сборке (Build) и задаче сборки (BuildRun) и YAML. |
| **Запустить сборку** | Когда сборка завершилась неудачно или исходный код изменился, вы можете нажать эту кнопку для повторного выполнения задачи сборки.   |
