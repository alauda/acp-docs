# Получение URL-адреса доступа к реестру Docker

> Служба Expose Service переключена, после ее включения, получив адрес доступа и учетную запись администратора, вы сможете управлять этим реестром изображений вне кластера.

1. Войдите в систему, перейдите на страницу **Управление платформой**.
2. Нажмите **Маркетплейс** > **Плагины кластера**, чтобы попасть на страницу списка **Плагинов кластера**.

## Служба Expose Service отключена

> Когда служба Expose Service отключена, но требуется управлять изображениями с помощью клиента, необходимо вручную настроить сопоставление между адресом ClusterIP службы реестра Docker и URL-адресом репозитория изображений в файле /etc/hosts текущего узла кластера. Чтобы получить адрес ClusterIP, выполните следующую команду: `kubectl get service -n cpaas-system internal-docker-registry -o jsonpath='{.spec.clusterIP}'`

1. Найдите плагин кластера **ACP Registry** и нажмите его название, чтобы перейти на страницу деталей **ACP Registry**.
2. Нажмите на иконку **Адрес реестра**, чтобы скопировать URL-адрес доступа к реестру Docker.
3. Вы можете получить доступ к реестру Docker из любого Pod в текущем кластере.

## Служба Expose Service включена

1. Найдите плагин кластера **ACP Registry** и нажмите **Обновить**, чтобы отредактировать плагин.
2. Переключите **Expose Service** на **Включено**.
3. После этого нажмите на название **ACP Registry**, чтобы перейти на страницу деталей **ACP Registry**.
4. Нажмите на иконку **Адрес реестра**, чтобы скопировать URL-адрес доступа к реестру Docker.
5. Вы можете получить доступ к реестру Docker вне текущего кластера.
