# Получение URL-адреса доступа к реестру Docker

> Включите переключатель Expose Service, который, когда он включен, позволяет управлять этим репозиторием образов извне кластера, получив адрес доступа и учетную запись администратора.

1. Войдите в систему и перейдите на страницу **Управление платформой**.
2. Нажмите **Marketplace** > **Плагины кластера**, чтобы перейти на страницу списка **Плагинов кластера**.

## Expose Service отключен

> Когда Expose Service отключен, но необходимо управлять образами с помощью клиента, необходимо вручную настроить сопоставление между адресом ClusterIP службы реестра Docker и URL-адресом репозитория образов в файле /etc/hosts текущего узла кластера. Чтобы получить адрес ClusterIP, выполните следующую команду: `kubectl get service -n cpaas-system internal-docker-registry -o jsonpath='{.spec.clusterIP}'`

1. Найдите плагин кластера **ACP Registry** и нажмите на его название, чтобы перейти на страницу деталей **ACP Registry**.
2. Нажмите на значок **Адрес реестра**, чтобы скопировать URL-адрес доступа к реестру docker.
3. Вы можете получить доступ к реестру Docker из любого Pod в текущем кластере.

## Expose Service включен

1. Найдите плагин кластера **ACP Registry** и нажмите **Обновить**, чтобы отредактировать плагин.
2. Переключите **Expose Service** на **Включено**.
3. Затем нажмите на название **ACP Registry**, чтобы перейти на страницу деталей **ACP Registry**.
4. Нажмите на значок **Адрес реестра**, чтобы скопировать URL-адрес доступа к реестру docker.
5. Вы можете получить доступ к реестру Docker извне текущего кластера.
