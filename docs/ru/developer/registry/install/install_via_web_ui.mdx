# Установка через веб-интерфейс

## Когда использовать этот метод?

**Рекомендуется для**:

- **Пользователей, впервые использующих систему**, которые предпочитают управляемый визуальный интерфейс.
- **Быстрых установок для проверки концепции** в непроизводственных средах.
- Команд с **ограниченной экспертизой в Kubernetes**, стремящихся упростить процесс развертывания.
- Сценариев, требующих **минимальной настройки** (например, конфигурации хранения по умолчанию).
- **Базовых сетевых настроек** без специфических правил входа.
- Конфигураций **StorageClass** для высокой доступности.

**Не рекомендуется для**:

- Производственных сред, требующих сложных конфигураций хранения (S3 хранилище).
- Сетевых настроек, нуждающихся в специфических правилах входа.

## Предварительные требования

- **Установите** плагин **Alauda Container Platform Registry** в целевом кластере с использованием механизма [Cluster Plugin](/extend/cluster_plugin.mdx).

## Установка плагина Alauda Container Platform Registry через веб-консоль

### Процедура

1. Войдите в систему и перейдите на страницу **Администратор**.
2. Нажмите **Marketplace** > **Cluster Plugins**, чтобы получить доступ к странице списка **Cluster Plugins**.
3. Найдите плагин **Alauda Container Platform Registry**, нажмите **Установить**, затем перейдите на страницу установки.
4. Настройте параметры в соответствии со следующими спецификациями и нажмите **Установить**, чтобы завершить развертывание.

Описание параметров:

| **Параметр**             | **Описание**                                                                                                                                                                                        |
| :------------------------ | :----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Expose Service**        | После включения администраторы могут управлять репозиторием изображений извне, используя адрес доступа. Это создает значительные риски безопасности и должно быть включено с крайней осторожностью.                    |
| **Enable IPv6**           | Включите эту опцию, если кластер использует сетевую архитектуру с единственным стеком IPv6.                                                                                                                                 |
| **NodePort**              | Когда Expose Service включен, настройте NodePort для обеспечения внешнего доступа к реестру через этот порт.                                                                                             |
| **Storage Type**          | Выберите тип хранения. Поддерживаемые типы: LocalVolume и StorageClass.                                                                                                                                  |
| **Nodes**                 | Выберите узел для запуска службы реестра для хранения и распределения изображений. (Доступно только при типе хранения LocalVolume)                                                                        |
| **StorageClass**          | Выберите StorageClass. Когда количество реплик превышает 1, выберите хранилище с возможностью RWX (ReadWriteMany) (например, файловое хранилище), чтобы обеспечить высокую доступность. (Доступно только при типе хранения StorageClass) |
| **Storage Size**          | Объем хранилища, выделенный для реестра (Единица: Gi).                                                                                                                                                 |
| **Replicas**              | Настройте количество реплик для Pod реестра: <ul><li>**LocalVolume**: По умолчанию 1 (фиксировано)</li><li>**StorageClass**: По умолчанию 3 (регулируется)</li></ul>                                     |
| **Resource Requirements** | Определите запросы и лимиты ресурсов CPU и памяти для Pod реестра.                                                                                                                               |

### Проверка

1. Перейдите в **Marketplace** > **Cluster Plugins** и подтвердите, что статус плагина отображается как **Установлен**.
2. Нажмите на имя плагина, чтобы просмотреть его детали.
3. Скопируйте **Адрес реестра** и используйте клиент Docker для загрузки/выгрузки изображений.

## Обновление/удаление Alauda Container Platform Registry

Вы можете обновить или удалить плагин **Alauda Container Platform Registry** как со страницы списка, так и со страницы деталей.
