# Установка

## Необходимые условия

1. **Установите** плагин кластера **ACP Registry** на целевой кластер, используя механизм плагинов кластера.

## Установка плагина кластера ACP Registry с использованием веб-консоли

### Процедура

1. Войдите в систему, перейдите на страницу **Управление платформой**.
2. Нажмите **Маркетплейс** > **Плагины кластера**, чтобы перейти на страницу списка **Плагинов кластера**.
3. Найдите плагин кластера **ACP Registry**, нажмите **Установить** и перейдите на страницу **Установка ACP Registry**.
4. Настройте параметры, используя следующие характеристики; нажмите **Установить** для завершения установки плагина кластера **ACP Registry**.

Описание параметров:

| **Параметр**                | **Описание**                                                                                                                                                                                                                                                                                       |
| :-------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Expose Service**          | После включения, получив адрес доступа и учетную запись администратора, вы можете управлять этим репозиторием изображений извне кластера. Это представляет собой значительный риск для безопасности и должно быть включено с большой осторожностью.                                                  |
| **Enable IPv6**             | Включите этот переключатель, когда сетевой протокол текущего кластера - IPv6 с одноуровневой стековой архитектурой.                                                                                                                                                                               |
| **NodePort**                | Когда служба Expose включена, настройка NodePort позволяет внешнему доступу к реестру через номер порта.                                                                                                                                                                                          |
| **Password**                | Пароль администратора, используемого для входа в систему и доступа к реестру.                                                                                                                                                                                                                     |
| **Storage Type**            | Выберите тип хранилища, при этом поддержка ограничена: LocalVolume и StorageClass.                                                                                                                                                                                                               |
| **Nodes**                   | Выберите узел, используемый для запуска службы Docker Registry и предоставления хранилища и распределения изображений. (Этот параметр можно выбрать только в результате выбора типа хранилища LocalVolume.)                                                                                       |
| **StorageClass**            | Выберите классы хранилища. Когда количество реплик больше 1, чтобы обеспечить высокую доступность Docker Registry, рекомендуется выбрать хранилище, поддерживающее возможности «RWX», такие как файловое хранилище. (Этот параметр можно выбрать только при выборе типа хранилища StorageClass.)   |
| **Storage Size**            | Размер хранилища для Docker Registry (единица: Gi).                                                                                                                                                                                                                                             |
| **Replicas**                | Настройка количества реплик для пода Docker Registry. <ul><li> **Тип хранилища - LocalVolume**: значение Replicas по умолчанию равно 1 и не может быть изменено. </li><li>**Тип хранилища - StorageClass**: значение Replicas по умолчанию равно 3, но может быть изменено.</li></ul>     |
| **Resource Requirements**   | Определяет требуемые ресурсы процессора и памяти для пода реестра.                                                                                                                                                                                                                                 |

### Проверка

1. На странице **Управление платформой** нажмите **Маркетплейс** > **Плагины кластера**, чтобы перейти на страницу списка **Плагинов кластера**. Найдите плагин кластера **ACP Registry**. Убедитесь, что его состояние **Установлен**.
2. Нажмите на название **ACP Registry**, чтобы перейти на страницу деталей плагина кластера **ACP Registry**.
3. Найдите значение поля **Адрес реестра**, скопируйте адрес и попытайтесь выполнить push/pull контейнерных изображений через этот адрес, используя клиент docker.

## Обновление/Удаление ACP Registry

Обновление или удаление **ACP Registry** можно выполнить как на странице списка, так и на странице деталей.
