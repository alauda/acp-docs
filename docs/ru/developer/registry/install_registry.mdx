# Установка

## Предварительные условия

1. **Установите** кластерный плагин **ACP Registry** на целевой кластер с использованием механизма плагинов для кластеров.

## Установка кластерного плагина ACP Registry с помощью веб-консоли

### Процедура

1. Войдите в систему, перейдите на страницу **Управление платформой**.
2. Нажмите **Маркетплейс** > **Кластерные плагины**, чтобы перейти на страницу списка **Кластерных плагинов**.
3. Найдите кластерный плагин **ACP Registry**, нажмите **Установить** и перейдите на страницу **Установка ACP Registry**.
4. Настройте параметры, обратившись к следующим спецификациям; нажмите **Установить**, чтобы завершить установку кластерного плагина **ACP Registry**.

Описание параметров:

| **Параметр**             | **Описание**                                                                                                                                                                                                                                                         |
| :------------------------ | :---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Expose Service**        | После активации, получив адрес доступа и учетную запись администратора, вы сможете управлять этим репозиторием изображений извне кластера. Это представляет собой значительный риск безопасности и должно быть включено с большой осторожностью.                                 |
| **Enable IPv6**           | Включите этот переключатель, когда сетевой протокол текущего кластера является одной стековой IPv6.                                                                                                                                                                   |
| **NodePort**              | Когда служба Expose включена, настройка NodePort позволяет внешнему доступу к кластеру к Реестру через номер порта.                                                                                                                                                    |
| **Password**              | Пароль пользователя admin для входа в систему и доступа к Реестру.                                                                                                                                                                                                    |
| **Storage Type**          | Выберите тип хранения, поддержка ограничена: LocalVolume и StorageClass.                                                                                                                                                                                             |
| **Nodes**                 | Выберите узел, который будет использоваться для работы службы Docker Registry и предоставления хранения и распределения изображений. (Этот вариант можно выбрать только при типе хранения LocalVolume.)                                                             |
| **StorageClass**          | Выберите StorageClass. Когда количество реплик больше 1, чтобы обеспечить высокую доступность Docker Registry, рекомендуется выбрать хранилище, поддерживающее возможности "RWX", такие как файловое хранилище. (Этот вариант можно выбрать только при типе хранения StorageClass.)   |
| **Storage Size**          | Размер хранения для Docker Registry (единица: Gi).                                                                                                                                                                                                                     |
| **Replicas**              | Настройка количества реплик для Pod Docker Registry. <ul><li> **Тип хранения LocalVolume**: значение Replicas по умолчанию равно 1 и не может быть изменено. </li><li>**Тип хранения StorageClass**: значение Replicas по умолчанию равно 3, но может быть изменено.</li></ul> |
| **Resource Requirements** | Определяет запросы и ограничения на ресурсы CPU и памяти для Pod реестра.                                                                                                                                                                                               |

### Проверка

1. На странице **Управление платформой** нажмите **Маркетплейс** > **Кластерные плагины**, чтобы перейти на страницу списка **Кластерных плагинов**. Найдите кластерный плагин **ACP Registry**. Убедитесь, что его состояние **Установлено**.
2. Нажмите на имя **ACP Registry**, чтобы перейти на страницу деталей кластерного плагина **ACP Registry**.
3. Найдите значение поля **Адрес реестра**, скопируйте адрес и попробуйте загрузить/выгрузить образы контейнеров через этот адрес, используя клиент docker.

## Обновление/Удаление ACP Registry

Обновление или удаление **ACP Registry** можно выполнить как на странице списка, так и на странице деталей.
