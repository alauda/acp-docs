# Создание стратегии изоляции узлов

Создайте политику изоляции узлов для текущего кластера, позволяющую указанным проектам иметь эксклюзивный доступ к узлам сгруппированных ресурсов внутри кластера, тем самым ограничивая узлы, на которых могут запускаться Pods, под данным проектом, достигая физической изоляции ресурсов между проектами.

## Создание стратегии изоляции узлов

1. В левой навигационной панели нажмите на **Безопасность** > **Стратегия изоляции узлов**.

2. Нажмите на **Создать стратегию изоляции узлов**.

3. Ознакомьтесь с инструкциями ниже, чтобы настроить соответствующие параметры.

   | Параметр               | Описание                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
   | ---------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
   | **Эксклюзивность проекта** | Включить или отключить переключатель для узлов, содержащихся в политике изоляции проекта, настроенной в стратегии; нажмите, чтобы переключить, по умолчанию включен. <br />Когда переключатель включен, только Pods под указанным проектом в политике могут работать на узлах, включенных в политику; когда выключен, Pods под другими проектами в текущем кластере также могут работать на узлах, включенных в политику, кроме указанного проекта.              |
   | **Проект**             | Проект, который настроен для использования узлов в политике.<br />Нажмите на выпадающий список **Проект** и установите флажок перед именем проекта, чтобы выбрать несколько проектов.<br />**Примечание**:<br />Проект может иметь только одну политику изоляции узлов; если у проекта уже назначена политика изоляции узлов, он не может быть выбран; <br />Поддерживается ввод ключевых слов в выпадающем списке для фильтрации и выбора проектов. |
   | **Узел**               | IP-адреса вычислительных узлов, выделенные для использования проектом в политике.<br />Нажмите на выпадающий список **Узел** и установите флажок перед именем узла, чтобы выбрать несколько узлов.<br />**Примечание**:<br />Узел может принадлежать только одной политике изоляции; если узел уже принадлежит другой политике изоляции, его нельзя выбрать; <br />Поддерживается ввод ключевых слов в выпадающем списке для фильтрации и выбора узлов.      |

4. Нажмите **Создать**.

   **Примечание**:

   - После создания политики существующие Pods в проекте, которые не соответствуют текущей политике, будут запланированы на узлы, включенные в текущую политику, после их перезапуска;

   - Когда **Эксклюзивность проекта** включена, текущие существующие Pods на узлах не будут автоматически выселены; требуется ручное планирование, если требуется выселение.

## Удаление стратегии изоляции узлов

**Примечание**: После удаления политики изоляции узлов проект больше не будет ограничен в запуске на определенных узлах, и узлы больше не будут использоваться эксклюзивно проектом.

1. В левой навигационной панели нажмите на **Безопасность** > **Стратегия изоляции узлов**.

2. Найдите политику изоляции узлов, нажмите ⋮ > **Удалить**.
