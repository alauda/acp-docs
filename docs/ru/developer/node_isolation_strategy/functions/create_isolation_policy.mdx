# Создание стратегии изоляции узлов

Создайте политику изоляции узлов для текущего кластера, позволяя указанным проектам иметь эксклюзивный доступ к узлам сгруппированных ресурсов в кластере, тем самым ограничивая работоспособные узлы для Pod’ов в рамках проекта, достигая физической изоляции ресурсов между проектами.

## Создание стратегии изоляции узлов

1. В левом навигационном меню нажмите **Безопасность** > **Стратегия изоляции узлов**.

2. Нажмите **Создать стратегию изоляции узлов**.

3. Ознакомьтесь с инструкциями ниже, чтобы настроить соответствующие параметры.

   | Параметр               | Описание                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
   | ----------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | **Эксклюзивность проекта** | Включить или отключить переключатель для узлов, содержащихся в политике изоляции проекта, настроенной в стратегии; нажмите, чтобы переключить между включенным и отключенным состоянием, по умолчанию включено. <br />Когда переключатель включен, только Pod’ы с указанным проектом в политике могут работать на узлах, включенных в политику; когда он выключен, Pod’ы других проектов в текущем кластере также могут работать на узлах, включенных в политику, кроме указанного проекта.       |
   | **Проект**             | Проект, который настроен на использование узлов в политике.<br />Нажмите на выпадающий список **Проект** и установите флажок перед названием проекта, чтобы выбрать несколько проектов.<br />**Примечание**:<br />У проекта может быть установлена только одна политика изоляции узлов; если у проекта уже назначена политика изоляции узлов, его нельзя выбрать; <br />Поддерживает ввод ключевых слов в выпадающем списке для фильтрации и выбора проектов.     |
   | **Узел**               | IP-адреса вычислительных узлов, выделенных для использования проектом в политике.<br />Нажмите на выпадающий список **Узел** и установите флажок перед названием узла, чтобы выбрать несколько узлов.<br />**Примечание**:<br />Узел может принадлежать только одной политике изоляции; если узел уже принадлежит к другой политике изоляции, его нельзя выбрать; <br />Поддерживает ввод ключевых слов в выпадающем списке для фильтрации и выбора узлов.        |

4. Нажмите **Создать**.

   **Примечание**:

   - После создания политики существующие Pod’ы в проекте, не соответствующие текущей политике, будут запланированы на узлы, включенные в текущую политику, после их перестройки;

   - Когда **Эксклюзивность проекта** включена, текущие Pod’ы на узлах не будут автоматически выселены; для выселения требуется ручное планирование.

## Удаление стратегии изоляции узлов

**Примечание**: После удаления политики изоляции узлов проект больше не будет ограничен в работе на конкретных узлах, и узлы больше не будут использоваться исключительно проектом.

1. В левом навигационном меню нажмите **Безопасность** > **Стратегия изоляции узлов**.

2. Найдите политику изоляции узлов, нажмите ⋮ > **Удалить**.
