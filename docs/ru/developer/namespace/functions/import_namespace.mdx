# Импорт пространств имен

## Предварительные условия

- Пространство имен не связано с каким-либо проектом на платформе.

- Пространства имен могут быть импортированы только в проекты, связанные с их целевым кластером. Если на платформе не существует проектов, связанных с целевым кластером, необходимо сначала создать проект, связанный с этим кластером.

## Обзор

**Возможности управления пространствами имен**:

- **Импорт пространств имен между кластерами**
  - Импорт пространства имен в проект позволяет централизованно управлять Kubernetes пространствами имен во всех кластерах, развернутых на платформе. Это позволяет администраторам объединить управление ресурсами и мониторинг в распределенных средах.

- **Разъединение пространства имен**
  - Функция "Разъединить пространство имен" позволяет отвязать пространство имен от его текущего проекта, сбрасывая его связь для повторного присвоения или очистки.

Импорт пространства имен в проект предоставляет ему эквивалентные возможности по сравнению с пространствами имен, созданными непосредственно на платформе. Это включает доступ к политике на уровне проекта (например, лимиты ресурсов), единому мониторингу и контроль управления.

**Примечание**:

- Пространство имен может быть связано только с одним проектом в любой момент времени.
- Если пространство имен уже связано с проектом, его нельзя импортировать в другой проект или переназначить без предварительного разъединения с оригинальным проектом.

## Сценарии использования

Общие сценарии использования включают:

- После подключения нового кластера к платформе вы можете связать его существующие Kubernetes пространства имен с проектом через функцию импорта пространств имен. Выберите целевой проект и кластер для импорта пространств имен. Это действие предоставляет проекту возможность управления пространствами имен, включая лимиты ресурсов, мониторинг и обеспечение соблюдения политик.

<img src="../../../../en/developer/building_application/functions/namespace/assets/nsimport.svg" width="400" />

- Пространство имен, разъединенное с проектом, может быть повторно связано с другим проектом через функцию импорта пространств имен для централизованного управления.

- Пространства имен, не связанные с каким-либо проектом (например, созданные через скрипты на уровне кластера), должны быть связаны с целевым проектом через функцию импорта пространств имен для включения управления на уровне платформы, включая видимость и централизованное управление.

## Процедура

1. В представлении **Управление проектом** нажмите на ***Имя проекта***, в котором находится пространство имен для импорта.

2. В левой навигационной панели нажмите **Пространства имен** > **Пространства имен**.

3. Нажмите кнопку **Выпадающий список** рядом с **Создать пространство имен**, затем нажмите **Импортировать пространство имен**.

4. Ознакомьтесь с описанием параметров [Создать пространство имен](./create_namespace), чтобы настроить соответствующие параметры.

5. Нажмите **Импорт**.
