# Управление членами пространства имен

## Импорт членов

Платформа поддерживает массовый импорт членов в пространство имен и назначение ролей, таких как администратор пространства имен или разработчик, для предоставления соответствующих разрешений.

### Ограничения и ограничения

- Члены могут быть импортированы в пространство имен только из **Членов проекта** проекта пространства имен.

- Платформа не поддерживает импорт пользователей-администраторов, созданных по умолчанию, или активного пользователя.

### Предварительные условия

Для импорта пользователей в качестве членов пространства имен их необходимо сначала добавить в проект пространства имен.

### Процедура

1. В представлении **Управление проектом** щелкните на ***Имя проекта***, в котором находятся члены, подлежащие импорту.

2. В левой боковой панели навигации щелкните **Пространство имен** > **Пространство имен**.

3. Щелкните на ***Имя пространства имен*** членов, подлежащих импорту.

4. На вкладке **Члены пространства имен** щелкните **Импортировать членов**.

5. Следуйте процедурам ниже, чтобы импортировать всех или некоторых пользователей из списка в пространство имен.

   :::tip
   Вы можете выбрать группу пользователей, используя выпадающий список в верхнем правом углу диалогового окна, и выполнить нечеткий поиск, введя имя пользователя в поисковом поле для имени пользователя.
   :::

   - Импортировать всех пользователей из списка в качестве членов пространства имен и назначить роли пользователям массово.

     1. Щелкните выпадающий список справа от элемента **Назначить роль** внизу диалогового окна и выберите имя роли для назначения.

     2. Щелкните **Импортировать всех**.

   - Импортировать одного или нескольких пользователей из списка в качестве членов пространства имен.

     1. Щелкните флажок перед именем пользователя/отображаемым именем, чтобы выбрать одного или нескольких пользователей.

     2. Щелкните выпадающий список справа от элемента **Назначить роль** внизу диалогового окна и выберите имя роли для назначения выбранным пользователям.

     3. Щелкните **Импортировать**.

## Добавить членов

После добавления идентификационного провайдера OICD платформа может добавить пользователей OIDC в качестве членов пространства имен.

Вы можете добавить действительные учетные записи OIDC, которые соответствуют требованиям ввода, в качестве ролей пространства имен и назначить соответствующие роли пространства имен пользователю.

**Примечание**: При добавлении членов система не проверяет действительность учетных записей. Пожалуйста, убедитесь, что добавляемые учетные записи действительны; в противном случае эти учетные записи не смогут успешно войти на платформу.

**Действительные учетные записи OIDC включают**: Действительные учетные записи в службе аутентификации личностей OIDC, настроенной через IDP для платформы, включая как те, которые успешно вошли на платформу, так и те, которые ещё не вошли.

**Предварительные условия**

На платформе должен быть добавлен идентификационный провайдер **OICD**.

### Процедура

1. В представлении **Управление проектом** щелкните на ***Имя проекта***, в котором находится член, который необходимо добавить.

2. В левой боковой панели навигации щелкните **Пространство имен** > **Пространство имен**.

3. Щелкните на ***Имя пространства имен*** члена, который необходимо добавить.

4. На вкладке **Члены пространства имен** щелкните **Добавить члена**.

5. В поле ввода **Имя пользователя** введите имя пользователя для существующей учетной записи сторонней платформы, поддерживаемой платформой.

   **Примечание**: Пожалуйста, убедитесь, что введенное имя пользователя соответствует существующей учетной записи на сторонней платформе; в противном случае эта учетная запись не сможет успешно войти на эту платформу.

6. В выпадающем списке **Роль** выберите имя роли, которое нужно настроить для этого пользователя.

7. Щелкните **Добавить**.\
   После успешного добавления вы можете просмотреть члена в списке членов пространства имен.\
   В то же время в списке пользователей (**Управление платформой > Управление пользователями**) вы можете увидеть этого пользователя. До успешного входа пользователя или синхронизации на этой платформе источник будет `-`, и его можно будет удалить; когда учетная запись успешно войдет или синхронизируется с платформой, платформа зафиксирует информацию о источнике учетной записи и отобразит её в списке пользователей.

## Удалить членов

Удаление указанных членов пространства имен и удаление их связанных ролей для отзыва разрешений пространства имен.

### Процедура

1. В представлении **Управление проектом** щелкните на ***Имя проекта***, в котором находится член, который необходимо удалить.

2. В левой боковой панели навигации щелкните **Пространство имен** > **Пространство имен**.

3. Щелкните на ***Имя пространства имен*** члена, который необходимо удалить.

4. На вкладке **Члены пространства имен** щелкните ⋮ справа от записи члена, который необходимо удалить > **Удалить**.

5. Щелкните **Удалить**.
