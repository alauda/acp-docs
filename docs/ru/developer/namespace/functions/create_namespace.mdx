# Создание пространства имен

В кластере, связанном с проектом, создайте новое пространство имен в соответствии с доступными квотами ресурсов проекта.

Новое пространство имен работает в рамках квот ресурсов, выделенных проекту (например, CPU, память), и все ресурсы в пространстве имен должны находиться в связанном кластере.

## Процедура через веб-консоль

1. В представлении **Управление проектом** нажмите на ***Имя проекта***, для которого вы хотите создать пространство имен.

2. В левой навигационной панели нажмите на **Пространства имен** > **Пространства имен**.

3. Нажмите на **Создать пространство имен**.

4. Настроить **Основную информацию**.

   | Параметр     | Описание                                                                    |
   | ------------- | ------------------------------------------------------------------------------ |
   | **Кластер**   | Выберите кластер, связанный с проектом, чтобы разместить пространство имен. |
   | **Пространство имен** | Имя пространства имен должно содержать обязательный префикс, который является именем проекта. |

5. (Необязательно) Настроить **Квоту ресурсов**.

   Каждый раз, когда для контейнера в пространстве имен указывается лимит ресурса (limits) для вычислительных или хранилищных ресурсов, или каждый раз, когда добавляется новый Pod или PVC, он будет потреблять квоту, установленную здесь.

   **ВНИМАНИЕ**:

   - Квота ресурсов пространства имен наследуется от выделенной квоты проекта в кластере. Максимально допустимая квота для типа ресурса не может превышать оставшуюся доступную квоту проекта. Если доступная квота какого-либо ресурса достигает 0, создание пространства имен будет заблокировано. Обратитесь к администратору вашей платформы для корректировки квот.

   - **Требования к настройке квоты GPU**:

     - Квоты GPU (vGPU или pGPU) могут быть настроены только в том случае, если ресурсы GPU предусмотрены в кластере.
     - При использовании vGPU также могут быть установлены квоты на память.

     **Определения единицы GPU**:

     - **Единицы vGPU**: 100 виртуальных единиц GPU (vGPU) = 1 физическое ядро GPU (pGPU).
       - Примечание: единицы pGPU учитываются только целыми числами (например, 1 pGPU = 1 ядро = 100 vGPU).
     - **Единицы памяти**:
       - 1 единица памяти = 256 MiB.
       - 1 GiB = 4 единицы памяти (1024 MiB = 4 × 256 MiB).

   - **Поведение квоты по умолчанию**:
     - Если для типа ресурса не указана квота, по умолчанию она неограниченная.
     - Это означает, что пространство имен может потреблять **все доступные ресурсы этого типа, выделенные проекту**, без явных ограничений.

6. (Необязательно) Настроить **Диапазон лимитов контейнера**; для получения дополнительных сведений см. [Диапазон лимитов контейнера](/developer/namespace/concepts/limit_range).

7. (Необязательно) Настроить **Политики безопасности Pod**; для получения конкретных сведений см. [Политики безопасности Pod](/developer/namespace/concepts/security_policies).

8. (Необязательно) В области **Дополнительные настройки** добавьте метки и аннотации для текущего пространства имен.

   **Подсказка**: Вы можете определить атрибуты пространства имен с помощью меток или дополнить пространство имен дополнительной информацией с помощью аннотаций; оба могут использоваться для фильтрации и сортировки пространств имен.

9. Нажмите на **Создать**.

## Создание пространства имен через CLI

### YAML пространства имен

```yaml
apiVersion: v1
kind: Namespace
metadata:
  name: <namespace-name>
```

### Команда создания пространства имен

```bash
kubectl create namespace <namespace-name>
```
