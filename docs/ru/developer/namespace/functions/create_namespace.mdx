# Создание пространства имен

В кластере, связанном с проектом, создайте новое пространство имен в соответствии с доступными квотами ресурсов проекта.

Новое пространство имен работает в пределах квот ресурсов, выделенных проекту (например, CPU, память), и все ресурсы в пространстве имен должны находиться в связанном кластере.

## Процедура через веб-консоль

1. На панели **Управление проектом** нажмите на ***Имя проекта***, для которого вы хотите создать пространство имен.

2. В левой навигационной панели нажмите на **Пространства имен** > **Пространства имен**.

3. Нажмите на **Создать пространство имен**.

4. Настройте **Основную информацию**.

   | Параметр     | Описание                                                                    |
   | ------------- | ------------------------------------------------------------------------------ |
   | **Кластер**   | Выберите кластер, связанный с проектом, для размещения пространства имен.     |
   | **Пространство имен** | Имя пространства имен должно включать обязательный префикс, которым является имя проекта. |

5. (По желанию) Настройте **Квоту ресурсов**.

   Каждый раз, когда для контейнера в пространстве имен задается ограничение ресурса (limits) или каждый раз, когда добавляется новый Pod или PVC, это будет потреблять квоту, установленную здесь.

   **ВНИМАНИЕ**:

   - Квота ресурсов пространства имен наследуется от выделенной квоты проекта в кластере. Максимально допустимая квота для типа ресурса не может превышать оставшуюся доступную квоту проекта. Если доступная квота какого-либо ресурса достигнет 0, создание пространства имен будет заблокировано. Обратитесь к администратору платформы для корректировки квот.

   - **Требования к конфигурации квот GPU**:

     - Квоты GPU (vGPU или pGPU) могут быть настроены только при условии, что ресурсы GPU выделены в кластере.
     - При использовании vGPU можно также установить квоты памяти.

     **Определения единиц GPU**:

     - **Единицы vGPU**: 100 виртуальных единиц GPU (vGPU) = 1 физическая единица GPU (pGPU).
       - Примечание: единицы pGPU считаются только целыми числами (например, 1 pGPU = 1 ядро = 100 vGPU).
     - **Единицы памяти**:
       - 1 единица памяти = 256 MiB.
       - 1 GiB = 4 единицы памяти (1024 MiB = 4 × 256 MiB).

   - **Поведение по умолчанию для квоты**:
     - Если квота для типа ресурса не указана, по умолчанию она является неограниченной.
     - Это означает, что пространство имен может потреблять **все доступные ресурсы этого типа, выделенные проекту**, без явных ограничений.

6. (По желанию) Настройте **Диапазон пределов контейнера**; для получения дополнительной информации обратитесь к [Диапазону пределов контейнера](/developer/namespace/concepts/limit_range).

7. (По желанию) Настройте **Политики безопасности Pod**; для получения конкретных сведений обратитесь к [Политикам безопасности Pod](/developer/namespace/concepts/security_policies).

8. (По желанию) В области **Дополнительная конфигурация** добавьте метки и аннотации для текущего пространства имен.

   **Совет**: Вы можете определить атрибуты пространства имен через метки или дополнить пространство имен дополнительной информацией через аннотации; оба могут быть использованы для фильтрации и сортировки пространств имен.

9. Нажмите на **Создать**.

## Создание пространства имен через CLI

### YAML пространства имен

```yaml
apiVersion: v1
kind: Namespace
metadata:
  name: <имя-пространства-имен>
```

### Команда создания пространства имен в командной строке

```bash
kubectl create namespace <имя-пространства-имен>
```
