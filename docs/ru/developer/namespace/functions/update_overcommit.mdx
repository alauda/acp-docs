# Обновление коэффициента перераспределения ресурсов

Обновите коэффициент перераспределения ресурсов для созданного пространства имен. Вы можете обратиться к требованиям ресурсов для CPU и памяти в текущем пространстве имен, чтобы индивидуально установить коэффициент перераспределения ресурсов для пространства имен, ограничивая тем самым значение лимита CPU и памяти (limits) и запроса (requests), установленные пользователями в пространстве имен, в разумных пределах для повышения использования вычислительных ресурсов.

## Предостережения

В кластерах с включенной виртуализацией узлов, если вам необходимо настроить коэффициент перераспределения ресурсов виртуальной машины, сначала необходимо отключить коэффициент перераспределения ресурсов кластера или пространства имен.

## Процедура

1. Перейдите в представление **Управление проектами**.

2. В левой навигационной панели нажмите **Пространства имен** > **Пространства имен**.

3. Нажмите на ***имя пространства имен***, которое необходимо обновить.

4. Нажмите на **Действия** в правом верхнем углу > **Обновить коэффициент перераспределения ресурсов**.

5. Выберите соответствующий метод **настройки коэффициента перераспределения ресурсов**, чтобы установить коэффициент перераспределения ресурсов CPU или памяти для пространства имен.

| Параметр                | Описание                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| ------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Наследовать от кластера** | <ul><li>Коэффициент перераспределения ресурсов пространства имен по умолчанию является **наследуемым от кластера**, **значение** отображает настройки коэффициента перераспределения ресурсов исходного кластера. Например, если коэффициенты перераспределения ресурсов CPU и памяти кластера оба установлены на 4, то коэффициенты перераспределения ресурсов CPU и памяти для текущего пространства имен по умолчанию будут равны 4.</li><br /><li>При создании приложения значение запроса контейнера фиксируется на **указанном лимитированном значении/коэффициенте перераспределения ресурсов кластера**. Если лимитированное значение не указано, оно устанавливается на **значение по умолчанию** в квоте контейнера пространства имен.</li><br /><li>Пример: Предположим, что коэффициент перераспределения ресурсов CPU кластера установлен на 4, и коэффициент перераспределения ресурсов пространства имен наследуется от кластера. Когда пользователь создает приложение и указывает значение лимита CPU 4c, значение запроса CPU будет 4c/4 = 1c.</li></ul>                                                                                                                                                                                                                                         |
| **Пользовательский**               | <ul><li>Коэффициент перераспределения ресурсов пространства имен **значение** определяется пользовательскими настройками администратора, поддерживает только положительные целые числа, большие 1; если оставлено пустым, это означает, что текущее пространство имен не имеет установленного коэффициента перераспределения ресурсов.<br />Например, если коэффициенты перераспределения ресурсов CPU и памяти кластера оба установлены на 4, и здесь указаны пользовательские значения, оба установленные на 2, то коэффициенты перераспределения ресурсов CPU и памяти текущего пространства имен будут равны 2; если здесь не указаны пользовательские значения, это означает, что CPU и память текущего пространства имен не имеют установленного коэффициента перераспределения ресурсов.</li><br /><li>При создании приложения значение запроса контейнера фиксируется на **указанном лимитированном значении/коэффициенте перераспределения ресурсов пространства имен**. Если лимитированное значение не указано, оно устанавливается на **значение по умолчанию** в квоте контейнера пространства имен.</li><br /><li>Пример: Предположим, что коэффициент перераспределения ресурсов CPU кластера установлен на 4, и коэффициент перераспределения ресурсов пространства имен настроен на 2. Когда пользователь создает приложение, указывая значение лимита CPU 4c, значение запроса CPU будет 4c/2 = 2c.</li></ul> |

5. Нажмите **Обновить**.

**Примечание**: Обновление коэффициента перераспределения ресурсов не влияет на значения запросов текущих контейнеров. Однако при восстановлении Pod значения запросов контейнеров будут пересчитаны на основе обновленного коэффициента перераспределения ресурсов.
