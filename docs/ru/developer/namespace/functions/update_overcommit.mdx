# Обновление коэффициента переполнения

Обновите коэффициент переполнения для созданного пространства имен. Вы можете обратиться к требованиям к ресурсам для ЦП и памяти в текущем пространстве имен, чтобы индивидуально установить коэффициент переполнения для пространства имен, тем самым ограничивая значения предела ЦП и памяти (limits) и запросов (requests), установленные пользователями в пространстве имен, в разумных пределах для повышения эффективности использования вычислительных ресурсов.

## Предостережения

В кластерах с включенной виртуализацией узлов, если вам нужно настроить коэффициент переполнения виртуальной машины, вы должны сначала отключить коэффициент переполнения кластера или пространства имен.

## Процедура

1. Перейдите в представление **Управление проектом**.

2. В левой панели навигации щелкните **Пространства имен** > **Пространства имен**.

3. Щелкните по названию ***пространства имен***, которое необходимо обновить.

4. Щелкните на **Действия** в верхнем правом углу > **Обновить коэффициент переполнения**.

5. Выберите подходящий способ **конфигурации коэффициента переполнения**, чтобы установить коэффициент переполнения ЦП или памяти для пространства имен.

| Параметр                | Описание                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| ----------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Наследовать от кластера** | <ul><li>Коэффициент переполнения пространства имен по умолчанию **наследуется от кластера**, **значение** отображает настройки коэффициента переполнения исходного кластера. Например, если коэффициенты переполнения ЦП и памяти кластера установлены на 4, то коэффициенты переполнения ЦП и памяти текущего пространства имен по умолчанию равны 4.</li><br /><li>При создании приложения значение запроса контейнера фиксируется на уровне **указанного предела значения/коэффициент переполнения кластера**. Если предельное значение не указано, значение предела установлено на **значение по умолчанию** в квоте контейнера пространства имен.</li><br /><li>Пример: Предположим, что коэффициент переполнения ресурсного ЦП кластера установлен на 4, и коэффициент переполнения пространства имен наследуется от кластера. Когда пользователь создает приложение и указывает предельное значение ЦП 4c, значение запроса ЦП будет 4c/4 = 1c.</li></ul>                                                                                                                                                                                                                                         |
| **Пользовательский**    | <ul><li>Коэффициент переполнения пространства имен **значение** определяется пользовательскими настройками администратора, поддерживаются только положительные целые числа, большие 1; если оставлено пустым, это означает, что в текущем пространстве имен коэффициент переполнения не установлен.<br />Например, если коэффициенты переполнения ЦП и памяти кластера установлены на 4, а пользовательские значения здесь установлены на 2, то коэффициенты переполнения ЦП и памяти текущего пространства имен равны 2; если пользовательские значения здесь не предоставлены, это означает, что для текущего пространства имен коэффициенты переполнения ЦП и памяти не установлены.</li><br /><li>При создании приложения значение запроса контейнера фиксируется на уровне **указанного предела значения/коэффициент переполнения пространства имен**. Если предельное значение не указано, значение предела установлено на **значение по умолчанию** в квоте контейнера пространства имен.</li><br /><li>Пример: Предположим, что коэффициент переполнения ресурсного ЦП кластера установлен на 4, а коэффициент переполнения пространства имен настроен на 2. Когда пользователь создает приложение с указанием предельного значения ЦП 4c, значение запроса ЦП будет 4c/2 = 2c.</li></ul> |

5. Нажмите **Обновить**.

**Примечание**: Обновление коэффициента переполнения не влияет на значения запросов текущих работающих контейнеров. Однако при повторном создании Pod значения запросов контейнера будут пересчитаны на основе обновленного коэффициента переполнения.
