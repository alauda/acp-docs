# Ограничения лимитов контейнеров

Смотрите официальную документацию Kubernetes: [Ограничения лимитов](https://kubernetes.io/docs/concepts/policy/limit-range/)

Использование Kubernetes LimitRange в качестве контроллера допуска - это **ограничения ресурсов на уровне контейнера или Pod**. Он устанавливает значения стандартных запросов, ограничений и максимальных значений для контейнеров или Pods, созданных после создания или обновления LimitRange, при этом постоянно отслеживая использование контейнеров, чтобы гарантировать, что ни один из ресурсов не превышает максимальные значения, определенные в пространстве имен.

> Запрос ресурсов контейнера - это соотношение между ограничениями ресурсов и перераспределением кластера. Значения запросов ресурсов служат ссылкой и критерием для планировщика при планировании контейнеров. Планировщик будет проверять доступные ресурсы для каждого узла (общие ресурсы - сумма запросов ресурсов контейнеров в Pods, запланированных на узле). Если общие запросы ресурсов нового контейнера Pod превышают оставшиеся доступные ресурсы узла, этот Pod не будет запланирован на этом узле.

LimitRange является контроллером допуска:

- Он применяет стандартные значения запросов и ограничений для всех контейнеров, которые не устанавливают требования к вычислительным ресурсам.
- Он отслеживает использование, чтобы гарантировать, что оно не превышает максимумы ресурсов и соотношение, определенные в любом LimitRange, присутствующем в пространстве имен.

**Включает следующие конфигурации**

| Ресурс  | Поле                                                      |   |
| ------- | --------------------------------------------------------- | - |
| CPU     | <ul><li>Стандартный запрос</li><li>Ограничение</li><li>Макс</li></ul> |   |
| Память  | <ul><li>Стандартный запрос</li><li>Ограничение</li><li>Макс</li></ul> |   |
