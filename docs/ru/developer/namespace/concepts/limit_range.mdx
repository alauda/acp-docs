# Ограничения диапазонов контейнеров

Смотрите официальную документацию Kubernetes: [Ограничения диапазонов](https://kubernetes.io/docs/concepts/policy/limit-range/)

Использование LimitRange в Kubernetes в качестве контроллера допуска — это **ограничения ресурсов на уровне контейнера или Pod**. Он устанавливает значения по умолчанию для запросов, лимитов и максимальных значений для контейнеров или Pod, созданных после создания или обновления LimitRange, в то время как постоянно отслеживает использование контейнеров, чтобы гарантировать, что ресурсы не превышают определенные максимальные значения в пространстве имен.

> Запрос ресурсов контейнера — это соотношение между лимитами ресурсов и переполнением кластера. Значения запросов ресурсов служат в качестве ориентира и критерия для планировщика при планировании контейнеров. Планировщик проверит доступные ресурсы для каждого узла (всего ресурсов - сумма запросов ресурсов контейнеров в Pods, запланированных на узле). Если общие запросы ресурсов нового контейнера Pod превышают оставшиеся доступные ресурсы узла, этот Pod не будет запланирован на этом узле.

LimitRange является контроллером допуска:

- Он применяет значения запросов и лимитов по умолчанию для всех контейнеров, которые не устанавливают требования к вычислительным ресурсам.
- Он отслеживает использование, чтобы гарантировать, что оно не превышает максимальные значения ресурсов и соотношения, определенные в любом LimitRange, присутствующем в пространстве имен.

**Включает следующие конфигурации**

| Ресурс   | Поле                                                       |   |
| -------- | --------------------------------------------------------- | - |
| CPU      | <ul><li>Запрос по умолчанию</li><li>Лимит</li><li>Макс</li></ul> |   |
| Память   | <ul><li>Запрос по умолчанию</li><li>Лимит</li><li>Макс</li></ul> |   |
