# Коэффициент перерасхода ресурсов

:::tip
Кластеру необходимо активировать коэффициент перерасхода ресурсов, чтобы установить коэффициент перерасхода для пространства имен.
:::

## Что такое коэффициент перерасхода ресурсов?

Вы можете установить коэффициент перерасхода ресурсов отдельно для пространства имен на основе требований к ресурсам для процессора и памяти в текущем пространстве имен. Это гарантирует, что значения пределов и запросов для контейнеров, установленные пользователями в этом пространстве имен, находятся в разумных пределах, тем самым улучшая использование вычислительных ресурсов.

- **Пределы**: Максимальное количество ресурсов, которые контейнер может использовать во время выполнения. Использование ресурсов во время работы контейнера не может превышать это значение.
- **Запросы**: Количество ресурсов, которые контейнер запрашивает при создании. Когда Kubernetes планирует контейнеры, он судит по этому значению и планирует контейнер на выполнение на узле, где доступные ресурсы соответствуют запросам контейнера. Если нет узла, который соответствует требованиям (например, когда значение запроса больше доступных ресурсов на планируемых узлах), контейнер не может нормально запуститься.
- **Коэффициент перерасхода**: Соотношение пределов к запросам (пределы/запросы).

## Пример

Предположим, что коэффициент перерасхода для пространства имен установлен на 2, при создании приложения и указании предела CPU в 4c соответствующее значение запроса CPU рассчитывается как:

Запрос CPU = Предел CPU / Коэффициент перерасхода

Таким образом, запрос CPU становится 4c / 2 = 2c.
