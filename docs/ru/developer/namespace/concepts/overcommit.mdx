# Коэффициент переочередности

:::tip
Кластер должен иметь включенный коэффициент переочередности, чтобы установить коэффициент переочередности для пространства имен.
:::

## Что такое коэффициент переочередности?

Вы можете установить коэффициент переочередности отдельно для пространства имен на основе требований к ресурсам для ЦП и памяти в текущем пространстве имен. Это гарантирует, что значения лимита и запроса для контейнеров, установленные пользователями в этом пространстве имен, находятся в разумных пределах, что, в свою очередь, улучшает использование вычислительных ресурсов.

- **Лимиты**: Максимальное количество ресурсов, которые контейнер может использовать во время работы. Использование ресурсов во время работы контейнера не может превышать это значение.
- **Запросы**: Количество ресурсов, которые контейнер запрашивает при создании. Когда Kubernetes планирует контейнеры, он оценивает на основе этого значения и планирует запуск контейнера на узле, где доступные ресурсы соответствуют запросу контейнера. Если нет узла, который удовлетворяет требованиям (например, когда значение запроса больше доступных ресурсов на планируемых узлах), контейнер не может нормально запуститься.
- **Коэффициент переочередности**: Отношение лимитов к запросам (лимиты/запросы).

## Пример

Предположим, что коэффициент переочередности пространства имен установлен на 2, когда создается приложение и указывается лимит ЦП 4c, соответствующее значение запроса ЦП рассчитывается как:

Запрос ЦП = Лимит ЦП / Коэффициент переочередности

Таким образом, запрос ЦП становится 4c / 2 = 2c.
