# Типы рабочих нагрузок

Помимо создания нативных приложений и компонентных приложений, рабочие нагрузки также могут быть непосредственно созданы в **Container Platform > Computing Components**:

- **Deployment**: Наиболее часто используемый контроллер рабочих нагрузок для развертывания безсостояний приложений. Он может гарантировать, что заданное количество реплик Pod’ов работает в кластере, поддерживая последовательные обновления и откаты, что делает его подходящим для сценариев безсостояний приложений, таких как веб-сервисы и API-сервисы.

- **DaemonSet**: Обеспечивает, чтобы каждый узел в кластере (или определенные узлы) запускали реплику Pod. Когда узел присоединяется к кластеру, Pod автоматически создается; когда узел удаляется из кластера, эти Pod’ы также удаляются. Подходит для сценариев, требующих ведения журналов, мониторинга и т.д., чтобы работать на каждом узле.

- **StatefulSet**: Контроллер рабочих нагрузок для управления с состоянием приложениями. Он поддерживает фиксированную идентичность для каждого Pod и предоставляет стабильное хранилище и сетевую идентичность, которые остаются неизменными даже если Pod перераспределяется. Подходит для приложений с состоянием, таких как базы данных и распределенные кэши.

- **Job**: Рабочая нагрузка для выполнения однократных задач. Job создает один или несколько Pod’ов и гарантирует, что заданное количество Pod’ов успешно завершает задачу перед завершением работы. Подходит для пакетной обработки, миграции данных и других сценариев одноразовых задач.

- **CronJob**: Используется для управления заданиями, запланированными на выполнение по времени. Вы можете установить выражение времени для выполнения задачи, и система автоматически создаст и выполнит Job в запланированное время. Подходит для периодических задач, таких как резервное копирование данных, генерация отчетов и периодическая очистка.

Помимо создания вышеуказанных вычислительных компонентов через страницу формы платформы, платформа также поддерживает создание Pod’ов и контейнеров с помощью инструментов командной строки (CLI):

- **Pod**: Самая маленькая поддерживаемая единица в Kubernetes, Pod может содержать один или несколько контейнеров, которые делят хранилище, сеть и декларации конфигурации. Pod обычно управляются контроллерами (такими как Deployments).
- **Container**: Стандартная программная единица, которая упаковывает код и все зависимости, позволяя приложениям быстро и надежно работать в различных вычислительных средах. Контейнеры работают внутри Pod’ов и делят ресурсы Pod.
