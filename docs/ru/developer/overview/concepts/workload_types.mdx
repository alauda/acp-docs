# Типы нагрузок

В дополнение к созданию нативных приложений и компонентных приложений, нагрузки также могут быть созданы непосредственно в **Container Platform > Computing Components**:

- **Deployment**: Наиболее часто используемый контроллер нагрузки для развертывания безсостояний приложений. Он может гарантировать, что заданное количество реплик Pod работает в кластере, поддерживает поэтапные обновления и откаты, подходит для сценариев безсостояний приложений, таких как веб-сервисы и API-сервисы.

- **DaemonSet**: Обеспечивает, что каждый узел кластера (или определенные узлы) выполняет реплику Pod. Когда узел присоединяется к кластеру, Pod создается автоматически; когда узел удаляется из кластера, эти Pods также утилизируются. Подходит для сценариев, требующих ведения журналов, мониторинга и т. д., чтобы работать на каждом узле.

- **StatefulSet**: Контроллер нагрузки для управления состоянием приложений. Он сохраняет фиксированную идентичность для каждого Pod и предоставляет стабильное хранилище и сетевую идентичность, которая не изменяется даже если Pod переназначается. Подходит для приложений с состоянием, таких как базы данных и распределенные кэши.

- **Job**: Нагрузка для выполнения одноразовых задач. Job создает один или несколько Pods и гарантирует, что указанное количество Pods успешно завершает задачу перед завершением. Подходит для пакетной обработки, миграции данных и других сценариев одноразовых задач.

- **CronJob**: Используется для управления Jobs, запланированными для выполнения по времени. Вы можете установить временное выражение для выполнения задачи, и система автоматически создаст и выполнит Job в запланированное время. Подходит для периодических задач, таких как резервное копирование данных, генерация отчетов и периодическая очистка.

В дополнение к созданию вышеуказанных вычислительных компонентов через страницу форм платформы, платформа также поддерживает создание Pods и Контейнеров через инструменты командной строки:

- **Pod**: Наименьшая развертываемая единица в Kubernetes, Pod может содержать один или несколько контейнеров, которые делят хранилище, сеть и конфигурационные декларации. Pods обычно управляются контроллерами (такими как Deployments).
- **Container**: Стандартная программная единица, которая упаковывает код и все зависимости, позволяя приложениям работать быстро и надежно в различных вычислительных средах. Контейнеры работают внутри Pods и делят ресурсы Pod.
