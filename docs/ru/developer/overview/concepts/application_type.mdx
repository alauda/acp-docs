# Типы приложений

В разделе **Container Platform > Application Management** платформы можно создать следующие типы приложений:

- **Приложение**: Полноценное бизнес-приложение, состоящее из одного или нескольких связанных вычислительных компонентов (Workloads), внутренних маршрутов (Services) и других нативных ресурсов Kubernetes. Оно поддерживает создание через редактирование в интерфейсе пользователя, оркестрацию на основе YAML и шаблоны, и может работать в средах разработки, тестирования или производства. Различные типы нативных приложений могут быть созданы следующими способами:
  - **Создать из образа**: Быстро создать приложения, используя существующие контейнерные образы.
  - **Создать из каталога**: Создать приложения, используя пакеты Helm Chart.
  - **Создать из YAML**: Создать приложения, используя файлы конфигурации YAML.
  - **Создать из кода**: Создать приложения, используя исходный код.
- **Приложение с поддержкой оператора**: На основе компонентов приложения (поддерживаемых оператором) вы можете быстро развернуть компонент приложения и использовать возможности операторов для автоматизации всего управления жизненным циклом приложения.
- **Приложение OAM**: Используется для определения модели облачно-родных приложений. По сравнению с контейнерной или оркестрационной логикой Kubernetes, OAM больше сосредоточен на самом "приложении". На основе OAM общие возможности приложений инкапсулируются в высокоуровневые интерфейсы для использования на протяжении всего процесса развертывания, разработки и эксплуатации приложений.
