# Типы приложений

На платформе в разделе **Контейнерная платформа > Управление приложениями** можно создавать следующие типы приложений:

- **Приложение**: Полноценное бизнес-приложение, состоящее из одного или нескольких связанных вычислительных компонентов (нагрузки), внутренних маршрутов (Сервисы) и других нативных ресурсов Kubernetes. Оно поддерживает создание с помощью редактирования через интерфейс, оркестрации YAML и шаблонов, и может работать в средах разработки, тестирования или производства. Разные типы нативных приложений можно создать следующими способами:
  - **Создать из образа**: Быстро создать приложения, используя существующие контейнерные образы.
  - **Создать из каталога**: Создать приложения, используя пакеты Helm Chart.
  - **Создать из YAML**: Создать приложения, используя файлы конфигурации YAML.
  - **Создать из кода**: Создать приложения, используя исходный код.
- **Приложение на основе оператора**: На основе компонентов приложения (поддерживается оператором) вы можете быстро развернуть компонент приложения и использовать возможности операторов для автоматизации полного управления жизненным циклом приложения.
- **Приложение OAM**: Используется для определения модели облачно-нативных приложений. В отличие от логики оркестрации контейнеров или Kubernetes, OAM больше сосредоточен на самом «приложении». На основе OAM общие возможности приложений инкапсулируются в высокоуровневые интерфейсы для использования в процессе развертывания, разработки и эксплуатации приложений.
