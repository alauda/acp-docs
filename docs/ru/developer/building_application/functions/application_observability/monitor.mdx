# Мониторинг панелей

- Поддерживает просмотр данных мониторинга ресурсов для компонентов рабочей нагрузки на платформе за последние 7 дней (с настраиваемым периодом хранения данных мониторинга). Включает статистику для приложений, рабочих нагрузок, подов и тенденций/рейтингов использования ЦП/памяти.
- Поддерживает мониторинг на уровне пространства имен.
- Поддерживаемый мониторинг на уровне рабочей нагрузки: **Приложения**, **Развертывания**, **DaemonSets**, **StatefulSets** и **Pods**.

## Предварительные условия

- [Установка плагинов мониторинга](../../../../observability/monitor/install_monitor.mdx)

## Мониторинг панелей на уровне пространства имен

### Процедура

1. **Платформа контейнеров**, нажмите **Наблюдать** > **Панели**.

2. Просмотрите данные мониторинга в пространстве имен. Предоставляются три панели: **Обзор приложений**, **Обзор рабочих нагрузок** и **Обзор подов**.

3. Переключайтесь между панелями для мониторинга целевого **Обзора**.

### Создание панели мониторинга на уровне пространства имен

1. **Управление платформой**, создайте специализированную панель мониторинга, обратившись к [Создание панели мониторинга](../../../../observability/monitor/functions/manage_dashboard.mdx#｛#create_dashboard｝) для создания специализированной панели мониторинга.
2. Настройте следующие метки для отображения панели мониторинга на уровне пространства имен на **Платформе контейнеров**:
   - `cpaas.io/dashboard.folder: container-platform`
   - `cpaas.io/dashboard.tag.overview: "true"`

## Мониторинг на уровне рабочей нагрузки

> Эта процедура демонстрирует, как просматривать мониторинг подов через интерфейс развертывания.

### Панель мониторинга по умолчанию

#### Процедура

1. **Платформа контейнеров**, нажмите **Рабочие нагрузки** > **Развертывания**.

2. Нажмите на имя развертывания из списка.

3. Перейдите на вкладку **Мониторинг**, чтобы просмотреть метрики мониторинга по умолчанию.

#### Интерпретация метрик

| **Мониторинг ресурса** | **Гранулярность метрик**               | **Техническое определение**                                                                                                                                                                 |
| :---------------------- | :------------------------------------- | :--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **ЦП**                 | **Использование**/**Загрузка**            | **Использование** = Загрузка/Лимит (лимиты)<br />Оцените конфигурацию лимитов контейнера. Высокая загрузка указывает на недостаточные лимиты.<br />**Загрузка** представляет собой фактическое потребление ресурсов. |
| **Память**              | **Использование**/**Загрузка**            | **Использование** = Загрузка/Лимит (лимиты)<br />Метод оценки такой же, как и для ЦП. Высокий уровень может вызвать нестабильность компонента.                                                                    |
| **Сетевой трафик**     | **Скорость входа**/**Скорость выхода**     | Сетевой трафик (байты/сек) поступающий/выходящий из подов.                                                                                                                                    |
| **Сетевой пакет**      | **Скорость приема**/**Скорость передачи** | Сетевые пакеты (количество/сек) полученные/отправленные подами.                                                                                                                                       |
| **Скорость диска**           | **Чтение**/**Запись**                   | Пропускная способность чтения/записи (байты/сек) смонтированных томов на каждую рабочую нагрузку.                                                                                                                       |
| **IOPS диска**           | **Чтение**/**Запись**                   | Операции ввода/вывода в секунду (IOPS) смонтированных томов на каждую рабочую нагрузку.                                                                                                               |

### Пользовательская панель мониторинга

4. Нажмите на **Иконку переключения**, чтобы переключиться на пользовательские панели. Обратитесь к [Добавить панель в пользовательскую панель](../../../../observability/monitor/functions/manage_dashboard.mdx) для создания специализированной панели мониторинга на уровне **Рабочей нагрузки**.

:::info
Наведите курсор на кривые графиков, чтобы просмотреть метрики по подам и выражения PromQL в определенные моменты времени. Если количество подов превышает 15, отображаются только 15 лучших записей, отсортированных в порядке убывания.
:::
