# Настройка Secrets

Создайте Secret в зависимости от ваших потребностей, выберите соответствующий тип Secret и настройте параметры.

## Процедура

1. **Платформа контейнеров**.

2. В боковом навигационном меню нажмите **Конфигурация** > **Secrets**.

3. Нажмите **Создать Secret**.

4. Настройте параметры.

   **Примечание**: В форме данные, такие как введенное имя пользователя и пароль, будут автоматически закодированы в формате Base64 перед сохранением в Secret. Преобразованные данные можно просмотреть в представлении YAML.

5. Нажмите **Создать**.

## Дальнейшие действия

При создании рабочих нагрузок для оригинальных приложений в том же пространстве имен вы можете ссылаться на уже созданные Secrets.

## Операции

Вы можете нажать (⋮) с правой стороны страницы списка или нажать **Действия** в верхнем правом углу страницы сведений, чтобы обновить или удалить Secret по мере необходимости.

| Операция   | Описание                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| ---------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Обновить** | После добавления или обновления Secret, рабочие нагрузки, которые ссылаются на этот Secret (или его элементы конфигурации) через переменные окружения, должны быть переработаны для того, чтобы новая конфигурация вступила в силу.                                                                                                                                                                                                                                                                                                                                                |
| **Удалить** | <ul><li>После удаления Secret, рабочие нагрузки, которые ссылаются на этот Secret (или его элементы конфигурации) через переменные окружения, могут быть затронуты из-за невозможности найти источник ссылки при переработке Pod.</li><li>Пожалуйста, не удаляйте Secrets, автоматически сгенерированные платформой, так как это может препятствовать правильному функционированию платформы. Например: Secrets типа service-account-token, которые содержат информацию для аутентификации ресурсов пространства имен и Secrets в системных пространствах имен (например, kube-system).</li></ul> |
