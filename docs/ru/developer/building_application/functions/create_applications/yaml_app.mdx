# Создание приложений из YAML

Если вы владеете синтаксисом YAML и предпочитаете использовать конфигурации, не определенные в формах или шаблонах, вы можете выбрать метод создания YAML в один клик. Этот подход позволяет более гибко настраивать основную информацию и ресурсы для вашего нативного приложения.

## Предварительные меры

Когда в кластере существуют узлы как Linux, так и Windows, для предотвращения развертывания приложения на несовместимых узлах необходимо настроить выбор узлов. Например:

```yaml
spec:
    spec:
      nodeSelector:
        kubernetes.io/os: linux
```

## Предпосылки

Убедитесь, что образы, определенные в YAML, могут быть загружены в текущем кластере. Вы можете проверить это с помощью команды `docker pull`.

## Процедура

1. **Платформа контейнеров**, и перейдите в **Приложение** > **Приложения**.

2. Нажмите **Создать**.

3. Выберите **Создать из YAML**.

4. Завершите настройку и нажмите **Создать**.

5. Соответствующий **Деплоймент** можно просмотреть на странице данных.

````yaml
# webapp-deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: webapp
  labels:
    app: webapp
    env: prod
spec:
  replicas: 3
  selector:
    matchLabels:
      app: webapp
  template:
    metadata:
      labels:
        app: webapp
        tier: frontend
    spec:
      containers:
      - name: webapp
        image: nginx:1.25-alpine
        ports:
        - containerPort: 80
        resources:
          requests:
            cpu: "100m"
            memory: "128Mi"
          limits:
            cpu: "250m"
            memory: "256Mi"
---
# webapp-service.yaml
apiVersion: v1
kind: Service
metadata:
  name: webapp-service
spec:
  selector:
    app: webapp
  ports:
    - protocol: TCP
      port: 80
      targetPort: 80
  type: ClusterIP
```yaml
````
