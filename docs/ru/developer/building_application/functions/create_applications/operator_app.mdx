# Создание приложений с использованием Operator Backed

Приложения, основанные на операторе, представляют собой наборы ресурсов, предоставляемых оператором. Основываясь на этих приложениях, вы можете быстро развертывать компонентные приложения и использовать возможности оператора для автоматизации всего управления жизненным циклом приложения.

## Процедура

1. **Платформа контейнеров**, перейдите в **Приложения** > **Приложения** в левой боковой панели.

2. Нажмите **Создать**.

3. Выберите **Создать из каталога** в качестве способа создания.

4. Выберите экземпляр, основанный на операторе, и настройте **Параметры пользовательских ресурсов**. Выберите экземпляр приложения, управляемого оператором, и настройте его спецификации пользовательского ресурса (CR) в манифесте CR, включая:
   - `spec.resources.limits` (ограничения ресурсов на уровне контейнера).
   - `spec.resourceQuota` (политики квоты, определенные оператором). Другие параметры, специфичные для CR, такие как `spec.replicas`, `spec.storage.className` и т. д.

5. Нажмите **Создать**.

Веб-консоль перейдет на страницу **Приложения** > **Приложения, основанные на операторе**.

:::info
**Примечание**:
Процесс создания ресурсов Kubernetes требует асинхронной согласования. Завершение может занять несколько минут в зависимости от условий кластера.
:::

## Устранение неполадок

Если создание ресурса завершается неудачей:

1. Проверьте ошибки согласования контроллера:
   ```bash
   kubectl get events --field-selector involvedObject.kind=<Ваш-Пользовательский-Ресурс> --sort-by=.metadata.creationTimestamp
   ```
2. Проверьте доступность API ресурсов:
   ```bash
   kubectl api-resources | grep <Ваш-Тип-Ресурса>
   ```
3. Повторите создание после проверки готовности CRD/Оператор:
   ```bash
   kubectl apply -f ваш-манифест-ресурса.yaml
   ```
