# Создание приложений с использованием приложений, поддерживаемых оператором

Приложения, поддерживаемые оператором, представляют собой набор ресурсов, предоставляемых оператором. На основе этих приложений, поддерживаемых оператором, вы можете быстро развернуть компонентное приложение и использовать возможности оператора для автоматизации всего жизненного цикла управления приложением.

## Процедура

1. **Контейнерная платформа**, перейдите в **Приложения** > **Приложения** в левой боковой панели.

2. Нажмите **Создать**.

3. Выберите **Создать из каталога** в качестве способа создания.

4. Выберите экземпляр, поддерживаемый оператором, и настройте **Параметры пользовательских ресурсов**. Выберите экземпляр приложения, управляемого оператором, и настройте его спецификации пользовательских ресурсов (CR) в манифесте CR, включая:
   - `spec.resources.limits` (ограничения ресурсов на уровне контейнера).
   - `spec.resourceQuota` (политики квот, определенные оператором). Другие параметры, специфичные для CR, такие как `spec.replicas`, `spec.storage.className` и т.д.

5. Нажмите **Создать**.

Веб-консоль перейдет на страницу **Приложения** > **Приложения, поддерживаемые оператором**.

:::info
**Примечание**:
Процесс создания ресурсов Kubernetes требует асинхронного согласования. Завершение может занять несколько минут в зависимости от условий кластера.
:::

## Устранение неполадок

Если создание ресурса не удалось:

1. Проверьте ошибки согласования контроллера:
   ```bash
   kubectl get events --field-selector involvedObject.kind=<Ваш-Идентификатор-Пользовательского-Ресурса> --sort-by=.metadata.creationTimestamp
   ```
2. Проверьте доступность API ресурсов:
   ```bash
   kubectl api-resources | grep <Ваш-Тип-Ресурса>
   ```
3. Повторите создание после проверки готовности CRD/оператора:
   ```bash
   kubectl apply -f ваш-манифест-ресурса.yaml
   ```
