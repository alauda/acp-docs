# Создание приложений из образа

## Предварительные условия

Получите адрес изображения. Источник изображений может быть как из репозитория изображений, интегрированного администратором платформы через цепочку инструментов, так и из репозиториев изображений сторонних платформ.

- В первом случае администратор обычно назначает репозиторий изображений для вашего проекта, и вы можете использовать изображения из него. Если требуемый репозиторий изображений не найден, пожалуйста, свяжитесь с администратором для его выделения.

- Если это репозиторий изображений сторонней платформы, убедитесь, что изображения могут быть напрямую получены из него в текущем кластере.

## Процедура 1 - Нагрузки

1. **Платформа контейнеров**, перейдите в **Приложения** > **Приложения** в левом боковом меню.

2. Нажмите **Создать**.

3. Выберите **Создать из образа** в качестве подхода к созданию.

4. **Выберите** или **Введите** изображение и нажмите **Подтвердить**.

:::info
**Примечание**: При использовании изображений из репозитория изображений, интегрированного в веб-консоль, вы можете фильтровать изображения по **Уже интегрированные**. **Имя интеграционного проекта**, например, изображения (docker-registry-projectname), что включает в себя имя проекта projectname в этой веб-консоли и имя проекта контейнеров в репозитории изображений.
:::

6. Обратитесь к следующим инструкциям для настройки связанных параметров.

## Операции управления приложениями

Чтобы изменить конфигурации приложений, используйте один из следующих методов:

1. Нажмите вертикальные троеточия (⋮) с правой стороны списка приложений.
2. Выберите **Действия** в верхнем правом углу страницы сведений приложения.

| :------------ | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Обновить**    | <ul><li>**Обновить**: Модифицирует только целевую нагрузку, используя ее определенную [стратегию обновления](#updatepolicy) (пример стратегии Развёртывания). Сохраняет существующее количество реплик и конфигурацию развертывания. </li><li>**Принудительное обновление**: Запускает полное развертывание приложения, используя стратегию обновления каждого компонента. <br />1. **Сценарии использования**: <ul><li>Пакетная конфигурация, требующая немедленной массовой пропаганды по всему кластеру (например, обновления ConfigMap/Secret, на которые ссылаются как на переменные окружения). </li><li>Координированные перезагрузки компонентов для критической безопасности. </li></ul> 2. **Предупреждение**: <ul><li>Может привести к временным сбоям в работе сервиса во время массовых перезагрузок. </li><li>Не рекомендуется для производственных сред без проверки непрерывности бизнеса.</li></ul></li><li>**Сетевые последствия**: <ul><li>Удаление правила Ingress: Внешний доступ остается доступным через `LB_IP:NodePort`, если: <br /> 1) Сервис LoadBalancer использует порты по умолчанию. <br /> 2) Сохранившиеся правила маршрутизации ссылаются на компоненты приложения. <br />Полное прекращение внешнего доступа требует удаления Сервиса.</li><li>Удаление Сервиса: Необратимая потеря сетевого подключения к компонентам приложения. Связанные правила Ingress становятся неработоспособными, несмотря на сохранение объекта API.</li></ul></li></ul> |
| **Удалить**    | <ul><li>**Каскадное удаление**: <br /> 1. Удаляет все дочерние ресурсы, включая Развёртывания, Сервисы и правила Ingress. <br /> 2. Запросы на постоянные тома (PVC) следуют правилам хранения, определенным в классе хранилища.</li><li>**Контрольный список перед удалением**: <br /> 1. Убедитесь, что нет активного трафика через связанные Сервисы. <br /> 2. Подтвердите, что резервное копирование данных для состояния завершено. <br /> 3. Проверьте взаимосвязи зависимых ресурсов с помощью `kubectl describe ownerReferences`.</li></ul>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |

### Параметры проверки состояния

#### Общие параметры

| **Параметры**        | **Описание**                                                                                                                              |
| :-------------------- | :------------------------------------------------------------------------------------------------------------------------------------------- |
| **Начальная задержка** | Период ожидания (секунды) перед началом проверок. По умолчанию: `300`.                                                                               |
| **Период**            | Интервал проверки (1-120s). По умолчанию: `60`.                                                                                                      |
| **Таймаут**           | Продолжительность тайм-аута проверки (1-300s). По умолчанию: `30`.                                                                                              |
| **Порог успеха** | Минимальное количество последовательных успешных проверок для отметки как здорового. По умолчанию: `0`.                                                                                 |
| **Порог неудачи** | Максимальное количество последовательных сбоев для выполнения действия:<br />- `0`: Отключает действия на основе сбоев<br />- По умолчанию: `5` сбоев → перезапуск контейнера. |

#### Параметры, специфичные для протокола

| **Параметр**    | **Применимые протоколы** | **Описание**                                                                                                                                              |
| :--------------- | :----------------------- | :----------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Протокол**     | HTTP/HTTPS               | Протокол проверки состояния                                                                                                                                        |
| **Порт**         | HTTP/HTTPS/TCP           | Целевой порт контейнера для проверки.                                                                                                                           |
| **Путь**         | HTTP/HTTPS               | Путь конечной точки (например, `/healthz`).                                                                                                                            |
| **HTTP Заголовки** | HTTP/HTTPS               | Пользовательские заголовки (Добавить пары ключ-значение).                                                                                                                        |
| **Команда**      | EXEC                     | Проверочная команда для контейнера (например, `sh -c "curl -I localhost:8080 \| grep OK"`).<br />**Примечание**: Экранируйте специальные символы и протестируйте работоспособность команды. |
