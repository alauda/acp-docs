# Создание приложений из изображения

## Предварительные требования

Получите адрес изображения. Источник изображений может быть из репозитория изображений, интегрированного администратором платформы через цепочку инструментов, или из репозиториев изображений сторонних платформ.

- В первом случае администратор обычно назначает репозиторий изображений вашему проекту, и вы можете использовать изображения в нем. Если необходимый репозиторий изображений не найден, пожалуйста, свяжитесь с администратором для его выделения.

- Если это репозиторий изображений сторонней платформы, убедитесь, что изображения могут быть извлечены непосредственно из него в текущем кластере.

## Процедура 1 - Нагрузки

1. В **Container Platform** перейдите в **Applications** > **Applications** в левой боковой панели.

2. Нажмите **Создать**.

3. Выберите **Создать из изображения** в качестве подхода к созданию.

4. **Выберите** или **Введите** изображение и нажмите **Подтвердить**.

:::info
**Примечание**: При использовании изображений из репозитория изображений, интегрированного в веб-консоль, вы можете фильтровать изображения по **Уже интегрировано**. **Имя интеграционного проекта**, например, изображения (docker-registry-projectname), которое включает имя проекта projectname в этой веб-консоли и имя проекта containers в репозитории изображений.
:::

6. Ознакомьтесь с приведенными ниже инструкциями для настройки связанных параметров.

   Контейнер инициализации:

   1. Запускается перед контейнерами приложения (последовательное выполнение).
   2. Освобождает ресурсы после завершения.
   3. Удаление разрешено, когда:
      - Pod имеет >1 контейнер приложения И ≥1 контейнер инициализации.
      - Не разрешено для Pod с одним контейнером приложения.

Однако убедитесь, что **Сервис** связан с нагрузкой под приложением; в противном случае обнаружение сервиса и доступ к нагрузке потерпят неудачу. |

## Операции управления приложениями

Чтобы изменить конфигурации приложения, используйте один из следующих методов:

1. Нажмите вертикальные многоточия (⋮) с правой стороны списка приложений.
2. Выберите **Действия** в правом верхнем углу страницы сведений о приложении.

| **Операция** | **Описание**                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| :------------ | :----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Обновить**    | <ul><li>**Обновить**: Изменяет только целевую нагрузку, используя ее определенную [стратегию обновления](#updatepolicy) (стратегия развертывания показана в качестве примера). Сохраняет существующее количество реплик и конфигурацию развертывания. </li><li>**Принудительное обновление**: Запускает полное развертывание приложения, используя стратегию обновления каждого компонента. <br />1. **Сценарии использования**: <ul><li>Пакетные изменения конфигурации, требующие немедленного распространения по всему кластеру (например, обновления ConfigMap/Secret, на которые ссылаются как на переменные окружения). </li><li>Согласованные перезапуски компонентов для критической безопасности. </li></ul> 2. **Предупреждение**: <ul><li>Может вызвать временное ухудшение обслуживания во время массовых перезапусков. </li><li>Не рекомендуется для производственных сред без проверки непрерывности бизнеса.</li></ul></li><li>**Сетевые последствия**: <ul><li>Удаление правила Ingress: Внешний доступ остается доступным через `LB_IP:NodePort`, если: <br /> 1) Сервис LoadBalancer использует порты по умолчанию. <br /> 2) Сохранившиеся правила маршрутизации ссылаются на компоненты приложения. <br />Полное прекращение внешнего доступа требует удаления сервиса.</li><li>Удаление сервиса: Необратимая потеря сетевой связи с компонентами приложения. Связанные правила Ingress становятся неработоспособными, несмотря на сохранение объекта API.</li></ul></li></ul> |
| **Удалить**    | <ul><li>**Каскадное удаление**: <br /> 1. Удаляет все дочерние ресурсы, включая развертывания, сервисы и правила Ingress. <br /> 2. Запросы на постоянные тома (PVC) следуют политике хранения, определенной в StorageClass</li><li>**Контрольный список перед удалением**: <br /> 1. Убедитесь, что нет активного трафика через связанные сервисы. <br /> 2. Подтвердите завершение резервного копирования данных для состоящих компонентов. <br /> 3. Проверьте зависимости ресурсов с помощью `kubectl describe ownerReferences`.</li></ul>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |

<a id="healthcheck" />

### Параметры проверки состояния

#### Общие параметры

| **Параметры**        | **Описание**                                                                                                                              |
| :-------------------- | :------------------------------------------------------------------------------------------------------------------------------------------- |
| **Начальная задержка**     | Период ожидания (в секундах) перед началом проверок. По умолчанию: `300`.                                                                               |
| **Период**            | Интервал проверки (1-120с). По умолчанию: `60`.                                                                                                      |
| **Тайм-аут**           | Продолжительность тайм-аута проверки (1-300с). По умолчанию: `30`.                                                                                              |
| **Порог успеха** | Минимальное количество последовательных успешных проверок для обозначения состояния как здорового. По умолчанию: `0`.                                                                                 |
| **Порог неудачи** | Максимальное количество последовательных неудач для инициирования действия:<br />- `0`: Отключает действия на основе неудач<br />- По умолчанию: `5` неудач → перезапуск контейнера. |

#### Параметры, специфичные для протокола

| **Параметр**    | **Применимые протоколы** | **Описание**                                                                                                                                              |
| :--------------- | :----------------------- | :----------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Протокол**     | HTTP/HTTPS               | Протокол проверки состояния                                                                                                                                        |
| **Порт**         | HTTP/HTTPS/TCP           | Целевой порт контейнера для проверки.                                                                                                                           |
| **Путь**         | HTTP/HTTPS               | Путь конечной точки (например, `/healthz`).                                                                                                                            |
| **HTTP-заголовки** | HTTP/HTTPS               | Пользовательские заголовки (добавьте пары ключ-значение).                                                                                                                        |
| **Команда**      | EXEC                     | Команда проверки, выполняемая контейнером (например, `sh -c "curl -I localhost:8080 \| grep OK"`).<br />**Примечание**: Экранируйте специальные символы и проверьте жизнеспособность команды. |
