# Создание приложений с использованием CLI

`kubectl` является основным интерфейсом командной строки (CLI) для взаимодействия с кластерами Kubernetes. Он функционирует как клиент для API-сервера Kubernetes - RESTful HTTP API, который служит программным интерфейсом управляющей подсистемы. Все операции Kubernetes выполняются через API-точки, и `kubectl` по сути переводит команды CLI в соответствующие API-запросы для управления ресурсами кластера и рабочими нагрузками приложений (деплойментами, StatefulSets и т. д.).

Инструменты CLI упрощают развертывание приложений, интеллектуально интерпретируя входные артефакты (образы, чарты и т. д.) и генерируя соответствующие объекты API Kubernetes. Созданные ресурсы варьируются в зависимости от типов входных данных:

- **Образ**: Прямо создает деплоймент.
- **Чарт**: Создает все объекты, определенные в Helm Chart.

## Предварительные требования

Плагин **ACP Web Terminal** установлен, и переключатель web-cli включен.

## Процедура

1. В **Контейнерной платформе** нажмите на значок терминала в нижнем правом углу.

2. Подождите инициализации сессии (1-3 сек).

3. Выполните команды kubectl в интерактивной оболочке:

```bash
   kubectl get pods -n ${CURRENT_NAMESPACE}
```

4. Просмотрите вывод команд в реальном времени.

## Пример

### YAML

```yaml
# webapp.yaml
apiVersion: app.k8s.io/v1beta1
kind: Application
metadata:
  name: webapp
spec:
  componentKinds:
    - group: apps
      kind: Deployment
    - group: ""
      kind: Service
  descriptor: {}

# webapp-deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: webapp
  labels:
    app: webapp
    env: prod
spec:
  replicas: 3
  selector:
    matchLabels:
      app: webapp
  template:
    metadata:
      labels:
        app: webapp
        tier: frontend
    spec:
      containers:
      - name: webapp
        image: nginx:1.25-alpine
        ports:
        - containerPort: 80
        resources:
          requests:
            cpu: "100m"
            memory: "128Mi"
          limits:
            cpu: "250m"
            memory: "256Mi"
---
# webapp-service.yaml
apiVersion: v1
kind: Service
metadata:
  name: webapp-service
spec:
  selector:
    app: webapp
  ports:
    - protocol: TCP
      port: 80
      targetPort: 80
  type: ClusterIP
```

### команды kubectl

```bash
kubectl apply -f webapp.yaml -n {CURRENT_NAMESPACE}
kubectl apply -f webapp-deployment.yaml -n {CURRENT_NAMESPACE}
kubectl apply -f webapp-service.yaml -n {CURRENT_NAMESPACE}
```
