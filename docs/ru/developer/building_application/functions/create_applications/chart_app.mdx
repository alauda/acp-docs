# Создание приложений с использованием Chart

Базируясь на Helm Chart, представляет собой нативный паттерн развертывания приложения. Helm Chart — это сборник файлов, которые определяют ресурсы Kubernetes, предназначенные для упаковки приложений и упрощения их распространения с возможностями контроля версий. Это обеспечивает бесшовный переход между окружениями, такими как миграции из разработческого в производственное окружение.

## Предостережения

Когда кластер содержит как узлы Linux, так и Windows, необходимо явно настроить выбор узлов, чтобы предотвратить конфликты планирования. Пример:

```
spec:
    spec:
      nodeSelector:
        kubernetes.io/os: linux
```

## Предварительные требования

Если шаблон исходит от приложения и ссылается на соответствующие ресурсы (например, секретные словари), убедитесь, что ресурсы, на которые следует сослаться, уже существуют в текущем пространстве имен до развертывания приложения.

## Процедура

1. **Платформа контейнеров**, перейдите в **Приложения** > **Приложения** в левом боковом меню.

2. Нажмите **Создать**.

3. Выберите **Создать из каталога** в качестве способа создания.

4. Выберите Chart и настройте параметры, такие как `resources.requests`, `resources.limits` и другие параметры, которые тесно связаны с графиком.

5. Нажмите **Создать**.

Веб-консоль перенаправит вас на страницу сведений **Приложение** > \[**Нативные приложения**]. Процесс займёт некоторое время, пожалуйста, набирайтесь терпения. В случае сбоя операции следуйте указаниям интерфейса для завершения операции.

## Справочник по анализу статуса

Нажмите на ***Название приложения***, чтобы отобразить подробный анализ статуса Chart в информации о деталях.

| **Тип**         | **Причина**                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| :-------------- | :----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Инициализировано** | Указывает на статус загрузки шаблона Chart. <ul><li>**True**: это означает, что шаблон Chart был успешно загружен.</li><li>**False**: это означает, что загрузка шаблона Chart не удалась; вы можете проверить конкретную причину сбоя в колонке сообщений.</li><ul><li>`ChartLoadFailed`: Загрузка шаблона Chart не удалась.</li><li>`InitializeFailed`: Произошло исключение в процессе инициализации перед загрузкой Chart.</li></ul></ul>                                                                                                                                                                                                                                        |
| **Проверен**    | Указывает на статус прав пользователя, зависимостей и других проверок для шаблона Chart. <ul><li>**True**: это означает, что все проверки валидности прошли.</li><li>**False**: это означает, что проверки валидности не прошли; вы можете проверить конкретную причину сбоя в колонке сообщений.</li><ul><li>`DependenciesCheckFailed`: Проверка зависимостей Chart не удалась.</li><li>`PermissionCheckFailed`: У текущего пользователя отсутствуют разрешения на выполнение операций с определенными ресурсами.</li><li>`ConsistentNamespaceCheckFailed`: При развертывании приложений через шаблоны в нативных приложениях Chart содержит ресурсы, которые требуют развертывания через несколько пространств имен.</li></ul></ul> |
| **Синхронизировано** | Указывает на статус развертывания шаблона Chart. <ul><li>**True**: это означает, что шаблон Chart был успешно развернут.</li><li>**False**: это означает, что развертывание шаблона Chart не удалось; в колонке причин отображается `ChartSyncFailed`, и вы можете проверить конкретную причину сбоя в колонке сообщений.</li></ul>                                                                                                                                                                                                                                                                                                                                                                         |

:::warning

- Если шаблон ссылается на ресурсы из различных пространств имен, обратитесь к Администратору за помощью в создании. После этого вы можете нормально [обновлять или удалять приложения](../application_operation/chart_upgrade.mdx) в веб-консоли.
- Если шаблон ссылается на ресурсы уровня кластера (например, StorageClasses), рекомендуется обратиться к Администратору за помощью в создании.
:::
