# Создание приложений с использованием Chart

На основе Helm Chart представляет собой шаблон развертывания нативных приложений. Helm Chart — это коллекция файлов, которые определяют ресурсы Kubernetes, предназначенные для упаковки приложений и упрощения распространения приложений с функциями контроля версий. Это обеспечивает бесшовные переходы между средами, такими как миграции из среды разработки в производственную.

## Предостережения

Когда кластер содержит как Linux, так и Windows узлы, явный выбор узлов ДОЛЖЕН быть настроен, чтобы избежать конфликтов при планировании. Пример:

```
spec:
    spec:
      nodeSelector:
        kubernetes.io/os: linux
```

## Предварительные требования

Если шаблон взят из приложения и ссылается на соответствующие ресурсы (например, секретные словари), убедитесь, что ссылаемые ресурсы уже существуют в текущем пространстве имен перед развертыванием приложения.

## Процедура

1. **Контейнерная платформа**, перейдите в **Приложения** > **Приложения** в левой боковой панели.

2. Нажмите **Создать**.

3. Выберите **Создать из каталога** в качестве подхода к созданию.

4. Выберите Chart и настройте параметры, выберите Chart и настройте необходимые параметры, такие как `resources.requests`, `resources.limits` и другие параметры, которые тесно связаны с chart.

5. Нажмите **Создать**.

Веб-консоль перенаправит вас на страницу деталей **Приложение** > \[**Нативные приложения**]. Процесс займет некоторое время, поэтому, пожалуйста, наберитесь терпения. В случае сбоя операции следуйте подсказкам интерфейса, чтобы завершить операцию.

## Справочник по анализу состояния

Нажмите на ***Имя приложения***, чтобы отобразить подробный анализ состояния Chart в информационной таблице.

| **Тип**         | **Причина**                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| :-------------- | :----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Инициализировано** | Указывает на статус загрузки шаблона Chart. <ul><li>**True**: Указывает на то, что шаблон Chart был успешно загружен.</li><li>**False**: Указывает на то, что загрузка шаблона Chart не удалась; вы можете проверить конкретную причину сбоя в столбце сообщения.</li><ul><li>`ChartLoadFailed`: Загрузка шаблона Chart не удалась.</li><li>`InitializeFailed`: Произошло исключение в процессе инициализации до загрузки Chart.</li></ul></ul>                                                                                                                                                                                                                                        |
| **Проверено**   | Указывает на статус прав доступа пользователя, зависимостей и других проверок для шаблона Chart. <ul><li>**True**: Указывает на то, что все проверки валидации прошли успешно.</li><li>**False**: Указывает на то, что имеются проверки валидации, которые не прошли; вы можете проверить конкретную причину сбоя в столбце сообщения.</li><ul><li>`DependenciesCheckFailed`: Проверка зависимостей Chart не удалась.</li><li>`PermissionCheckFailed`: У текущего пользователя недостаточно прав для выполнения операций с определенными ресурсами.</li><li>`ConsistentNamespaceCheckFailed`: При развертывании приложений через шаблоны в нативных приложениях Chart содержит ресурсы, требующие развертывания между пространствами имен.</li></ul></ul> |
| **Синхронизировано**      | Указывает на статус развертывания шаблона Chart. <ul><li>**True**: Указывает на то, что шаблон Chart был успешно развернут.</li><li>**False**: Указывает на то, что развертывание шаблона Chart не удалось; в столбце с причиной указано `ChartSyncFailed`, и вы можете проверить конкретную причину сбоя в столбце сообщения.</li></ul>                                                                                                                                                                                                                                                                                                                                                                         |

:::warning

- Если шаблон ссылается на ресурсы между пространствами имен, обратитесь к администратору за помощью в создании. После этого вы можете обычно [обновлять или удалять приложения](../application_operation/chart_upgrade.mdx) в веб-консоли.
- Если шаблон ссылается на ресурсы уровня кластера (например, StorageClasses), рекомендуется обратиться к администратору за помощью в создании.
  :::
