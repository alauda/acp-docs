# Создание Jobs

Job предоставляет различные способы определения задач, которые выполняются до завершения, а затем останавливаются. Вы можете использовать Job для определения задачи, которая выполняется до завершения, всего один раз.

- **Атомарная единица выполнения**: Каждый Job управляет одним или несколькими Pods до успешного завершения.
- **Механизм повторных попыток**: Управляется `spec.backoffLimit` (по умолчанию: 6).
- **Отслеживание завершенности**: Используйте `spec.completions` для определения необходимого количества успешных завершений.

## Пример

```yaml
apiVersion: batch/v1
kind: Job
metadata:
  name: data-processing-job
spec:
  completions: 1          # Количество необходимых успешных завершений
  parallelism: 1          # Максимальное количество параллельных Pods
  backoffLimit: 3         # Максимальное количество попыток повторного выполнения
  template:
    spec:
      restartPolicy: Never # Политика конкретного Job (Never/OnFailure)
      containers:
      - name: processor
        image: alpine:3.14
        command: ["/bin/sh", "-c"]
        args:
          - echo "Обработка данных..."; sleep 30; echo "Работа завершена"
```

## Обзор выполнения

Каждое выполнение Job в Kubernetes создает отдельный объект Job, позволяя пользователям:

- Отслеживать жизненный цикл задания с помощью `kubectl get jobs`
- Просматривать детали выполнения через `kubectl describe job/<job-name>`
- Смотреть логи Pod с помощью `kubectl logs <pod-name>`
