# Создание Jobs

Job предоставляет различные способы определения задач, которые выполняются до завершения, а затем останавливаются. Вы можете использовать Job для определения задачи, которая выполняется до завершения только один раз.

- **Атомарная единица выполнения**: Каждый Job управляет одним или несколькими Pod'ами до успешного завершения.
- **Механизм повторных попыток**: Контролируется с помощью `spec.backoffLimit` (по умолчанию: 6).
- **Отслеживание завершения**: Используйте `spec.completions` для определения необходимого количества успешных завершений.

## Пример

```yaml
apiVersion: batch/v1
kind: Job
metadata:
  name: data-processing-job
spec:
  completions: 1          # Количество необходимых успешных завершений
  parallelism: 1          # Максимальное количество параллельных Pods
  backoffLimit: 3         # Максимальное количество попыток повторения
  template:
    spec:
      restartPolicy: Never # Политика, специфичная для Job (Never/OnFailure)
      containers:
      - name: processor
        image: alpine:3.14
        command: ["/bin/sh", "-c"]
        args:
          - echo "Обработка данных..."; sleep 30; echo "Job завершен"
```

## Обзор выполнения

Каждое выполнение Job в Kubernetes создает выделенный объект Job, позволяя пользователям:

- Отслеживать жизненный цикл задания через `kubectl get jobs`
- Просмотреть детали выполнения с помощью `kubectl describe job/<job-name>`
- Посмотреть логи Pod'ов с использованием `kubectl logs <pod-name>`
