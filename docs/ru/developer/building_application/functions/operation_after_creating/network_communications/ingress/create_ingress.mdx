# Создание Ingress'ов

Создайте Ingress для управления внешним доступом HTTP/HTTPS к Сервису.

:::warning
При создании нескольких ingress'ов в одном пространстве имен разные ingress'ы **НЕ ДОЛЖНЫ** иметь одинаковые
**Домен**, **Протокол** и **Путь** (то есть дубликаты точек доступа не допускаются).
:::

## Предварительные требования

- В текущем пространстве имен должен быть доступен **Сервис**.

- Пожалуйста, подтвердите с администратором, что для проекта, связанного с текущим пространством имен, выделено используемое доменное имя.

- Чтобы получить доступ к домену через HTTPS, вам сначала нужно сохранить сертификат HTTPS в качестве TLS-секрета.

## Шаги

1. Получите доступ к **Платформе Контейнеров**.

2. В левой навигационной панели нажмите **Сеть** > **Ingress**.

3. Нажмите **Создать Ingress**.

4. Сообщите о инструкциях ниже для настройки некоторых параметров.

   | Параметр                   | Описание                                                                                                                                                                                                                                                                                                               |
   | --------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | **Класс Ingress**          | Ingress'ы могут реализовываться различными контроллерами с различными именами `IngressClass`. Если на платформе доступно несколько контроллеров ingress, пользователь может выбрать, какой из них использовать с этой опцией.                                                                                                           |
   | **Доменное имя**           | Хосты могут быть точными совпадениями (например, `foo.bar.com`) или подстановочными символами (например, `*.foo.com`). Доменные имена выделяются администратором платформы.                                                                                                                                                     |
   | **Сертификаты**            | TLS-секрет или сертификаты, выделенные администратором платформы.                                                                                                                                                                                                                                                           |
   | **Тип сопоставления** и **Путь** | <ul><li>**Префикс**: Сопоставляет префиксы путей, например, `/abcd` может совпадать с `/abcd/efg` или `/abcde`.</li><li>**Точно**: Сопоставляет точные пути, например, `/abcd`.</li><li>**Специфично для реализации**: Если вы используете пользовательский контроллер Ingress для управления правилами Ingress, вы можете выбрать, чтобы контроллер решил.</li></ul> |
   | **Сервис**                 | Внешний трафик будет перенаправлен на этот Сервис.                                                                                                                                                                                                                                                                       |
   | **Порт сервиса**           | Укажите, на какой порт Сервиса будет перенаправлен трафик.                                                                                                                                                                                                                                                              |

5. Нажмите **Создать**.

:::note
Администратор платформы может выделять экземпляры ALB проектам.
Если у ingress нет класса Ingress, все экземпляры ALB, выделенные для этого проекта, будут обрабатывать этот ingress.
:::
