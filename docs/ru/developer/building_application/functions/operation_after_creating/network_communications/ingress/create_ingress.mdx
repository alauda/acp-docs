# Создание Ingress

Создайте Ingress, чтобы управлять внешним доступом HTTP/HTTPS к Сервису.

:::warning
При создании нескольких Ingress в одном пространстве имен различные Ingress **ДОЛЖНЫ** иметь разные
**Домены**, **Протоколы** и **Пути** (т.е. дублирование точек доступа не допускается).
:::

## Предварительные требования

- В текущем пространстве имен должен быть доступен **Сервис**.

- Пожалуйста, подтвердите с администратором, что для проекта, связанного с текущим пространством имен, выделено пригодное доменное имя.

- Чтобы получить доступ к домену через HTTPS, вам сначала необходимо сохранить сертификат HTTPS в виде TLS-секрета.

## Шаги

1. Получите доступ к **Платформе Контейнеров**.

2. В левой панели навигации нажмите **Сеть** > **Ingress**.

3. Нажмите **Создать Ingress**.

4. Ознакомьтесь с приведенными ниже инструкциями для настройки определенных параметров.

   | Параметр                   | Описание                                                                                                                                                                                                                                                                                                               |
   | --------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | **Ingress Class**           | Ingress могут реализовываться различными контроллерами с разными именами `IngressClass`. Если на платформе доступно несколько контроллеров Ingress, пользователь может выбрать, какой из них использовать с этой опцией.                                                                                                           |
   | **Доменное имя**             | Хосты могут быть точными совпадениями (например, `foo.bar.com`) или использовать подстановочный знак (например, `*.foo.com`). Доменные имена выделяются администратором платформы.                                                                                                                                                     |
   | **Сертификаты**            | TLS-секрет или сертификаты, выделенные администратором платформы.                                                                                                                                                                                                                                                           |
   | **Тип совпадения** и **Путь** | <ul><li>**Префикс**: Соответствует префиксам путей, например, `/abcd` может соответствовать `/abcd/efg` или `/abcde`.</li><li>**Точный**: Соответствует точным путям, например, `/abcd`.</li><li>**Специфичный для реализации**: Если вы используете пользовательский контроллер Ingress для управления правилами Ingress, вы можете выбрать, чтобы контроллер решал это.</li></ul> |
   | **Сервис**                 | Внешний трафик будет перенаправлен на этот Сервис.                                                                                                                                                                                                                                                                       |
   | **Порт сервиса**            | Укажите, на какой порт Сервиса будет перенаправлен трафик.                                                                                                                                                                                                                                                              |

5. Нажмите **Создать**.

:::note
Администратор платформы может выделять экземпляры ALB проектам.
Если у Ingress нет класса Ingress, все экземпляры ALB, выделенные для этого проекта, будут обрабатывать этот Ingress.
:::
