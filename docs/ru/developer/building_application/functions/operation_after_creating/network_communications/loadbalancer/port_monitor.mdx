# Мониторинг портов

Когда протокол прослушивающего порта - это HTTPS или HTTP, вы можете просматривать данные мониторинга для входящего/исходящего трафика, запросов и ответов.

**Примечание**: Данные мониторинга для протокола gRPC не поддерживаются.

## Шаги для выполнения

1. Перейдите в **Платформа контейнеров**.

2. В левой панели навигации нажмите **Сеть** > **Балансировка нагрузки**.

3. Щелкните по имени балансировщика нагрузки.

4. Щелкните по имени прослушиваемого порта.

5. В разделе **Мониторинг** просмотрите данные мониторинга для порта.

   Формула для расчета данных: `Статистическое значение данных = Общее количество данных за период статистического времени / Период статистического времени`. Значение трафика будет преобразовано в точное значение в зависимости от установленной единицы; если значения запросов и ответов имеют десятичные знаки, они будут округлены, а десятичная часть будет игнорироваться, единица измерения - `единиц/секунда`.

   Например: Если период статистического времени в определенный момент составляет 4 часа, и общее количество запросов и ответов с кодом состояния 5xx за эти 4 часа составляет 10003, то статистическое значение для кода состояния 5xx в этот момент времени = 10003 / 4*60*60, округленное статистическое значение составляет 7 единиц/секунда.

   | Временной диапазон   | Интервал временной оси | Период статистического времени |
   | --------------------- | ---------------------- | ------------------------------- |
   | **Последние 15 минут** | 2 минуты              | 30 секунд                      |
   | **Последний 1 час**    | 5 минут               | 1 минута                      |
   | **Последний 1 день**   | 3 часа                | 30 минут                      |
   | **Последние 7 дней**   | 1 день                | 4 часа                        |
   | **Последние 30 дней**  | 2 дня                 | 12 часов                      |
