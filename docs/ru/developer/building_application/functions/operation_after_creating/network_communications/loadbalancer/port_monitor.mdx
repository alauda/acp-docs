# Мониторинг портов

Когда протокол прослушивающего порта является HTTPS или HTTP, вы можете просматривать данные мониторинга для входящего/исходящего трафика, запросов и ответов.

**Примечание**: Данные мониторинга для протокола gRPC не поддерживаются.

## Шаги для работы

1. Перейдите в **Container Platform**.

2. В левом навигационном меню нажмите **Network** > **Load Balancing**.

3. Щелкните по имени балансировщика нагрузки.

4. Щелкните по имени прослушивающего порта.

5. В разделе **Monitoring** просмотрите данные мониторинга для порта.

   Формула для расчета данных: `Статистическое значение данных = Общий объем данных за период статистики / Период статистики`. Значение трафика будет преобразовано в точное значение на основе установленной единицы измерения; если значения запросов и ответов содержат десятичные дроби, они будут округлены, а десятичная часть будет проигнорирована, единица измерения - `единицы/секунда`.

   Например: если период статистики в определенный момент времени составляет 4 часа, и общее количество запросов и ответов со статусом 5xx за эти 4 часа составляет 10003, то статистическое значение для статус-кода 5xx в этот момент времени = 10003 / 4*60*60, округленное статистическое значение составляет 7 единиц/секунда.

   | Период времени       | Интервал по оси времени | Период статистики          |
   | -------------------- | ----------------------- | --------------------------- |
   | **Последние 15 минут** | 2 минуты               | 30 секунд                   |
   | **Последний 1 час**   | 5 минут                 | 1 минута                   |
   | **Последний 1 день**   | 3 часа                  | 30 минут                   |
   | **Последние 7 дней**   | 1 день                  | 4 часа                     |
   | **Последние 30 дней**  | 2 дня                   | 12 часов                   |
