# Настройка VPA

Независимо от того, являются ли приложения без состояния или с состоянием, VPA (Вертикальный Автоскейлер Подов) поддерживает рекомендации по более подходящим ограничениям ресурсов CPU и памяти, основываясь на ваших бизнес-потребностях, помогая вам настраивать ресурсы подов и обеспечивая поды достаточными ресурсами для повышения Utilization кластерных ресурсов.

## Предварительные требования

- Пожалуйста, убедитесь, что в текущем кластере развернуты компоненты мониторинга и что компоненты мониторинга работают корректно. Вы можете проверить статус развертывания и состояния компонентов мониторинга, нажав в верхнем правом углу платформы <span style={{ display: 'inline-flex', alignItems: 'center', gap: '4px', verticalAlign: 'middle' }}><img src="../../../../../en/developer/building_application/functions/operation_after_creating/assets/question.png" alt="expand" style={{ verticalAlign: 'middle' }} /> <span>  > **Состояние Здоровья Платформы**, чтобы просмотреть развертывание и состояние компонентов мониторинга.</span></span>.
- Пожалуйста, убедитесь, что администратор развернул плагин **Вертикального Автоскейлинга** для текущего кластера.

## Шаги для выполнения

1. В левой панели навигации нажмите **Рабочие Нагрузки** > **Развертывания**.

2. Нажмите на ***Имя Развертывания***.

3. Прокрутите вниз до области **Эластичное Масштабирование** и нажмите **Обновить** справа.

4. Выберите **Вертикальное Масштабирование** и настройте правила масштабирования.

   | Параметр            | Описание                                                                                                                                                                                                                                                                                                                                                                                                                                            |
   | -------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
   | **Режим Масштабирования**     | В настоящее время поддерживается режим **Ручного Масштабирования**, который предоставляет рекомендуемые конфигурации ресурсов, анализируя прошлое использование ресурсов. Вы можете вручную настроить значения в соответствии с рекомендованными. Изменения приведут к пересозданию и перезапуску подов, поэтому пожалуйста, выберите подходящее время, чтобы избежать влияния на работающие приложения.<br />Как правило, после того, как поды работают более 8 дней, рекомендованные значения станут точными. |
   | **Целевой Контейнер** | По умолчанию выбирается первый контейнер рабочей нагрузки. Вы можете выбрать, чтобы включить рекомендации по ограничениям ресурсов для одного или нескольких контейнеров по мере необходимости.                                                                                                                                                                                                                                                                                                         |

5. Нажмите **Обновить**.

## Дальнейшие действия

После настройки рекомендованные значения для ограничений ресурсов CPU и памяти целевого контейнера можно просмотреть в области **Эластичного Масштабирования**. В области **Контейнеры** выберите вкладку целевого контейнера и нажмите на значок справа от **Ограничения Ресурсов**, чтобы обновить ограничения ресурсов в соответствии с рекомендованными значениями.
