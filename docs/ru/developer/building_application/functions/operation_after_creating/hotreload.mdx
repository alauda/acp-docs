# Загрузка конфигурации ConfigMap

После обновления смонтированного ConfigMap приложение должно загрузить обновленную конфигурацию, чтобы она вступила в силу. Для приложений, которые не могут автоматически загружать конфигурацию, **Загрузить конфигурацию** предоставляет возможность вручную инициировать загрузку ConfigMap приложением.

## Предварительные условия

- Приложение должно поддерживать горячие обновления конфигурации.

- ConfigMap должен быть смонтирован в качестве хранилища тома внутри пода, и не может быть использован SubPath. ConfigMaps, на которые ссылаются через переменные окружения, не поддерживают загрузку конфигурации.

## Шаги

1. Перейдите в **Платформу контейнеров**.

2. В левом навигационном меню нажмите **Конфигурация** > **ConfigMaps**.

3. Нажмите на ***Имя ConfigMap***.

4. Нажмите **Добавить элемент конфигурации** или нажмите **Добавить бинарный элемент конфигурации**; также при необходимости можно нажать <span style={{ display: 'inline-flex', alignItems: 'center', gap: '4px', verticalAlign: 'middle' }}><img src="../../../../../en/developer/building_application/functions/operation_after_creating/assets/blackpencil.png" alt="expand" style={{ verticalAlign: 'middle' }} /> <span>Обновить элемент конфигурации и подтвердить.</span></span>.

5. Переключитесь на вкладку **Рабочие нагрузки** и нажмите <span style={{ display: 'inline-flex', alignItems: 'center', gap: '4px', verticalAlign: 'middle' }}><img src="../../../../../en/developer/building_application/functions/operation_after_creating/assets/fold.png" alt="expand" style={{ verticalAlign: 'middle' }} /> <span>Развернуть список его контейнеров.</span></span>.

6. Нажмите **Загрузить конфигурацию**, введите пользователя и выполните команду, затем подтвердите. По умолчанию выполняется вход в контейнер под пользователем **root**.

7. Введите соответствующую команду для загрузки конфигурации в зависимости от типа приложения.

   - Пример 1: Nginx

     ```
     nginx -s reload
     ```

   - Пример 2: Logstash

     ```
     kill -SIGHUP <ИД процесса Logstash>
     ```
