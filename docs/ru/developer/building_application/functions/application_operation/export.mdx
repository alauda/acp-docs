# Экспорт нативных приложений

Для стандартизации процесса экспорта приложений между средами разработки, тестирования и производства, а также для ускорения миграции бизнеса в новые среды, вы можете экспортировать нативные приложения в качестве шаблонов приложений (Charts) или экспортировать упрощенные YAML-файлы, которые можно использовать непосредственно для развертывания. Это позволяет нативному приложению работать в различных средах или пространствах имен. Вы также можете экспортировать YAML-файлы в репозиторий кода для быстрого развертывания приложений по кластерам с использованием функционала GitOps.

## Экспорт Helm Charts

### Шаги

1. Перейдите в **Container Platform**.

2. В левой навигационной панели щелкните на **Application Management** > **Native Applications**.

3. Щелкните на ***имя приложения*** типа `Custom Application`.

4. Щелкните на **Actions** > **Export**; вы также можете экспортировать конкретную версию с страницы сведений о приложении.

5. Выберите один из методов экспорта по мере необходимости и обратитесь к следующим инструкциям для настройки соответствующей информации.

   - Экспорт Helm Charts в репозиторий шаблонов с правами управления

     **Примечание**: Репозиторий шаблонов добавляется администратором платформы. Пожалуйста, свяжитесь с администратором платформы, чтобы получить действительный репозиторий шаблонов типа **Chart** или **OCI Chart** с правами **Management**.

     | Параметр                 | Описание                                                                                                                                                                                                                                                                                                       |
     | ------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
     | **Целевая локация**      | Выберите **Template Repository**, чтобы непосредственно синхронизировать шаблон с репозиторием шаблонов типа **Chart** или **OCI Chart** с правами **Management**. Владелец проекта, назначенный этому **Template Repository**, может непосредственно использовать этот шаблон.                             |
     | **Каталог шаблонов** | Когда выбранный тип репозитория шаблонов - OCI Chart, вам необходимо выбрать или ввести вручную каталог для хранения Helm Chart.<br />**Примечание**: При ручном вводе нового каталога шаблона платформа создаст этот каталог в репозитории шаблонов, но существует риск, что создание может не удастся. |
     | **Версия**              | Номер версии шаблона приложения.<br />Формат должен быть `v<Major>.<Minor>.<Patch>`. Значение по умолчанию - текущая версия приложения или текущая версия снимка.                                                                                                                                           |
     | **Иконка**              | Поддерживает форматы изображений JPG, PNG и GIF, размер файла не более 500KB. Рекомендуемые размеры - 80*60 пикселей.                                                                                                                                                                                       |
     | **Описание**            | Описание будет отображено в списке шаблонов приложений в каталоге приложений.                                                                                                                                                                                                                               |
     | **README**              | Файл описания. Поддерживает редактирование в формате Markdown и будет отображаться на странице сведений о шаблоне приложения.                                                                                                                                                                               |
     | **ЗАМЕТКИ**             | Файл помощи шаблона. Поддерживает стандартное редактирование простого текста; после завершения развертывания шаблона он будет отображаться на странице сведений о шаблоне приложения.                                                                                                                      |

   - Экспорт Helm Charts на локальный компьютер для ручной загрузки в репозиторий шаблонов: выберите **Local** в качестве целевой локации и выберите **Helm Chart** в качестве формата файла для генерации пакета Helm Chart, который будет загружен локально для оффлайн-передачи.

6. Щелкните **Export**.

### Дальнейшие действия

- Если вы экспортируете Helm Chart на локальный, вам нужно будет [добавить шаблон в репозиторий шаблонов с правами управления](../helm_chart/manage_helmchart.mdx).

- Независимо от выбранного метода экспорта, вы можете обратиться к [Создание нативных приложений - Метод шаблона](../create_applications/chart_app.mdx), чтобы создать нативное приложение типа `Template Application` в **некорректном** пространстве имен.

## Экспорт YAML на локальный

### Шаги

#### **Метод 1**

1. Перейдите в **Container Platform**.

2. В левой навигационной панели щелкните на **Application Management** > **Native Applications**.

3. Щелкните на ***имя приложения***.

4. Щелкните на **Actions** > **Export**; вы также можете экспортировать конкретную версию с страницы сведений о приложении.

5. Выберите **Local** в качестве целевой локации и **YAML** в качестве формата файла, после чего вы сможете экспортировать упрощенный YAML-файл, который можно развернуть непосредственно в других средах.

6. Щелкните **Export**.

#### **Метод 2**

1. Перейдите в **Container Platform**.

2. В левой навигационной панели щелкните на **Application Management** > **Native Applications**.

3. Щелкните на ***имя приложения***.

4. Щелкните на вкладку **YAML**, настройте параметры по мере необходимости и просмотрите YAML-файл.

   | Тип                  | Описание                                                                                                                                                                                                                                                                                                                                   |
   | --------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | **Полный YAML**       | По умолчанию **Предварительный просмотр упрощенного YAML** не выбран, отображая YAML-файл с **скрытыми полями managedFields**. Вы можете просмотреть его и экспортировать напрямую; вы также можете отменить выбор **Скрыть поля managedFields**, чтобы экспортировать полный YAML-файл.<br />**Примечание**: Полный YAML в основном используется для операций и устранения неполадок и не может быть использован для быстрой настройки нативных приложений на платформе. |
   | **Упрощенный YAML**   | Выберите **Предварительный просмотр упрощенного YAML**, после чего вы сможете просмотреть и экспортировать упрощенный YAML-файл, который можно развернуть непосредственно в других средах.                                                                                                                                                                                              |

5. Щелкните **Export**.

### Дальнейшие действия

После экспорта упрощенного YAML вы можете обратиться к [Создание нативных приложений - Метод YAML](../create_applications/yaml_app.mdx), чтобы создать нативное приложение типа `Custom Application` в **некорректном** пространстве имен.

## Экспорт YAML в репозиторий кода (Alpha)

### Предостережения

- Только администраторы платформы и администраторы проектов могут непосредственно экспортировать YAML-файлы нативных приложений в репозиторий кода.

- `Template Applications` не поддерживают экспорт файлов конфигурации приложений, отформатированных Kustomize, или прямой экспорт YAML-файлов в репозиторий кода; сначала вы можете **отключиться от шаблона** и конвертировать его в `Custom Application`.

### Шаги

1. Перейдите в **Container Platform**.

2. В левой навигационной панели щелкните на **Application Management** > **Native Applications**.

3. Щелкните на ***имя приложения*** типа `Custom`.

4. Щелкните на **Actions** > **Export**; вы также можете экспортировать конкретную версию с страницы сведений о приложении.

5. Выберите один из методов экспорта по мере необходимости и обратитесь к следующим инструкциям для настройки соответствующей информации.

   - Экспорт YAML в репозиторий кода:

     | Параметр                    | Описание                                                                                                                                                                                                                                                                                                                                   |
     | ---------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
     | **Целевая локация**          | Выберите **Code Repository**, чтобы напрямую синхронизировать YAML-файл в указанный Git репозиторий кода. Владелец проекта, назначенный этому **Code Repository**, может непосредственно использовать YAML-файл.                                                                                                                                                                       |
     | **Название интеграционного проекта** | Название проекта инструмента интеграции, назначенного или связанного с вашим проектом администратором платформы.                                                                                                                                                                                                                                                                                   |
     | **Адрес репозитория**       | Адрес репозитория, назначенный для вашего использования в рамках интегрированного проекта инструмента.                                                                                                                                                                                                                                                                                             |
     | **Метод экспорта**            | <ul><li>**Существующая ветка**: экспортируйте YAML приложения в выбранную ветку.</li><li>**Новая ветка**: создайте новую ветку на основе выбранного **Branch/Tag/Commit ID** и экспортируйте YAML приложения в новую ветку.</li><ul><li>Если **Подать PR (Запрос на изменение)** отмечено, платформа создаст новую ветку и подаст Запрос на изменение.</li><li>Если **Автоматически удалить исходную ветку после слияния PR** отмечено, исходная ветка будет автоматически удалена после того, как вы сольете PR в репозитории кода Git.</li></ul></ul> |
     | **Путь к файлу**              | Конкретное место, где файл должен быть сохранен в репозитории кода; вы также можете ввести путь к файлу, и платформа создаст новый путь в репозитории кода на основе введенного.                                                                                                                                                                                                                       |
     | **Сообщение о коммите**       | Заполните информацию о коммите для идентификации содержания этого представления.                                                                                                                                                                                                                                                                                                               |
     | **Предварительный просмотр**                  | Просмотрите YAML-файл, который будет представлен, и сравните различия с существующим YAML в репозитории кода, отображаемое с цветовым различием.                                                                                                                                                                                                                                                |

   - Экспорт файлов типа Kustomize на локальный для ручной загрузки в репозиторий кода: выберите **Local** в качестве целевой локации и выберите **Kustomize** в качестве формата файла для экспорта файла конфигурации приложения типа Kustomize на локальный. Этот файл поддерживает дифференцированные конфигурации и подходит для развертывания приложений между кластерами.

6. Щелкните **Export**.

### Дальнейшие действия

После экспорта YAML в репозиторий Git кода вы можете обратиться к [Создание GitOps приложений](/gitops/functions/create_argocd_application/create_application_via_platform.mdx), чтобы создать нативное приложение типа `Custom Application` GitOps приложения по кластерам.
