# Экспорт приложений

Для стандартизации процесса экспорта приложений между средами разработки, тестирования и продуктивной эксплуатации, а также для упрощения быстрой миграции бизнеса в новые среды, вы можете экспортировать нативные приложения в виде шаблонов приложений (Charts) или экспортировать упрощенные YAML файлы, которые могут быть использованы непосредственно для развертывания. Это позволяет нативному приложению работать в разных средах или пространствах имен. Вы также можете экспортировать YAML файлы в репозиторий кода, чтобы быстро развернуть приложения по всему кластеру, используя функциональность GitOps.

## Экспорт Helm Charts

### Процедура

1. Получите доступ к **Container Platform**.

2. В левой навигационной панели нажмите на **Application Management** > **Native Applications**.

3. Нажмите на ***имя приложения*** типа `Custom Application`.

4. Нажмите на **Actions** > **Export**; вы также можете экспортировать конкретную версию со страницы сведений о приложении.

5. Выберите один из методов экспорта по необходимости и обратитесь к следующим инструкциям для настройки соответствующей информации.

   - Экспорт Helm Charts в репозиторий шаблонов с правами управления

     **Примечание**: Репозиторий шаблонов добавляется администратором платформы. Пожалуйста, свяжитесь с администратором платформы, чтобы получить действующий репозиторий шаблонов типа **Chart** или **OCI Chart** с правами **Management**.

     | Параметр               | Описание                                                                                                                                                                                                                                                                                                              |
     | ---------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
     | **Целевое Местоположение** | Выберите **Template Repository**, чтобы напрямую синхронизировать шаблон с репозиторием шаблонов типа **Chart** или **OCI Chart** с правами **Management**. Владелец проекта, назначенный этому **Template Repository**, может напрямую использовать шаблон.                                                                      |
     | **Каталог Шаблона**   | Когда выбранный тип репозитория шаблонов - OCI Chart, вам нужно выбрать или вручную ввести каталог для хранения Helm Chart.<br />**Примечание**: При ручном вводе нового каталога шаблона платформа создаст этот каталог в репозитории шаблонов, но существует риск сбоев при создании. |
     | **Версия**            | Номер версии шаблона приложения.<br />Формат должен быть `v<Major>.<Minor>.<Patch>`. Значение по умолчанию – текущая версия приложения или текущая версия снимка.                                                                                                                              |
     | **Иконка**            | Поддерживает форматы изображений JPG, PNG и GIF, с размером файла не более 500КБ. Рекомендуемые размеры - 80\*60 пикселей.                                                                                                                                                                                                |
     | **Описание**          | Описание будет отображаться в списке шаблонов приложений в каталоге приложений.                                                                                                                                                                                                                 |
     | **README**            | Файл описания. Поддерживает редактирование в формате Markdown и будет отображаться на странице деталей шаблона приложения.                                                                                                                                                                                             |
     | **НОТЫ**              | Файл помощи по шаблону. Поддерживает стандартное редактирование в формате простого текста; после завершения развертывания шаблона он будет отображаться на странице деталей шаблона приложения.                                                                                                                                                      |

   - Экспорт Helm Charts на локальный диск для ручной загрузки в репозиторий шаблонов: выберите **Local** в качестве целевого местоположения и выберите **Helm Chart** в качестве формата файла, чтобы создать пакет Helm Chart, который будет загружен локально для оффлайн передачи.

6. Нажмите **Export**.

### Последующие действия

- Если вы экспортируете Helm Chart на локальный диск, вам необходимо [добавить шаблон в репозиторий шаблонов с правами управления](../working_with_helm_charts.mdx).

- Независимо от выбранного метода экспорта, вы можете обратиться к [Создание нативных приложений - метод шаблона](../create_applications/chart_app.mdx) для создания нативного приложения типа `Template Application` в **немодерируемом** пространстве имен.

## Экспорт YAML на локальный диск

### Шаги

#### **Метод 1**

1. Получите доступ к **Container Platform**.

2. В левой навигационной панели нажмите на **Application Management** > **Native Applications**.

3. Нажмите на ***имя приложения***.

4. Нажмите на **Actions** > **Export**; вы также можете экспортировать конкретную версию со страницы сведений о приложении.

5. Выберите **Local** в качестве целевого местоположения и **YAML** в качестве формата файла, после чего вы можете экспортировать упрощенный YAML файл, который может быть непосредственно развернут в других средах.

6. Нажмите **Export**.

#### **Метод 2**

1. Получите доступ к **Container Platform**.

2. В левой навигационной панели нажмите на **Application Management** > **Native Applications**.

3. Нажмите на ***имя приложения***.

4. Нажмите на вкладку **YAML**, настройте параметры по необходимости и просмотрите YAML файл.

   | Тип                  | Описание                                                                                                                                                                                                                                                                                                                                                                                                  |
   | ------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
   | **Полный YAML**     | По умолчанию **Preview Simplified YAML** не выбран, отображая YAML файл с **скрытыми полями managedFields**. Вы можете предварительно просмотреть его и экспортировать напрямую; также вы можете убрать отметку с **Скрыть поля managedFields**, чтобы экспортировать полный YAML файл.<br />**Примечание**: Полный YAML в основном используется для операций и устранения неполадок и не может быть использован для быстрого создания нативных приложений на платформе. |
   | **Упрощенный YAML** | Убедитесь, что **Preview Simplified YAML** выбран, после чего вы можете предварительно просмотреть и экспортировать упрощенный YAML файл, который может быть непосредственно развернут в других средах.                                                                                                                                                                                                                                                     |

5. Нажмите **Export**.

### Последующие действия

После экспорта упрощенного YAML вы можете обратиться к [Создание нативных приложений - метод YAML](../create_applications/yaml_app.mdx) для создания нативного приложения типа `Custom Application` в **немодерируемом** пространстве имен.

## Экспорт YAML в репозиторий кода (Alpha)

### Предостережения

- Только администраторы платформы и администраторы проектов могут напрямую экспортировать YAML файлы нативного приложения в репозиторий кода.

- `Template Applications` не поддерживают экспорт файлов конфигурации приложений в формате Kustomize или прямой экспорт YAML файлов в репозиторий кода; сначала вы можете **отменить привязку к шаблону** и преобразовать его в `Custom Application`.

### Шаги

1. Получите доступ к **Container Platform**.

2. В левой навигационной панели нажмите на **Application Management** > **Native Applications**.

3. Нажмите на ***имя приложения*** типа `Custom`.

4. Нажмите на **Actions** > **Export**; вы также можете экспортировать конкретную версию со страницы сведений о приложении.

5. Выберите один из методов экспорта по необходимости и обратитесь к следующим инструкциям для настройки соответствующей информации.

   - Экспорт YAML в репозиторий кода:

     | Параметр                    | Описание                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
     | ---------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
     | **Целевое Местоположение**  | Выберите **Code Repository**, чтобы напрямую синхронизировать YAML файл с заданным репозиторием кода Git. Владелец проекта, назначенный этому **Code Repository**, может непосредственно использовать YAML файл.                                                                                                                                                                                                                                                                                                                                                                   |
     | **Имя Интеграционного Проекта** | Имя проекта интеграционного инструмента, назначенное или связанное с вашим проектом администратором платформы.                                                                                                                                                                                                                                                                                                                                                                                                                                         |
     | **Адрес Репозитория**       | Адрес репозитория, назначенный для вашего использования в рамках проекта интеграционного инструмента.                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
     | **Метод Экспорта**          | <ul><li>**Существующая Ветка**: Экспортировать YAML приложения в выбранную ветку.</li><li>**Новая Ветка**: Создать новую ветку на основе выбранной **Branch/Tag/Commit ID** и экспортировать YAML приложение в новую ветку.</li><ul><li>Если выбран параметр **Submit PR (Pull Request)**, платформа создаст новую ветку и отправит запрос на слияние.</li><li>Если выбран параметр **Автоматически удалить исходную ветку после слияния PR**, исходная ветка будет автоматически удалена после слияния запроса на слияние в репозитории кода Git.</li></ul></ul> |
     | **Путь Файла**              | Конкретное местоположение, где файл должен быть сохранен в репозитории кода; вы также можете ввести путь к файлу, и платформа создаст новый путь в репозитории кода на основе введенного.                                                                                                                                                                                                                                                                                                                                                          |
     | **Сообщение Коммита**       | Заполните информацию о коммите, чтобы идентифицировать содержание этого представления.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
     | **Предварительный Просмотр** | Предварительный просмотр YAML файла, который будет представлен, и сравнение различий с существующим YAML в репозитории кода, отображаемым с цветовым различием.                                                                                                                                                                                                                                                                                                                                                                                                       |

   - Экспорт файлов типа Kustomize на локальный диск для ручной загрузки в репозиторий кода: выберите **Local** в качестве целевого местоположения и выберите **Kustomize** в качестве формата файла, чтобы экспортировать конфигурационный файл приложения типа Kustomize локально. Этот файл поддерживает дифференцированные конфигурации и подходит для развертывания приложений через кластеры.

6. Нажмите **Export**.

### Последующие действия

После экспорта YAML в репозиторий кода Git вы можете обратиться к [Создание GitOps приложений](/gitops/functions/create_argocd_application/create_application_via_platform.mdx) для создания приложения GitOps типа `Custom Application` по всем кластерам.
