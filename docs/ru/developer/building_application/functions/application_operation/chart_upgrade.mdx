# Обновление и удаление шаблонных приложений

В связи с пересечением функционала между текущими шаблонными приложениями и нативными приложениями, а также с улучшенными операционными возможностями, доступными в нативных приложениях, независимое управление шаблонными приложениями больше не будет предложено в будущих версиях. Пожалуйста, как можно скорее обновите ваши успешно развернутые шаблонные приложения до нативных приложений.

## Важные замечания

Эта функция **будет прекращена**. Пожалуйста, как можно скорее обновите ваши успешно развернутые шаблонные приложения до нативных приложений.

## Предварительные условия

Пожалуйста, свяжитесь с администратором платформы, чтобы активировать функции, связанные с шаблонными приложениями.

## Описание анализа состояния

Нажмите на ***Имя шаблонного приложения***, чтобы отобразить подробный анализ статуса развертывания диаграммы в детальной информации.

| Тип             | Причина                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| --------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Инициализировано** | Указывает на состояние загрузки шаблона диаграммы. <ul><li>Когда статус истинный, это означает, что загрузка шаблона диаграммы была успешной.</li><li>Когда статус ложный, это означает, что загрузка шаблона диаграммы не удалась, и причину неудачи можно увидеть в колонке сообщения.</li><ul><li>ChartLoadFailed: Не удалось загрузить шаблон диаграммы.</li><li>InitializeFailed: Во время инициализации произошла ошибка перед загрузкой диаграммы.</li></ul></ul>                                                                                                                                                                                                                                             |
| **Проверено**    | Указывает на состояние проверки прав и зависимостей пользователя для шаблона диаграммы. <ul><li>Когда статус истинный, это означает, что все проверки валидации прошли успешно.</li><li>Когда статус ложный, это означает, что некоторые проверки валидации не прошли, и причину неудачи можно увидеть в колонке сообщения.</li><ul><li>DependenciesCheckFailed: Не удалось проверить зависимости диаграммы.</li><li>PermissionCheckFailed: У текущего пользователя нет разрешений на выполнение определенных операций с ресурсами.</li><li>ConsistentNamespaceCheckFailed: При развертывании шаблонного приложения как нативного приложения диаграмма содержит ресурсы, требующие развертывания между пространствами имен.</li></ul></ul> |
| **Синхронизировано** | Указывает на состояние развертывания шаблона диаграммы. <ul><li>Когда статус истинный, это означает, что развертывание шаблона диаграммы прошло успешно.</li><li>Когда статус ложный, это означает, что развертывание шаблона диаграммы не удалось, причина указывается как ChartSyncFailed, а конкретную причину можно увидеть в колонке сообщения.</li></ul>                                                                                                                                                                                                                                                                                        |
