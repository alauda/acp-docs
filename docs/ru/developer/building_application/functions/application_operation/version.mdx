# Управление версиями для нативных приложений

После обновления приложения через интерфейс платформы автоматически создается запись об исторической версии. Для обновлений приложения, инициируемых неинтерфейсными операциями, такими как обновление приложения через вызовы API, вы можете вручную создать снимок версии для записи изменений.

**Примечание**: Когда количество записей о снимках версий превышает 6, платформа сохраняет только последние 6 записей и автоматически удаляет остальные, приоритетно удаляя самые старые записи о снимках версий.

## Создание снимка версии

### Шаги

1. Получите доступ к **Платформе контейнеров**.

2. В левой навигационной панели нажмите на **Управление приложениями** > **Нативные приложения**.

3. Нажмите на ***Название приложения***.

4. На вкладке **Снимок версии** нажмите **Создать снимок версии**.

5. Настройте информацию и нажмите **Подтвердить**.

   **Примечание**: Вы также можете [Распространить приложение](./export.mdx), что позволяет вам распространять снимок версии приложения в виде чарта, упрощая быструю развертку одного и того же приложения на нескольких кластерах и пространствах имен на платформе.

## Возврат к исторической версии

Вернуть конфигурацию текущего приложения к исторической версии.

### Шаги

1. Получите доступ к **Платформе контейнеров**.

2. В левой навигационной панели нажмите на **Управление приложениями** > **Нативные приложения**.

3. Нажмите на ***Название приложения***.

4. На вкладке **Исторические версии** нажмите на ***Номер версии***.

5. Нажмите ⋮ > **Вернуться к этой версии**.

6. Нажмите **Вернуться**.
