# Управление версиями для нативных приложений

После обновления приложения через интерфейс платформы автоматически создается историческая запись версии. Для обновлений приложения, инициированных не через интерфейс, таких как обновление приложения с помощью вызовов API, вы можете вручную создать снимок версии для записи изменений.

**Примечание**: Когда количество записей снимков версии превышает 6, платформа сохраняет только последние 6 записей и автоматически удаляет остальные, приоритет отдается удалению самых старых записей снимков версии.

## Создание снимка версии

### Шаги

1. Перейдите в **Платформу контейнеров**.

2. В левой боковой панели навигации нажмите **Управление приложениями** > **Нативные приложения**.

3. Нажмите на ***Название приложения***.

4. На вкладке **Снимок версии** нажмите **Создать снимок версии**.

5. Настройте информацию и нажмите **Подтвердить**.

   **Примечание**: Вы также можете [Распределить приложение](./export.mdx), что позволяет распределять снимок версии приложения в виде диаграммы, что облегчает быстрое развертывание одного и того же приложения на нескольких кластерах и пространствах имен на платформе.

## Возврат к исторической версии

Верните конфигурацию текущего приложения к исторической версии.

### Шаги

1. Перейдите в **Платформу контейнеров**.

2. В левой боковой панели навигации нажмите **Управление приложениями** > **Нативные приложения**.

3. Нажмите на ***Название приложения***.

4. На вкладке **Исторические версии** нажмите на ***Номер версии***.

5. Нажмите ⋮ > **Вернуться к этой версии**.

6. Нажмите **Вернуться**.
