# Обновление приложений

Пользовательские приложения значительно упрощают единое управление рабочими нагрузками, сетями, хранилищами и конфигурациями, но не все ресурсы принадлежат приложению.

- Ресурсы, добавленные в процессе создания приложения или добавленные через обновления приложения, по умолчанию ассоциируются с приложением и не требуют дополнительного импорта.

- Ресурсы, созданные вне приложения, не принадлежат приложению и не могут быть найдены в деталях приложения. Однако, если определения ресурсов соответствуют бизнес-требованиям, бизнес может функционировать нормально. В этом случае рекомендуется импортировать ресурсы в приложение для единого управления.

- **Управление изображениями**
  - Развертывание новых контейнерных изображений с контролем версии/патча
  - Настройка imagePullPolicy (Always/IfNotPresent/Never)

- **Конфигурация времени выполнения**
  - Изменение переменных окружения через ConfigMaps/Secrets
  - Обновление запросов/лимитов ресурсов (CPU/Память)

- **Оркестрация ресурсов**
  - Импорт существующих ресурсов Kubernetes (Deployments/Services/Ingresses)
  - Синхронизация конфигураций между пространствами имен с использованием `kubectl apply -f`

Ресурсы, импортированные в приложение, могут воспользоваться следующими функциями:

| Функция                      | Описание                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| ---------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Снимок версии**           | При [создании снимка версии](./version.mdx) для приложения также будет создан снимок для ресурсов внутри приложения. <ul><li>Если приложение будет откатано, ресурсы также откатятся к состоянию в снимке.</li><li>Если конкретная версия приложения будет распространена, платформа автоматически создаст ресурсы, зафиксированные в снимке, при повторном развертывании приложения.</li></ul> |
| **Удаляется с приложением**  | Если приложение больше не нужно, удаление приложения автоматически удалит все ресурсы, связанные с приложением, включая вычислительные компоненты, внутренние маршруты и входящие правила.                                                                                                                                                                                                                                             |
| **Легче найти**              | В информации о деталях приложения вы можете быстро просмотреть ресурсы, связанные с приложением. <br />Например: Внешний трафик может получить доступ к *Deployment D* через *Service S*, который принадлежит *Application A*, но соответствующий адрес доступа можно быстро найти в деталях приложения, только если *Service S* также принадлежит *Application A*.                                                                                |

## Импорт ресурсов

> Пакетный импорт связанных ресурсов в пространстве имен, где находится приложение; ресурс может принадлежать только одному приложению.

1. Войдите в **Платформу контейнеров**.

2. В левой навигационной панели нажмите **Управление приложениями** > **Нативные приложения**.

3. Нажмите на ***Имя приложения***.

4. Нажмите **Действия** > **Управление ресурсами**.

5. В **Типе ресурса** внизу выберите тип ресурсов для импорта.

   **Примечание**: Общие типы ресурсов включают Deployment, DaemonSet, StatefulSet, Job, CronJob, Service, Ingress, PVC, ConfigMap, Secret и HorizontalPodAutoscaler, которые отображаются вверху; другие ресурсы расположены в алфавитном порядке, и вы можете быстро найти конкретные типы ресурсов, используя ключевые слова для поиска.

6. В разделе **Ресурсы** выберите ресурсы для импорта.

   **Внимание**: Для ресурсов типа **Job** поддерживается импорт только задач, созданных через YAML.

7. Нажмите **Импортировать ресурсы**.

## Удаление/пакетное удаление ресурсов

> Удаление / пакетное удаление ресурсов из приложения только разъединяет приложение с ресурсами и не удаляет ресурсы.

Если между ресурсами в приложении существуют взаимосвязи, удаление любого ресурса из приложения не изменит ассоциации между ресурсами. Например, даже если *Service S* удален из *Application A*, внешний трафик все равно может получить доступ к *Deployment D* через *Service S*.

1. Войдите в **Платформу контейнеров**.

2. В левой навигационной панели нажмите **Управление приложениями** > **Нативные приложения**.

3. Нажмите на ***Имя приложения***.

4. Нажмите **Действия** > **Управление ресурсами**.

5. Нажмите **Удалить** справа от ресурса, чтобы удалить его; или выберите несколько ресурсов сразу и нажмите **Удалить** в верхней части таблицы, чтобы пакетно удалить ресурсы.
