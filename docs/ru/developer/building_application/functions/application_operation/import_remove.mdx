# Импорт и удаление ресурсов для нативных приложений

Нативные приложения значительно упрощают унифицированное управление вычислительными компонентами, сетями, хранилищем и конфигурациями, но не все ресурсы, связанные с бизнесом, принадлежат приложению.

- Ресурсы, добавленные в процессе создания приложения или добавленные через обновления приложения, по умолчанию ассоциируются с приложением и не требуют дополнительного импорта.

- Ресурсы, созданные вне приложения, не принадлежат приложению и не могут быть найдены в деталях приложения. Однако, пока определения ресурсов соответствуют бизнес-требованиям, бизнес может функционировать нормально. В этом случае рекомендуется импортировать ресурсы в приложение для унифицированного управления.

Ресурсы, импортированные в приложение, могут воспользоваться следующими возможностями:

| Особенность                  | Описание                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| ---------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Снимок версии**           | При [создании снимка версии](./version.mdx) для приложения снимок также будет сгенерирован для ресурсов внутри приложения. <ul><li>Если приложение будет возвращено к предыдущей версии, ресурсы также вернутся к состоянию в снимке.</li><li>Если конкретная версия приложения будет распространена, платформа автоматически создаст ресурсы, записанные в снимке, при повторном развертывании приложения.</li></ul> |
| **Удалены вместе с приложением** | Если приложение больше не требуется, удаление приложения автоматически удалит все ресурсы, связанные с приложением, включая вычислительные компоненты, внутренние маршруты и входящие правила.                                                                                                                                                                                                                                                                        |
| **Легче найти**             | В информации о деталях приложения вы можете быстро просмотреть ресурсы, связанные с приложением. <br />Например: Внешний трафик может получить доступ к *Deployment D* через *Service S*, который принадлежит *Application A*, но соответствующий адрес доступа можно быстро найти в деталях приложения, только если *Service S* также принадлежит *Application A*.                                                            |

## Импорт ресурсов

Пакетный импорт связанных ресурсов в пространстве имен, где находится приложение; ресурс может принадлежать только одному приложению.

### Шаги для выполнения

1. Войдите в **Платформу контейнеров**.

2. В левом навигационном меню нажмите **Управление приложениями** > **Нативные приложения**.

3. Нажмите на ***Имя приложения***.

4. Нажмите **Действия** > **Управление ресурсами**.

5. В поле **Тип ресурса** внизу выберите тип ресурсов для импорта.

   **Примечание**: Общие типы ресурсов включают Deployment, DaemonSet, StatefulSet, Job, CronJob, Service, Ingress, PVC, ConfigMap, Secret и HorizontalPodAutoscaler, которые отображаются вверху; другие ресурсы расположены в алфавитном порядке, и вы можете быстро найти конкретные типы ресурсов, используя ключевые слова.

6. В разделе **Ресурсы** выберите ресурсы для импорта.

   **Внимание**: Для ресурсов типа **Job** поддерживается импорт только задач, созданных через YAML.

7. Нажмите **Импортировать ресурсы**.

## Удаление / Пакетное удаление ресурсов

Удаление / пакетное удаление ресурсов из приложения просто разъединяет приложение с ресурсами и не удаляет сами ресурсы.

Если между ресурсами в приложении существуют взаимосвязи, удаление любого ресурса из приложения не изменит связи между ресурсами. Например, даже если *Service S* удален из *Application A*, внешний трафик все равно может получить доступ к *Deployment D* через *Service S*.

### Шаги для выполнения

1. Войдите в **Платформу контейнеров**.

2. В левом навигационном меню нажмите **Управление приложениями** > **Нативные приложения**.

3. Нажмите на ***Имя приложения***.

4. Нажмите **Действия** > **Управление ресурсами**.

5. Нажмите **Удалить** справа от ресурса, чтобы удалить его; или выберите несколько ресурсов одновременно и нажмите **Удалить** вверху таблицы, чтобы пакетно удалить ресурсы.
