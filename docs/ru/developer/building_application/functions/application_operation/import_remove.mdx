# Импорт и удаление ресурсов для нативных приложений

Нативные приложения значительно упрощают единое управление вычислительными компонентами, сетями, хранилищами и конфигурациями, но не все ресурсы, связанные с бизнесом, принадлежат приложению.

- Ресурсы, добавленные в процессе создания приложения или добавленные через обновления приложения, по умолчанию ассоциируются с приложением и не требуют дополнительного импорта.

- Ресурсы, созданные вне приложения, не принадлежат приложению и не могут быть найдены в деталях приложения. Тем не менее, при соблюдении определений ресурсов бизнес может работать нормально. В этом случае рекомендуется импортировать ресурсы в приложение для единого управления.

Импортированные в приложение ресурсы могут воспользоваться следующими функциями:

| Функция                      | Описание                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| ---------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Снимок версии**           | При [создании снимка версии](./version.mdx) для приложения будет также создан снимок для ресурсов внутри приложения. <ul><li>Если приложение будет откатиться, ресурсы также откатятся к состоянию в снимке.</li><li>Если конкретная версия приложения будет распространена, платформа автоматически создаст ресурсы, зафиксированные в снимке, при повторном развертывании приложения.</li></ul> |
| **Удалены с приложением**   | Если приложение больше не нужно, его удаление автоматически удалит все ресурсы, связанные с приложением, включая вычислительные компоненты, внутренние маршруты и входящие правила.                                                                                                                                                                                                                                             |
| **Легче найти**             | В информации о деталях приложения вы можете быстро просмотреть ресурсы, связанные с приложением. <br />Например: Внешний трафик может получить доступ к *Развертыванию D* через *Сервис S*, который принадлежит *Приложению A*, но соответствующий адрес доступа можно быстро найти в деталях приложения, только если *Сервис S* также принадлежит *Приложению A*.                                                                                |

## Импорт ресурсов

Пакетный импорт связанных ресурсов в пространстве имен, где находится приложение; ресурс может принадлежать только одному приложению.

### Этапы операции

1. Войдите в **Платформу контейнеров**.

2. В левой навигационной панели выберите **Управление приложениями** > **Нативные приложения**.

3. Щелкните по ***имени приложения***.

4. Нажмите **Действия** > **Управление ресурсами**.

5. В **Типе ресурса** внизу выберите тип ресурсов для импорта.

   **Примечание**: Обычные типы ресурсов включают Развертывание, DaemonSet, StatefulSet, Job, CronJob, Сервис, Ingress, PVC, ConfigMap, Secret и HorizontalPodAutoscaler, которые отображаются сверху; другие ресурсы упорядочены в алфавитном порядке, и вы можете быстро запросить конкретные типы ресурсов, используя ключевые слова.

6. В разделе **Ресурсы** выберите ресурсы для импорта.

   **Внимание**: Для ресурсов типа **Job** поддерживаются только задачи, созданные через YAML, для импорта.

7. Нажмите **Импортировать ресурсы**.

## Удаление / Пакетное удаление ресурсов

Удаление / пакетное удаление ресурсов из приложения только разъединяет приложение с ресурсами и не удаляет ресурсы.

Если между ресурсами в приложении существует взаимосвязь, удаление любого ресурса из приложения не изменит взаимосвязи между ресурсами. Например, даже если *Сервис S* будет удален из *Приложения A*, внешний трафик все равно сможет получить доступ к *Развертыванию D* через *Сервис S*.

### Этапы операции

1. Войдите в **Платформу контейнеров**.

2. В левой навигационной панели выберите **Управление приложениями** > **Нативные приложения**.

3. Щелкните по ***имени приложения***.

4. Нажмите **Действия** > **Управление ресурсами**.

5. Нажмите **Удалить** справа от ресурса, чтобы удалить его; или выберите несколько ресурсов сразу и нажмите **Удалить** вверху таблицы, чтобы пакетно удалить ресурсы.
