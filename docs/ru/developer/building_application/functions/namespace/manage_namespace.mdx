# Обновление пространств имен

## Обновление квоты

> [Квота ресурсов](./resource_quota.mdx)

### Обновление квоты ресурсов с помощью веб-консоли

1. Перейдите в **Управление проектом** и выберите **Пространства имен** > **Список пространств имен** в левой боковой панели.

2. Нажмите на целевое ***имя пространства имен***.

3. Нажмите **Действия** > **Обновить квоту**.

4. Отрегулируйте квоты ресурсов (ЦП, память, поды и т. д.) и нажмите **Обновить**.

### Обновление квоты ресурсов с помощью CLI

> [Пример YAML файла квоты ресурсов](./create_namespace.mdx#namespace-yaml)

```bash
# Шаг 1: Измените квоту пространства имен  
kubectl edit resourcequota <quota-name> -n <namespace-name>  

# Шаг 2: Проверьте изменения  
kubectl get resourcequota <quota-name> -n <namespace-name> -o yaml  
```

## Обновление лимита контейнера

> [Лимит диапазона](./limit_range.mdx)

### Обновление LimitRange с помощью веб-консоли

1. Перейдите в **Управление проектом** и выберите **Пространства имен** > **Список пространств имен** в левой боковой панели.

2. Нажмите на целевое ***имя пространства имен***.

3. Нажмите **Действия** > **Обновить лимит контейнера**.

4. Отрегулируйте диапазон лимитов контейнера (`defaultRequest`, `default`, `max`) и нажмите **Обновить**.

### Обновление LimitRange с помощью CLI

> [Пример YAML файла Limit Range](./create_namespace.mdx#namespace-yaml)

```bash
# Шаг 1: Измените LimitRange  
kubectl edit limitrange <limitrange-name> -n <namespace-name>  

# Шаг 2: Проверьте изменения  
kubectl get limitrange <limitrange-name> -n <namespace-name> -o yaml  
```
