# Импорт пространств имен

## Обзор

**Возможности управления жизненным циклом пространств имен**:

- Импорт пространств имен между кластерами: Импорт пространств имен в проект централизует их управление во всех кластерах Kubernetes, предоставленных платформой. Это предоставляет администраторам единые возможности управления ресурсами и мониторинга в распределенных средах.

**Разъединение пространств имен**:

- Функция разъединения пространства имен позволяет вам отвязать пространство имен от его текущего проекта, сбрасывая его ассоциацию для последующего переназначения или очистки.

- Импорт пространства имен в проект предоставляет ему возможности, эквивалентные возможностям пространств имен, созданных нативно на платформе. Это включает унаследованные политики уровня проекта (например, квоты ресурсов), единое мониторинг и централизованные средства управления.

**Важные примечания**:

- Пространство имен может быть связано только с одним проектом в любой момент времени.
- Если пространство имен уже связано с проектом, его нельзя импортировать или переназначить в другой проект без предварительного разъединения с его оригинальным проектом.

## Сценарии использования

Распространенные сценарии управления **пространствами имен** включают:

- При подключении нового **кластера Kubernetes** к платформе вы можете использовать функцию **Импорт пространства имен**, чтобы связать его существующие **пространства имен Kubernetes** с проектом. Просто выберите целевой проект и кластер, чтобы инициировать импорт. Это действие предоставляет **проекту** управление этими **пространствами имен**, включая **квоты ресурсов**, мониторинг и соблюдение политик.

<img src="./assets/nsimport.svg" width="400" />

- Пространство имен, которое было разъединено от одного **проекта**, может быть без проблем повторно связано с другим проектом с помощью функции **Импорт пространства имен** для продолжения централизованного управления.

- Пространства имен, которые в настоящее время не управляются ни одним **проектом** (например, те, которые созданы с помощью скриптов на уровне кластера), должны быть связаны с целевым **проектом** с помощью функции **Импорт пространства имен**, чтобы включить управление на уровне платформы, включая видимость и централизованное управление.

## Предварительные условия

- Пространство имен в настоящее время не управляется ни одним существующим проектом на платформе.
- Пространства имен могут быть импортированы только в проект, который уже связан с их целевым кластером Kubernetes. Если такого проекта не существует, вам сначала необходимо создать проект, связанный с этим кластером.

## Процедура

1. В **Управлении проектами** нажмите на ***Имя проекта***, в который должно быть импортировано пространство имен.
2. Перейдите в **Пространства имен** > **Пространства имен**.
3. Нажмите на кнопку **Выпадающий список** рядом с **Создать пространство имен**, затем выберите **Импорт пространства имен**.
4. Обратитесь к документации [Создание пространств имен](./create_namespace.mdx) для получения деталей конфигурации параметров.
5. Нажмите **Импорт**.
