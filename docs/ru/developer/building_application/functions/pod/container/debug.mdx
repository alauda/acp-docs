# Дебаг-контейнер (альфа)

Функция дебага предоставляет соответствующие инструменты для отладки работающих контейнеров, включая системные, сетевые и дисковые утилиты.

## Принцип реализации

Функция дебага реализована с помощью Эфемерных контейнеров. Эфемерный контейнер - это тип контейнера, который разделяет ресурсы с бизнес-контейнерами. Вы можете добавить Эфемерный контейнер (например, *Контейнер A-debug*) в под и использовать инструменты отладки в этом контейнере. Результаты отладки будут напрямую применены к бизнес-контейнеру (например, *Контейнер A*).

## Заметки

- Вы не можете добавить Эфемерный контейнер, напрямую обновив конфигурацию пода; убедитесь, что функция дебага активирована для добавления Эфемерного контейнера.

- Эфемерные контейнеры, активированные функцией дебага, не имеют гарантий по ресурсам или планированию и не будут перезапускаться автоматически. Пожалуйста, избегайте запуска бизнес-приложений в них, за исключением целей отладки.

- Пожалуйста, осторожно используйте функцию дебага, если ресурсы на узле, где расположен под, на грани исчерпания, так как это может привести к выселению пода.

## Сценарии использования

Хотя вы также можете войти в контейнеры и отлаживать с помощью функции EXEC, многие образы контейнеров не включают необходимые инструменты отладки (такие как bash, net-tools и т.д.) для сокращения размера образа. В отличие от этого, функция дебага, которая идет с предустановленными инструментами отладки, больше подходит для следующих сценариев.

- Диагностика неисправностей: Если бизнес-контейнер сталкивается с проблемой, помимо проверки событий и журналов, вам может понадобиться провести более детальное устранение неисправностей и решение проблемы в контейнере.

- Настройка конфигурации: Если в текущем бизнес-решении есть недостатки, вы можете захотеть выполнить настройку конфигурации бизнес-компонентов внутри контейнера, чтобы разработать новую схему конфигурации, которая помогает бизнесу работать более эффективно.

## Процедура

1. Войдите в **Платформу контейнеров**.

2. В левой навигационной панели нажмите **Нагрузки** > **Поды**.

3. Найдите под и нажмите ⋮ > **Дебаг**.

4. Выберите контейнер, который хотите отлаживать.

5. (Необязательно) Если интерфейс запрашивает, что **требуется инициализация**, нажмите **Инициализировать**.

   **Примечание**: После инициализации функции дебага, пока под не будет воссоздан, вы можете напрямую войти в Эфемерный контейнер (например, *Контейнер A-debug*) для отладки.

6. Дождитесь готовности окна отладки, а затем начните отладку.

   **Совет**: Нажмите на запрос команды в верхнем правом углу, чтобы просмотреть общие инструменты и их использование.

7. После завершения закройте окно отладки.
