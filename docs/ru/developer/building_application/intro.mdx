# Введение

Создание и управление приложениями на платформе Alauda Container Platform. С помощью <Term name="product" textCase="lower" /> вы можете создавать, редактировать, удалять и управлять приложениями, используя веб-консоль или интерфейс командной строки (CLI).

## Преимущества

### Создание приложений

В <Term name="product" textCase="lower" /> приложения можно создавать несколькими способами. Вот некоторые распространенные методы:

- Создание из образов: Если у вас уже есть образ контейнера, вы можете напрямую создать приложение из него.
- Создание из каталога: <Term name="product" textCase="lower" /> предоставляет каталоги приложений, позволяя пользователям выбирать предопределенные шаблоны приложений (Helm Charts или Operator-Backed) для создания.
- Создание из YAML файла: Импортируя YAML файл, создайте пользовательское приложение со всеми включенными ресурсами за один шаг.
- Создание из кода: Выберите или введите адрес репозитория кода для импорта кода, автоматически создается образ и создается приложение.

### Управление приложениями

Требуется непрерывное управление работой, включая журналы, события, мониторинг и т. д.

- Просмотр журналов подов.
- Просмотр событий в пространстве имен.
- Просмотр панелей мониторинга через платформу Container Platform (панель мониторинга на уровне пространства имен).

### Полное управление жизненным циклом приложений

<Term name="product" textCase="lower" /> предоставляет поддержку управления жизненным циклом приложений, начиная от создания приложения, выполнения, обновлений до его окончательной деактивации. Операции, такие как поэтапные обновления и откаты, могут выполняться с использованием ресурсов, таких как приложения и другие рабочие нагрузки Kubernetes.
