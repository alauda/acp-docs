# Введение

<Term name="product" /> предоставляет единый интерфейс для создания, редактирования, удаления и управления облачными приложениями как через веб-консоль, так и через интерфейс командной строки (CLI). Приложения могут быть развернуты в нескольких пространствах имен с политиками RBAC.

## Управление пространствами имен

Пространства имен обеспечивают логическую изоляцию ресурсов Kubernetes. Ключевые операции включают:

- [Создание пространств имен](./functions/namespace/create_namespace.mdx): Определение квот ресурсов и политик безопасности подов.
- [Импорт пространств имен](./functions/namespace/import_namespace.mdx): Импорт существующих пространств имен Kubernetes в <Term name="product" /> обеспечивает полное соответствие возможностям платформы с пространствами имен, созданными нативно.

## Управление жизненным циклом приложений

<Term name="product" /> поддерживает управление жизненным циклом от начала до конца, включая:

### Шаблоны создания приложений

В <Term name="product" /> приложения можно создавать несколькими способами. Вот некоторые распространенные методы:

- [Создание из образов](./functions/create_applications/image_app.mdx): Создание пользовательских приложений с использованием заранее собранных образов контейнеров. Этот метод поддерживает создание полного приложения, включающего `Deployments`, `Services`, `ConfigMaps` и другие ресурсы Kubernetes.
- [Создание из каталога](./functions/create_applications/chart_app.mdx): <Term name="product" /> предоставляет каталоги приложений, позволяя пользователям выбирать предопределенные шаблоны приложений (Helm Charts или Operator Backed) для создания.
- [Создание из YAML](./functions/create_applications/yaml_app.mdx): Импортируя YAML файл, создайте пользовательское приложение со всеми включенными ресурсами за один шаг.
- [Создание из кода](./functions/create_applications/s2i_app.mdx): Создание образов с помощью Source to Image (S2I).

### Операции с приложениями

- [Обновление приложений](./functions/application_operation/application_upgrade.mdx): Обновление версии образа приложения, переменных окружения и других конфигураций или импорт существующих ресурсов Kubernetes для централизованного управления.
- [Экспорт приложений](./functions/application_operation/export.mdx): Экспорт приложений в форматах YAML, Kustomize или Helm Chart, затем импорт для создания новых экземпляров приложений в других пространствах имен или кластерах.
- [Управление версиями](./functions/application_operation/version.mdx): Поддержка автоматического или ручного создания версий приложений, а в случае проблем доступно однонажатие для отката к конкретной версии для быстрого восстановления.
- [Удаление приложений](./functions/application_operation/application_delete.mdx): Удаление приложения, которое одновременно удаляет само приложение и все его непосредственно содержащие ресурсы Kubernetes. Кроме того, это действие разрывает любые ассоциации, которые приложение могло иметь с другими ресурсами Kubernetes, которые не были непосредственно частью его определения.

### Наблюдаемость приложений

Для непрерывного управления работой платформа предоставляет журналы, события, мониторинг и т. д.

- [Журналы](./functions/application_observability/log.mdx): Поддержка просмотра журналов в реальном времени из текущего работающего пода, а также предоставление журналов из предыдущих перезапусков контейнеров.
- [События](./functions/application_observability/event.mdx): Поддержка просмотра информации о событиях для всех ресурсов в пространстве имен.
- [Панели мониторинга](./functions/application_observability/monitor.mdx): Предоставление панелей мониторинга на уровне пространства имен, включая специальные представления для приложений, рабочих нагрузок и подов, а также поддержка настройки панелей мониторинга для удовлетворения конкретных операционных требований.

## Управление рабочими нагрузками Kubernetes

Поддержка основных типов рабочих нагрузок:

- [Deployments](./functions/application_workloads/deployment_create.mdx): Управление статeless приложениями с поэтапными обновлениями.
- [StatefulSets](./functions/application_workloads/statefulset_create.mdx): Запуск stateful приложений с стабильными сетевыми идентификаторами.
- [DaemonSets](./functions/application_workloads/deamonset_create.mdx): Развертывание сервисов на уровне узла (например, сборщиков журналов).
- [CronJobs](./functions/application_workloads/cronjob_create.mdx): Планирование пакетных заданий с политиками повторных попыток.
