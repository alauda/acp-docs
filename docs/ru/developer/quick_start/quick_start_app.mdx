## Процедура

### Создать пространство имен \{#namespace}

> Пространства имен обеспечивают логическую изоляцию для группировки ресурсов и управления квотами.

**Предварительные условия**

- Разрешения на создание, обновление и удаление пространств имен (например, роли администратора или администратора проекта)
- Настроенный kubectl с доступом к кластеру

**Процесс создания**

1. Войдите в систему и перейдите в **Управление проектом** > **Пространства имен**

2. Выберите **Создать пространство имен**

3. Настройте основные параметры:

   | \*\* Параметр \*\* | **Описание**                                     |
   | :------------------ | :-------------------------------------------------- |
   | Кластер             | Целевой кластер из ассоциированных кластеров проекта     |
   | Пространство имён   | Уникальный идентификатор (автоматически префиксированный именем проекта) |

4. Завершите создание с использованием стандартных ограничений ресурсов

### Настроить репозиторий образов \{#image}

<Term name="product" textCase="capitalize" /> поддерживает несколько стратегий получения образов:

#### Метод 1: Интегрированный реестр через инструментальный комплекс

1. Перейдите в **Управление платформой** > **Инструментальный комплекс** > **Интеграция**

2. Начните новую интеграцию:

   | Параметр    | Требование                              |
   | ------------ | ---------------------------------------- |
   | Имя         | Уникальный идентификатор интеграции            |
   | API Endpoint | URL сервиса реестра (HTTP/HTTPS)        |
   | Secret       | Существующая или вновь созданная учетные данные |

3. Назначьте реестр целевому проекту платформы

#### Метод 2: Внешние реестровые службы

- Используйте публично доступные URL-адреса реестров (например, Docker Hub)
- Пример: `index.docker.io/library/nginx:latest`

**Требование для проверки**

- Сеть кластера должна иметь доступ к исходящим соединениям с конечными точками реестра

### Создать приложение через развертывание \{#deployment}

> Развертывания предоставляют декларативные обновления для повторяющихся наборов Pod.

**Процесс создания**

1. На экране **Платформа контейнеризации**:
   - Используйте селектор пространства имен для выбора целевой границы изоляции
2. Перейдите в **Рабочие нагрузки** > **Развертывания**
3. Нажмите **Создать развертывание**
4. Укажите источник образа:
   - Выберите интегрированный реестр *или*
   - Введите внешний URL образа (например, `index.docker.io/library/nginx:latest`)
5. Настройте идентификатор рабочей нагрузки и запустите

**Операции управления**

- Мониторинг статуса реплик
- Просмотр событий и логов
- Просмотр YAML-манифестов
- Анализ метрик ресурсов, оповещений

### Открыть службу через NodePort \{#service}

Службы обеспечивают сетевую доступность для групп Pod.

**Процесс создания**

1. Перейдите в **Сеть** > **Службы**

2. Нажмите **Создать службу** с параметрами:

   | Параметр    | Значение                                        |
   | :----------- | :--------------------------------------------- |
   | Тип         | NodePort                                       |
   | Селектор     | Название целевого развертывания                       |
   | Port Mapping | Порт службы: Порт контейнера (например, 8080:80) |

3. Подтвердите создание.

**Критично**

- Виртуальный IP, видимый для кластера
- Диапазон выделения NodePort (30000-32767)

Внутренние маршруты обеспечивают обнаружение служб для рабочих нагрузок, предоставляя единый IP-адрес или хост-порт для доступа.

1. Нажмите на **Сеть** > **Служба**.

2. Нажмите на **Создать службу**.

3. Настройте **Детали** на основе параметров ниже, оставив другие параметры по умолчанию.

   | Параметр     | Описание                                                                                                                                                                                                                |
   | ------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | Имя          | Введите имя службы.                                                                                                                                                                                             |
   | Тип          | `NodePort`                                                                                                                                                                                                                 |
   | Название рабочей нагрузки | Выберите ранее созданное `Развертывание`.                                                                                                                                                                                |
   | Порт          | **Порт службы**: Номер порта, открытого службой внутри кластера, т.е. Порт, например, `8080`.<br />**Порт контейнера**: Номер порта (или имя), целевое, сопоставленный с портом службы, т.е. targetPort, например, `80`. |

4. Нажмите на **Создать**. На этом этапе служба успешно создана.

### Проверить доступность приложения \{#application}

**Метод проверки**

1. Получите компоненты открытой конечной точки:
   - **Node IP**: Публичный адрес рабочего узла
   - **NodePort**: Выделенный внешний порт
2. Составьте URL-адрес доступа: `http://<Node_IP>:<NodePort>`
3. Ожидаемый результат: страница приветствия Nginx
