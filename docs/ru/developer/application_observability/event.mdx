# События

Информация о событиях, созданная изменениями состояния ресурсов Kubernetes и обновлениями оперативного состояния, с интегрированным визуальным интерфейсом запросов. Когда приложения, рабочие нагрузки или другие ресурсы сталкиваются с исключениями, анализ событий в реальном времени помогает устранить коренные причины.

## Процедура

> Эта процедура демонстрирует, как просматривать события времени выполнения контейнера через интерфейс развертывания.

1. **Платформа контейнеров**, нажмите **Рабочие нагрузки** > **Развертывание**.

2. Нажмите имя Развертывания из списка.

3. Перейдите на вкладку **События**, чтобы просмотреть подробные записи.

## Интерпретация записей событий

**Записи событий ресурсов**: Под панелью сводки событий перечислены все совпадающие события в заданном диапазоне времени. Нажмите на карточки событий, чтобы просмотреть полную информацию о событии. Каждая карточка отображает:

- **Тип ресурса**: Тип ресурса Kubernetes, представленный сокращениями иконок:
  - `P` = Под
  - `RS` = ReplicaSet
  - `D` = Развертывание
  - `SVC` = Служба

- **Имя ресурса**: Название целевого ресурса.

- **Причина события**: Причина, сообщенная Kubernetes (например, FailedScheduling).

- **Уровень события**: Серьезность события.
  - `Normal` : Информационное
  - `Warning` : Требует немедленного внимания

- **Время**: Время последнего возникновения, количество случаев.

:::info
Kubernetes позволяет администраторам настраивать периоды хранения событий через контроллер TTL событий с периодом хранения по умолчанию 1 час. Просроченные события автоматически удаляются системой. Для получения обширных исторических записей перейдите в [Все события](../../observability/event/event.mdx).
:::
