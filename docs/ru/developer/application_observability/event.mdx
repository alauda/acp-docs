# События

Информация о событиях, генерируемая изменениями состояния ресурсов Kubernetes и обновлениями статуса работы, с интегрированным визуальным интерфейсом для запросов. Когда приложения, рабочие нагрузки или другие ресурсы сталкиваются с исключениями, анализ событий в реальном времени помогает выявить коренные причины.

## Процедура

> Эта процедура демонстрирует, как просмотреть события выполнения контейнеров через интерфейс Deployment.

1. **Платформа контейнеров**, нажмите **Рабочие нагрузки** > **Развертывания**.

2. Нажмите на имя развертывания из списка.

3. Перейдите на вкладку **События**, чтобы просмотреть подробные записи.

## Интерпретация записей событий

**Записи событий ресурсов**: Под панелью сводки событий перечислены все соответствующие события в указанном диапазоне времени. Нажмите на карточки событий, чтобы просмотреть полные детали событий. Каждая карточка отображает:

- **Тип ресурса**: Тип ресурса Kubernetes, представленный сокращениями иконок:
  - `P` = Pod
  - `RS` = ReplicaSet
  - `D` = Deployment
  - `SVC` = Service

- **Имя ресурса**: Название целевого ресурса.

- **Причина события**: Причина,reported Kubernetes (например, FailedScheduling).

- **Уровень события**: Степень серьезности события.
  - `Normal` : Информационное
  - `Warning` : Требует немедленного внимания

- **Время**: Время последнего возникновения, количествоOccurrences.

:::info
Kubernetes позволяет администраторам настраивать периоды хранения событий через контроллер TTL событий с периодом хранения по умолчанию 1 час. Просроченные события автоматически удаляются системой. Для получения полных исторических записей перейдите к [Все события](../../observability/event/event.mdx).
:::
