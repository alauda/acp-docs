# Мониторинг панелей

- Поддерживает просмотр данных мониторинга ресурсов для компонентов рабочей нагрузки на платформе за последние 7 дней (с настраиваемым сроком хранения данных мониторинга). Включает статистику для приложений, рабочих нагрузок, подов, а также тенденции/рейтинги использования CPU/памяти.
- Поддерживает мониторинг на уровне пространства имен.
- Поддерживаемый мониторинг на уровне рабочей нагрузки: **Приложения**, **Развертывания**, **DaemonSets**, **StatefulSets** и **Pods**.

## Предварительные условия

- [Установка плагинов мониторинга](../../observability/monitor/install_monitor.mdx)

## Панели мониторинга на уровне пространства имен

### Процедура

1. **Платформа контейнеров**, щелкните **Наблюдать** > **Панели мониторинга**.

2. Просмотрите данные мониторинга под пространством имен. Предоставляются три панели мониторинга: **Обзор приложений**, **Обзор рабочих нагрузок** и **Обзор подов**.

3. Переключайтесь между панелями для мониторинга целевого **Обзора**.

### Создание панели мониторинга на уровне пространства имен

1. **Управление платформой**, создайте специальную панель мониторинга по адресу [Создание панели мониторинга](../../observability/monitor/functions/manage_dashboard.mdx#｛#create_dashboard｝) для создания специальной панели мониторинга.
2. Настройте следующие метки для отображения панели мониторинга на уровне пространства имен на **Платформе контейнеров**:
   - `cpaas.io/dashboard.folder: container-platform`
   - `cpaas.io/dashboard.tag.overview: "true"`

## Мониторинг на уровне рабочей нагрузки

> Эта процедура демонстрирует, как просмотреть мониторинг подов через интерфейс развертывания.

### Панель мониторинга по умолчанию

#### Процедура

1. **Платформа контейнеров**, щелкните **Рабочие нагрузки** > **Развертывания**.

2. Щелкните на имя развертывания из списка.

3. Перейдите на вкладку **Мониторинг**, чтобы увидеть стандартные метрики мониторинга.

#### Интерпретация метрик

| **Мониторинг ресурса** | **Гранулярность метрик**               | **Техническое определение**                                                                                                                                                                 |
| :---------------------- | :------------------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **CPU**                 | **Использование**/**Загрузка**        | **Загрузка** = Использование/Лимит (лимиты)<br />Оцените конфигурацию лимитов контейнера. Высокая загрузка указывает на недостаточные лимиты.<br />**Использование** представляет собой фактическое потребление ресурсов. |
| **Память**              | **Использование**/**Загрузка**        | **Загрузка** = Использование/Лимит (лимиты)<br />Метод оценки такой же, как и для CPU. Высокий уровень может вызвать неустойчивость компонента.                                           |
| **Сетевой трафик**     | **Скорость входящего трафика**/**Скорость исходящего трафика** | Сетевой трафик (байты/сек) поступающий/выходящий из подов.                                                                                                                                   |
| **Сетевой пакет**      | **Скорость получения**/**Скорость передачи** | Сетевые пакеты (количество/сек) полученные/отправленные подами.                                                                                                                                 |
| **Дисковая скорость**   | **Чтение**/**Запись**                 | Скорость чтения/записи (байты/сек) смонтированных томов на каждую рабочую нагрузку.                                                                                                           |
| **IOPS диска**         | **Чтение**/**Запись**                 | Операции ввода/вывода в секунду (IOPS) смонтированных томов на каждую рабочую нагрузку.                                                                                                        |

### Пользовательская панель мониторинга

4. Щелкните значок **Переключить** для переключения на пользовательские панели мониторинга. Обратитесь к [Добавить панель в пользовательскую панель мониторинга](../../observability/monitor/functions/manage_dashboard.mdx), чтобы создать специальную панель мониторинга на уровне рабочей нагрузки.

:::info
Наведите курсор на кривые графиков, чтобы увидеть метрики по подам и выражения PromQL в определенные временные метки. Если количество подов превышает 15, отображаются только 15 записей, отсортированных в порядке убывания.
:::
