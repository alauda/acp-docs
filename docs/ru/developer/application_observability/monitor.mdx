# Мониторинг панелей

- Поддерживает просмотр данных мониторинга ресурсов для компонентов рабочих нагрузок на платформе за последние 7 дней (с настраиваемым периодом хранения данных мониторинга). Включает статистику для приложений, рабочих нагрузок, подов и тенденций/рейтингов использования ЦП/памяти.
- Поддерживает мониторинг на уровне пространства имен.
- Поддерживаемый мониторинг на уровне рабочих нагрузок: **Приложения**, **Развертывания**, **DaemonSets**, **StatefulSets** и **Подов**.

## Предварительные требования

- [Установка плагинов мониторинга](../../observability/monitor/install_monitor.mdx)

## Панели мониторинга на уровне пространства имен

### Процедура

1. **Платформа контейнеров**, нажмите **Наблюдение** > **Панели мониторинга**.

2. Просмотрите данные мониторинга в пространстве имен. Предоставляется три панели: **Обзор приложений**, **Обзор рабочих нагрузок** и **Обзор подов**.

3. Переключайтесь между панелями для мониторинга целевого **Обзора**.

### Создание панели мониторинга на уровне пространства имен

1. **Управление платформой**, создайте выделенную панель мониторинга, обратившись к [Создание панели мониторинга](../../observability/monitor/functions/manage_dashboard.mdx#｛#create_dashboard｝) для создания выделенной панели мониторинга.
2. Настройте следующие метки для отображения панели мониторинга на уровне пространства имен на **Платформе контейнеров**:
   - `cpaas.io/dashboard.folder: контейнер-платформа`
   - `cpaas.io/dashboard.tag.overview: "true"`

## Мониторинг на уровне рабочих нагрузок

> Эта процедура демонстрирует, как просматривать мониторинг подов через интерфейс развертывания.

### Панель мониторинга по умолчанию

#### Процедура

1. **Платформа контейнеров**, нажмите **Рабочие нагрузки** > **Развертывания**.

2. Нажмите на имя развертывания из списка.

3. Перейдите на вкладку **Мониторинг**, чтобы просмотреть стандартные метрики мониторинга.

#### Интерпретация метрик

| **Мониторинг ресурса** | **Гранулярность метрик**              | **Техническое определение**                                                                                                                                                                   |
| :---------------------- | :------------------------------------ | :--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **ЦП**                  | **Использование**/**Потребление**     | **Использование** = Потребление/Лимит (ограничения)<br />Оцените конфигурацию лимитов контейнера. Высокое использование указывает на недостаточные лимиты.<br />**Потребление** представляет собой фактическое потребление ресурсов. |
| **Память**             | **Использование**/**Потребление**     | **Использование** = Потребление/Лимит (ограничения)<br />Метод оценки такой же, как и для ЦП. Высокая скорость может вызвать нестабильность компонентов.                                                                                                       |
| **Сетевой трафик**    | **Скорость входа**/**Скорость выхода** | Сетевой трафик (байты/с), поступающий в поды и выходящий из них.                                                                                                                                   |
| **Сетевой пакет**      | **Скорость получения**/**Скорость передачи** | Сетевые пакеты (количество/с), полученные и отправленные подами.                                                                                                                                      |
| **Скорость диска**     | **Чтение**/**Запись**                 | Пропускная способность чтения/записи (байты/с) примонтированных томов для каждой рабочей нагрузки.                                                                                                 |
| **Диск IOPS**          | **Чтение**/**Запись**                 | Операции ввода/вывода в секунду (IOPS) примонтированных томов для каждой рабочей нагрузки.                                                                                                        |

### Кастомизированная панель мониторинга

4. Нажмите на **Иконку переключения**, чтобы переключиться на пользовательские панели. Обратитесь к [Добавить панель в пользовательскую панель](../../observability/monitor/functions/manage_dashboard.mdx), чтобы создать выделенную панель мониторинга на уровне **Рабочих нагрузок**.

:::info
Наведите курсор на кривые графика, чтобы посмотреть метрики для каждого пода и PromQL выражения в определенные временные метки. Если количество подов превышает 15, отображаются только 15 лучших записей, отсортированных в порядке убывания.
:::
