# Логи

Агрегированные журналы времени выполнения контейнеров с возможностями визуального запроса. Когда приложения, рабочие нагрузки или другие ресурсы проявляют аномальное поведение, анализ журналов помогает диагностировать коренные причины.

## Процедура

> Эта процедура демонстрирует, как просмотреть журналы времени выполнения контейнеров через интерфейс Развертывания.

1. **Платформа контейнеров**, нажмите **Рабочие нагрузки** > **Развертывания**.

2. Щелкните по имени развертывания из списка.

3. Перейдите на вкладку **Логи**, чтобы просмотреть подробные записи.

   | Операция              | Описание                                                                                                                       |
   | :-------------------- | :----------------------------------------------------------------------------------------------------------------------------- |
   | **Pod**/**Container** | Переключение между Pod'ами и контейнерами с помощью выборщика выпадающего списка для просмотра соответствующих журналов.         |
   | **Предыдущие логи**   | Просмотр журналов от завершенных контейнеров (доступно, когда количество перезапусков контейнера > 0).                          |
   | **Строки**            | Настройка размера буфера отображения логов: 1K/10K/100K строк.                                                                |
   | **Перенос строк**     | Переключение обёртки строк для длинных записей журналов (включено по умолчанию).                                              |
   | **Найти**             | Полнотекстовый поиск с выделением совпадений и передачей для навигации.                                                      |
   | **Сырые**             | Неподготовленные потоки логов, захваченные непосредственно из интерфейсов времени выполнения контейнеров (CRI) без форматирования, фильтрации или усечения. |
   | **Экспорт**           | Скачайте сырые логи.                                                                                                         |
   | **Во весь экран**     | Щелкните по усеченной строке, чтобы просмотреть полный контент в модальном диалоговом окне.                                   |

:::warning

- **Обработка усечения**: Записи логов, превышающие 2000 символов, будут усечены с многоточием `...`
  - Обрезанные части не могут быть найдены с помощью функции поиска страницы.
  - Щелкните по маркеру многоточия `...` в усеченных строках, чтобы просмотреть полный контент в модальном диалоговом окне.
- **Надежность копирования**: Избегайте прямого копирования из визуализатора логов, когда вы видите маркеры усечения (...) или коды цветов ANSI. Всегда используйте функцию **Экспорт**, **Сырые** для получения полных логов.
- **Политика хранения**: Живые логи следуют конфигурации ротации логов Kubernetes. Для исторического анализа используйте [Логи](../../observability/log/functions/log.mdx) в разделе Наблюдение.
:::
