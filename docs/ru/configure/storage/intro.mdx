# Введение

Kubernetes предлагает гибкий и масштабируемый механизм хранения для управления сохранением данных в контейнеризованных средах. Благодаря абстракции ресурсов хранения, таких как тома, постоянные тома и запросы на постоянные тома, Kubernetes отделяет приложения от базовых систем хранения, обеспечивая динамическое распределение, автоматическое подключение и постоянные данные между узлами.

Ключевые функции включают поддержку нескольких базовых систем хранения (например, локальные диски, NFS, облачные сервисы хранения), динамическое распределение, контроль режима доступа (например, разрешения на чтение/запись) и управление жизненным циклом — что удовлетворяет потребности хранения устойчивых приложений. Для рабочих нагрузок на уровне предприятий, требующих высокой доступности, сохранения данных и изоляции многопользовательской системы, хранение Kubernetes является важной основополагающей возможностью.

Хранение Kubernetes предназначено для разработчиков, инженеров по операциям и команд платформы, помогая им эффективно и безопасно управлять данными в контейнеризованных рабочих нагрузках.
