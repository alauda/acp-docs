# Создание класса хранения типа CephFS File Storage

CephFS файловое хранилище — это встроенная система файлового хранения Ceph, которая предоставляет платформе метод доступа к хранилищу на основе интерфейса хранения контейнеров (CSI), предлагая безопасную, надежную и масштабируемую службу совместного файлового хранения, подходящую для таких сценариев, как совместное использование файлов и резервное копирование данных. Перед тем как продолжить, вам необходимо сначала создать класс хранения CephFS.

После привязки класса хранения в заявке на постоянный объем (PVC) платформа динамически создаст постоянные объемы на узлах в соответствии с заявкой на постоянный объем для бизнес-приложений.

## Развертывание плагина для объемов

После нажатия на **Развернуть** на странице **Распределенное хранилище**, [Создать службу хранения](/storage/storagesystem_ceph/installation/create_service_stand.mdx) или [Получить доступ к службе хранения](/storage/storagesystem_ceph/functions/access_storage_service.mdx).

## Создание класса хранения

1. Перейдите в **Управление платформой**.

2. В левой навигационной панели нажмите **Управление хранилищем** > **Классы хранения**.

3. Нажмите **Создать класс хранения**.

   **Примечание**: Следующее содержимое предоставлено в виде примера в формате формы; вы также можете выбрать создание с использованием YAML.

4. Выберите **CephFS File Storage** и нажмите **Далее**.

5. Настройте соответствующие параметры в соответствии со следующими инструкциями.

   | Параметр             | Описание                                                                                                                                                                                                                                                                                  |
   | -------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
   | **Политика возврата**| Политика возврата для постоянных объемов.<br />- Удалить: Когда заявка на постоянный объем удаляется, связанный постоянный объем также будет удален.<br />- Сохранить: Связанный постоянный объем останется, даже если заявка на постоянный объем будет удалена.                           |
   | **Режимы доступа**   | Все режимы доступа, поддерживаемые текущим хранилищем. Только один из этих режимов может быть выбран при объявлении постоянных объемов позже.<br />- ReadWriteOnce (RWO): Может быть смонтирован как доступ на чтение и запись одним узлом.<br />- ReadWriteMany (RWX): Может быть смонтирован как доступ на чтение и запись несколькими узлами. |
   | **Выделить проект**  | Пожалуйста, выделите проекты, которые могут использовать этот тип хранилища.<br />Если в данный момент нет проектов, которым необходимо использовать этот тип хранилища, вы можете выбрать не выделять их сейчас и обновить позже.                                                                                   |

   **Совет**: Следующие параметры необходимо установить в распределенном хранилище, и они будут применены непосредственно здесь.

   - Кластер хранения: Встроенный кластер хранения Ceph в текущем кластере.

   - Пул хранения: Логическое разделение, используемое для хранения данных в кластере хранения.

6. Нажмите **Создать**.
