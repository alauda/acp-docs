# Создание класса хранения CephFS File Storage Type

CephFS файловое хранилище — это встроенная система хранения файлов Ceph, которая предоставляет платформе способ доступа к хранилищу на основе интерфейса контейнерного хранения (CSI), предлагая безопасное, надежное и масштабируемое совместное файловое хранилище, подходящее для таких сценариев, как обмен файлами и резервное копирование данных. Перед продолжением необходимо сначала создать класс хранения CephFS.

После привязки класса хранения в заявке на постоянный объем (PVC) платформа динамически создаст постоянные объемы на узлах в соответствии с заявкой на постоянный объем для бизнес-приложений.

## Развертывание плагина объема

После нажатия на **Развернуть** на странице **Распределенное Хранилище**, [Создание службы хранения](/storage/storagesystem_ceph/installation/create_service_stand.mdx) или [Доступ к службе хранения](/storage/storagesystem_ceph/functions/access_storage_service.mdx).

## Создание класса хранения

1. Перейдите в **Управление платформой**.

2. В левой навигационной панели нажмите **Управление хранилищем** > **Классы хранения**.

3. Нажмите **Создать класс хранения**.

   **Примечание**: Следующее содержание предоставлено в виде примера в формате формы; вы также можете выбрать создание с помощью YAML.

4. Выберите **CephFS File Storage** и нажмите **Далее**.

5. Настройте соответствующие параметры в соответствии со следующими инструкциями.

   | Параметр              | Описание                                                                                                                                                                                                                                                                               |
   | --------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | **Политика восстановления**   | Политика восстановления для постоянных объемов.<br />- Удалить: Когда заявка на постоянный объем удаляется, связанный постоянный объем также будет удален.<br />- Сохранить: Связанный постоянный объем останется, даже если заявка на постоянный объем будет удалена.                          |
   | **Режимы доступа**     | Все режимы доступа, поддерживаемые текущим хранилищем. Только один из этих режимов может быть выбран при объявлении постоянных объемов позже.<br />- ReadWriteOnce (RWO): Может быть смонтирован как доступ на чтение и запись только одним узлом.<br />- ReadWriteMany (RWX): Может быть смонтирован как доступ на чтение и запись несколькими узлами. |
   | **Выделить проект** | Пожалуйста, выделите проекты, которые могут использовать этот тип хранилища.<br />Если в настоящее время нет проектов, которым необходимо использовать этот тип хранилища, вы можете выбрать, чтобы пока не выделять их и обновить позже.                                                                                         |

   **Совет**: Следующие параметры необходимо установить в распределенном хранилище и они будут применены напрямую здесь.

   - Хранилище кластера: Встроенный кластер хранения Ceph в текущем кластере.

   - Пул хранения: Логическая разделение, используемая для хранения данных в кластере хранения.

6. Нажмите **Создать**.
