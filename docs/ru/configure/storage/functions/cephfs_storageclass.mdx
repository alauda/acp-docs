# Создание класса хранения типа CephFS File Storage

Файловое хранилище CephFS — это встроенная файловая система хранения данных Ceph, которая предоставляет платформе метод доступа к хранилищу на основе интерфейса хранения контейнеров (CSI), обеспечивая безопасный, надежный и масштабируемый сервис совместного файлового хранения, подходящий для таких сценариев, как обмен файлами и резервное копирование данных. Прежде чем продолжить, необходимо создать класс хранения CephFS.

После привязки класса хранения к заявке на постоянный объем (PVC) платформа динамически создаст постоянные объемы на узлах в соответствии с заявкой на постоянный объем для бизнес-приложений.

## Развертывание плагина для объемов

После нажатия **Развернуть**, на странице **Распределенное хранилище**, [Создать сервис хранения](/storage/storagesystem_ceph/installation/create_service_stand.mdx) или [Доступ к сервису хранения](/storage/storagesystem_ceph/functions/access_storage_service.mdx).

## Создание класса хранения

1. Перейдите в **Управление платформой**.

2. В левом навигационном меню нажмите **Управление хранилищем** > **Классы хранения**.

3. Нажмите **Создать класс хранения**.

   **Примечание**: Следующий контент предоставлен в формате примера; вы также можете выбрать создание с использованием YAML.

4. Выберите **CephFS File Storage** и нажмите **Далее**.

5. Настройте соответствующие параметры в соответствии со следующими инструкциями.

   | Параметр            | Описание                                                                                                                                                                                                                                                                                     |
   | -------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | **Политика восстановления** | Политика восстановления для постоянных объемов.<br />- Удалить: Когда заявка на постоянный объем удаляется, связанный постоянный объем также будет удален.<br />- Сохранить: Связанный постоянный объем останется, даже если заявка на постоянный объем будет удалена.                       |
   | **Режимы доступа**   | Все режимы доступа, поддерживаемые текущим хранилищем. При объявлении постоянных объемов позже может быть выбран только один из этих режимов.<br />- ReadWriteOnce (RWO): Может быть смонтирован в режиме чтения и записи только одним узлом.<br />- ReadWriteMany (RWX): Может быть смонтирован в режиме чтения и записи несколькими узлами. |
   | **Проект распределения** | Пожалуйста, выделите проекты, которые могут использовать этот тип хранения.<br />Если в данный момент нет проектов, которым необходимо использовать этот тип хранения, вы можете выбрать, чтобы не выделять их сейчас и обновить позже.                                                                                          |

   **Совет**: Следующие параметры необходимо настроить в распределенном хранилище и они будут применены непосредственно здесь.

   - Кластер хранения: Встроенный кластер хранения Ceph в текущем кластере.

   - Хранилищный пул: Логическая разделение, используемое для хранения данных в кластере хранения.

6. Нажмите **Создать**.
