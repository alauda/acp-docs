# Создание класса блочного хранилища CephRBD

Блочное хранилище CephRBD - это встроенное блочное хранилище Ceph для платформы, предоставляющее метод доступа к хранилищу на основе интерфейса хранения контейнеров (CSI), который может обеспечить высокую IOPS и низкую задержку хранения, подходя для таких сценариев, как базы данных и виртуализация. Перед использованием необходимо создать класс блочного хранилища CephRBD.

Как только Запрос Постоянного Объема (PVC) связан с классом хранилища, платформа динамически создаст Постоянный Объем на основе Запроса Постоянного Объема для использования бизнес-приложениями.

## Развертывание плагина объема

После нажатия **Развернуть** на странице **Распределенное хранилище**, [создайте сервис хранилища](/storage/storagesystem_ceph/installation/create_service_stand.mdx) или [доступ к сервису хранилища](/storage/storagesystem_ceph/functions/access_storage_service.mdx).

## Создание класса хранилища

1. Перейдите в **Управление платформой**.

2. В левом навигационном меню нажмите **Управление хранилищем** > **Классы хранилища**.

3. Нажмите **Создать класс хранилища**.

   **Примечание**: Следующее содержимое является примером в формате формы, вы также можете выбрать YAML для завершения операции.

4. Выберите **Блочное хранилище CephRBD** и нажмите **Далее**.

5. Настройте параметры по необходимости.

   | Параметр          | Описание                                                                                                                                                                                                                                                   |
   | ------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | **Файловая система**    | По умолчанию **EXT4**, это журналируемая файловая система для Linux, способная предоставлять хранение расширенных файлов и обрабатывать большие файлы. Вместимость файловой системы может достигать 1 EiB, поддерживаемые размеры файлов - до 16 TiB.                                  |
   | **Политика возмещения** | Политика возмещения для постоянных объемов.<br />- Удалить: Связанный постоянный объем будет удален вместе с запросом постоянного объема.<br />- Сохранить: Связанный постоянный объем будет сохранен, даже если запрос постоянного объема будет удален. |
   | **Режимы доступа**   | Поддерживает только ReadWriteOnce (RWO): его может монтировать только один узел в режиме чтения-записи.                                                                                                                                                             |
   | **Назначить проект** | Пожалуйста, назначьте проекты, которые могут использовать этот тип хранения.<br />Если в настоящее время нет проектов, нуждающихся в этом типе хранения, вы можете не назначать их и обновить позже.                                                                     |

   **Совет**: Следующие параметры необходимо установить в распределенном хранилище и они будут напрямую применены здесь.

   - Хранилище кластера: Встроенный кластер хранилища Ceph в текущем кластере.

   - Хранилище пула: Логическое разделение, использующееся для хранения данных в пределах кластера хранения.

6. Нажмите **Создать**.
