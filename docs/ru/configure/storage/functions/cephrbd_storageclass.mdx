# Создание класса блочного хранилища CephRBD

Блочное хранилище CephRBD является встроенным блочным хранилищем Ceph для платформы, предоставляющим метод доступа к хранилищу через интерфейс хранения контейнеров (CSI), который может обеспечить высокую производительность ввода-вывода и низкую задержку, что делает его подходящим для таких сценариев, как базы данных и виртуализация. Перед использованием этого хранилища необходимо создать класс блочного хранилища CephRBD.

После того как запрос на постоянный объем (PVC) будет связан с классом хранилища, платформа динамически создаст постоянный объем на основе запроса на постоянный объем для использования бизнес-приложениями.

## Развертывание плагина объема

После нажатия **Развернуть** на странице **Распределенное хранилище**, [создайте сервис хранения](/storage/storagesystem_ceph/installation/create_service_stand.mdx) или [доступ к сервису хранения](/storage/storagesystem_ceph/functions/access_storage_service.mdx).

## Создание класса хранилища

1. Перейдите в **Управление платформой**.

2. В левой навигационной панели нажмите **Управление хранилищем** > **Классы хранилища**.

3. Нажмите **Создать класс хранилища**.

   **Примечание**: Следующее содержимое является примером в формате формы, вы также можете выбрать YAML для выполнения операции.

4. Выберите **Блочное хранилище CephRBD** и нажмите **Далее**.

5. Настройте параметры по мере необходимости.

   | Параметр           | Описание                                                                                                                                                                                                                                               |
   | ------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | **Файловая система**| По умолчанию **EXT4**, что является журналируемой файловой системой для Linux, способной обеспечивать хранение и обработку больших файлов. Вместимость файловой системы может достигать 1 EiB, поддерживаемые размеры файлов до 16 TiB.                                  |
   | **Политика восстановления** | Политика восстановления для постоянных объемов.<br />- Удалить: Связанный постоянный объем будет удален вместе с запросом на постоянный объем.<br />- Сохранить: Связанный постоянный объем будет сохранен, даже если запрос на постоянный объем будет удален. |
   | **Режимы доступа** | Поддерживает только ReadWriteOnce (RWO): его может монтировать только один узел в режиме чтения-записи.                                                                                                                                                                 |
   | **Назначить проект** | Пожалуйста, назначьте проекты, которые могут использовать этот тип хранилища.<br />Если в настоящее время нет проектов, нуждающихся в этом типе хранилища, вы можете не назначать его и обновить позже.                                                                     |

   **Совет**: Следующие параметры необходимо установить в распределенном хранилище и они будут непосредственно применены здесь.

   - Кластер хранения: Встроенный кластер хранения Ceph в текущем кластере.

   - Пул хранения: Логическое разделение, используемое для хранения данных в пределах кластера хранения.

6. Нажмите **Создать**.
