# Создание класса блочного хранилища CephRBD

Блочное хранилище CephRBD является встроенным блочным хранилищем Ceph для платформы, предоставляющим метод доступа к хранилищу, основанный на интерфейсе хранения контейнеров (CSI), который может обеспечить высокую производительность ввода-вывода и услуги по хранению с низкой задержкой, подходящие для таких сценариев, как базы данных и виртуализация. Перед использованием этого хранилища необходимо создать класс блочного хранилища CephRBD.

После того как Запрос постоянного объема (PVC) будет связан с классом хранилища, платформа динамически создаст Постоянный объем на основе Запроса постоянного объема для использования бизнес-приложениями.

## Установка плагина Volume

После нажатия **Развернуть**, на странице **Распределенное хранилище**, [создайте сервис хранения](/storage/storagesystem_ceph/installation/create_service_stand.mdx) или [доступ к сервису хранения](/storage/storagesystem_ceph/functions/access_storage_service.mdx).

## Создание класса хранилища

1. Перейдите в **Управление платформой**.

2. В левом навигационном баре нажмите **Управление хранилищем** > **Классы хранения**.

3. Нажмите **Создать класс хранения**.

   **Примечание**: Следующий контент является примером в формате формы, вы также можете выбрать YAML для выполнения операции.

4. Выберите **Блочное хранилище CephRBD** и нажмите **Далее**.

5. Настройте параметры по мере необходимости.

   | Параметр          | Описание                                                                                                                                                                                                                                               |
   | ------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | **Файловая система**    | По умолчанию **EXT4**, которая является журналируемой файловой системой для Linux, способной предоставлять расширенное файловое хранилище и обрабатывать большие файлы. Вместимость файловой системы может достигать 1 EiB, при этом поддерживаемые размеры файлов - до 16 TiB.                                  |
   | **Политика восстановления** | Политика восстановления для постоянных объемов.<br />- Удалить: Связанный постоянный объем будет удален вместе с запросом постоянного объема.<br />- Сохранить: Связанный постоянный объем будет сохранен даже если запрос постоянного объема удален. |
   | **Режимы доступа**   | Поддерживает только ReadWriteOnce (RWO): он может быть смонтирован одним узлом в режиме чтения-записи.                                                                                                                                                                 |
   | **Назначить проект** | Пожалуйста, назначьте проекты, которые могут использовать этот тип хранения.<br />Если в настоящее время нет проектов, которым требуется этот тип хранения, вы можете выбрать, чтобы не назначать один и обновить его позже.                                                                     |

   **Совет**: Следующие параметры необходимо установить в распределенном хранилище и они будут применены здесь напрямую.

   - Кластер хранения: Встроенный кластер хранения Ceph в текущем кластере.

   - Пул хранения: Логическая часть, используемая для хранения данных внутри кластера хранения.

6. Нажмите **Создать**.
