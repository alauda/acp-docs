# Режим сети ALB

Экземпляр ALB может быть развернут в двух режимах: режим сети хоста и режим сети контейнеров.

## Режим сети хоста

Непосредственное использование сетевого стека узла, совместное использование IP-адреса и порта с узлом.

В этом режиме экземпляр балансировщика нагрузки напрямую связывается с портом узла, без сопоставления портов или аналогичного преобразования сетевой инкапсуляции контейнера.

**Примечание**: Чтобы избежать конфликтов портов, на одном узле может быть развернут только один экземпляр ALB.

<img src="../../../../en/configure/networking/assets/albhostnet.png" width="500" />

В режиме сети хоста экземпляр ALB по умолчанию будет прослушивать все сетевые интерфейсы узла.

### Преимущества:

1. Лучшая сетевое производительность.
2. Доступ через IP-адрес узла.

### Недостатки:

1. На одном узле может быть развернут только один экземпляр ALB.
2. Порт может конфликтовать с другими процессами.

## Режим сети контейнеров

<img src="../../../../en/configure/networking/assets/albcontainnet.png" width="500" />

В отличие от режима сети хоста, режим сети контейнеров развертывает ALB с использованием сетевого взаимодействия контейнеров.

### Преимущества:

1. Поддерживает развертывание нескольких экземпляров ALB на одном узле.
2. ALB предоставляет интеграцию с MetalLB, который может предоставить VIP для ALB.
3. Порт не будет конфликтовать с другими процессами.

### Недостатки:

1. Немного менее высокая производительность.
2. Доступ к ALB обязателен через сервис LoadBalancer.
