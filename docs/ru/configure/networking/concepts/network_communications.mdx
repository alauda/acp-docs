# Сравнение различных методов Ingress

Платформа контейнеров Alauda поддерживает несколько спецификаций входящего трафика в экосистеме Kubernetes. Этот документ сравнивает их ([Служба](../functions/configure_service.mdx), [Ingress](../functions/configure_ingress.mdx), [Gateway API](./gatewayapi.mdx) и [Правило ALB](../how_to/create_loadbalancer.mdx#alb_rule)), чтобы помочь пользователям сделать правильный выбор.

## Для трафика L4 (TCP/UDP)

Службы типа LoadBalancer, Gateway API и Правила ALB могут все открывать L4 трафик для внешнего доступа. Здесь мы рекомендуем использовать подход с типом службы LoadBalancer. Оба Gateway API и Правила ALB реализуются с помощью ALB, который является прокси в пространстве пользователей, и их производительность значительно ухудшается при обработке L4 трафика по сравнению с службами типа LoadBalancer.

## Для трафика L7 (HTTP/HTTPS)

Хотя Ingress, GatewayAPI и Правила ALB могут все открывать L7 трафик для внешнего доступа, они различаются по своим возможностям и моделям изоляции.

### Ingress

Ingress является стандартной спецификацией, принятой сообществом Kubernetes, и рекомендуется для использования по умолчанию. Ingress обрабатывается экземплярами ALB, которые управляются администратором платформы.

### GatewayAPI

GatewayAPI предоставляет более гибкий режим изоляции, однако они не так зрелы, как Ingress. Используя GatewayAPI, разработчик может создать свои собственные изолированные экземпляры ALB для обработки правил GatewayAPI. Поэтому, если вам нужно делегировать создание и управление экземплярами ALB разработчикам, вам нужно выбрать использование GatewayAPI.

### Правило ALB

Правило ALB (Балансировщик нагрузки в интерфейсе) предоставляет самые гибкие правила сопоставления трафика и наибольшие возможности. На самом деле, как Ingress, так и GatewayAPI реализуются путем их преобразования в Правила ALB. Однако Правило ALB более сложное, чем Ingress и GatewayAPI, и не является стандартным API сообщества. Поэтому мы рекомендуем использовать его только в том случае, если Ingress и GatewayAPI не соответствуют вашим требованиям.
