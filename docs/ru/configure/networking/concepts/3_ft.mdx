# ALB Фронтенд

Мы определяем ресурс, называемый фронтендом (сокращенно ft), который используется для объявления всех портов, на которых должны прослушиваться все alb.

```yaml
apiVersion: crd.alauda.io/v1
kind: Frontend
metadata:
  labels:
    alb2.cpaas.io/name: alb-demo # обязательно, указывает на экземпляр ALB, к которому принадлежит этот фронтенд
  name: alb-demo-00080           # формат $alb_name-$port
spec:
  backendProtocol: "http"
  certificate_name: ""  # формат $secret_ns/$secret_name
  port: 80              
  protocol: http        # протокол самого этого фронтенда. http|https|grpc|grpcs для l7 прокси. tcp|udp для l4 прокси
  serviceGroup:         # может содержать несколько сервисов.
    services:
    - name: hello-world
      namespace: default
      port: 80
      weight: 100
```

## Как L4 прокси

Для l4 прокси serviceGroup является обязательным.

## Как L7 прокси

Для l7 прокси serviceGroup является необязательным. Когда запрос поступает, ALB сначала попытается сопоставить его с правилами, связанными с этим фронтендом. Только если запрос не соответствует ни одному правилу, ALB перенаправит его на группу сервисов по умолчанию, указанную в конфигурации фронтенда.
