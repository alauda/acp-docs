# Понимание ALB

ALB (Another Load Balancer) — это шлюз Kubernetes, работающий на базе OpenResty и имеющий многолетний опыт эксплуатации от Alauda.

## Основные компоненты

![](../assets/alb_architecture.png)

- **ALB Operator**: Оператор, который управляет жизненным циклом экземпляров ALB. Он отвечает за наблюдение за ALB CR и затем создает и обновляет экземпляры ALB для различных арендаторов.
- **ALB Instance**: Экземпляр ALB включает Openresty, который действует как план данных, и контроллер на Go в качестве плана управления. 
  Контроллер на Go отслеживает различные CR (Ingress, Gateway, Rule и т.д.) и преобразует их в специфические для ALB правила DSL. Затем OpenResty использует эти правила DSL для сопоставления и обработки входящих запросов.
