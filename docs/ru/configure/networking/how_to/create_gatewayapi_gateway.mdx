# Создание шлюза GatewayAPI

GatewayAPI — это новый API для Kubernetes, который предоставляет более гибкий и расширяемый способ управления входящим трафиком. Он позволяет вам определять правила маршрутизации, политики трафика и другие конфигурации более декларативным образом.\
Этот документ предоставляет пошаговое руководство по созданию шлюза GatewayAPI в кластере Kubernetes платформы Alauda Container.

# Требования

## Развертывание MetalLB

Шлюз GatewayAPI требует MetalLB для выделения IP-адреса. Пожалуйста, обратитесь к [Создание MetalLB](../functions/create_metallb.mdx) для получения инструкций по развертыванию MetalLB.

## Установите политики безопасности Pod в режим привилегий

Если пространство имен, в котором вы хотите развернуть шлюз, создано через интерфейс пользователя, вам необходимо обновить его Политику безопасности Pod (PSP) до режима привилегий.\
![Обновить PSP](../assets/update_psp.png)

# Процедура

1. Перейдите в **Управление платформой**.

2. В левой боковой панели нажмите на **Управление сетью** > **Входящие шлюзы**.

3. Нажмите на **Создать входящие шлюзы**.

4. Следуйте приведенным ниже инструкциям, чтобы завершить настройку сети:

   | Параметр         | Описание                                                                                                                                                                                            |
   | ----------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
   | **Имя**          | Имя шлюза.                                                                                                                                                                               |
   | **GatewayClass**  | Встроенный `exclusive-gateway` предоставляется платформой Alauda Container и поддерживается ALB. Он создаст ALB в режиме контейнерной сети для реализации спецификации шлюза GatewayAPI.            |
   | **Спецификация** | Установите спецификации в соответствии с вашими бизнес-потребностями. Вы также можете обратиться к [Как правильно выделить ресурсы CPU и памяти](../how_to/decide_performance_selection.mdx) для получения рекомендаций. |

5. Нажмите **Создать**. Процесс создания может занять некоторое время; пожалуйста, наберитесь терпения.

![Создать шлюз](../assets/create_gatewayapi_gateway.png)
