# Автоматическое соединение подслоёв и наложенных подсетей

Если кластер имеет как подслои, так и наложенные подсети, по умолчанию, Pods в наложенной подсети могут получать доступ к IP-адресам Pods в подслое через шлюз, используя NAT. Однако Pods в подслое должны настроить маршрутизацию узлов для доступа к Pods в наложенной подсети.

Для достижения автоматического соединения между подслоем и наложенной подсетью вы можете вручную изменить YAML файл подслоя. После настройки Kube-OVN также будет использовать дополнительный IP-адрес подслоя для соединения подслоя и логического маршрутизатора ovn-cluster, устанавливая соответствующие правила маршрутизации для включения соединения.

### Шаги

1. Перейдите в **Управление платформой**.

2. В левой панели навигации нажмите **Управление кластером** > **Управление ресурсами**.

3. Введите **Подсеть** для фильтрации объектов ресурса.

4. Нажмите ⋮ > **Обновить** рядом с подслоем, который нужно изменить.

5. Измените YAML файл, добавив поле `u2oInterconnection: true` в `Spec`.

6. Нажмите **Обновить**.

**Примечание**: Существующие вычислительные компоненты в подслое необходимо воссоздать, чтобы изменения вступили в силу.
