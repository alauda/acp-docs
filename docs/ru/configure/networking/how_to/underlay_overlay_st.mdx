# Автоматическое соединение подсетей Underlay и Overlay

Если кластер имеет как подсети Underlay, так и подсети Overlay, по умолчанию Pods в подсети Overlay могут получать доступ к IP-адресам Pods в подсети Underlay через шлюз, используя NAT. Однако Pods в подсети Underlay должны настроить маршрутизацию узлов для доступа к Pods в подсети Overlay.

Чтобы достичь автоматического соединения между подсетями Underlay и Overlay, вы можете вручную изменить YAML-файл подсети Underlay. После настройки Kube-OVN также будет использовать дополнительный IP-адрес Underlay для подключения подсети Underlay и логического маршрутизатора ovn-cluster, устанавливая соответствующие правила маршрутизации для обеспечения соединения.

### Шаги

1. Перейдите в раздел **Управление платформой**.

2. В левой панели навигации кликните на **Управление кластером** > **Управление ресурсами**.

3. Введите **Подсеть**, чтобы отфильтровать объекты ресурсов.

4. Нажмите на ⋮ > **Обновить** рядом с подсетью Underlay, которую нужно изменить.

5. Измените YAML-файл, добавив поле `u2oInterconnection: true` в раздел `Spec`.

6. Нажмите **Обновить**.

**Примечание**: Существующие вычислительные компоненты в подсети Underlay необходимо воссоздать, чтобы изменения вступили в силу.
