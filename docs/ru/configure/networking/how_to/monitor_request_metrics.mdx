# Мониторинг ALB

## Терминология

| Термин | Описание                                             |
| ------ | ---------------------------------------------------- |
| ALB    | Самостоятельно разработанный балансировщик нагрузки уровня 7 платформы. |

## Шаги операции

1. Перейдите в **Управление платформой**.

2. В левом навигационном меню нажмите на **Операционный центр** > **Мониторинг** > **Мониторинговая панель**.

3. Нажмите на **Кластер** в верхней части страницы, чтобы переключиться на кластер, который вы хотите мониторить.

4. Нажмите на **Переключить** в правом верхнем углу страницы.

5. Вы можете войти в мониторинговую панель **Статус ALB** двумя способами:

   - Метод 1: Нажмите на карточку **container-platform**, чтобы развернуть каталог мониторинга, затем нажмите на название **Статус ALB**, чтобы войти в мониторинговую панель. При необходимости вы можете установить эту мониторинговую панель в качестве основной.

   - Метод 2: Введите ключевое слово (например, alb) в строку поиска и выполните поиск, затем нажмите на название **Статус ALB**, чтобы войти в мониторинговую панель. При необходимости вы можете установить эту мониторинговую панель в качестве основной.

6. Просмотрите различные метрики мониторинга через панель.

   - **Выберите пространство имен для мониторинга**: Нажмите на **пространство имен** в верхней части страницы, чтобы выбрать пространство имен для мониторинга, по умолчанию выбирается все, что означает мониторинг всех пространств имен.

   - **Выберите ALB для мониторинга**: Нажмите на **имя** в верхней части страницы, чтобы выбрать ALB для мониторинга, по умолчанию выбирается все, что означает мониторинг всех ALB.

## Метрики мониторинга

Отображает метрики мониторинга общего трафика, использования ресурсов, Ingress (входящие правила), HTTPRoute (правила маршрутизации типа HTTPRoute) и Rule (правила, которые не являются ни Ingress, ни HTTPRoute) для выбранного ALB за **последние 5 минут**.

**Примечание**: Все данные являются данными мониторинга, собранными за **последние 5 минут**.

### Мониторинг трафика ALB

| Метрика мониторинга       | Описание                                                                                          |
| ------------------------- | ------------------------------------------------------------------------------------------------ |
| **Активные соединения**   | Количество активных соединений на выбранном ALB.                                                |
| **Запросы в секунду**    | Общее количество запросов, полученных за секунду на выбранном ALB.                                |
| **Уровень ошибок**       | Доля запросов с ошибками 4XX (например, 404) и 5XX, возникающих за секунду на выбранном ALB. |
| **Задержка**             | Средняя задержка запросов на выбранном ALB.                                                     |

### Использование ресурсов ALB

| Метрика мониторинга            | Описание                                     |
| ------------------------------ | --------------------------------------------- |
| **Использование CPU**          | Использование CPU выбранного ALB.            |
| **Использование памяти**       | Использование памяти выбранного ALB.         |
| **Сетевой прием/передача**    | Пропускная способность сетевого ввода/вывода выбранного ALB. |
| **Скорость чтения/записи на диск** | Пропускная способность ввода/вывода на диск выбранного ALB. |

### Мониторинг трафика Ingress, HTTPRoute, Rule

| Метрика мониторинга                            | Описание                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| ---------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **QPS (Запросы в секунду)**                   | Количество запросов, полученных за секунду от Ingress/HTTPRoute/Rule на выбранном ALB, с единицей измерения по умолчанию req/s.                                                                                                                                                                                                                                                                                                                                                                                                                      |
| **Запрос BPS (Байты в секунду)**              | Общий размер запросов, полученных за секунду от Ingress/HTTPRoute/Rule на выбранном ALB.                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| **Ответ BPS (Байты в секунду)**               | Общий размер ответов, отправленных Ingress/HTTPRoute/Rule на выбранном ALB.                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| **Уровень ошибок**                             | Процент ошибок, возникших при обработке запросов Ingress/HTTPRoute/Rule на выбранном ALB.                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| **P50, P90, P99**                             | Время ответа для запросов на выбранном ALB, в частности, медианное время ответа. Это означает, что 50%, 90% и 99% запросов имеют время ответа меньше или равно этому значению. <br /><br /><b>Примечание</b>: Принцип P50, P90 и P99 заключается в сортировке собранных данных от наименьшего к наибольшему и взятии значений данных на 50%, 90% и 99% позициях; таким образом, 50%, 90% и 99% собранных данных находятся ниже этого значения. Перцентили помогают анализировать распределение данных и выявлять различные экстремальные ситуации. |
| **Upstream P50, Upstream P90, Upstream P99** | Время ответа запросов для upstream-сервисов. Это означает, что 50%, 90% и 99% запросов, отправленных на upstream-сервисы, имеют время ответа меньше или равно этому значению.                                                                                                                                                                                                                                                                                                                                                                     |
