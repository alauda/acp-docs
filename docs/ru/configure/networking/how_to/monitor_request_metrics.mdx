# Мониторинг ALB

## Терминология

| Термин | Описание                                                   |
| ------ | --------------------------------------------------------- |
| ALB    | Самостоятельно разработанный балансировщик нагрузки уровня 7 платформы. |

## Шаги операции

1. Перейдите в раздел **Управление платформой**.

2. В левой боковой панели навигации кликните на **Операционный центр** > **Мониторинг** > **Мониторинговая панель**.

3. Нажмите на **Кластер** в верхней части страницы, чтобы переключиться на нужный вам кластер для мониторинга.

4. Нажмите на **Переключить** в правом верхнем углу страницы.

5. Вы можете войти на панель мониторинга **Статус ALB** двумя способами:

   - Метод 1: Нажмите на карточку **container-platform**, чтобы развернуть каталог мониторинга, затем кликните на название **Статус ALB**, чтобы перейти на панель мониторинга. При необходимости вы можете установить эту панель мониторинга в качестве основной.

   - Метод 2: Введите ключевое слово (например, alb) в строку поиска и выполните поиск, затем нажмите на название **Статус ALB**, чтобы перейти на панель мониторинга. При необходимости вы можете установить эту панель мониторинга в качестве основной.

6. Просматривайте различные метрики мониторинга через панель инструментов.

   - **Выберите пространство имен для мониторинга**: Нажмите на **пространство имен** в верхней части страницы, чтобы выбрать пространство имен для мониторинга, по умолчанию выбрано все, что означает мониторинг всех пространств имен.

   - **Выберите ALB для мониторинга**: Нажмите на **имя** в верхней части страницы, чтобы выбрать ALB для мониторинга, по умолчанию выбрано все, что означает мониторинг всех ALB.

## Метрики мониторинга

Отображаются метрики мониторинга общего трафика, использования ресурсов, Ingress (входящие правила), HTTPRoute (маршрутизирующие правила типа HTTPRoute) и Rule (правила, которые не являются ни Ingress, ни HTTPRoute) для выбранного ALB за **последние 5 минут**.

**Примечание**: Все данные являются данными мониторинга, собранными за **последние 5 минут**.

### Мониторинг трафика ALB

| Метрика мониторинга       | Описание                                                                                                  |
| ------------------------- | ---------------------------------------------------------------------------------------------------------- |
| **Активные соединения**   | Количество активных соединений на выбранном ALB.                                                          |
| **Запросы в секунду**    | Общее количество запросов, поступивших за секунду на выбранный ALB.                                      |
| **Уровень ошибок**       | Доля ошибочных запросов 4XX (таких как 404) и 5XX, происходящих за секунду на выбранном ALB.             |
| **Задержка**              | Средняя задержка запросов на выбранном ALB.                                                               |

### Использование ресурсов ALB

| Метрика мониторинга            | Описание                                          |
| ------------------------------ | ------------------------------------------------ |
| **Использование ЦП**          | Использование ЦП выбранного ALB.                 |
| **Использование памяти**       | Использование памяти выбранного ALB.              |
| **Сетевой прием / передача**  | Пропускная способность сетевого ввода-вывода выбранного ALB. |
| **Скорость чтения/записи диска** | Пропускная способность ввода-вывода диска выбранного ALB.  |

### Мониторинг трафика Ingress, HTTPRoute, Rule

| Метрика мониторинга                          | Описание                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| -------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **QPS (Запросы в секунду)**                 | Количество запросов, поступивших за секунду на Ingress/HTTPRoute/Rule на выбранном ALB, при этом основной единицей является req/s.                                                                                                                                                                                                                                                                                                                                                                                  |
| **Размер запросов в BPS (Байты в секунду)** | Общий размер запросов, поступивших за секунду на Ingress/HTTPRoute/Rule на выбранном ALB.                                                                                                                                                                                                                                                                                                                                                                                                                             |
| **Размер ответов в BPS (Байты в секунду)**  | Общий размер ответов, отправленных Ingress/HTTPRoute/Rule на выбранном ALB.                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| **Уровень ошибок**                           | Процент ошибок, произошедших при обработке запросов Ingress/HTTPRoute/Rule на выбранном ALB.                                                                                                                                                                                                                                                                                                                                                                                                                          |
| **P50, P90, P99**                           | Время отклика для запросов на выбранном ALB, в частности, медианное время ответа. Это означает, что 50%, 90% и 99% запросов имеют время ответа меньше или равно этому значению.<br /><br /><b>Примечание</b>: Принцип P50, P90 и P99 заключается в том, чтобы отсортировать собранные данные от меньшего к большему и взять значения данных на позициях 50%, 90% и 99%; таким образом, 50%, 90% и 99% собранных данных находятся ниже этого значения. Перцентили помогают проанализировать распределение данных и выявить различные экстремальные ситуации. |
| **Отправка P50, Отправка P90, Отправка P99** | Время отклика запросов к upstream-сервисам. Это означает, что 50%, 90% и 99% запросов, отправленных к upstream-сервисам, имеют время ответа меньше или равно этому значению.                                                                                                                                                                                                                                                                                                                                            |
