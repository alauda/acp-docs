# Мониторинг ALB

## Терминология

| Термин | Описание                                             |
| ------ | ----------------------------------------------------- |
| ALB    | Собственный балансировщик нагрузки 7 уровня, разработанный платформой. |

## Шаги операции

1. Перейдите в **Управление платформой**.

2. В левой навигационной панели нажмите на **Центр операций** > **Мониторинг** > **Приборная панель мониторинга**.

3. Нажмите на **Кластер** в верхней части страницы, чтобы переключиться на кластер, который вы хотите мониторить.

4. Нажмите на **Переключить** в правом верхнем углу страницы.

5. Вы можете войти в приборную панель мониторинга состояния **ALB** двумя способами:

   - Метод 1: Нажмите на карточку **container-platform**, чтобы развернуть каталог мониторинга, затем нажмите на имя **ALB Status**, чтобы перейти в приборную панель мониторинга. При необходимости вы можете установить эту приборную панель как основную.

   - Метод 2: Введите ключевое слово (например, alb) в поле поиска и выполните поиск, затем нажмите на имя **ALB Status**, чтобы перейти в приборную панель мониторинга. При необходимости вы можете установить эту приборную панель как основную.

6. Просмотрите различные метрики мониторинга через приборную панель.

   - **Выберите пространство имен для мониторинга**: Нажмите на **namespace** в верхней части страницы, чтобы выбрать пространство имен для мониторинга, по умолчанию это все, что означает мониторинг всех пространств имен.

   - **Выберите ALB для мониторинга**: Нажмите на **name** в верхней части страницы, чтобы выбрать ALB для мониторинга, по умолчанию это все, что означает мониторинг всех ALB.

## Метрики мониторинга

Отображает метрики мониторинга общего трафика, использования ресурсов, Ingress (входящие правила), HTTPRoute (правила маршрутизации типа HTTPRoute) и Rule (правила, которые не являются ни Ingress, ни HTTPRoute) для выбранного ALB за **последние 5 минут**.

**Примечание**: Все данные являются данными мониторинга, собранными за **последние 5 минут**.

### Мониторинг трафика ALB

| Метрика мониторинга      | Описание                                                                                          |
| ------------------------ | -------------------------------------------------------------------------------------------------- |
| **Активные соединения**  | Количество активных соединений на выбранном ALB.                                                  |
| **Запросы в секунду**   | Общее количество запросов, полученных за секунду на выбранном ALB.                                |
| **Процент ошибок**       | Доля запросов с ошибками 4XX (такими как 404) и 5XX, возникающих за секунду на выбранном ALB.  |
| **Задержка**            | Средняя задержка запросов на выбранном ALB.                                                       |

### Использование ресурсов ALB

| Метрика мониторинга        | Описание                                     |
| -------------------------- | --------------------------------------------- |
| **Использование CPU**      | Использование CPU выбранного ALB.             |
| **Использование памяти**   | Использование памяти выбранного ALB.          |
| **Сеть Прием/Передача**    | Пропускная способность сетевого ввода/вывода выбранного ALB. |
| **Скорость чтения/записи диска** | Пропускная способность ввода/вывода диска выбранного ALB.   |

### Мониторинг трафика Ingress, HTTPRoute, Rule

| Метрика мониторинга                            | Описание                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| ---------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **QPS (Запросы в секунду)**                   | Количество запросов, полученных в секунду Ingress/HTTPRoute/Rule на выбранном ALB, единицей по умолчанию является req/s.                                                                                                                                                                                                                                                                                                                                                                                                                      |
| **Запрос BPS (Байты в секунду)**              | Общий размер запросов, полученных за секунду Ingress/HTTPRoute/Rule на выбранном ALB.                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| **Ответ BPS (Байты в секунду)**               | Общий размер ответов, отправленных Ingress/HTTPRoute/Rule на выбранном ALB.                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| **Процент ошибок**                            | Процент ошибок, возникающих при обработке запросов Ingress/HTTPRoute/Rule на выбранном ALB.                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| **P50, P90, P99**                             | Время ответа для запросов на выбранном ALB, конкретно медианное время ответа. Это означает, что 50%, 90% и 99% запросов имеют время ответа меньше или равно этому значению. <br /><br /><b>Примечание</b>: Принцип P50, P90 и P99 заключается в сортировке собранных данных от наименьших к наибольшим и взятии значений данных на 50%, 90% и 99% позициях; таким образом, 50%, 90% и 99% собранных данных находятся ниже этого значения. Квантели помогают анализировать распределение данных и выявлять различные экстремальные ситуации. |
| **Upstream P50, Upstream P90, Upstream P99** | Времена отклика запроса дляUpstream-сервисов. Это означает, что 50%, 90% и 99% запросов, отправленных в Upstream-сервисы, имеют времена отклика меньше или равны этому значению.                                                                                                                                                                                                                                                                                                                                                                     |
