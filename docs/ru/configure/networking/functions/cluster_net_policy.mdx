# Настройка сетевых политик кластера

Сетевые политики кластера отвечают за управление правилами контроля доступа на **уровне проекта**. Когда эта функция включена, проекты по умолчанию изолированы друг от друга, и вычислительные компоненты в разных проектах не могут взаимодействовать по сети. Коммуникация может быть достигнута путем добавления правил **доступа к одному проекту** или **доступа по IP-сегменту**.

После настройки сетевые политики кластера будут синхронизированы с пространствами имен в кластере и могут быть просмотрены в модуле функции **Сетевые политики** контейнерной платформы.

## Заметки

- Эффективность сетевых политик кластера зависит от того, поддерживает ли сетевой плагин, используемый в кластере, сетевые политики.

  - Kube-OVN и Calico поддерживают сетевые политики.
  - Flannel не поддерживает сетевые политики.
  - При доступе к кластеру или использовании пользовательского сетевого плагина вы можете обратиться к соответствующей документации для подтверждения поддержки.

- Функция находится в альфа-стадии под сетевым режимом Kube-OVN.

## Шаги

1. Перейдите в **Управление платформой**.

2. В левой навигационной панели нажмите **Управление сетью** > **Сетевые политики кластера**.

3. Нажмите **Настроить сейчас**.

4. Следуйте инструкциям ниже, чтобы завершить соответствующую настройку.

   | Элемент настройки                     | Описание                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
   | ------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
   | **Полная изоляция между проектами**  | Включение переключателя полной изоляции между проектами, который включен по умолчанию и может быть отключен по нажатию. При включении достигается сетевая изоляция между всеми проектами в текущем кластере, и другим ресурсам не разрешается доступ к каким-либо проектам в кластере (например, внешним IP-адресам, балансировщикам нагрузки). Это не влияет на доступ проектов к ресурсам вне кластера.                                                  |
   | **Доступ к одному проекту**          | Этот параметр эффективен только при включенном переключателе **Полная изоляция между проектами**.<br />Настройте **исходный проект** и **целевой проект** для одностороннего доступа.<br />Нажмите **Добавить**, чтобы добавить запись конфигурации, поддерживающую несколько записей.<br />В выпадающем списке **исходный проект** выберите проект, который будет получать доступ к целевому проекту, или выберите все проекты; в выпадающем списке **целевой проект** выберите целевой проект, к которому следует получить доступ.     |
   | **Доступ по IP-сегменту**            | Этот параметр эффективен только при включенном переключателе **Полная изоляция между проектами**.<br />Настройте конкретный **IP/сегмент** и **целевой проект** для одностороннего доступа.<br />Нажмите **Добавить**, чтобы добавить запись конфигурации, поддерживающую несколько записей.<br />В поле ввода **исходный IP-сегмент** введите IP-адрес или CIDR-сегмент для доступа к целевому проекту; в выпадающем списке **целевой проект** выберите целевой проект, к которому следует получить доступ.          |

5. Нажмите **Настроить**.
