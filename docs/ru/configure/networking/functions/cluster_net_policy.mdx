# Настройка сетевых политик кластера

Сетевые политики кластера отвечают за управление правилами контроля доступа на уровне **проекта**. Когда эта функция активирована, различные проекты изолированы друг от друга по умолчанию, и вычислительные компоненты в разных проектах не могут взаимодействовать друг с другом через сеть. Связь может быть установлена путем добавления правил **доступа к одному проекту** или **доступа по IP-сегменту**.

После настройки сетевые политики кластера будут синхронизированы с пространствами имен в кластере и могут быть просмотрены в модуле функциональности **Сетевые политики** контейнерной платформы.

## Примечания

- Эффективность сетевых политик кластера зависит от того, поддерживает ли используемый кластером сетевой плагин сетевые политики.

  - Kube-OVN и Calico поддерживают сетевые политики.
  - Flannel не поддерживает сетевые политики.
  - При доступе к кластеру или использовании пользовательского сетевого плагина вы можете обратиться к соответствующей документации для подтверждения поддержки.

- Данная функциональность имеет альфа-стадию зрелости в сетевом режиме Kube-OVN.

## Шаги

1. Перейдите в **Управление платформой**.

2. В левой навигационной панели нажмите на **Управление сетью** > **Сетевые политики кластера**.

3. Нажмите **Настроить сейчас**.

4. Следуйте приведенным ниже инструкциям для завершения соответствующей настройки.

   | Пункт настройки                         | Описание                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
   | --------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
   | **Полная изоляция между проектами**    | Нужно ли включить переключатель полной изоляции между проектами, который включен по умолчанию и может быть выключен путем нажатия. При включении достигается сетевая изоляция между всеми проектами в текущем кластере, и другим ресурсам не разрешается доступ к любому проекту внутри кластера (например, внешние IP-адреса, балансировщики нагрузки). Это не влияет на доступ проектов к ресурсам вне кластера.                                                     |
   | **Доступ к одному проекту**            | Этот параметр имеет эффект только когда переключатель **Полная изоляция между проектами** включен.<br />Настройте **исходный проект** и **целевой проект** для одностороннего доступа.<br />Нажмите **Добавить**, чтобы добавить запись конфигурации, поддерживающую несколько записей.<br />В раскрывающемся списке **исходный проект** выберите проект, который будет иметь доступ к целевому проекту, или выберите все проекты; в раскрывающемся списке **целевой проект** выберите целевой проект для доступа. |
   | **Доступ по IP-сегменту**              | Этот параметр имеет эффект только когда переключатель **Полная изоляция между проектами** включен.<br />Настройте конкретный **IP/сегмент** и **целевой проект** для одностороннего доступа.<br />Нажмите **Добавить**, чтобы добавить запись конфигурации, поддерживающую несколько записей.<br />В поле ввода **исходного IP-сегмента** введите IP-адрес или CIDR-сегмент для доступа к целевому проекту; в раскрывающемся списке **целевой проект** выберите целевой проект для доступа.          |

5. Нажмите **Настроить**.
