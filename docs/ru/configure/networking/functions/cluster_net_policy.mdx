# Настройка сетевых политик кластера

Сетевые политики кластера отвечают за управление правилами контроля доступа на уровне **проекта**. Когда эта функция включена, различные проекты по умолчанию изолированы друг от друга, и вычислительные компоненты в разных проектах не могут взаимодействовать друг с другом по сети. Связь может быть установлена путем добавления правил **Доступ между проектами** или **Доступ с использованием IP-адресов**.

После настройки сетевые политики кластера будут синхронизированы с пространствами имен в кластере и могут быть просмотрены в модуле функции **Сетевые политики** контейнерной платформы.

## Примечания

- Эффективность сетевых политик кластера зависит от того, поддерживает ли сетевой плагин, используемый кластером, сетевые политики.

  - Kube-OVN и Calico поддерживают сетевые политики.
  - Flannel не поддерживает сетевые политики.
  - При доступе к кластеру или использовании пользовательского сетевого плагина вы можете обратиться к соответствующей документации для подтверждения поддержки.

- Функциональность находится на стадии Alpha в режиме сети Kube-OVN.

## Шаги

1. Перейдите в **Управление платформой**.

2. В левой навигационной панели нажмите **Сеть** > **Сетевые политики кластера**.

3. Нажмите **Настроить сейчас**.

4. Следуйте приведенным ниже инструкциям, чтобы завершить соответствующую настройку.

   | Элемент конфигурации          | Описание                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
   | ------------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | **Изоляция между проектами**   | Включает ли полный переключатель изоляции между проектами, который по умолчанию отключен и может быть включен нажатием. При включении достигается сетевая изоляция между всеми проектами в текущем кластере, и другим ресурсам не разрешается доступ к любому проекту внутри кластера (например, внешние IP-адреса, балансировщики нагрузки). Это не влияет на доступ проектов к ресурсам вне кластера.                                            |
   | **Доступ между проектами**     | Этот параметр эффективен только при включенном переключателе **Изоляция между проектами**.<br />Настройте **Исходный проект** и **Целевой проект** для одностороннего доступа.<br />Нажмите **Добавить**, чтобы добавить запись конфигурации, поддерживающую несколько записей.<br />В выпадающем списке **Исходный проект** выберите проект, который будет получать доступ к целевому проекту, или выберите все проекты; в выпадающем списке **Целевой проект** выберите целевой проект для доступа. |
   | **Доступ с использованием IP-адресов** | Этот параметр эффективен только при включенном переключателе **Изоляция между проектами**.<br />Настройте конкретный **Исходный IP-сегмент** и **Целевой проект** для одностороннего доступа.<br />Нажмите **Добавить**, чтобы добавить запись конфигурации, поддерживающую несколько записей.<br />В поле ввода **Исходный IP-сегмент** введите IP или CIDR-сегмент для доступа к целевому проекту; в выпадающем списке **Целевой проект** выберите целевой проект для доступа.   |

5. Нажмите **Настроить**.
