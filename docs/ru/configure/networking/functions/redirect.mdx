# HTTP Перенаправление

## Основная концепция

HTTP перенаправление — это функция, предоставляемая ALB. Она напрямую возвращает код HTTP 30x для запроса, соответствующего правилу. Заголовок Location будет использоваться для указания клиенту на перенаправление на новый URL.

ALB поддерживает конфигурацию перенаправления на уровне порта и правил.

## CRD

```yaml
redirect:
  properties:
    code:
      type: integer
    host:
      type: string
    port:
      type: integer
    prefix_match:
      type: string
    replace_prefix:
      type: string
    scheme:
      type: string
    url:
      type: string
  type: object
```

Перенаправление может быть настроено на:

- Фронтенд: `.spec.config.redirect`
- Правило: `.spec.config.redirect`

## Аннотация Ingress

| Аннотация                                          | Описание                                                        |
| --------------------------------------------------- | ---------------------------------------------------------------- |
| nginx.ingress.kubernetes.io/permanent-redirect      | Соответствует URL в CR, по умолчанию устанавливает код 301       |
| nginx.ingress.kubernetes.io/permanent-redirect-code | Соответствует коду в CR                                        |
| nginx.ingress.kubernetes.io/temporal-redirect       | Соответствует URL в CR, по умолчанию устанавливает код 302       |
| nginx.ingress.kubernetes.io/temporal-redirect-code  | Соответствует коду в CR                                        |
| nginx.ingress.kubernetes.io/ssl-redirect            | Соответствует схеме в CR, по умолчанию устанавливает схему HTTPS |
| nginx.ingress.kubernetes.io/force-ssl-redirect      | Соответствует схеме в CR, по умолчанию устанавливает схему HTTPS |

### SSL-Redirect

1. SSL-redirect и force-ssl-redirect различаются тем, что SSL-redirect вступает в силу только тогда, когда у ingress есть сертификат для соответствующего домена, в то время как force-ssl-redirect вступает в силу независимо от наличия сертификата.
2. Для портов HTTPS, если настроен только SSL-redirect, перенаправление не будет установлено.

## Перенаправление на уровне порта

Когда перенаправление настроено на уровне порта, *все запросы* к этому порту будут перенаправлены в соответствии с конфигурацией перенаправления.

## Перенаправление на уровне правила

Когда перенаправление настроено на уровне правила, запрос, соответствующий этому правилу, будет перенаправлен в соответствии с конфигурацией перенаправления.
