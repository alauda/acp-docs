# Создание сертификата

После того как администратор платформы импортирует сертификат TLS и назначит его определенному проекту, разработчики с соответствующими правами доступа к проекту могут использовать сертификат, импортированный и назначенный администратором платформы, при использовании входящих правил и функций балансировки нагрузки. В дальнейшем, в таких сценариях, как истечение срока действия сертификата, администратор платформы может централизованно обновить сертификат.

## Примечания

Функциональность сертификата в настоящее время не поддерживается для использования в кластерах публичного облака. Вы можете создавать секретные словари типа TLS по мере необходимости в пределах указанного пространства имен.

## Шаги

1. Перейдите в **Управление платформой**.

2. В левой навигационной панели нажмите **Управление сетью** > **Сертификаты**.

3. Нажмите **Создать сертификат**.

4. Ознакомьтесь с инструкциями ниже для настройки соответствующих параметров.

   | Параметр          | Описание                                                                                                                                                                                                                                                                                                                                                                                                     |
   | ------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | **Назначить проект** | <ul><li>Все проекты: Назначить сертификат для использования во всех проектах, связанных с текущим кластером.</li><li>Указанный проект: Назначить сертификат для использования в указанном проекте.</li><li>Без назначения: Не назначать проект на данный момент. После завершения создания сертификата вы можете обновить проекты, которые могут использовать сертификат, с помощью операции **Обновить проект**.</li></ul> |
   | **Открытый ключ**     | Это относится к tls.crt. При импорте открытого ключа двоичные файлы не поддерживаются.                                                                                                                                                                                                                                                                                                                          |
   | **Закрытый ключ**    | Это относится к tls.key. При импорте закрытого ключа двоичные файлы не поддерживаются.                                                                                                                                                                                                                                                                                                                         |

5. Нажмите **Создать**.
