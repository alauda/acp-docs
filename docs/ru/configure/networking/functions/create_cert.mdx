# Создание сертификатов

После того как администратор платформы импортирует TLS-сертификат и назначит его конкретному проекту, разработчики с соответствующими разрешениями смогут использовать сертификат, импортированный и назначенный администратором платформы, при использовании входящих правил и функций балансировки нагрузки. В дальнейшем, в таких ситуациях, как истечение срока действия сертификата, администратор платформы может обновить сертификат централизованно.

:::note
Функциональность сертификата в настоящее время не поддерживается для использования в кластерах публичного облака. Вы можете создавать секретные словари типа TLS по мере необходимости в указанном пространстве имен.
:::

## Создание сертификата с помощью веб-консоли

1. Перейдите в **Управление платформой**.

2. В левом навигационном меню щелкните **Управление сетью** > **Сертификаты**.

3. Нажмите **Создать сертификат**.

4. Ознакомьтесь с инструкциями ниже, чтобы настроить соответствующие параметры.

   | Параметр           | Описание                                                                                                                                                                                                                                                                                                                                                                                                          |
   | ------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | **Назначить проект** | <ul><li>Все проекты: Назначить сертификат для использования во всех проектах, связанных с текущим кластером.</li><li>Указанный проект: Назначить сертификат для использования в указанном проекте.</li><li>Без назначения: Временно не назначать проект. После завершения создания сертификата вы можете обновить проекты, которые могут использовать сертификат через операцию **Обновить проект**.</li></ul>      |
   | **Открытый ключ**  | Это относится к tls.crt. При импорте открытого ключа двоичные файлы не поддерживаются.                                                                                                                                                                                                                                                                                                                           |
   | **Закрытый ключ**  | Это относится к tls.key. При импорте закрытого ключа двоичные файлы не поддерживаются.                                                                                                                                                                                                                                                                                                                        |

5. Нажмите **Создать**.
