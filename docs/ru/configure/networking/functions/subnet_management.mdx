# Управление подсетями

## Управление сетями Kube-OVN\{#ovn}

Управляйте подсетями в сети Kube-OVN, включая транспортные сети Underlay и Overlay.

### Обновление шлюза

Это включает в себя изменение метода исходящего трафика, узлов шлюза и конфигурации NAT.

1. Перейдите в **Управление платформой**.

2. В левой боковой панели выберите **Управление сетью** > **Подсети**.

3. Нажмите на имя подсети.

4. Выберите **Действие** > **Обновить шлюз**.

5. Обновите параметры конфигурации; для получения подробной информации смотрите [Описание параметров](./subnet_kubeovn_overlay.mdx).

6. Нажмите **ОК**.

### Обновление зарезервированных IP

IP-адрес шлюза не может быть удален из зарезервированных IP, в то время как другие зарезервированные IP могут быть отредактированы, удалены или добавлены свободно.

1. Перейдите в **Управление платформой**.

2. В левой боковой панели выберите **Управление сетью** > **Подсети**.

3. Нажмите на имя подсети.

4. Выберите **Действие** > **Обновить зарезервированный IP**.

5. После завершения обновлений нажмите **Обновить**.

### Назначение проектов

Назначение подсетей конкретным проектам помогает командам лучше управлять и изолировать сетевой трафик для различных проектов, обеспечивая каждому проекту достаточные сетевые ресурсы.

1. Перейдите в **Управление платформой**.

2. В левой боковой панели выберите **Управление сетью** > **Подсети**.

3. Нажмите на имя подсети.

4. Выберите **Действие** > **Назначить проект**.

5. После добавления или удаления проектов нажмите **Назначить**.

### Назначение пространств имен

Назначение подсетей конкретным пространствам имен позволяет осуществлять более детальную сетевую изоляцию.

**Примечание**: Процесс назначения пересоздаст шлюз, и исходящие данные будут отброшены! Пожалуйста, убедитесь, что в данный момент никакие бизнес-приложения не обращаются к внешним кластерам.

1. Перейдите в **Управление платформой**.

2. В левой боковой панели выберите **Управление сетью** > **Подсети**.

3. Нажмите на имя подсети.

4. Выберите **Действие** > **Назначить пространство имен**.

5. После добавления или удаления пространств имен нажмите **Назначить**.

### Расширение подсетей

Когда диапазон зарезервированных IP подсети достигает предела использования или собирается истощиться, его можно расширить на основе исходного диапазона подсети без влияния на нормальную работу существующих сервисов.

1. Перейдите в **Управление платформой**.

2. В левой боковой панели выберите **Управление сетью** > **Подсети**.

3. Нажмите на имя подсети.

4. Выберите **Действие** > **Расширить подсеть**.

5. Завершите конфигурацию и нажмите **Обновить**.
