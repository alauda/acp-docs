# Как решить проблемы межузловой связи в средах ARM?

При использовании более ранних версий ядра и определенных отечественных сетевых карт может возникнуть проблема, когда сетевой адаптер неверно вычисляет контрольные суммы после включения оффлода контрольных сумм. Это может привести к сбоям в связи между узлами в сети Kube-OVN Overlay. Конкретные решения следующие:

- **Решение 1: Обновите версию ядра**. Рекомендуется обновить версию ядра до 4.19.90-25.16.v2101 или более поздней версии.

- **Решение 2: Отключите оффлод контрольных сумм**. Если обновление версии ядра невозможно сделать немедленно и возникают проблемы межузловой связи, вы можете отключить оффлод контрольных сумм для физической сетевой карты, используя следующую команду.

  ```
  ethtool -K eth0 tx off
  ```
