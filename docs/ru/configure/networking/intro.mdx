# Введение

Сетевое решение контейнеров представляет собой комплексное решение для сетевого взаимодействия, предназначенное для облачно-ориентированных приложений, обеспечивая бесперебойную коммуникацию в направлении восток-запад внутри кластеров и эффективное управление трафиком в направлении север-юг через внешние сети, одновременно предоставляя необходимые сетевые функциональности. Оно состоит из следующих основных компонентов:

- Интерфейсы сетей контейнеров (CNI) для управления трафиком в направлении восток-запад внутри кластера.
- Контроллер шлюза входа ALB для управления входящим HTTPS-трафиком.
- MetalLB для обработки служб типа LoadBalancer.
- Кроме того, оно предоставляет надежные функции сетевой безопасности и шифрования для обеспечения безопасной связи.

## Преимущества

Сетевое решение контейнеров предлагает следующие основные преимущества:

- **Гибкое управление сетью**

  С поддержкой нескольких CNI, сетевое решение контейнеров поддерживает как наложенные, так и подложенные сети и режимы маршрутизации, предоставляя гибкость для адаптации к разнообразным сетевым окружениям. Оно также предлагает детальную IP-распределение и надежное управление исходящим трафиком. Как основная команда Kube-OVN, мы обладаем обширным практическим опытом в создании и поддержке крупномасштабных сетей, обеспечивая надежное и продуктивное соединение.

- **Изоляция, многоарендность и гибкость API для шлюза входа**

  С оператором ALB можно создавать и управлять несколькими экземплярами ALB в одном кластере. Каждому арендатору может быть предоставлена специализированная группа экземпляров ALB в качестве шлюза входа, что гарантирует эффективную изоляцию и управление ресурсами. Кроме того, пользователи могут гибко выбирать между Ingress и Gateway API в зависимости от своих предпочтений и операционных требований, обеспечивая бесперебойное управление трафиком и повышенную гибкость. Как основная команда ALB, мы можем гарантировать надежное и масштабируемое решение.

- **Комплексная сетевая безопасность**

  Сетевое решение контейнеров предоставляет многоуровневую структуру безопасности для обеспечения защиты на всех уровнях. На уровне CNI мы поддерживаем несколько моделей политик безопасности, включая NetworkPolicy и AdminNetworkPolicy, для обеспечения детального контроля доступа к сети. Для безопасной передачи данных сеть включает надежное шифрование трафика. На уровне шлюза входа мы предоставляем расширенные механизмы защиты, такие как завершение TLS и поддержку ModSecurity, обеспечивая всестороннюю защиту для приложений, работающих с внешним-facing. С встроенным обеспечением политик сети, шифрованием и мониторингом трафика, она обеспечивает защиту от несанкционированного доступа и поддерживает соответствие стандартам безопасности.

## Сценарии применения

Сетевое решение контейнеров особенно подходит для следующих сценариев:

- **Управление трафиком в направлении восток-запад**

  Использование CNI для обеспечения эффективного взаимодействия между контейнерами внутри кластеров, с поддержкой как наложенных, так и подложенных сетевых режимов для удовлетворения различных потребностей в развертывании.

- **Контроль трафика в направлении север-юг**

  Использование ALB в качестве контроллера шлюза входа для управления внешним HTTPS-трафиком, с гибкими выборами API и возможностями изоляции многоаренды для различных команд.

- **Вывод услуги балансировщика нагрузки**

  Использование MetalLB для обеспечения высокой доступности для служб типа LoadBalancer, позволяя надежный внешний доступ к службам кластера через виртуальные IP-адреса.

- **Сетевая безопасность и шифрование**

  Реализация комплексной безопасности через NetworkPolicy, AdminNetworkPolicy и шифрование трафика для обеспечения безопасной связи по всей сетевой инфраструктуре.

## Ограничения использования

Несмотря на то, что сетевое решение контейнеров предоставляет обширные функциональные возможности, следует отметить следующие ограничения:

- **Требования к подложной сети**

  Некоторые возможности подложной сети, такие как Kube-OVN Underlay Subnet, Egress IP и MetalLB, требуют поддержки подлежащей L2 сети. Эти функции не могут исполняться в публичных облачных провайдерах и некоторых виртуализированных средах, таких как AWS и GCP.

С своим универсальным дизайном и комплексным набором функций, сетевое решение контейнеров позволяет организациям разрабатывать, масштабировать и управлять безопасными, надежными и высокопроизводительными контейнеризированными приложениями.
