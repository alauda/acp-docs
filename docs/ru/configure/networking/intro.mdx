# Введение

Сетевая инфраструктура контейнеров — это комплексное сетевое решение, разработанное для облачно-ориентированных приложений, обеспечивающее бесшовную восточно-западную связь внутри кластеров и эффективное управление северо-южным трафиком через внешние сети, при этом предоставляя основные сетевые функции. Она состоит из следующих ключевых компонентов:

- Интерфейсы сетей контейнеров (CNI) для управления восточно-западным трафиком внутри кластера.
- Контроллер шлюза входа ALB для управления HTTPS-трафиком входа.
- MetalLB для обработки служб типа LoadBalancer.
- Кроме того, она предоставляет надежные функции сетевой безопасности и шифрования для обеспечения безопасной связи.

## Преимущества

Сетевая инфраструктура контейнеров предлагает следующие основные преимущества:

- **Гибкое управление сетью**

  С поддержкой нескольких CNI, сетевая инфраструктура контейнеров поддерживает как режимы наложения, так и подложки, а также маршрутизации, обеспечивая гибкость для адаптации к разнообразным сетевым средам. Она также предлагает детализированное распределение IP-адресов и надежное управление исходящим трафиком. Как команда-основатель Kube-OVN, мы обладаем обширным практическим опытом в создании и поддержке крупных сетей, обеспечивая надежное и производительное соединение.

- **Изоляция, многопользовательская среда и гибкость API для шлюза входа**

  С оператором ALB можно создавать и управлять несколькими экземплярами ALB в одном кластере. Каждый арендатор может иметь выделенную группу экземпляров ALB в качестве шлюза входа, обеспечивая эффективную изоляцию и управление ресурсами. Кроме того, пользователи могут гибко выбирать между Ingress и Gateway API в зависимости от своих предпочтений и операционных требований, обеспечивая бесшовное управление трафиком и повышенную гибкость. Как команда-основатель ALB, мы можем гарантировать надежное и масштабируемое решение.

- **Комплексная сетевая безопасность**

  Сетевая инфраструктура контейнеров предоставляет многоуровневую структуру безопасности для обеспечения защиты на всех уровнях. На уровне CNI мы поддерживаем несколько моделей политик безопасности, включая NetworkPolicy и AdminNetworkPolicy, для обеспечения детализированного контроля доступа к сети. Для безопасной передачи данных сеть включает надежное шифрование трафика. На уровне шлюза входа мы предоставляем продвинутые механизмы безопасности, такие как завершение TLS и поддержку ModSecurity, предлагая комплексную защиту для внешних приложений. С встроенным обеспечением соблюдения сетевых политик, шифрованием и мониторингом трафика она обеспечивает защиту от несанкционированного доступа и соблюдение стандартов безопасности.

## Сценарии применения

Сетевая инфраструктура контейнеров особенно подходит для следующих сценариев:

- **Управление восточно-западным трафиком**

  Используя CNI для обеспечения эффективной связи между подами внутри кластеров, с поддержкой как режимов наложения, так и подложки для удовлетворения различных потребностей развертывания.

- **Контроль северо-южного трафика**

  Используя ALB в качестве контроллера шлюза входа для управления внешним HTTPS-трафиком, с гибкими выборами API и возможностями изоляции для разных команд.

- **Вывод службы балансировщика нагрузки**

  Используя MetalLB для обеспечения высокой доступности служб типа LoadBalancer, позволяя надежный внешний доступ к службам кластера через виртуальные IP-адреса.

- **Сетевая безопасность и шифрование**

  Реализуя комплексную безопасность через NetworkPolicy, AdminNetworkPolicy и шифрование трафика для обеспечения безопасной связи по всей сетевой инфраструктуре.

## Ограничения использования

Хотя сетевая инфраструктура контейнеров предоставляет обширные функции, следует отметить следующие ограничения:

- **Требование к подложной сети**

  Некоторые возможности подложной сети, такие как Kube-OVN Underlay Subnet, Egress IP и MetalLB, требуют поддержки базовой L2 сети. Эти функции не могут быть использованы у публичных облачных провайдеров и в определенных виртуализированных средах, таких как AWS и GCP.

С своим универсальным дизайном и комплексным набором функций сетевая инфраструктура контейнеров предоставляет организациям возможность создавать, масштабировать и управлять безопасными, надежными и высокопроизводительными контейнеризованными приложениями.
