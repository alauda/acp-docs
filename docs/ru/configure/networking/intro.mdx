# Введение

Сеть контейнеров - это комплексное сетевое решение, разработанное для облачно-нативных приложений, обеспечивая бесшовную восточно-западную связь внутри кластеров и эффективное управление северо-южным трафиком через внешние сети, при этом предоставляя основные сетевые функции. Она состоит из следующих основных компонентов:

- Сетевые интерфейсы контейнеров (CNI) для управления восточно-западным трафиком внутри кластера.
- Контроллер шлюза входа ALB для управления HTTPS трафиком на входе.
- MetalLB для обработки услуг типа LoadBalancer.
- Кроме того, она обеспечивает надежные функции сетевой безопасности и шифрования для обеспечения безопасной связи.

## Преимущества

Сеть контейнеров предлагает следующие основные преимущества:

- **Гибкое управление сетью**

  С поддержкой нескольких CNI, сеть контейнеров поддерживает как оверлейные, так и подложные режимы, обеспечивая гибкость для адаптации к разнообразным сетевым средам. Она также предлагает тонкую настройку распределения IP и надежное управление уходящим трафиком. В качестве основателей Kube-OVN, мы обладаем обширным практическим опытом в создании и обслуживании крупных сетей, обеспечивая надежную и производительную связь.

- **Изоляция, многопользовательский доступ и гибкость API для шлюза входа**

  С оператором ALB можно создать и управлять несколькими экземплярами ALB в одном кластере. Каждый арендатор может иметь выделенную группу экземпляров ALB в качестве шлюза входа, что обеспечивает эффективную изоляцию и управление ресурсами. Кроме того, пользователи могут гибко выбирать между API управления входом и шлюзом в зависимости от своих предпочтений и эксплуатационных требований, что обеспечивает бесшовное управление трафиком и повышенную гибкость. Как основатели ALB, мы можем гарантировать надежное и масштабируемое решение.

- **Комплексная сетевая безопасность**

  Сеть контейнеров предоставляет многоуровневую структуру безопасности для обеспечения защиты на всех уровнях. На уровне CNI мы поддерживаем несколько моделей политики безопасности, включая NetworkPolicy и AdminNetworkPolicy, для обеспечения тонкого управления доступом к сети. Для безопасной передачи данных сеть включает надежное шифрование трафика. На уровне шлюза входа мы предоставляем расширенные механизмы безопасности, такие как завершение TLS и поддержку ModSecurity, обеспечивая всестороннюю защиту внешних приложений. С встроенным принудительным выполнением политик сети, шифрованием и мониторингом трафика, она защищает от несанкционированного доступа и поддерживает соблюдение стандартов безопасности.

## Сценарии применения

Сеть контейнеров особенно подходит для следующих сценариев:

- **Управление восточно-западным трафиком**

  Используя CNI для обеспечения эффективной связи между подами внутри кластеров, с поддержкой как оверлейных, так и подложных сетевых режимов для удовлетворения различных потребностей в развертывании.

- **Контроль северо-южного трафика**

  Используя ALB в качестве контроллера шлюза входа для управления внешним HTTPS трафиком, с гибкими выборами API и возможностями многопользовательской изоляции для разных команд.

- **Вывод услуги с балансировкой нагрузки**

  Используя MetalLB для обеспечения высокой доступности услуг типа LoadBalancer, позволяя надежный внешний доступ к кластерным службам через виртуальные IP-адреса.

- **Сетевая безопасность и шифрование**

  Реализуя комплексную безопасность через [NetworkPolicy](./functions/configure_network_policy.mdx), [AdminNetworkPolicy](./functions/admin_network_policy.mdx) и шифрование трафика для обеспечения безопасной связи по всей сетевой инфраструктуре.

## Ограничения использования

Хотя сеть контейнеров предоставляет широкий спектр функций, следует учитывать следующие ограничения:

- **Требование к подложной сети**

  Некоторые функциональные возможности подложной сети, такие как Kube-OVN Underlay Subnet, Egress IP и MetalLB, требуют поддержки L2 сети. Эти функции нельзя использовать в публичных облачных провайдерах и определенных виртуализованных средах, таких как AWS и GCP.

С своим универсальным дизайном и комплексным набором функций сеть контейнеров предлагает организациям возможность строить, масштабировать и управлять безопасными, надежными и высокопроизводительными контейнеризованными приложениями.
