# Введение

<Term name="product" /> (<Term name="productShort" />) — это облачная платформа, в которой большинство функциональностей реализуются через операторы Kubernetes. Документация API Reference для <Term name="productShort" /> предоставляет исчерпывающую информацию о доступных API в <Term name="productShort" />.

## Типы API

<Term name="productShort" /> предлагает два типа API:

### Kubernetes API

Большинство возможностей <Term name="productShort" /> доступны через API Kubernetes. Эти API доступны с помощью стандартных клиентов Kubernetes, таких как kubectl, client-go и других. Для полного списка этих API обратитесь к разделу [Kubernetes API](../kubernetes_apis/).

Для получения информации о том, как вызывать эти API, включая стандартные шаблоны и примеры, пожалуйста, обратитесь к [Руководству по использованию API Kubernetes](./kubernetes_api_usage).

### Расширенные API

Некоторые функциональности, такие как запросы логов, не подходят для экспонирования через API Kubernetes. Для этих случаев <Term name="productShort" /> предоставляет RESTful API, к которым можно получить доступ через HTTP-запросы. Для полного списка этих API обратитесь к разделу [Расширенные API](../advanced_apis/).

## Аутентификация API

Как Kubernetes API, так и Расширенные API <Term name="productShort" /> используют токен пользователя <Term name="productShort" /> для аутентификации и авторизации. Вам нужно только добавить токен в заголовок `Authorization` вашего HTTP-запроса:

```bash
curl -X GET "http://<acp-api-server-address>/apis/v1/namespaces/default/pods" \
     -H "Authorization: Bearer <your-token>"
```

### Получение токена пользователя <Term name="productShort" />

Чтобы получить токен пользователя <Term name="productShort" />, выполните следующие шаги:

1. Войдите в веб-консоль <Term name="productShort" />, нажмите на имя пользователя в верхнем правом углу для раскрытия выпадающего меню и нажмите на пункт меню **Профиль**, чтобы получить доступ к странице личного кабинета.

2. На странице личного кабинета нажмите на вкладку **API токены**, а затем нажмите кнопку **Добавить API токен** для создания нового токена.

3. В появившемся диалоговом окне введите описание токена и установите время истечения, затем нажмите кнопку **Добавить**, чтобы создать токен.

4. После успешного создания система сгенерирует токен и отобразит его на странице.

5. Скопируйте токен в буфер обмена и надежно сохраните его для будущего использования при вызове API ACP.
