# Введение

<Term name="product" /> (<Term name="productShort" />) - это облачная платформа, в которой большинство функциональных возможностей реализованы с помощью операторов Kubernetes. Документация по API <Term name="productShort" /> Reference предоставляет полную информацию о доступных API в <Term name="productShort" />.

## Типы API

<Term name="productShort" /> предлагает два типа API:

### Kubernetes API

Большинство возможностей <Term name="productShort" /> предоставляются через API Kubernetes. Эти API доступны с помощью стандартных клиентов Kubernetes, таких как kubectl, client-go и других. Для получения полного списка этих API обратитесь к разделу [Kubernetes APIs](../kubernetes_apis/).

Для информации о том, как вызывать эти API, включая стандартные схемы и примеры, пожалуйста, обратитесь к [Kubernetes API Usage Guide](./kubernetes_api_usage).

### Расширенные API

Некоторые функциональные возможности, такие как запросы журналов, не подходят для предоставления через Kubernetes API. Для этих случаев <Term name="productShort" /> предоставляет RESTful API, к которым можно получить доступ через HTTP-запросы. Для полного списка этих API обратитесь к разделу [Advanced APIs](../advanced_apis/).

## Аутентификация API

Как Kubernetes API, так и Расширенные API <Term name="productShort" /> используют токен пользователя <Term name="productShort" /> для аутентификации и авторизации. Вам нужно только добавить токен в заголовок `Authorization` в вашем HTTP-запросе:

```bash
curl -X GET "http://<acp-api-server-address>/apis/v1/namespaces/default/pods" \
     -H "Authorization: Bearer <your-token>"
```

### Получение токена пользователя <Term name="productShort" />

Чтобы получить токен пользователя <Term name="productShort" />, выполните следующие шаги:

1. Войдите в веб-консоль <Term name="productShort" />, кликните на ваше имя пользователя в правом верхнем углу, чтобы открыть выпадающее меню, и кликните на пункт **Профиль**, чтобы получить доступ к странице личного центра.

2. На странице личного центра кликните на вкладку **API Tokens** и нажмите кнопку **Добавить токен API**, чтобы создать новый токен.

3. В появившемся диалоговом окне введите описание токена и установите время истечения срока, затем нажмите кнопку **Добавить**, чтобы создать токен.

4. После успешного создания система сгенерирует токен и отобразит его на странице.

5. Скопируйте токен в буфер обмена и сохраните его в безопасном месте для последующего использования при вызове API ACP.
