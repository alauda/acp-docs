# Обзор

Этот гид по обновлению охватывает две части: обновление **глобального кластера** и обновление **кластеров нагрузки**.

Для глобальных и кластеров нагрузки на странице деталей кластера доступна вкладка **Компоненты функций**. Эта вкладка перечисляет все компоненты, которые можно обновить при выполнении обновления кластера, включая:

- версию Kubernetes
- контейнерный рантайм
- плагины кластера и операторы, предоставленные платформой

Когда новая версия любого из перечисленных компонентов становится доступной, кнопка **Обновить** будет активирована. Пользователи могут инициировать процесс обновления, нажав на кнопку.

<Directive type="info" title="ИНФОРМАЦИЯ">
  - **Версия Kubernetes:** Обновления Kubernetes поддерживаются только для **локальных кластеров**. Для **управляемых кластеров** (например, Amazon Elastic Kubernetes Service, Azure Kubernetes Service) обновления Kubernetes должны выполняться через соответствующего облачного провайдера. Для получения дополнительной информации о определениях и различиях между **локальными кластерами** и **управляемыми кластерами** см. [Тип кластера](../configure/clusters/overview.mdx#cluster_type).
  - **Оператор:** Только **операторы, предоставленные платформой**, перечислены и могут быть обновлены с помощью функции обновления кластера. **Операторы третьих сторон или установленные пользователем** управляются через компонент **OLM** на рынке и не включены в этот процесс обновления.
  - **Плагин кластера:** Плагины, предоставленные платформой, могут быть обновлены через функцию обновления кластера как на **локальных**, так и на управляемых кластерах, при условии, что они установлены.
  - **DR:** DR - это сокращение от *Окружение восстановления после катастроф*. Он включает как основной глобальный кластер, так и резервный глобальный кластер, в то время как стандартная среда <Term name="productShort" /> будет иметь только один глобальный кластер.
  - **основной глобальный кластер:** Резервный глобальный кластер является приблизительной копией основного глобального кластера. Чтобы различать их, определим основной глобальный кластер как тот, к которому разрешается доменное имя доступа <Term name="productShort" />.
  - **резервный глобальный кластер:** Определим резервный глобальный кластер как тот, к которому доменное имя доступа <Term name="productShort" /> НЕ разрешается.
</Directive>
