# Мониторинг ресурсов GPU

## Обзор функции

Функция мониторинга ресурсов позволяет осуществлять реальный и исторический отслеживание использования GPU и объема памяти на узлах и подах в контейнерной платформе. Эта функциональность помогает администраторам и разработчикам:

- **Мониторинг производительности GPU**: Определение узких мест в распределении ресурсов GPU.
- **Устранение неполадок**: Анализ тенденций использования GPU для отладки проблем, связанных с ресурсами.
- **Оптимизация рабочих нагрузок**: Принятие обоснованных на фактах решений для улучшения распределения рабочих нагрузок.

**Применимые сценарии**:

- Мониторинг приложений с высокой нагрузкой на GPU в реальном времени.
- Исторический анализ использования GPU для планирования ресурсов.
- Сравнение производительности GPU на нескольких узлах/подах.

**Доставляемая ценность**:

- Улучшенная видимость потребления ресурсов GPU.
- Повышенная эффективность кластера за счет действенных рекомендаций.

## Основные функции

- **Мониторинг на уровне узлов**: Отслеживание использования GPU и памяти по узлам.
- **Мониторинг на уровне подов**: Мониторинг метрик GPU для отдельных подов.
- **Пользовательские временные рамки**: Анализ данных за период от 30 минут до 7 дней.

## Преимущества функции

- **Визуализация в реальном времени**: Интерактивные панели управления с возможностями автоматического обновления.
- **Многомерная фильтрация**: Узкая фильтрация метрик по типу GPU, пространству имен или поду.

## Мониторинг узлов

Мониторьте ресурсы GPU на уровне узлов с помощью следующих шагов:

<Steps>
  ### Доступ к панелям управления GPU

  1. Перейдите в раздел **Управление платформой**
  2. Перейдите в **Центр операций → Мониторинг → Панели управления**
  3. Перейдите в каталог **GPU**

  ### Выбор метрик узлов

  1. Выберите панель управления **Мониторинг узлов**
  2. Выберите целевой узел из выпадающего списка
  3. Выберите временной диапазон:
     - Последние 30 минут
     - Последние 1/6/12/24 часов
     - Последние 2/7 дней
     - Пользовательский диапазон

  ### Интерпретация метрик

  | Метрика               | Описание                                         |
  | --------------------- | ------------------------------------------------- |
  | **Использование GPU**  | Процент использования вычислительных мощностей GPU (0-100%)  |
  | **Использование памяти GPU** | Объем памяти, используемой по сравнению с доступной памятью (в GiB) |
</Steps>

## Мониторинг подов

Анализируйте использование GPU на уровне подов с детальной фильтрацией:

<Steps>
  ### Доступ к метрикам подов

  1. Перейдите к панелям управления каталога **GPU**
  2. Выберите **Мониторинг подов**

  ### Настройка фильтров

  1. Выберите тип GPU:
     - pGPU
     - GPU-Manager(vGPU)
     - MPS
  2. Выберите пространство имен, в котором находятся поды GPU
  3. Выберите конкретный под

  ### Ключевые метрики

  | Метрика                    | Описание                                         |
  | -------------------------- | ------------------------------------------------- |
  | **Использование GPU пода** | Использование вычислительных мощностей GPU выбранным подом  |
  | **Память GPU пода**       | Выделение памяти для выбранного пода             |
</Steps>

## Выбор временного диапазона

```yaml

- Последние 30 минут
- Последний 1 час
- Последние 6 часов
- Последние 12 часов
- Последние 24 часа
- Последние 2 дня
- Последние 7 дней
- Пользовательский диапазон
```
