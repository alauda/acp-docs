# Мониторинг ресурсов GPU

## Обзор функционала

Функция мониторинга ресурсов позволяет отслеживать использование GPU и оперативную память в реальном времени и в историческом контексте по узлам и подам внутри Контейнерной Платформы. Эта функциональность помогает администраторам и разработчикам:

- **Мониторить производительность GPU**: Определять узкие места в распределении ресурсов GPU.
- **Устранять проблемы**: Анализировать тенденции использования GPU для отладки проблем, связанных с ресурсами.
- **Оптимизировать рабочие нагрузки**: Принимать обоснованные решения для улучшения распределения трудозатрат.

**Применимые сценарии**:

- Мониторинг приложений с интенсивным использованием GPU в реальном времени.
- Исторический анализ использования GPU для планирования емкости.
- Сравнение производительности GPU на нескольких узлах/падах.

**Доставляемая ценность**:

- Улучшенная видимость потребления ресурсов GPU.
- Повышенная эффективность кластера за счет ценной информации.

## Основные функции

- **Мониторинг на уровне узла**: Отслеживание использования GPU и оперативной памяти по узлам.
- **Мониторинг на уровне пода**: Мониторинг метрик GPU для отдельных подов.
- **Настраиваемые временные диапазоны**: Анализ данных от 30 минут до 7 дней.

## Преимущества функции

- **Визуализация в реальном времени**: Интерактивные приборные панели с автоматическим обновлением.
- **Многоуровневая фильтрация**: Уточнение метрик по типу GPU, пространству имен или поду.

## Мониторинг узлов

Мониторинг ресурсов GPU на уровне узлов осуществляется следующими шагами:

<Steps>
  ### Доступ к панелям мониторинга GPU

  1. Перейдите в представление **Управление платформой**
  2. Перейдите в **Операционный центр → Мониторинг → Панели управления**
  3. Переключитесь в директорию **GPU**

  ### Выбор метрик узла

  1. Выберите панель мониторинга **Мониторинг узлов**
  2. Выберите целевой узел из выпадающего списка
  3. Выберите временной диапазон:
     - Последние 30 минут
     - Последние 1/6/12/24 часа
     - Последние 2/7 дней
     - Настраиваемый диапазон

  ### Интерпретация метрик

  | Метрика               | Описание                                         |
  | --------------------- | ------------------------------------------------ |
  | **Использование GPU**  | Процент использованной вычислительной мощности GPU (0-100%)  |
  | **Использование памяти GPU** | Общая память, использованная по сравнению с доступной памятью (в GiB) |
</Steps>

## Мониторинг подов

Анализ использования GPU на уровне подов с детальной фильтрацией:

<Steps>
  ### Доступ к метрикам подов

  1. Перейдите в панели управления директории **GPU**
  2. Выберите **Мониторинг подов**

  ### Настройка фильтров

  1. Выберите тип GPU:
     - pGPU
     - GPU-менеджер (vGPU)
     - MPS
  2. Выберите пространство имен, содержащее поды GPU
  3. Выберите конкретный под

  ### Ключевые метрики

  | Метрика                   | Описание                         |
  | ------------------------- | -------------------------------- |
  | **Использование GPU пода** | Использование вычислительных ресурсов GPU выбранным подом  |
  | **Память GPU пода**      | Выделение памяти для выбранного пода |
</Steps>

## Выбор временного диапазона

Обе панели поддерживают гибкие временные окна:

```yaml
Доступные предустановки:
- Последние 30 минут
- Последний 1 час
- Последние 6 часов
- Последние 12 часов
- Последние 24 часа
- Последние 2 дня
- Последние 7 дней
- Настраиваемый диапазон
```
