Мониторинг ресурсов является критически важным компонентом набора средств аппаратного ускорения Kubernetes, предназначенным для обеспечения комплексной видимости использования ресурсов GPU в ваших контейнеризированных рабочих нагрузках. Этот модуль предоставляет информацию как о **использовании вычислительных ресурсов**, так и о **потреблении памяти GPU** на двух основных уровнях:

- **Мониторинг на уровне узла**: Отслеживание общего использования ресурсов GPU на всех узлах Kubernetes
- **Мониторинг на уровне пода**: Анализ потребления GPU для каждой рабочей нагрузки с детализированным уровнем пода

Интегрированный с основными модулями платформы (pGPU/vGPU(GPU-Manager)/MPS), это решение для мониторинга позволяет пользователям оптимизировать выделение GPU, обеспечивать соблюдение квот ресурсов и устранять узкие места в производительности в рабочих нагрузках AI/ML, службах реального времени и т.д.

## Преимущества

Основные преимущества мониторинга ресурсов заключаются в следующем:

- **Многомерная наблюдаемость**

  Одновременный мониторинг как вычислительных единиц (ядра CUDA), так и использования памяти на физических/виртуальных GPU, предоставляющий целостные представления о паттернах использования ускорителей.

- **Иерархический сбор метрик**

  Сбор данных как на уровне узла, так и на уровне пода, позволяющий коррелировать тенденции использования ресурсов на уровне кластера с требованиями отдельных рабочих нагрузок.

- **Нативная интеграция**

  Бесшовно работает со всеми модулями ускорителей (pGPU/vGPU/MPS) без необходимости установки дополнительных агентов, используя нативные конвейеры метрик Kubernetes.

- **Исторический анализ**

  Хранение метрик GPU с настраиваемыми периодами хранения (по умолчанию 7 дней) для планирования мощности и анализа паттернов использования с помощью интегрированных инструментов визуализации.

## Сценарии применения

Основные сценарии применения мониторинга ресурсов следующие:

- **Оптимизация производительности**

  Определение недоиспользуемых GPU в обучающих кластерах и оптимизация запросов ресурсов для задач глубокого обучения. Например, обнаружение подов, которые постоянно используют \<30% выделенной памяти GPU для оптимизации выделений памяти.

- **Управление многопользовательскими средами**

  Обеспечение соблюдения квот GPU в совместных средах путем мониторинга потребления vGPU между командами. Отслеживание суммарного использования в рамках выделенных квот в развертываниях платформы AI.

- **Атрибуция затрат**

  Генерация отчетов об использовании GPU по каждому пространству имен для моделей оплаты и показов в корпоративных средах Kubernetes, коррелируя метрики на уровне подов с организационными единицами.

- **Диагностика ошибок**

  Анализ инцидентов OOM (недостаток памяти) в рабочих нагрузках с ускорением GPU путем изучения тенденций использования памяти перед сбоями контейнеров. Перекрестная проверка с событиями Kubernetes для анализа причин.

- **Планирование мощности**

  Анализ исторических паттернов использования GPU (например, периоды пикового спроса на вычислительные ресурсы) для информирования решений о масштабировании инфраструктуры и распределении бюджета для инфраструктуры AI.

## Ограничения использования

При использовании мониторинга ресурсов обратите внимание на следующие ограничения:

- **Зависимости модуля**
  - Требуется, чтобы как минимум один модуль ускорителя (pGPU/vGPU/MPS) был развернут в кластере
