# Введение

## Введение в мониторинг ресурсов

Мониторинг ресурсов является критически важным компонентом набора аппаратных ускорителей Kubernetes, предназначенным для обеспечения всесторонней видимости использования ресурсов GPU в контейнеризованных рабочих нагрузках. Этот модуль предоставляет данные как по **использованию вычислительных ресурсов**, так и по **потреблению памяти GPU** на двух основных уровнях:

- **Мониторинг на уровне узла**: Отслеживайте общую нагрузку на ресурсы GPU по всем узлам Kubernetes.
- **Мониторинг на уровне пода**: Анализируйте потребление GPU на каждую рабочую нагрузку с гранулярностью на уровне пода.

Интегрированный с основными модулями ускорителей платформы (pGPU/vGPU(GPU-Manager)/MPS), это решение для мониторинга позволяет пользователям оптимизировать распределение GPU, регулировать квоты ресурсов и устранять узкие места в производительности в рабочих нагрузках AI/ML, службах реального времени и т. д.

## Преимущества

Основные преимущества мониторинга ресурсов следующие:

- **Многоаспектная наблюдаемость**

  Одновременный мониторинг как вычислительных единиц (CUDA-ядра), так и использования памяти на физических/виртуальных GPU, предоставляющий целостное представление о паттернах использования ускорителей.

- **Иерархический сбор метрик**

  Сбор данных как на уровне узла, так и на уровне пода, позволяющий увязать общие тенденции использования ресурсов кластера с потребностями отдельных рабочих нагрузок.

- **Нативная интеграция**

  Работает без швов со всеми модулями ускорителей (pGPU/vGPU/MPS) без необходимости в дополнительных агентах, используя нативные метрики Kubernetes.

- **Исторический анализ**

  Храните метрики GPU с настраиваемыми сроками хранения (по умолчанию 7 дней) для планирования емкости и анализа паттернов использования с помощью интегрированных инструментов визуализации.

## Сценарии применения

Основные сценарии применения мониторинга ресурсов следующие:

- **Оптимизация производительности**

  Выявление недостаточно используемых GPU в кластерах для обучения и правильная настройка запросов на ресурсы для рабочих нагрузок глубокого обучения. Например, обнаружьте поды, которые постоянно используют \<30% выделенной памяти GPU, чтобы оптимизировать распределение памяти.

- **Управление мультиарендными средами**

  Обеспечение соблюдения квот на GPU в совместных средах путем мониторинга потребления vGPU среди команд. Отслеживание совокупного использования по сравнению с выделенными квотами в развертываниях AI-платформ.

- **Атрибуция затрат**

  Генерирование отчетов по использованию GPU на уровне пространства имен для моделей перераспределения/отчетности в корпоративных средах Kubernetes, коррелирующих метрики на уровне пода с организационными единицами.

- **Диагностика ошибок**

  Исследование инцидентов OOM (вытекание памяти) в рабочих нагрузках с ускорением GPU, анализируя тенденции использования памяти перед сбоями контейнеров. Сравнение с событиями Kubernetes для анализа коренных причин.

- **Планирование емкости**

  Анализ исторических паттернов использования GPU (например, периоды пикового спроса на вычислительные ресурсы) для информирования решений о масштабировании инфраструктуры и распределениях бюджета для AI-инфраструктуры.

## Ограничения использования

При использовании мониторинга ресурсов следует учитывать следующие ограничения:

- **Зависимости модулей**
  - Требуется, чтобы как минимум один модуль ускорителя (pGPU/vGPU/MPS) был развернут в кластере.
