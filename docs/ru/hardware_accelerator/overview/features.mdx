# Функции

## vGPU (На основе Opensource GPU-Manager)

- **Тонкая настройка ресурсов**\
  Разделяет физические ядра GPU на квоты от 1 до 100. Поддерживает динамическое распределение для многопользовательских сред, таких как ИИ-инференс и виртуальные рабочие столы.

- **Учет топологии при планировании**\
  Автоматически приоритизирует графические процессоры, соединенные через NVLink/C2C, чтобы минимизировать задержку передачи данных между сокетами. Обеспечивает оптимальное сочетание GPU для распределенных обучающих задач.

## pGPU (Плагин устройства NVIDIA)

- **Оптимизированное распределение NUMA**\
  Обеспечивает соответствие 1:1 между GPU и Pod с привязкой к узлу NUMA, уменьшая конфликт PCIe шины для задач высокопроизводительных вычислений (HPC), таких как обучение LLM.

- **Эксклюзивный доступ к аппаратному обеспечению**\
  Обеспечивает полную физическую изоляцию GPU через прямую передачу PCIe, что идеально подходит для критически важных приложений, требующих предсказуемой производительности (например, обработка медицинских изображений).

## MPS (Плагин многоprocessной службы NVIDIA)

- **Оптимизированное время выполнения**\
  Позволяет слияние CUDA-ядр между процессами, уменьшая время ожидания инверсии на 30-50% для приложений в реальном времени, таких как анализ видео.

- **Совместное использование ресурсов с ограничениями**\
  Позволяет одновременному выполнению GPU-контекста с соблюдением ограничений на вычисления (0-100%) и использование памяти на основе переменных окружения.
