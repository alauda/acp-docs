# Возможности

## vGPU (основанный на открытом GPU-менеджере)

- **Тонкая настройка ресурсов**\
  Разделяет физические ядра GPU на квоты от 1 до 100. Поддерживает динамическое распределение для мульти-пользовательских сред, таких как ИИ инференс и виртуальные рабочие столы.

- **Учет топологии при планировании**\
  Автоматически приоритизирует GPU, соединенные по NVLink/C2C, чтобы минимизировать задержку передачи данных между сокетами. Обеспечивает оптимальное связывание GPUs для распределенных учебных нагрузок.

## pGPU (плагин устройства NVIDIA)

- **Оптимизированное распределение NUMA**\
  Обеспечивает сопоставление 1:1 между GPU и Pod с привязкой к узлам NUMA, уменьшая конкуренцию на шине PCIe для задач высокопроизводительных вычислений (HPC), таких как обучение LLM.

- **Эксклюзивный доступ к оборудованию**\
  Обеспечивает полную изоляцию физических GPU через прямой доступ PCIe, идеален для критически важных приложений, требующих детерминированной производительности (например, обработка медицинских изображений).

## MPS (плагин многопроцессорной службы NVIDIA)

- **Оптимизированное выполнение с низкой задержкой**\
  Позволяет слияние CUDA-ядер между процессами, снижая задержку инференса на 30-50% для приложений реального времени, таких как видеоанализ.

- **Общий доступ к ресурсам с ограничениями**\
  Позволяет одновременному выполнению контекста GPU с обеспечением вычислительных (0-100%) и лимитов памяти для каждой процесса через переменные окружения.
