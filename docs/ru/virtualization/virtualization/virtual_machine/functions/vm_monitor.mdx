# Мониторинг и Оповещения

Мониторьте и уведомляйте о виртуальных машинах в аспектах ЦП, памяти, хранилища и сети. Чтобы обеспечить своевременные оповещения, можно также настроить политики уведомлений.

Интуитивно представленные данные мониторинга могут быть использованы для поддержки принятия решений в инспекции операций или настройке производительности, в то время как всеобъемлющий механизм оповещения и уведомления поможет гарантировать стабильную работу виртуальных машин.

## Мониторинг

По умолчанию платформа собирает общепринятые метрики мониторинга производительности для виртуальных машин, включая ЦП, память, хранилище и сеть. Перейдите в **Виртуализация** > **Виртуальные Машины**, и на вкладке **Мониторинг** в деталях виртуальной машины вы можете просмотреть данные мониторинга в реальном времени для метрик.

## Оповещения

### Настройка Политик Оповещения

Чтобы включить оповещения, сначала необходимо создать политику оповещения. Политика оповещения описывает объекты, которые вы хотите контролировать, условия, при которых вы хотите получать оповещения, и как вы будете уведомлены о соответствующих оповещениях. Перейдите в **Платформа Контейнеров** > **Виртуализация** > **Виртуальные Машины**, и в деталях виртуальной машины нажмите **Создать Политику Оповещения** на вкладке **Оповещения**, чтобы завершить настройку.

| Параметр             | Описание                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| :-------------------- | :---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Тип Оповещения**    | - Оповещение по Метрике: Контролируемый объект - это метрика, предопределенная платформой, такая как *Уровень Использования Памяти*. <br />- Оповещение по Событию: Контролируемый объект - причина события, то есть причина, по которой виртуальная машина перешла в текущее состояние, например, BackOff, Pulling, Failed.                                                                                                                                                                                                                                                                                                                                                          |
| **Условие Сигнала**   | Составлено из операторов сравнения, порогов оповещения и продолжительности. Сравнивая результаты мониторинга в реальном времени с установленными порогами, определяется необходимость оповещения. <br />Если установлена продолжительность, платформа также будет сравнивать продолжительность, в течение которой контролируемый объект находился в состоянии оповещения.                                                                                                                                                                                                                                                                                                         |
| **Уровень Оповещения** | - Подсказка: У контролируемого объекта есть ожидаемые проблемы, которые не оказывают немедленного влияния на бизнес-операции, но представляют потенциальные риски. Например, если использование ЦП превышает 70% в течение 3 минут. <br />- Предупреждение: У контролируемого объекта есть операционные риски, которые могут повлиять на обычные бизнес-операции, если не будет принято своевременное решение. Например, если использование ЦП превышает 80% в течение 3 минут. <br />- Серьезное: У контролируемого объекта есть известные проблемы, которые могут привести к сбоям функциональности платформы, затрудняя обычные бизнес-операции. <br />- Катастрофа: У контролируемого объекта произошел сбой, что привело к перерывам в сервисе платформы, потере данных и значительному воздействию. |

**Совет**: Функция оповещения виртуальных машин аналогична общей функции оповещения платформы. Для более подробных настроек обратитесь к общей [документации по Оповещениям](#).

### Обработка Оповещений

Перейдите на вкладку **Оповещения**, и если указаны стратегии состояния оповещения, пожалуйста, устраните их немедленно.

### Привязка Политик Уведомления

Кроме оповещений в реальном времени на вкладке **Оповещения**, платформа также поддерживает отправку информации об оповещении по электронной почте, СМС и другим средствам для соответствующего персонала, уведомляя их о необходимых действиях для разрешения проблем или предотвращения неполадок. Политика уведомлений должна быть настроена путем обращения к администратору.
