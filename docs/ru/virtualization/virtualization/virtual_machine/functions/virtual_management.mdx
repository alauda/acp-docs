# Управление виртуальными машинами

## Сброс пароля

Сбросьте пароль пользователя **root**. Этот пароль также используется как пароль для входа в виртуальную машину при аутентификации с помощью пароля.

### Процедура

1. Получите доступ к **Платформе контейнеров**.

2. В левом навигационном меню нажмите **Виртуализация** > **Виртуальные машины**.

3. Найдите виртуальную машину и выберите ⋮ > **Сброс пароля**.

4. Установите новый пароль.

5. Нажмите **Сбросить**.

   **Примечание**: Пожалуйста, храните свой пароль в безопасности. Для обеспечения безопасности среды платформа шифрует и хранит ваш пароль, и вы не сможете увидеть его в открытом виде.

## Обновление ключа

Обновите SSH-ключи.

### Процедура

1. Получите доступ к **Платформе контейнеров**.

2. В левом навигационном меню нажмите **Виртуализация** > **Виртуальные машины**.

3. Найдите виртуальную машину и выберите ⋮ > **Обновить ключ**.

4. Выберите один или несколько связанных ключей, или **Создать ключ**.

5. Выберите, следует ли немедленно перезагрузить; обновление ключей требует перезагрузки виртуальной машины для применения изменений.

6. Нажмите **Обновить**.

## Обновление характеристик

1. Получите доступ к **Платформе контейнеров**.

2. В левом навигационном меню нажмите **Виртуализация** > **Виртуальные машины**.

3. Найдите целевую виртуальную машину и нажмите ⋮ > **Обновить характеристики**.

4. Измените соответствующие ресурсы на основе рекомендуемых сценариев платформы или индивидуальных нужд.

5. Выберите, следует ли **Немедленно перезапустить**; конфигурация вступит в силу после перезагрузки.

6. Нажмите **Обновить**.

## Горячая миграция

**Примечание**: Если вам нужна документация по операциям горячей миграции, пожалуйста, свяжитесь с администратором для получения помощи.

1. Получите доступ к **Платформе контейнеров**.

2. В левом навигационном меню нажмите **Виртуализация** > **Виртуальные машины**.

3. Найдите целевую виртуальную машину и нажмите ⋮ > **Горячая миграция**.

4. Нажмите **Подтвердить**.

## Обновление конфигурации сети NAT

При использовании режима сети NAT платформа по умолчанию открывает порт 22 для SSH-сервисов, и вы можете открывать другие порты по мере необходимости.

### Процедура

1. Получите доступ к **Платформе контейнеров**.

2. В левом навигационном меню нажмите **Виртуализация** > **Виртуальные машины**.

3. Нажмите ***Имя виртуальной машины***.

4. В разделе **Основная информация** нажмите на значок справа от **Открыть порт**.

5. Введите номер порта и нажмите клавишу Enter для подтверждения.

6. Выберите, следует ли **Немедленно перезапустить**; конфигурация вступит в силу после перезагрузки.

7. Нажмите **Обновить**.

## Обновление тегов и аннотаций

1. Получите доступ к **Платформе контейнеров**.

2. В левом навигационном меню нажмите **Виртуализация** > **Виртуальные машины**.

3. Нажмите ***Имя виртуальной машины***.

4. В разделе **Основная информация** нажмите на значок справа от **Теги** или **Аннотации**.

5. Настройте по мере необходимости и нажмите **Обновить**.

## Добавление внутренних маршрутов

1. Получите доступ к **Платформе контейнеров**.

2. В левом навигационном меню нажмите **Виртуализация** > **Виртуальные машины**.

3. Нажмите ***Имя виртуальной машины***.

4. В разделе **Информация для входа** нажмите на значок справа от Внутреннего маршрута.

5. Обратитесь к странице [Создание внутреннего маршрута](/developer/building_application/functions/operation_after_creating/network_communications/service/create_service.mdx) для быстрого добавления внутренних маршрутов для виртуальной машины.

6. Нажмите **Подтвердить**.

## Переустановка операционной системы

Настоятельно рекомендуется сделать резервную копию данных перед переустановкой операционной системы, чтобы избежать потери данных.

**Примечание**: Эта операция приведет к удалению всех данных на **системном диске** виртуальной машины, а также всех **снимков**, и она необратима. Пожалуйста, будьте осторожны!

### Процедура

1. Получите доступ к **Платформе контейнеров**.

2. В левом навигационном меню нажмите **Виртуализация** > **Виртуальные машины**.

3. Найдите виртуальную машину и выберите ⋮ > **Переустановка операционной системы**.

4. В окне **Переустановка операционной системы** настройте следующие параметры.

   - **Метод развертывания**: в настоящее время поддерживаются публичные образы.

   - **Выбор образа**: по умолчанию будет использоваться текущий образ операционной системы для переустановки. Если вы хотите переустановить новую операционную систему, сначала выберите операционную систему образа виртуальной машины, затем выберите образ виртуальной машины, относящийся к этой операционной системе.

5. Нажмите **Переустановить**.

## Настройка IP

Назначьте IP виртуальной машине с использованием динамического распределения (DHCP) или привяжите фиксированный IP к виртуальной машине; новый IP вступит в силу после перезагрузки виртуальной машины.

### Процедура

1. Получите доступ к **Платформе контейнеров**.

2. В левом навигационном меню нажмите **Виртуализация** > **Виртуальные машины**.

3. Найдите целевую виртуальную машину и нажмите ⋮ > **Настроить IP**.

4. Настройте **IP-адрес**.

   - Заполните доступный IP: привязка фиксированного IP означает, что даже после перезагрузки виртуальная машина будет постоянно использовать этот IP-адрес.

   - Оставьте поле пустым: это позволит использовать динамическое распределение (DHCP) для получения IP, присваивая IP, когда виртуальная машина запускается, и освобождая его, когда виртуальная машина останавливается.

5. Выберите, следует ли **Немедленно перезапустить**; конфигурация вступит в силу после перезагрузки.

6. Нажмите **Настроить**.
