# Управление виртуальными машинами

## Сброс пароля

Сбросьте пароль пользователя **root**. Этот пароль также является паролем для входа в виртуальную машину при входе с использованием пароля.

### Процедура

1. Получите доступ к **Container Platform**.

2. В левом навигационном меню нажмите **Виртуализация** > **Виртуальные машины**.

3. Найдите виртуальную машину и выберите ⋮ > **Сброс пароля**.

4. Установите пароль.

5. Нажмите **Сброс**.

   **Примечание**: Пожалуйста, храните свой пароль в безопасности. Для обеспечения безопасности среды платформа шифрует и хранит ваш пароль, и вы больше не сможете увидеть исходный текст пароля.

## Обновление ключа

Обновите SSH-ключи.

### Процедура

1. Получите доступ к **Container Platform**.

2. В левом навигационном меню нажмите **Виртуализация** > **Виртуальные машины**.

3. Найдите виртуальную машину и выберите ⋮ > **Обновить ключ**.

4. Выберите один или несколько связанных ключей или **Создать ключ**.

5. Выберите, нужно ли перезагружать немедленно; обновление ключей требует перезагрузки виртуальной машины для вступления в силу.

6. Нажмите **Обновить**.

## Обновление спецификаций

1. Получите доступ к **Container Platform**.

2. В левом навигационном меню нажмите **Виртуализация** > **Виртуальные машины**.

3. Найдите целевую виртуальную машину и нажмите ⋮ > **Обновить спецификации**.

4. Измените соответствующие ресурсы в соответствии с рекомендованными сценариями платформы или индивидуальными потребностями.

5. Выберите, нужно ли **Перезагрузить немедленно**; конфигурация вступит в силу после перезагрузки.

6. Нажмите **Обновить**.

## Живое перемещение

**Примечание**: Если вам нужна документация по операциям живого перемещения, пожалуйста, обратитесь к администратору за помощью.

1. Получите доступ к **Container Platform**.

2. В левом навигационном меню нажмите **Виртуализация** > **Виртуальные машины**.

3. Найдите целевую виртуальную машину и нажмите ⋮ > **Живое перемещение**.

4. Нажмите **Подтвердить**.

## Обновление конфигурации NAT-сети

При использовании режима сети NAT платформа по умолчанию открывает порт 22 для SSH-сервисов, и вы можете открыть другие порты по мере необходимости.

### Процедура

1. Получите доступ к **Container Platform**.

2. В левом навигационном меню нажмите **Виртуализация** > **Виртуальные машины**.

3. Нажмите ***Имя виртуальной машины***.

4. В разделе **Основная информация** нажмите на иконку справа от **Открыть порт**.

5. Введите номер порта и нажмите клавишу Enter для подтверждения.

6. Выберите, нужно ли **Перезагрузить немедленно**; конфигурация вступит в силу после перезагрузки.

7. Нажмите **Обновить**.

## Обновление тегов и аннотаций

1. Получите доступ к **Container Platform**.

2. В левом навигационном меню нажмите **Виртуализация** > **Виртуальные машины**.

3. Нажмите ***Имя виртуальной машины***.

4. В разделе **Основная информация** нажмите на иконку справа от **Теги** или **Аннотации**.

5. Настройте по мере необходимости и нажмите **Обновить**.

## Добавить внутренние маршруты

1. Получите доступ к **Container Platform**.

2. В левом навигационном меню нажмите **Виртуализация** > **Виртуальные машины**.

3. Нажмите ***Имя виртуальной машины***.

4. В разделе **Информация для входа** нажмите на иконку справа от Внутренний маршрут.

5. Обратитесь к странице [Создать внутренний маршрут](/developer/building_application/functions/operation_after_creating/network_communications/service/create_service.mdx) для быстрого добавления внутренних маршрутов для виртуальной машины.

6. Нажмите **Подтвердить**.

## Переустановка операционной системы

Настоятельно рекомендуется сделать резервную копию данных перед переустановкой операционной системы, чтобы предотвратить потерю данных.

**Примечание**: Эта операция удалит все данные на **системном диске** виртуальной машины, а также все **снимки**, и это необратимо. Пожалуйста, действуйте осторожно!

### Процедура

1. Получите доступ к **Container Platform**.

2. В левом навигационном меню нажмите **Виртуализация** > **Виртуальные машины**.

3. Найдите виртуальную машину и выберите ⋮ > **Переустановить операционную систему**.

4. В окне **Переустановка операционной системы** настройте следующие параметры.

   - **МетодProvisioning**: В настоящее время поддерживает публичные образы.

   - **Выбрать образ**: По умолчанию будет использоваться текущий образ операционной системы для переустановки. Если вы хотите переустановить новую операционную систему, сначала выберите операционную систему образа виртуальной машины, затем выберите образ виртуальной машины, который принадлежит этой операционной системе.

5. Нажмите **Переустановить**.

## Настройка IP

Назначьте IP-адрес виртуальной машине, используя динамическое распределение (DHCP) или привяжите фиксированный IP-адрес к виртуальной машине; новый IP-адрес вступит в силу после перезагрузки виртуальной машины.

### Процедура

1. Получите доступ к **Container Platform**.

2. В левом навигационном меню нажмите **Виртуализация** > **Виртуальные машины**.

3. Найдите целевую виртуальную машину и нажмите ⋮ > **Настроить IP**.

4. Настройте **IP-адрес**.

   - Заполните доступный IP: привязывание фиксированного IP означает, что даже после перезагрузки виртуальная машина будет последовательно использовать этот IP-адрес.

   - Оставьте опцию пустой: это позволит использовать динамическое распределение (DHCP) для получения IP, назначая IP при запуске виртуальной машины и освобождая его, когда виртуальная машина останавливается.

5. Выберите, нужно ли **Перезагрузить немедленно**; конфигурация вступит в силу после перезагрузки.

6. Нажмите **Настроить**.
