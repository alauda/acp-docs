# Настройка виртуальных машин для использования режима сетевого связывания с поддержкой IPv6

Режим сетевого связывания представляет собой механизм расширения плагина для сети виртуальных машин. По умолчанию платформа использует плагин под названием ManagedTap для включения поддержки IPv6 для виртуальных машин. Этот плагин позволяет виртуальным машинам получать IP-адреса через DHCP-сервер CNI. Следовательно, пока DHCP-сервер CNI поддерживает IPv6, виртуальные машины также получат возможности IPv6.

В настоящее время мы используем Kube-OVN в качестве CNI. Поскольку DHCP-сервер Kube-OVN полностью поддерживает IPv6, виртуальные машины могут добиться надежной функциональности IPv6 благодаря сочетанию ManagedTap и Kube-OVN.

## Предварительные требования

- Версия ACP должна быть v4.0.0 или выше.
- Используется Kube-OVN в качестве CNI, и подсеть виртуальной машины настроена как Underlay.

## Процедура

<Steps>
  ### Добавьте конфигурацию IPv6 в подсеть виртуальной машины

  ```bash
  kubectl edit subnet <subnet-name>
  ```

  Добавьте следующие параметры под `spec`:

  ```yaml
  spec:
    enableDHCP: true
    enableIPv6RA: true
    u2oInterconnection: true
  ```

  ### Создание виртуальной машины с использованием режима сетевого связывания в веб-консоли

  При создании виртуальной машины выберите **Сетевое связывание** в качестве сетевого режима.

  ### Доступ к виртуальной машине через VNC и настройка сетевого интерфейса

  Для систем CentOS отредактируйте файл `/etc/sysconfig/network-scripts/ifcfg-enp1s0` и добавьте следующую конфигурацию:

  ```bash
  IPV6INIT=yes
  DHCPV6C=yes
  IPV6_AUTOCONF=yes
  ```

  перезапустите сеть

  ```bash
  systemctl restart network
  ```

  ### Настройка маршрута по умолчанию для IPv6

  Если коммутатор настроен на отправку сообщений об объявлении маршрутизатора (RA), ручная настройка маршрута не требуется. Маршрут по умолчанию может быть автоматически выучен через сообщения RA от коммутатора.

  ```bash
  ip r r default via <subnet-v6-gateway>
  ```
</Steps>
