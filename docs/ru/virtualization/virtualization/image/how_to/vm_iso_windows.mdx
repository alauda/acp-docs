# Создание образов Windows на основе ISO с использованием KubeVirt

В этом документе рассматривается решение для виртуальных машин на основе компонента с открытым исходным кодом KubeVirt, с использованием технологий виртуализации KubeVirt для создания изображения операционной системы Windows через файл ISO-образа.

## Предварительные требования

- Все компоненты в кластере функционируют правильно.

- Пожалуйста, заранее подготовьте образ Windows и [последние virtio-win-tools](https://fedorapeople.org/groups/virt/virtio-win/direct-downloads/archive-virtio/).

- Пожалуйста, подготовьте репозиторий для хранения образа. В этом документе в качестве примера используется репозиторий build-harbor.example.cn, и пожалуйста, замените его в соответствии с вашей реальной средой.

## Ограничения и лимитации

- При запуске KubeVirt размер файловой системы настраиваемого образа повлияет на скорость записи образа на диск в PVC. Если файловая система слишком большая, это может привести к увеличению времени создания.

- Рекомендуется сохранять корневой раздел Linux или диск C Windows ниже 100 Гб, чтобы минимизировать начальный размер. Последующее расширение может быть выполнено через cloud-init (для систем Windows его необходимо расширять вручную после создания).

## Процедура

<Steps>
  ### Создание образа

  Создайте образ Docker на основе подготовленных образов Windows и virtio-win ISO, и загрузите его в репозиторий. В этом документе используется Windows Server 2019 в качестве примера.

  **Создание образа Docker из ISO Windows**

  1. Перейдите в каталог, где хранится ISO-образ, и выполните следующую команду в терминале, чтобы переименовать ISO-образ в win.iso.

     ```bash
     mv <ISO image name> win.iso # Замените <ISO image name> на фактическое имя образа, например, mv en_windows_server_2019_x64_dvd_4cb967d8.iso win.iso
     ```

  2. Выполните следующую команду, чтобы создать Dockerfile.

     ```bash
     touch Dockerfile
     ```

  3. Отредактируйте Dockerfile, добавьте следующее содержимое и сохраните его.

     ```docker
     FROM scratch
     ADD --chown=107:107 win.iso /disk/
     ```

  4. Выполните следующую команду, чтобы собрать образ Docker.

     ```bash
     docker build -t build-harbor.example.cn/3rdparty/vmdisks/winiso:2019 . # Замените репозиторий в соответствии с вашей реальной средой
     ```

  5. Выполните следующую команду, чтобы загрузить образ в репозиторий.

     ```bash
     docker push  build-harbor.example.cn/3rdparty/vmdisks/winiso:2019 # Замените репозиторий в соответствии с вашей реальной средой
     ```

  **Создание образа Docker из ISO virtio-win**

  1. Перейдите в каталог, где хранится ISO-образ, и выполните следующую команду в терминале, чтобы создать Dockerfile.

     ```bash
     touch Dockerfile
     ```

  2. Отредактируйте Dockerfile, добавьте следующее содержимое и сохраните его.

     ```docker
     FROM scratch
     ADD --chown=107:107 virtio-win.iso  /disk/
     ```

  3. Выполните следующую команду, чтобы собрать образ Docker.

     ```bash
     docker build -t build-harbor.example.cn/3rdparty/vmdisks/win-virtio:latest . # Замените репозиторий в соответствии с вашей реальной средой
     ```

  4. Выполните следующую команду, чтобы загрузить образ в репозиторий.

     ```bash
     docker push  build-harbor.example.cn/3rdparty/vmdisks/win-virtio:latest # Замените репозиторий в соответствии с вашей реальной средой
     ```

  ### Создание виртуальной машины

  1. Получите доступ к **Container Platform**.

  2. В левой навигационной панели нажмите на **Виртуализация** > **Виртуальные машины**.

  3. Нажмите на **Создать виртуальную машину**.

  4. Заполните необходимые параметры, такие как **Имя**, **Образ** и т.д., на странице формы. Для получения подробных параметров и конфигураций, пожалуйста, обратитесь к [Создание виртуальной машины](/virtualization/virtualization/virtual_machine/functions/create_virtual_machine.mdx).

  5. Переключитесь на YAML.

  6. Замените конфигурацию в поле spec.template.spec.domain.devices.disks следующим содержимым.

     ```yaml
       domain:
         devices:
           disks:
             - disk:
                 bus: virtio
               name: cloudinitdisk
             - bootOrder: 1
               cdrom:
                 bus: sata
               name: containerdisk
             - cdrom:
                 bus: sata
               name: virtio
             - disk:
                 bus: sata
               name: rootfs
               bootOrder: 10
     ```

  7. Добавьте следующее содержимое в поле spec.template.spec.volumes.

     ```yaml
             - containerDisk:
                 image: registry.example.cn:60070/3rdparty/vmdisks/winiso:2019 # Замените образ в соответствии с вашей реальной средой
               name: containerdisk
             - containerDisk:
                 image: registry.example.cn:60070/3rdparty/vmdisks/win-virtio # Замените образ в соответствии с вашей реальной средой
               name: virtio
     ```

  11.       **Примечание**: Если на верхней части страницы деталей виртуальной машины появится сообщение **Текущая виртуальная машина имеет изменения конфигурации, которые требуют перезапуска для применения, пожалуйста, перезапустите**, это сообщение можно проигнорировать; перезагрузка не требуется.

  ### Установка операционной системы Windows

  1. Следуйте инструкциям по установке для установки системы после входа на страницу установки.

     **Примечание**: Во время шага выбора раздела шина должна быть sata, чтобы диск был правильно распознан. Поэтому вам нужно выбрать каждый раздел по очереди и нажать **Удалить**, чтобы удалить все разделы, позволяя системе обрабатывать это автоматически.

  2. После настройки пароля учетной записи администратора нажмите **Отправить удаленную команду** в верхнем левом углу страницы, затем выберите **Ctrl-Alt-Delete** из выпадающего меню.

  3. Когда появится сообщение **Сочетание Ctrl+Alt+Delete перезагрузит сервер, подтвердите перезапуск**, нажмите **ОК**.

  4. Введите пароль для доступа к рабочему столу Windows; на данном этапе установка операционной системы Windows завершена.

  ### Установка virtio-win-tools

  Этот инструмент в основном содержит необходимые драйверы.

  1. Откройте проводник файлов.

  2. Дважды щелкните на **Дисковод CD(E:) virtio-win-\<version>**, перейдите в каталог **virtio-win-guest-tools** для входа на страницу установки и следуйте инструкциям по установке. Часть \<version> должна основываться на реальной ситуации.

  3. После завершения установки выключите систему Windows.

  ### Экспорт настраиваемого образа Windows

  Пожалуйста, обратитесь к [Экспорт образа виртуальной машины](./vm_image_export.mdx) для конкретной операции.

  ### Использование образа Windows

  1. Получите доступ к **Container Platform**.

  2. В левой навигационной панели нажмите на **Виртуализация** > **Виртуальные машины**.

  3. Нажмите на **Создать виртуальную машину**.

  4. Заполните необходимые параметры на странице формы. Для образа выберите экспортированный образ Windows. Для получения подробных параметров и конфигураций, пожалуйста, обратитесь к [Создание виртуальной машины](/virtualization/virtualization/virtual_machine/functions/create_virtual_machine.mdx).

  5. (По желанию) Если вы используете более новую операционную систему, такую как Windows 11, включите такие функции, как часы, UEFI, TPM и т.д. Переключитесь на YAML и замените оригинальный YAML файл на следующий YAML файл.

     ```yaml
     apiVersion: kubevirt.io/v1
     kind: VirtualMachineInstance
     metadata:
       labels:
         special: vmi-windows
       name: vmi-windows
     spec:
       domain:
         clock:
           timer:
             hpet:
               present: false
             hyperv: {}
             pit:
               tickPolicy: delay
             rtc:
               tickPolicy: catchup
           utc: {}
         cpu:
           cores: 2
         devices:
           disks:
           - disk:
               bus: sata
             name: pvcdisk
           interfaces:
           - masquerade: {}
             model: e1000
             name: default
           tpm: {}
         features:
           acpi: {}
           apic: {}
           hyperv:
             relaxed: {}
             spinlocks:
               spinlocks: 8191
             vapic: {}
           smm: {}
         firmware:
           bootloader:
             efi:
               secureBoot: true
           uuid: 5d307ca9-b3ef-428c-8861-06e72d69f223
         resources:
           requests:
             memory: 4Gi
       networks:
       - name: default
         pod: {}
       terminationGracePeriodSeconds: 0
       volumes:
       - name: pvcdisk
         persistentVolumeClaim:
           claimName: disk-windows
       - name: winiso
         persistentVolumeClaim:
           claimName: win11cd-pvc
     ```

  6. Нажмите **Создать**.

  ### Добавить внутренний маршрут

  Настроив внутренний маршрут типа NodePort, откройте порт для удаленных подключений к рабочему столу.

  1. Получите доступ к **Container Platform**.

  2. В левой навигационной панели нажмите на **Виртуализация** > **Виртуальные машины**.

  3. Щелкните по имени виртуальной машины, созданной с образом Windows, в списке, чтобы перейти на страницу деталей.

  4. Нажмите на значок **Добавить** рядом с **Внутренний маршрут** в области **Информация для входа**.

  5. Настройте параметры в соответствии со следующими инструкциями.

     | Параметр | Описание                                                                                                                                         |
     | --------- | --------------------------------------------------------------------------------------------------------------------------------------------------- |
     | **Тип**  | Выберите **NodePort**.                                                                                                                                |
     | **Порт**  | <ul><li>Протокол: выберите TCP.</li><li>Порт службы: используйте 3389.</li><li>Порт виртуальной машины: используйте 3389.</li><li>Имя порта службы: используйте rdp.</li></ul> |

  6. Нажмите **ОК**, чтобы вернуться на страницу деталей.

  7. Щелкните по ссылке **Внутренний маршрут** в области **Информация для входа**.

  8. Сохраните информацию о **Виртуальном IP** из основной области информации и информацию о **Хост-порте** из области порта.
</Steps>

## Удаленный доступ

В этом документе рассматривается использование операционной системы Windows для удаленного подключения в качестве примера. Другие операционные системы могут использовать программное обеспечение, поддерживающее протокол RDP, для подключения.

1. Откройте **Подключение к удаленному рабочему столу**.

2. Введите сохраненный Виртуальный IP и Хост-порт из шага **Добавить внутренний маршрут**, отформатированных как **Виртуальный IP:Хост-порт**, например: 192.1.1.1:3389.

3. Нажмите **Подключиться**.
