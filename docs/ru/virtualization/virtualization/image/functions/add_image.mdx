# Добавление образов виртуальных машин

Платформа поддерживает добавление образов виртуальных машин архитектуры **X86\_64** и **ARM64 (Alpha)**, что позволяет разработчикам быстро создавать виртуальные машины для существующих сервисов и облегчает миграцию бизнес-систем.

## Процедура

1. Перейдите в **Управление платформой**.

2. В левой навигационной панели нажмите **Управление виртуализацией** > **Образы виртуальных машин**.

3. Нажмите **Добавить образ виртуальной машины**.

4. Ознакомьтесь с приведенными ниже инструкциями, чтобы настроить соответствующие параметры.

   | Параметр                | Описание                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
   | -----------------------  | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | **Метод развертывания**  | В настоящее время поддерживается только метод **Публичный образ**, что означает, что добавленный образ может использоваться в назначенных проектах.                                                                                                                                                                                                                                                                                                                                                                                                      |
   | **Операционная система**  | Поддерживаемые операционные системы: **CentOS**/**Ubuntu**/**RedHat**/**Debian**/**TLinux**/**Другие Linux**/**Windows (Alpha)**.<br />Поддерживаемые архитектуры систем: **X86\_64** и **ARM64 (Alpha)**.                                                                                                                                                                                                                                                                                                     |
   | **Источник**             | <ul><li>**Репозиторий образов**: Образы виртуальных машин, хранящиеся в репозитории контейнеров.</li><li>**HTTP**: Образы виртуальных машин, хранящиеся на файловом сервере с использованием протокола HTTP.</li><li>**Объектное хранилище (S3)**: Образы виртуальных машин, которые можно получить с помощью протокола объектного хранилища (S3). Если они не требуют аутентификации, используйте HTTP в качестве источника.</li></ul>                                                                                                                                                              |
   | **Архитектура CPU**      | Укажите информацию об архитектуре CPU. Для источников репозитория образов поддерживается множественный выбор; для других источников разрешен только односторонний выбор.                                                                                                                                                                                                                                                                                                                                                                             |
   | **Адрес образа**         | Поддерживает образы виртуальных машин KVM, включая форматы qcow2/raw. <ul><li>Если из репозитория образов, введите `repository_address:image_version`, например, `index.docker.io/library/ubuntu:latest`.</li><li>Если из HTTP-источника, введите URL файла образа, который должен начинаться с `http://` или `https://`, например, `http://192.168.0.1/vm_image/centos_7.8.qcow2`.</li><li>Если из объектного хранилища (S3), введите адрес образа, который можно получить с помощью протокола объектного хранилища (S3), например, `https://endpoint/bucket/centos.qcow2`.</li></ul> |
   | **Аутентификация**       | В зависимости от того, требует ли репозиторий образов аутентификации, можно включить или отключить переключатель. Если включено, вы можете выбрать из существующих учетных данных образа или нажать **Добавить учетные данные**, поддерживается только тип учетных данных **Имя пользователя/Пароль**.<br />**Заметка**: Когда источник - объектное хранилище (S3), аутентификацию невозможно отключить.                                                                                                                                                   |
   | **Назначенный проект**    | Назначьте права доступа к этому образу для проектов. <ul><li>Все проекты: назначает права доступа к образу для всех проектов.</li><li>Конкретный проект: назначает права доступа к образу для указанного проекта.</li><li>Без назначения: пока не назначайте никаким проектам. После создания образа вы можете назначить его через операцию **Обновить образ**.</li></ul>                                                                                                                         |

5. Нажмите **Добавить**.
