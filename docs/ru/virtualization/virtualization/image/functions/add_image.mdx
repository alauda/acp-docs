# Добавление образов виртуальных машин

Платформа поддерживает добавление образов виртуальных машин архитектуры **X86_64** и **ARM64 (Alpha)**, что позволяет разработчикам быстро создавать виртуальные машины для существующих сервисов и облегчать миграцию бизнес-систем.

## Процедура

1. Перейдите в **Управление платформой**.

2. В левой навигационной панели нажмите **Управление виртуализацией** > **Образы виртуальных машин**.

3. Нажмите **Добавить образ виртуальной машины**.

4. Ознакомьтесь с нижеуказанными инструкциями для настройки соответствующих параметров.

   | Параметр               | Описание                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
   | ---------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | **Метод развертывания** | В настоящее время поддерживается только метод **Общий образ**, что означает, что добавленный образ может использоваться в назначенных проектах.                                                                                                                                                                                                                                                                                                                                                                                                                                |
   | **Операционная система**    | Поддерживаемые операционные системы: **CentOS**/**Ubuntu**/**RedHat**/**Debian**/**TLinux**/**Другие Linux**/**Windows (Alpha)**.<br />Поддерживаемые архитектуры систем: **X86_64** и **ARM64 (Alpha)**.                                                                                                                                                                                                                                                                                                                                      |
   | **Источник**              | <ul><li>**Репозиторий изображений**: Образы виртуальных машин, хранящиеся в репозитории контейнерных изображений.</li><li>**HTTP**: Образы виртуальных машин, хранящиеся на файловом сервере с использованием протокола HTTP.</li><li>**Объектное хранилище (S3)**: Образы виртуальных машин, которые могут быть получены с помощью протокола объектного хранилища (S3). Если они не требуют аутентификации, пожалуйста, используйте HTTP в качестве источника.</li></ul>                                                                                                                                       |
   | **Архитектура ЦП**    | Укажите информацию об архитектуре ЦП. Для источников из репозитория изображений поддерживаются множественные выборы; для других источников допускается только одиночный выбор.                                                                                                                                                                                                                                                                                                                                                                                        |
   | **Адрес изображения**       | Поддерживает образы виртуальных машин KVM, включая форматы qcow2/raw. <ul><li>Если из репозитория изображений, введите `repository_address:image_version`, например, `index.docker.io/library/ubuntu:latest`.</li><li>Если из HTTP источника, введите URL файла изображения, который должен начинаться с `http://` или `https://`, например, `http://192.168.0.1/vm_image/centos_7.8.qcow2`.</li><li>Если из объектного хранилища (S3), введите адрес изображения, который можно получить через протокол объектного хранилища (S3), например, `https://endpoint/bucket/centos.qcow2`.</li></ul> |
   | **Аутентификация**      | В зависимости от того, требует ли репозиторий изображений аутентификации, вы можете переключить этот параметр включенным или выключенным. Если включено, вы можете выбрать из существующих учетных данных для изображений или нажать **Добавить учетные данные**, поддерживается только тип аутентификации **Имя пользователя/Пароль**.<br />**Примечание**: Когда источник - объектное хранилище (S3), аутентификация не может быть отключена.                                                                                                                                                                                           |
   | **Назначенный проект**    | Назначьте разрешения на использование этого изображения для проектов. <ul><li>Все проекты: Назначает разрешения на использование изображения для всех проектов.</li><li>Конкретный проект: Назначает разрешения на использование изображения для указанного проекта.</li><li>Без назначения: Пока не назначайте ни одному проекту. После создания образа вы можете назначить его через операцию **Обновить образ**.</li></ul>                                                                                                                                                        |

5. Нажмите **Добавить**.
