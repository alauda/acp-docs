# Оператор

## Обзор

Операторы — это метод упаковки, развертывания и управления приложениями Kubernetes, способный управлять полным жизненным циклом ресурсов Kubernetes.

## Введение

Оператор — это метод упаковки, развертывания и управления приложениями Kubernetes, который может автоматизировать весь жизненный цикл приложения (например, создание, обновления, удаление). Администраторы платформы могут развертывать Операторы из OperatorHub на уровне каждого кластера.

## Ключевые концепции

- **OperatorHub**: Представляет собой интерфейс веб-консоли, который отображает список доступных Операторов на платформе. Через OperatorHub администраторы платформы могут просматривать описания Операторов и подробную информацию, а также устанавливать и управлять Операторами.

- **Ресурсы**: CRD ([CustomResourceDefinition](https://kubernetes.io/docs/tasks/access-kubernetes-api/custom-resources/custom-resource-definitions/)) ресурсы, принадлежащие Оператору. В API Kubernetes ресурс CRD является конечной точкой для хранения коллекции объектов API (CR, пользовательский ресурс) определенного типа.

- **Пользовательские ресурсы (CR)**: Определенные типы объектов API, созданные CRD, которые принадлежат Операторам, развернутым в кластер. После успешной установки Оператора пользователи во всех проектах, связанных с кластером, могут добавлять CR в свои пространства имен.

- **Экземпляры ресурсов**: Экземпляры ресурсов, созданные в пространстве имен, путем вызова API Kubernetes на основе CR. Создание экземпляров ресурсов предоставляет Оператору возможности в пространстве имен.

## Процесс установки

1. В представлении управления платформой нажмите на левое навигационное меню: **Управление платформой** > **Маркетплейс** > **OperatorHub**, чтобы просмотреть всех Операторов, предоставленных платформой.

2. Найдите нужный Оператор. Если Оператор отображает статус "отсутствует", вам нужно:
   - Посетить Портал настроек, чтобы загрузить соответствующий пакет Оператора. Если у вас нет доступа к Порталу настроек, свяжитесь с технической поддержкой.
   - Использовать инструмент `violet`, чтобы опубликовать пакет в кластер, где вы хотите установить Оператора. Для получения подробных инструкций по использованию этого инструмента обратитесь к [документации CLI](/ui/cli_tools/index.mdx).
   - Перейдите в **Управление платформой** > **Маркетплейс** > **Загрузить пакеты**, переключитесь на вкладку **Оператор** и найдите загруженное имя Оператора. На странице сведений об Операторе будут отображены версия(и) пакета, загруженного на платформу.

3. Если Оператор отображает статус "готов", нажмите **Установить**, чтобы начать процесс установки. Для получения конкретных инструкций по использованию после установки, пожалуйста, обратитесь к пользовательской документации для данного Оператора.

## Обновление Операторов

Чтобы обновить существующий Оператор до новой версии:

1. Загрузите новую версию Оператора на платформу, используя тот же процесс, описанный в разделе установки.

2. Перейдите в **Управление платформой** > **Маркетплейс** > **Загрузить пакеты**, переключитесь на вкладку **Оператор** и проверьте данные о пакете соответствующего Оператора, чтобы подтвердить, что новая версия была загружена.

3. Поведение обновления зависит от стратегии обновления подписки Оператора:

   - **Автоматическая стратегия обновления**: Если стратегия обновления установлена на автоматическую, Оператор будет обновлен автоматически, как только новый пакет версии будет загружен.

   - **Ручная стратегия обновления**: Если стратегия обновления установлена на ручную, у вас есть два варианта:
     - **Пакетное обновление**: Для обновления нескольких Операторов в рамках кластера перейдите в **Управление платформой** > **Кластеры** > **Кластеры**. Кластеры с установленными Операторами, которые могут быть обновлены, будут отображать значок обновления. Введите информацию о кластере, переключитесь на вкладку **Функции** и нажмите активированную кнопку обновления, чтобы выполнить пакетное обновление нескольких Операторов.
     - **Индивидуальное обновление**: Чтобы обновить только конкретного Оператора, проверьте сведения о OperatorHub для этого Оператора. Вы увидите уведомление, ожидающее подтверждения. После вашего подтверждения Оператор начнет процесс обновления.

**Примечание**: Операторы с источником, помеченным как "Alauda", не могут иметь стратегию обновления своей подписки, установленную на автоматическую. Их необходимо обновлять вручную.

## Поддержка

Если у вас возникли какие-либо проблемы во время процесса установки или обновления, пожалуйста, свяжитесь с технической поддержкой для получения помощи.
