# Оператор

## Обзор

Операторы представляют собой метод упаковки, развертывания и управления приложениями Kubernetes, способные управлять полным жизненным циклом ресурсов Kubernetes.

## Введение

Оператор — это метод упаковки, развертывания и управления приложениями Kubernetes, который может автоматизировать весь жизненный цикл приложения (например, создание, обновления, удаление). Администраторы платформы могут развертывать Операторы из OperatorHub на основе каждого кластера.

## Ключевые концепции

- **OperatorHub**: Представлено в виде веб-консольного интерфейса, который отображает список доступных Операторов на платформе. Через OperatorHub администраторы платформы могут просматривать описания Операторов и подробную информацию, а также устанавливать и управлять Операторами.

- **Ресурсы**: CRD ([CustomResourceDefinition](https://kubernetes.io/docs/tasks/access-kubernetes-api/custom-resources/custom-resource-definitions/)) ресурсы, принадлежащие Оператору. В API Kubernetes ресурс CRD представляет собой конечную точку для хранения набора объектов API (CR, Пользовательский ресурс) конкретного типа.

- **Пользовательские ресурсы (CR)**: Конкретные типы объектов API, созданные CRD, которые принадлежат Операторам, развернутым в кластер. После успешной установки Оператора пользователям во всех проектах, связанных с кластером, разрешается добавлять CR в свои пространства имен.

- **Экземпляры ресурсов**: Экземпляры ресурсов, созданные в пространстве имен путем вызова API Kubernetes на основе CR. Создание экземпляров ресурсов предоставляет функции Оператора пространству имен.

## Процесс установки

1. В представлении управления платформой щелкните по левому навигационному меню: **Управление платформой** > **Маркетплейс** > **OperatorHub**, чтобы просмотреть всех Операторов, предоставленных платформой.

2. Найдите желаемого Оператора. Если Оператор показывает статус "отсутствует", вам необходимо:
   - Посетить пользовательский портал, чтобы скачать соответствующий пакет Оператора. Если у вас нет доступа к пользовательскому порталу, обратитесь в техническую поддержку.
   - Использовать инструмент `violet`, чтобы опубликовать пакет в кластер, где вы хотите установить Оператора. Для получения подробных инструкций по использованию этого инструмента обратитесь к [документации CLI](/ui/cli_tools/index.mdx).
   - Перейдите в **Управление платформой** > **Маркетплейс** > **Загрузить пакеты**, переключитесь на вкладку **Оператор** и найдите загруженное имя Оператора. На странице деталей Оператора отобразятся версия(ии) пакета, загруженного на платформу.

3. Если Оператор показывает статус "готов", нажмите **Установить**, чтобы начать процесс установки. Для получения конкретных инструкций по использованию после установки обратитесь к пользовательской документации для данного Оператора.

## Обновление Операторов

Для обновления существующего Оператора до более новой версии:

1. Загрузите новую версию Оператора на платформу, используя тот же процесс, описанный в разделе установки.

2. Перейдите в **Управление платформой** > **Маркетплейс** > **Загрузить пакеты**, переключитесь на вкладку **Оператор** и проверьте детали пакета соответствующего Оператора, чтобы подтвердить, что новая версия была загружена.

3. Поведение обновления зависит от стратегии обновления подписки Оператора:

   - **Автоматическая стратегия обновления**: Если стратегия обновления установлена на автоматическую, Оператор автоматически обновится, как только новая версия пакета будет загружена.

   - **Ручная стратегия обновления**: Если стратегия обновления установлена на ручную, у вас есть два варианта:
     - **Пакетное обновление**: Чтобы обновить несколько Операторов в кластере, перейдите в **Управление платформой** > **Кластеры** > **Кластеры**. Кластеры с установленными Операторами, которые могут быть обновлены, будут отображать значок обновления. Введите данные кластера, переключитесь на вкладку **Функции** и нажмите активированную кнопку обновления, чтобы выполнить пакетное обновление нескольких Операторов.
     - **Индивидуальное обновление**: Чтобы обновить только конкретного Оператора, проверьте детали OperatorHub для этого Оператора. Вы увидите запрос на одобрение. После вашего одобрения Оператор начнет процесс обновления.

**Примечание**: Операторы, имеющие источник, помеченный как "Alauda", не могут иметь стратегию обновления подписки, установленную на автоматическую. Их необходимо обновлять вручную.

## Поддержка

Если у вас возникли проблемы в процессе установки или обновления, пожалуйста, обратитесь в техническую поддержку за помощью.
