# Установка Alauda Container Platform GitOps

## Предварительные требования

1. **Скачайте** установочный пакет плагина кластера **Alauda Container Platform GitOps**, соответствующий архитектуре вашей платформы.
2. **Загрузите** установочный пакет с помощью механизма загрузки пакетов.
3. **Установите** установочный пакет в `global` кластер с помощью механизма плагинов кластера.

:::info
Загрузка пакетов:
**Управление платформой** > **Маркетплейс** > страница **Загрузка пакетов**.
Нажмите **Документация помощи** справа, чтобы получить инструкции о том, как опубликовать плагин кластера в `global` кластере. Для получения дополнительной информации см. [CLI](/ui/cli_tools/index.mdx).
:::

## Установка плагина кластера Alauda Container Platform GitOps

### Ограничения и ограничения

- Поддерживает установку только в `global` кластере.
- После установки плагина экземпляр ArgoCD в `argocd-operator` будет ограничен в операциях.

### Процедура

1. Войдите в систему и перейдите на страницу **Управление платформой**.
2. Нажмите **Маркетплейс** > **Плагины кластера**, чтобы перейти на страницу списка **Плагинов кластера**.
3. Найдите плагин кластера **GitOps**, нажмите **Установить** и перейдите на страницу **Установка плагина GitOps**.
4. Рекомендуется использовать предложенную конфигурацию по умолчанию; просто нажмите **Установить**, чтобы завершить установку плагина кластера **Alauda Container Platform GitOps**.

### Проверка

1. На странице **Управление платформой**, в разделе **Кластер** в левом навигационном меню будет отображен элемент **Config**. Вы можете использовать возможности управления конфигурацией кластера.
2. Получите доступ к **Платформе контейнеров**, в левом навигационном меню будет отображен элемент **GitOps Applications**, где вы можете создать приложение GitOps, чтобы сразу оценить [Создание приложения Argo CD через веб-консоль](../how_to/create_gitops_app.mdx).
