# Установка Alauda Build of Argo CD

## Предварительные требования

1. **Скачайте** пакет установки **Alauda Build of Argo CD** оператора, соответствующий архитектуре вашей платформы.
2. **Загрузите** пакет установки с помощью механизма загрузки пакетов.

## Процедура

> Установите в кластер, где вы хотите использовать функциональность GitOps.

### Установка оператора Alauda Build of Argo CD

1. Войдите в систему и перейдите на страницу **Управление платформой**.
2. Нажмите **Маркетплейс** > **OperatorHub**, чтобы перейти на страницу **OperatorHub**.
3. Найдите оператора **Alauda Build of Argo CD**, нажмите **Установить**, и перейдите на страницу **Установка Argo CD**.

Параметры конфигурации:

| **Параметр**         | **Рекомендуемая конфигурация**                                                                                                            |
| :-------------------- | :---------------------------------------------------------------------------------------------------------------------------------------- |
| **Канал**            | Канал по умолчанию - `alpha`.                                                                                                             |
| **Режим установки**   | `Cluster`: Все пространства имен в кластере используют единственный экземпляр оператора для создания и управления, что приводит к снижению расхода ресурсов. |
| **Пространство имен** | Выберите `Рекомендуемое пространство имен`: автоматически создается, если оно не существует.                                             |
| **Стратегия обновления** | `Auto`: OperatorHub автоматически обновит оператора до последней версии, когда будет доступна новая версия.                             |

4. Рекомендуется использовать предложенную конфигурацию по умолчанию; просто нажмите **Установить**, чтобы завершить установку оператора **Alauda Build of Argo CD**.

### Создание экземпляра Argo CD

1. Нажмите **Маркетплейс** > **OperatorHub**.

2. Найдите оператора **Alauda Build of Argo CD**, нажмите на него, чтобы перейти на страницу с деталями **Argo CD**.

3. Нажмите **Все экземпляры**,

4. Нажмите **Создать экземпляр**, выберите карточку экземпляра **Argo CD**.

5. Нажмите **Создать экземпляр**.

:::info
На странице параметров конфигурации используйте конфигурацию по умолчанию, если только нет специфических требований.
**Примечание**: Если глобальный кластер не имеет высокой доступности (например, в нем только один управляющий узел), пожалуйста, переключитесь на режим YAML при создании экземпляра и установите значение поля ha.enabled в false.
:::

6. Нажмите **Создать**.

### Создание экземпляра AppProject

:::info
**Совет**: Если вам не нужно использовать управляемую платформой функцию **Управление конфигурацией кластера**, вам не нужно выполнять следующие шаги.
:::

1. Найдите оператора **Alauda Build of Argo CD**, нажмите на него, чтобы перейти на страницу с деталями **Alauda Argo CD**.

2. Нажмите **Все экземпляры**, **Создать экземпляр**, выберите карточку экземпляра **AppProject**.

3. Переключитесь на режим YAML и перезапишите существующее содержимое YAML на интерфейсе следующим кодом.

   ```
   apiVersion: argoproj.io/v1alpha1
   kind: AppProject
   metadata:
     name: cpaas-system
     namespace: argocd
   spec:
     clusterResourceWhitelist:
     - group: '*'
       kind: '*'
     destinations:
     - namespace: '*'
       server: '*'
     sourceRepos:
     - '*'
   ```

4. Нажмите **Создать**.

После завершения вышеуказанной процедуры вы успешно установили Argo CD. Немедленно [Создание приложения Argo CD через панель управления Argo CD](../how_to/create_application_by_argocd_dashoard.mdx), чтобы начать ваше путешествие по GitOps.
