# Установка Argo CD

## Предварительные условия

1. **Скачайте** пакет установки **Argo CD** оператора, соответствующий архитектуре вашей платформы.
2. **Загрузите** пакет установки **Argo CD** с помощью механизма загрузки пакетов.

## Процедура

> Установите в кластер, в котором вы хотите использовать функциональность GitOps.

### Установка оператора Argo CD

1. Войдите в систему, перейдите на страницу **Управление платформой**.
2. Нажмите **Маркетплейс** > **OperatorHub**, чтобы перейти на страницу **OperatorHub**.
3. Найдите оператора **Argo CD**, нажмите **Установить** и перейдите на страницу **Установка Argo CD**.

Параметры конфигурации:

| **Параметр**         | **Рекомендуемая конфигурация**                                                                                                      |
| :-------------------- | :------------------------------------------------------------------------------------------------------------------------------------- |
| **Канал**            | Канал по умолчанию — `alpha`.                                                                                                         |
| **Режим установки**   | `Cluster`: Все пространства имен в кластере используют один экземпляр оператора для создания и управления, что приводит к меньшему потреблению ресурсов. |
| **Пространство имен** | Выберите `Рекомендуемое пространство имен`: Автоматически создается, если его нет.                                                   |
| **Стратегия обновления** | `Auto`: OperatorHub автоматически обновит оператора до последней версии, когда станет доступна новая версия.                        |

4. Рекомендуется использовать предложенную конфигурацию по умолчанию; просто нажмите **Установить**, чтобы завершить установку оператора **Argo CD**.

### Создание экземпляра Argo CD

1. Нажмите **Маркетплейс** > **OperatorHub**.

2. Найдите оператора **Argo CD**, нажмите его, чтобы перейти на страницу подробной информации о **ArgoCD**.

3. Нажмите **Все экземпляры**,

4. Нажмите **Создать экземпляр**, выберите карточку экземпляра **Argo CD**.

5. Нажмите **Создать экземпляр**.

:::info
На странице параметров конфигурации экземпляра используйте конфигурацию по умолчанию, если нет конкретных требований.
**Примечание**: Если глобальный кластер не обладает высокой доступностью (например, в нем только один управляющий узел), пожалуйста, переключитесь на вид YAML при создании экземпляра и установите значение поля ha.enabled в false.
:::

6. Нажмите **Создать**.

### Создание экземпляра AppProject

:::info
**Совет**: Если вам не нужно использовать функцию **Управление конфигурацией кластера**, управляемую платформой, выполнять следующие шаги не нужно.
:::

1. Найдите оператора **Argo CD**, нажмите его, чтобы перейти на страницу подробной информации о **Argo CD**.

2. Нажмите **Все экземпляры**, **Создать экземпляр**, выберите карточку экземпляра **AppProject**.

3. Переключитесь на вид YAML и замените существующее содержимое YAML на интерфейсе кодом ниже.

   ```
   apiVersion: argoproj.io/v1alpha1
   kind: AppProject
   metadata:
     name: cpaas-system
     namespace: argocd
   spec:
     clusterResourceWhitelist:
     - group: '*'
       kind: '*'
     destinations:
     - namespace: '*'
       server: '*'
     sourceRepos:
     - '*'
   ```

4. Нажмите **Создать**.

После завершения вышеуказанной процедуры вы успешно установили Argo CD. Немедленно [Создайте приложение Argo CD через панель управления Argo CD](../how_to/create_application_by_argocd_dashoard.mdx), чтобы начать свой путь в GitOps.
