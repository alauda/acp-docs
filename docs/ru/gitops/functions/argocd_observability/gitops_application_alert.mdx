# GitOps Приложения Операции

## Обзор

Управляющие возможности веб-консоли **GitOps Приложений** позволяют просматривать мониторинг, журналы и события **GitOps Приложений**. Вы также можете создавать политики оповещения для **GitOps Приложений**. Когда в **GitOps Приложениях** происходят аномалии, активируются предварительные уведомления для быстрого определения, анализа и решения проблем.

## Предварительные требования

- GitOps приложение было создано в веб-консоли. [Создание приложения Argo CD через веб-консоль](../../how_to/create_gitops_app.mdx)
- [Установка мониторинговых плагинов](/observability/monitor/install_monitor.mdx)

## Оповещение

> Создайте правило оповещения заранее для настройки правил. Когда GitOps приложения сталкиваются с аномалиями, активируются предварительные уведомления для быстрого определения, анализа и решения проблем.

1. **Платформа Контейнеров**, щелкните на **GitOps Приложения**.

2. Выберите имя GitOps приложения из списка, для которого вы хотите создать правило оповещения.

3. Перейдите на вкладку **Оповещения**.

4. Нажмите **Создать правило** и заполните основную информацию по мере необходимости.

5. Нажмите **Добавить условие оповещения**, перейдите на страницу **Условия оповещения**. Соответствующие описания метрик следующие:

:::info
Для других параметров конфигурации и настроек оповещения смотрите [Управление Оповещениями](/observability/monitor/functions/manage_alert.mdx).
:::

| **Название метрики**                                         | **Описание правила**                                                                                                    |
| :---------------------------------------------------------- | :---------------------------------------------------------------------------------------------------------------------- |
| Состояние здоровья GitOps приложения `gitops.applicationset.healthy` | Состояние здоровья GitOps приложения:<br /> - 0: Неизвестно, Утеряно, Ухудшено или Приостановлено <br /> - 1: Синхронизация <br /> - 2: Здорово |
| Состояние синхронизации GitOps приложения `gitops.applicationset.synced` | Состояние синхронизации GitOps приложения:<br /> - 0: Ошибка синхронизации или Ожидание<br /> - 1: Синхронизация <br /> - 2: Синхронизировано |

6. Нажмите **Создать**.

## Журналы

> Просмотр журналов для всех ресурсов нагрузки, созданных GitOps приложениями. Журналы позволяют быстро определить информацию о сбое системы без полагания на плагин кластера **Журнал**.

- На странице подробностей GitOps приложения, в разделе **Ресурсы Kubernetes**, щелкните на любом названии нагрузки, чтобы просмотреть информацию **Журналы** для этого ресурса справа.

## События

> Просмотр событий для всех ресурсов, распределенных по **GitOps Приложениям**. События позволяют быстро определить информацию о событиях сбоя системы без полагания на плагин кластера **Журнал**.

- На странице подробностей GitOps приложения перейдите на вкладку **События**, чтобы просмотреть агрегированные события для всех ресурсов.
- На странице подробностей GitOps приложения, в разделе **Ресурсы Kubernetes**, щелкните на любом названии ресурса, чтобы просмотреть события для этого ресурса справа.
