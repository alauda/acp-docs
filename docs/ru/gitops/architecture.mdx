# Архитектура

## GitOps и Argo CD

GitOps — это современная концепция для непрерывной доставки и операций, в то время как Argo CD является мощным инструментом, который реализует GitOps, контролируя конфигурационные файлы в репозитории Git и автоматически синхронизируя их с целевой средой. Этот подход улучшает скорость, надежность и безопасность доставки программного обеспечения, включая весь процесс доставки в систему контроля версий Git.

**ACP GitOps**, основанный на Argo CD, использует репозиторий Git в качестве единственного доверенного источника для хранения приложений, конфигураций инфраструктуры и других файлов для быстрой и точной дистрибуции и развертывания в один или несколько кластеров Kubernetes.

## Архитектура GitOps

Основные отличия между GitOps и традиционными методами управления приложениями:

- Вместо непосредственного манипулирования средой выполнения GitOps контролирует её, поддерживая репозиторий конфигурации приложения в Git.
- Argo CD постоянно извлекает данные из репозитория и исправляет несоответствия между средой выполнения и репозиторием конфигурации приложения, гарантируя, что среда соответствует ожиданиям, предотвращая дрейф конфигурации и позволяя быстро восстановиться в случае сбоя.

## Архитектура ACP GitOps

**ACP GitOps** устанавливается в качестве кластерного плагина на кластере `global` и использует Argo CD для распределения приложений и предоставления инфраструктуры через несколько бизнес-кластеров.
