# GitOps

## Введение

GitOps — это практика использования репозитория Git в качестве авторитетного источника конфигурации инфраструктуры и приложений. Все оперативные изменения контролируются версиями, автоматизируются и могут быть проверены через Git. Это основано на декларативных конфигурациях, хранящихся в Git, где любые изменения должны быть зафиксированы для запуска автоматических процессов развертывания.

## Основные принципы

- **Декларативная конфигурация**: GitOps в основном требует декларативных инструментов, рассматривая Git как единственный источник правды. Это позволяет обеспечить согласованное развертывание приложений в кластерах Kubernetes и восстановление, не зависящее от платформы, в случае сбоя.
- **Версионированное и неизменяемое состояние**: Версии инфраструктуры и приложений напрямую сопоставлены с коммитами Git. Откаты выполняются с помощью `git revert`, обеспечивая неизменяемую историю версий.
- **Автоматическое согласование**: Смешанные декларативные состояния автоматически применяются к кластерам. Это исключает ручное вмешательство, предотвращает ошибки человека и поддерживает одобрение безопасности в потоках развертывания.
- **Самовосстановление**: Контроллеры (например, Argo CD) постоянно согласовывают состояния кластеров с состояниями, определенными в Git, что позволяет системе восстанавливаться автономно.

## Преимущества <a id="gitops-advantage" />

- **Ускоренное сотрудничество и доставка**: Декларативные определения инфраструктуры, конфигураций и целевых состояний, хранящиеся в Git, позволяют автоматизировать развертывания. Команды достигают развертывания окружений одним нажатием кнопки после проверки, упрощая сотрудничество и доставку.
- **Быстрый откат и восстановление**: Используя контроль версий Git, аномалии вызывают мгновенные откаты. Контроллеры GitOps обеспечивают самовосстановление через автоматическое согласование.
- **Управление многооблачными средами**: Git как единственный источник правды в сочетании с конфигурационными наложениями позволяет точно выполнять массовые развертывания в гибридных/многооблачных средах.
- **Повышенная безопасность и соблюдение норм**: RBAC Git, журналы аудита, защиты веток и шифрование защищают чувствительные конфигурации, обеспечивая соблюдение норм.

## Популярные инструменты GitOps

- **Argo CD**: Декларативный инструмент GitOps, оптимизированный для Kubernetes, для определения, версионирования и автоматизации жизненного цикла приложений с возможностью аудита.
- **Flux**: Легкий оператор GitOps для Kubernetes, который постоянно синхронизирует репозитории Git с кластерами.
- **Jenkins X**: Платформа CI/CD с интеграцией GitOps для автоматизированных конвейеров и развертываний, управляемых Git.
