# GitOps

## Введение

GitOps — это практика использования Git-репозитория в качестве авторитетного источника для конфигураций инфраструктуры и приложений. Все операционные изменения находятся под версионным контролем, автоматизированы и подлежат аудиту через Git. Она основывается на декларативных конфигурациях, хранящихся в Git, где любые изменения должны быть зафиксированы, чтобы запустить процессы автоматического развертывания.

## Основные принципы

- **Декларативная конфигурация**: GitOps в корне требует использования декларативных инструментов, рассматривая Git как единственный источник истины. Это позволяет обеспечивать последовательное развертывание приложений на кластерах Kubernetes и независимое восстановление в случае сбоев.
- **Версифицированное и неизменяемое состояние**: версии инфраструктуры и приложений напрямую соотносятся с коммитами в Git. Откаты выполняются через `git revert`, что обеспечивает неизменяемую историю версий.
- **Автоматизированная согласованность**: объединенные декларативные состояния автоматически применяются к кластерам. Это исключает необходимость ручного вмешательства, предотвращает человеческие ошибки и поддерживает одобрения безопасности в рабочих процессах развертывания.
- **Самовосстановление**: контроллеры (например, Argo CD) постоянно сверяют состояния кластера с состояниями, определенными в Git, позволяя системе автономно восстанавливаться.

## Преимущества <a id="gitops-advantage" />

- **Ускоренная совместная работа и доставка**: декларативные определения инфраструктуры, конфигураций и целевых состояний, хранящиеся в Git, позволяют выполнять автоматизированные развертывания. Команды достигают развертывания окружений одним кликом после валидации, упрощая совместную работу и доставку.
- **Быстрый откат и восстановление**: используя систему версионного контроля Git, аномалии вызывают мгновенные откаты. Контроллеры GitOps обеспечивают самовосстановление через автоматизированную согласованность.
- **Управление многоуровневыми средами**: Git как единственный источник истины в сочетании с конфигурационными наложениями обеспечивает точные массовые развертывания в гибридных/многооблачных средах.
- **Повышенная безопасность и соответствие требованиям**: RBAC Git, журналы аудита, защиты веток и шифрование защищают конфиденциальные конфигурации, обеспечивая соблюдение стандартов.

## Популярные инструменты GitOps

- **Argo CD**: Декларативный инструмент GitOps, ориентированный на Kubernetes, для определения, версионирования и автоматизации жизненного цикла приложений с возможностью аудита.
- **Flux**: Легковесный оператор GitOps для Kubernetes, который постоянно синхронизирует Git-репозитории с кластерами.
- **Jenkins X**: Платформа CI/CD с интеграцией GitOps для автоматизированных конвейеров и развертываний, управляемых Git.
