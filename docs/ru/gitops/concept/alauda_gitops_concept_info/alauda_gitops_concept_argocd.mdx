# Введение

ACP GitOps — это родное для Kubernetes решение GitOps, построенное на Argo CD. Оно отслеживает конфигурационные манифесты (приложения, определения инфраструктуры и т.д.) в репозиториях Git и автоматически синхронизирует их с целевыми кластерами Kubernetes, реализуя поставку на основе Git. Кодificando весь процесс доставки в системе контроля версий Git, оно повышает скорость развертывания, надежность и безопасность, обеспечивая точное распределение приложений между несколькими кластерами.

Решение нативно интегрирует Argo CD Operator для автоматизации операций жизненного цикла развертывания, включая выделение ресурсов, обновления и откаты.

## Почему Argo CD?

Argo CD является ведущим открытым движком GitOps в отрасли благодаря своим уникальным преимуществам:

| **Технические преимущества**                                                                                                                                                          | **Операционные преимущества**                                                                                           |
| :--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | :----------------------------------------------------------------------------------------------------------------------- |
| **Декларативный движок GitOps** <ul><li> Примирение состояния через CRD (Application, ApplicationSet) </li><li> Поддержка нескольких источников (Helm, Kustomize, сырые YAML) </li></ul> | **Ускоренное развертывание** <ul><li> Развертывание на 70% быстрее благодаря автоматизации на основе Git</li></ul>         |
| **Родная архитектура для Kubernetes** <ul><li> Глубокая интеграция с API-сервером Kubernetes ≈ Родная поддержка изоляции пространства имен и RBAC </li></ul>                               | **Готовность для предприятий** <ul><li> Встроенная мультиарендность и возможности аудита </li></ul>                      |
| **Управление несколькими кластерами** <ul><li> Централизованная управляющая плоскость для гибридных/мультиоблачных развертываний </li><li> Специфическая для кластера конфигурация через ApplicationSets </li></ul>| **Операционная эффективность** <ul><li> Снижение ошибок развертывания на 60% за счет декларативного контроля </li></ul>  |
| **Расширяемая система плагинов** <ul><li> Сертифицированные интеграции с Helm, Kustomize, Istio </li><li> Определения пользовательских ресурсов (CRD) для сложных рабочих процессов </li></ul>  | **Оптимизация затрат** <ul><li> Снижение облачных затрат на 40% за счет точной оркестрации ресурсов </li></ul>              |
| **Активная экосистема CNCF** <ul><li> Более 3500 звезд на GitHub </li><li> Более 200 активных участников </li></ul>                                                                        | **Будущее-обеспечение** <ul><li> Непрерывные инновации через сообщество с открытым исходным кодом </li></ul>             |

## Преимущества <a id="alauda-gitops-advantage" />

> В дополнение к внутренним преимуществам GitOps, ACP GitOps предлагает следующие улучшенные преимущества:

- **Оператор Argo CD уровня предприятия**

  - Обеспечивает полный набор функциональных возможностей родного оператора Argo CD, охватывающих развертывание приложений, обновления, откаты и все основные функции Argo CD.

- **Служба безопасности оператора Argo CD**

  - Предлагает специализированную техническую поддержку для оператора Argo CD, касающуюся ответов на сбои, патчей уязвимостей безопасности и общей стабильности системы.

- **Управление распределением GitOps приложений в нескольких средах визуально**

  - Использует возможности управления несколькими кластерами и дифференцированной конфигурации платформы для достижения стилевого согласования, визуального управления приложениями GitOps и управления конфигурацией кластеров, упрощая точное распределение в многоволновых и многослойных настройках.

- **Управление операциями и обслуживанием приложений GitOps визуально**

  - Обеспечивает прямой доступ к реальным журналам и событиям ресурсов рабочей нагрузки Kubernetes в рамках приложений GitOps. В случае аномалий в приложениях GitOps пользователи могут быстро анализировать и решать проблемы, используя информацию о аномалиях Argo CD и журналы реальной рабочей нагрузки, не покидая текущий интерфейс.

- **Управление конфигурацией кластера GitOps визуально**

  - Управляет конфигурациями кластера через GitOps, достигая единого управления и распределения конфигураций кластера визуальным образом.

- **Управление приложениями GitOps с замкнутым циклом, интегрированное со всеми продуктами платформы**

  - Интегрирует возможности DevOps платформы для непрерывной сборки, управления артефактами и серого выпуска микросервисов для реализации полностью автоматизированного управления приложениями GitOps, формируя полный замкнутый цикл сотрудничества непрерывной интеграции и непрерывной доставки.
  - Сочетается с продуктами платформы, такими как CrossPlane, MySQL и портал разработчиков, для достижения полного процесса, охватывающего инициализацию инфраструктуры, инициализацию бизнес-приложений (включая код, конвейер, инициализацию приложений GitOps и т.д.) и разработку и развертывание кода приложения.
