# Здоровье

## Введение

Каково состояние приложения, работает ли оно корректно? Может ли оно обрабатывать запросы?

## Область здоровья

| **Статус здоровья** | **Описание**                                                                                                           |
| :------------------ | :--------------------------------------------------------------------------------------------------------------------- |
| **Здоровье**        | Ресурс работает корректно.                                                                                            |
| **Развивается**     | Ресурс пока нездоров, но продолжает развиваться и может скоро вернуться к нормальному состоянию.                     |
| **Снижено**         | Ресурс работает не в полную силу.                                                                                     |
| **Приостановлено**  | Ресурс приостановлен и ожидает внешнего события для возобновления (например, приостановленный CronJob или приостановленный Deployment). |

## Ссылка

Argo CD предоставляет встроенную оценку состояния здоровья для нескольких стандартных типов Kubernetes, что затем отображается в общем статусе здоровья приложения. Конечно, Argo CD также поддерживает пользовательские проверки состояния здоровья.

Для более подробных объяснений, пожалуйста, обратитесь к: [Здоровье](https://argo-cd.readthedocs.io/en/stable/operator-manual/health/)
