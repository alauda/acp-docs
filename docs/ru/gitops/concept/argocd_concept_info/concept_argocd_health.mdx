# Состояние здоровья

## Введение

Каково состояние здоровья приложения? Работает ли оно корректно? Может ли оно обрабатывать запросы?

## Область применения состояния здоровья

| **Состояние здоровья** | **Описание**                                                                                                                  |
| :-------------------- | :----------------------------------------------------------------------------------------------------------------------------- |
| **Здоровое**          | Ресурс здоров.                                                                                                               |
| **В процессе**        | Ресурс пока не здоров, но все же делает успехи и может стать здоровым в ближайшее время.                                      |
| **Деградированное**   | Ресурс деградирован.                                                                                                        |
| **Приостановлено**    | Ресурс приостановлен и ждет какого-либо внешнего события для возобновления (например, приостановленный CronJob или приостановленный Deployment). |

## Ссылки

Argo CD предоставляет встроенную оценку состояния здоровья для нескольких стандартных типов Kubernetes, которая затем отображается как общее состояние здоровья приложения в целом. Конечно, Argo CD также поддерживает пользовательские проверки состояния здоровья.

Для более подробных объяснений, пожалуйста, обратитесь к: [Состояние здоровья](https://argo-cd.readthedocs.io/en/stable/operator-manual/health/)
