# Синхронизация

## Обзор синхронизации

Синхронизация является основной функциональностью Argo CD, отвечающей за сравнение **Желательного Состояния** приложения с его **Текущим Состоянием** и за принятие мер по устранению несоответствий. В сущности, Синхронизация обеспечивает соответствие состояния приложений в вашем кластере Kubernetes состоянию, определенному в Git-репозитории.\
Вы можете инициировать синхронизацию вручную или настроить Argo CD для автоматического выполнения этой операции. Авто-синхронизация может быть активирована путем мониторинга изменений в Git-репозитории (например, коммиты, публикация тегов) или выполняться с заданными интервалами.

## Обзор статуса синхронизации

Статус синхронизации указывает на состояние синхронизации приложения, отражая, соответствует ли его **Текущее Состояние** **Желательному Состоянию**. Статус синхронизации включает следующие состояния:

- `Синхронизировано`: **Текущее Состояние** приложения точно соответствует **Желательному Состоянию**.
- `Не синхронизировано`: **Текущее Состояние** приложения расходится с **Желательным Состоянием**.
- `Синхронизация`: приложение проходит процесс синхронизации, при этом **Текущее Состояние** приближается к **Желательному Состоянию**.

## Обзор статуса операции синхронизации

Статус операции синхронизации представляет собой состояние выполнения операции синхронизации Argo CD, указывая на то, завершилась ли операция успешно. Статус операции синхронизации включает следующие состояния:

- `Успешно`: операция синхронизации завершилась успешно.
- `Не удалось`: операция синхронизации не удалась по причинам, таким как конфликты ресурсов Kubernetes, недостаточные права доступа и т.д.
- `Запуск`: операция синхронизации в процессе выполнения.

## Обзор обновления

Эта операция загружает последнюю конфигурацию приложения из Git-репозитория и сравнивает ее с фактическим состоянием в кластере Kubernetes. Обновление можно инициировать вручную или настроить для автоматического выполнения через заданные интервалы.
