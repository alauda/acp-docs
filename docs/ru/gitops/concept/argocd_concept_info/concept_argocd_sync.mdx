# Sync

## Обзор Sync

Sync является основной функциональностью Argo CD, отвечающей за сравнение **Желаемого состояния** приложения с его **Текущим состоянием** и принятие мер для разрешения несоответствий. В сущности, Sync обеспечивает соответствие состояния приложений в вашем кластере Kubernetes состоянию, определенному в репозитории Git.\
Вы можете вручную инициировать Sync или настроить Argo CD для его автоматического выполнения. Авто-Sync может быть инициирован путем мониторинга изменений в репозитории Git (например, коммиты, теги) или выполняться в запланированные интервалы.

## Обзор состояния Sync

Состояние Sync указывает на состояние синхронизации приложения, отражая, соответствует ли его **Текущее состояние** **Желаемому состоянию**. Состояние Sync включает в себя следующие состояния:

- `Synced`: **Текущее состояние** приложения точно соответствует **Желаемому состоянию**.
- `OutOfSync`: **Текущее состояние** приложения отличается от **Желаемого состояния**.
- `Syncing`: В приложении происходит процесс синхронизации, **Текущее состояние** приближается к **Желаемому состоянию**.

## Обзор статуса операции Sync

Статус операции Sync представляет собой состояние выполнения операции синхронизации, инициированной Argo CD, указывая, завершилась ли операция успешно. Статус операции Sync включает в себя следующие состояния:

- `Succeeded`: Операция синхронизации завершилась успешно.
- `Failed`: Операция синхронизации потерпела неудачу по таким причинам, как конфликты ресурсов Kubernetes, недостаточные разрешения и т. д.
- `Running`: Операция синхронизации выполняется.

## Обзор обновления

Эта операция получает последнюю конфигурацию приложения из репозитория Git и сравнивает ее с фактическим состоянием в кластере Kubernetes. Обновление можно инициировать вручную или настроить для автоматического выполнения через определенные промежутки времени.
