# Введение

Argo CD — это очень популярный инструмент GitOps с открытым исходным кодом. Чтобы использовать Argo CD, необходимо понять следующие основные концепции:

1. Приложение: группа ресурсов Kubernetes, определяемых манифестом. Это определение пользовательского ресурса (CRD). [Приложение](./concept_argocd_application.mdx)
2. Набор приложений: контроллер Kubernetes, поддерживающий CRD Набор приложений, позволяющий массовую генерацию приложений из одного шаблона. Рассматривайте это как фабрику приложений, которая создает экземпляры на основе параметров. [Набор приложений](./concept_argocd_applicationset.mdx)
3. Инструмент: указывает инструмент управления конфигурациями для источников приложения (например, Kustomize, Helm). [Инструмент](./concept_argocd_tool.mdx)
4. Синхронизация: процесс согласования живого состояния приложения с его желаемым состоянием (например, применение изменений к кластерам Kubernetes). [Синхронизация](./concept_argocd_sync.mdx)
5. Состояние: указывает на операционное состояние приложения, включая готовность и способность обслуживать запросы. [Состояние](./concept_argocd_health.mdx)

## Резюме различий между приложением и набором приложений

| **Атрибут**     | **Приложение**                                  | **Набор приложений**                                                             |
| :--------------- | :----------------------------------------------- | :------------------------------------------------------------------------------- |
| Определение      | Развертывание одного приложения                  | Шаблон для генерации нескольких экземпляров приложения.                         |
| Конфигурация     | Статические YAML определения                     | Динамическая генерация шаблонов на основе параметров                             |
| Развертывание    | Одно приложение                                 | Несколько аналогичных приложений                                                  |
| Сценарии использования | Простые развертывания в одном окружении   | Сложные развертывания в нескольких окружениях/кластерах, требующие параметризованных экземпляров |
| Основные концепции | Git Repo, целевой кластер, стратегии развертывания | Генераторы, шаблоны, параметры, заполнители                                       |
| Роль в Argo CD   | Основная единица развертывания                  | Продвинутый уровень управления оптом                                             |

## Статусы синхронизации Argo CD

| **Статус синхронизации** | **Описание**                                                          |
| :----------------------- | :-------------------------------------------------------------------- |
| **Синхронизировано**     | Живое состояние приложения полностью совпадает с желаемым состоянием.   |
| **Не синхронизировано**  | Живое состояние расходится с желаемым состоянием; требуется синхронизация. |
| **Синхронизация**        | В активном процессе синхронизации; живое состояние приближается к желаемому состоянию. |
