# Введение

Argo CD — это очень популярный инструмент GitOps с открытым исходным кодом. Чтобы использовать Argo CD, необходимо понять следующие ключевые концепции:

1. Приложение: Группа ресурсов Kubernetes, определенная манифестом. Это определение пользовательского ресурса (CRD). [Приложение](./concept_argocd_application.mdx)
2. ApplicationSet: Контроллер Kubernetes, поддерживающий CRD ApplicationSet, который позволяет массовую генерацию приложений из единого шаблона. Рассматривайте это как фабрику приложений, создающую экземпляры на основе параметров. [ApplicationSet](./concept_argocd_applicationset.mdx)
3. Инструмент: Определяет инструмент управления конфигурацией для источников приложений (например, Kustomize, Helm). [Инструмент](./concept_argocd_tool.mdx)
4. Синхронизация: Процесс согласования живого состояния приложения с его желаемым состоянием (например, применение изменений в кластерах Kubernetes). [Синхронизация](./concept_argocd_sync.mdx)
5. Состояние здоровья: Указывает операционный статус приложения, включая готовность и способность обрабатывать запросы. [Состояние здоровья](./concept_argocd_health.mdx)

## Сводка различий между Приложением и ApplicationSet

| **Атрибут**     | **Приложение**                              | **ApplicationSet**                                                              |
| :-------------- | :------------------------------------------ | :------------------------------------------------------------------------------- |
| Определение      | Развертывание единственного приложения      | Шаблон для генерации нескольких экземпляров приложений.                         |
| Конфигурация    | Статические определения YAML               | Динамическое создание шаблонов на основе параметров                             |
| Развертывание    | Одно приложение                            | Несколько схожих приложений                                                     |
| Сценарии использования | Простые развертывания в одном окружении | Сложные развертывания в нескольких окружениях/кластерах, требующие параметризованных экземпляров |
| Основные концепции | Git Repo, целевой кластер, стратегии развертывания | Генераторы, шаблоны, параметры, заполнители                                      |
| Роль в Argo CD.  | Фундаментальная единица развертывания     | Продвинутый уровень управления пакетами                                         |

## Статусы синхронизации Argo CD

| **Статус синхронизации** | **Описание**                                                           |
| :----------------------- | :--------------------------------------------------------------------- |
| **Синхронизировано**     | Живое состояние приложения полностью соответствует желаемому состоянию. |
| **Не синхронизировано**  | Живое состояние отличается от желаемого состояния; требуется синхронизация. |
| **Синхронизация**       | В процессе активной синхронизации; живое состояние приближается к желаемому состоянию. |
