# Helm

## Введение

Helm — это инструмент управления пакетами для Kubernetes, позволяющий пользователям определять, устанавливать и обновлять сложные приложения Kubernetes. **Helm Chart** — это пакет конфигурации на основе шаблонов, содержащий определения ресурсов Kubernetes (файлы YAML).

## Основные концепции Helm

- **Chart**: Helm Chart — это пакет конфигурации на основе шаблонов, содержащий определения ресурсов Kubernetes (файлы YAML).
- **Release**: Helm Release — это экземпляр развернутого Helm Chart, представляющий конкретную конфигурацию ресурсов Kubernetes.
- **Values**: Helm Values — это параметризованные конфигурации для Helm Chart, позволяющие пользователям настраивать определения ресурсов Kubernetes.

Интеграция Argo CD с Helm улучшает практики GitOps, позволяя осуществлять декларативную непрерывную доставку через веб-консоль, панель управления Argo CD или CLI. Пример:

```yaml
apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: sealed-secrets
  namespace: argocd
spec:
  project: default
  source:
    chart: sealed-secrets
    repoURL: https://bitnami-labs.github.io/sealed-secrets
    targetRevision: 1.16.1
    helm:
      releaseName: sealed-secrets
  destination:
    server: "https://kubernetes.default.svc"
    namespace: kubeseal
```

Пример OCI Helm Chart:

```yaml
apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: nginx
spec:
  project: default
  source:
    chart: nginx
    repoURL: registry-1.docker.io/bitnamicharts  # обратите внимание: синтаксис oci:// не включен.
    targetRevision: 15.9.0
  destination:
    name: "in-cluster"
    namespace: nginx
```

:::info
\*\* Цикл жизни приложения управляется Argo CD, а не Helm. \*\*
Когда предоставляется несколько источников значений, порядок приоритетов следующий: `parameters` > `valuesObject` > `values` > `valueFiles` > helm-репозиторий `values.yaml`.
:::

## Преимущества

- **Шаблонизация**: Helm использует движок шаблонов Go (gotpl) для динамической генерации файлов ресурсов Kubernetes.

- **Управление пакетами**: Helm упаковывает приложения в виде Charts (включая шаблоны, значения по умолчанию и зависимости), что упрощает распространение и управление версиями.

- **Управление зависимостями**: Поддерживает зависимости между Charts.

- **Управление жизненным циклом**: Предоставляет команды для полного управления жизненным циклом, такие как установка, обновление и откат.

## Сценарии использования

- **Развертывание сложных приложений**: Идеально подходит для сценариев, требующих динамической генерации конфигураций (например, переменные среды или ввод пользователя).

- **Развертывания в нескольких средах**: Поддерживает конфигурации, специфичные для среды, через файлы values.yaml.

- **Распространение приложений**: Позволяет упаковывать Charts для распределения в репозитории Helm или OCI-реестры.
