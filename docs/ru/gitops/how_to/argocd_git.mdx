# Интеграция кодовых репозиториев через панель управления Argo CD

Используйте родную панель управления Argo CD для интеграции кодовых репозиториев и их распределения, позволяя разработчикам управлять приложениями GitOps на протяжении всего их жизненного цикла через визуальный интерфейс.

## Примеры использования

- Упростить процесс создания **GitOps-приложений**, выбрав связанный репозиторий через веб-консоль при их создании.
- При создании **Приложения** через родную панель управления Argo CD вы можете выбрать использование связанного репозитория.

## Предварительные требования

- [Установка ACP GitOps](../install_gitops/install_gitops_plugin.mdx) и включен переключатель **Родной UI Argo CD**.
- Доступ к URL **Родного UI Argo CD** вместе с именем пользователя и паролем.
  - Администраторы могут напрямую получить доступ к URL через страницу деталей плагина GitOps кластера.

## Процедура

Следуйте этим шагам, чтобы использовать функции:

1. Подключение к кодовому репозиторию

   - Войдите в **Argo CD**, используя URL доступа.

   - Нажмите на **Настройки** в левой боковой панели навигации.

   - Нажмите на карточку **РЕПО**.

   - Нажмите **ПОДКЛЮЧИТЬ РЕПО** в верхнем левом углу страницы.

   - Выберите метод подключения к репозиторию и заполните соответствующие параметры по мере необходимости.

   - Нажмите **ПОДКЛЮЧИТЬ**.

2. Связать проект

   - Нажмите на **Настройки** в левой боковой панели навигации.

   - Нажмите на карточку **Проекты**.

   - Нажмите на проект, в котором необходимо создать приложение GitOps.

   Примечание: Argo CD автоматически синхронизирует проекты в кластере, поэтому нет необходимости создавать их вручную.

   - Нажмите **ИЗМЕНИТЬ** в разделе **ИСТОЧНИКИ РЕПОЗИТОРИЕВ**.

   - Нажмите **ДОБАВИТЬ ИСТОЧНИК**, введите URL репозитория из шага **Подключите репозиторий**, и свяжите с проектом.

   - Нажмите **СОХРАНИТЬ**.

## Результат операции

- Вернитесь к веб-консоли и перейдите к **Платформа контейнеров** > **Приложения GitOps**. На странице **Создать** вы увидите связанные репозитории.
