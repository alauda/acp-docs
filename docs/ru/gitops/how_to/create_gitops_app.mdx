# Создание приложения Argo CD через веб-консоль

В этой статье будет представлен полный процесс создания приложения Argo CD через веб-консоль в разделе **GitOps Applications**, что позволит управлять приложением с помощью GitOps.

## Сценарии использования

- Создать приложение Argo CD на базе SpringBoot с использованием веб-консоли, чтобы ознакомиться с полным процессом управления приложениями через GitOps.

## Предварительные условия

- [Установка ACP GitOps](../install_gitops/install_gitops_plugin.mdx)
- Проекты и пространства имен были выделены

## Процедура

Следуйте этим шагам, чтобы использовать функции:

### Конфигурация репозитория кода

> Если вы не видите Интегрированный репозиторий кода на странице деталей **Создать GitOps Applications**, вы можете сначала интегрировать репозиторий кода:

- [Интеграция репозиториев кода через панель управления Argo CD](../how_to/argocd_git.mdx)

:::info
Если у вас нет доступного репозитория кода, вы можете использовать демонстрационный репозиторий в качестве примера.
**URL репозитория**: `https://github.com/argoproj/argocd-example-apps.git`
**Описание**: Этот репозиторий содержит примеры приложений, которые можно использовать для демонстрации и тестирования функций Argo CD.
:::

### Создание приложения Argo CD с использованием GitOps Applications

1. **Платформа контейнеров**, нажмите на **GitOps Applications**.

2. Нажмите на **Создать GitOps Application**.

3. В разделах **Основная информация** и **Репозиторий кода** настройте соответствующую информацию в соответствии с приведенными ниже инструкциями.

| **Параметр**               | **Вводимое содержимое**  |
| :-------------------------- | :---------------------- |
| **Тип**                    | `Приложение`            |
| **Источник**               | `Интеграция Argo CD`    |
| **Имя интегрированного проекта** | `argocd-example-apps`   |
| **Идентификатор версии**   | `Ветка` <br />`master`  |
| **Тип файла источника**    | `Helm`                  |
| **Каталог файла источника**| `helm-guestbook`        |
| **Пользовательские значения** | `values.yaml`          |

4. В пункте **Распределение** используйте рекомендуемое платформой ***Namespace*** или выберите другое пространство имен.

5. Установите политику синхронизации по умолчанию на **Ручная синхронизация**.

6. Нажмите на **Создать**.
