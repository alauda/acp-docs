# Создание приложения Argo CD через веб-консоль

В этой статье будет представлен полный процесс создания приложения Argo CD с помощью раздела **GitOps Приложения** веб-консоли, что позволит управлять приложением с использованием GitOps.

## Примеры использования

- Создать приложение Argo CD на базе SpringBoot, используя веб-консоль, чтобы ознакомится с полным процессом управления приложениями через GitOps.

## Предварительные условия

- [Установка ACP GitOps](../install_gitops/install_gitops_plugin.mdx)
- Проекты и пространства имен были распределены

## Процедура

Следуйте этим шагам для использования функциональных возможностей:

### Настройка репозитория кода

> Если вы не видите Интегрированный репозиторий кода на странице деталей **Создать GitOps Приложение**, вы можете сначала интегрировать репозиторий кода:

- [Интеграция репозиториев кода через панель управления Argo CD](../how_to/argocd_git.mdx)

:::info
Если у вас нет доступного репозитория кода, вы можете использовать демонстрационный репозиторий для демонстрационных целей.
**URL репозитория**: `https://github.com/argoproj/argocd-example-apps.git`
**Описание**: Этот репозиторий содержит пример приложений, которые можно использовать для демонстрации и тестирования функциональности Argo CD.
:::

### Создание приложения Argo CD с помощью GitOps приложений

1. **Платформа контейнеров**, нажмите на **GitOps Приложения**.

2. Нажмите на **Создать GitOps приложение**.

3. В разделах **Основная информация** и **Репозиторий кода** настройте соответствующую информацию согласно следующим инструкциям.

| **Параметр**               | **Вводимое содержание**   |
| :------------------------- | :----------------------- |
| **Тип**                   | `Приложение`             |
| **Источник**              | `Интеграция Argo CD`     |
| **Интегрированное имя проекта** | `argocd-example-apps`   |
| **Идентификатор версии**  | `Ветка` <br />`master`   |
| **Тип исходного файла**   | `Helm`                  |
| **Каталог исходного файла** | `helm-guestbook`        |
| **Пользовательские значения** | `values.yaml`           |

4. В разделе **Распределение** используйте рекомендованное платформой ***Namespace*** или выберите другое пространство имен.

5. Установите политику синхронизации по умолчанию на **Ручная синхронизация**.

6. Нажмите на **Создать**.
