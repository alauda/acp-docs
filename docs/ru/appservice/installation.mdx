## Установка Alauda Application Services Essentials

::: tip Опциональная установка
Плагин служит фундаментальным компонентом веб-консоли Alauda Application Services. Установка не обязательна, если веб-консоль не требуется.
:::

### Предварительные требования

1. Скачайте установочный пакет Alauda Application Services Essentials, соответствующий платформе.
2. Используйте возможности публикации приложений платформы для публикации Alauda Application Services Essentials в кластер `global`.

### Процесс

<Tabs>
  <Tab label="Веб-консоль">
    1. В левом навигационном меню перейдите в раздел **Marketplace** > **Плагины кластера**.
    2. Выберите кластер `global`.
    3. Нажмите кнопку действия рядом с плагином **Alauda Application Services Essentials** > **Установить**.
  </Tab>
</Tabs>

## Установка Alauda Application Services RDS Framework

::: tip Опциональная установка
Плагин служит фундаментальным компонентом веб-консоли Alauda Application Services. Установка не обязательна, если веб-консоль не требуется.
:::

### Предварительные требования

1. Скачайте установочный пакет Alauda Application Services RDS Framework, соответствующий платформе.
2. Используйте возможности публикации приложений платформы для публикации Alauda Application Services RDS Framework в любой кластер, где требуются графические возможности Application Services.

### Процесс

<Tabs>
  <Tab label="Веб-консоль">
    1. Войдите в платформу и перейдите на страницу **Управление платформой**.
    2. В левом навигационном меню выберите **Marketplace** -> **OperatorHub**, чтобы перейти на страницу **OperatorHub**.
    3. Найдите **Alauda Application Services RDS Framework**, нажмите **Установить**, чтобы перейти на страницу установки.

    Параметры конфигурации:

    | **Параметр**             | **Рекомендуемая конфигурация**                                                                                                                  |
    | :------------------------ | :--------------------------------------------------------------------------------------------------------------------------------------------- |
    | **Канал**                | Канал по умолчанию — **stable**.                                                                                                                 |
    | **Версия**               | Выберите желаемую версию для установки.                                                                                                         |
    | **Режим установки**       | **Cluster**: Все пространства имен в кластере используют один оператор для создания и управления экземплярами, что приводит к меньшему потреблению ресурсов.       |
    | **Место установки**       | Выберите **Рекомендуемое**: Если его нет, он будет создан автоматически.                                                                  |
    | **Стратегия обновления** | **Ручная**: Когда новая версия доступна в OperatorHub, требуется ручное подтверждение для обновления оператора до последней версии. |

    4. На странице **Установка оператора** выберите **Стандартная конфигурация**, затем нажмите **Установить**, чтобы завершить установку **Alauda Application Services RDS Framework**.
  </Tab>
</Tabs>

## Установка бизнес-плагины Application Services

### Предварительные требования

1. Скачайте установочный пакет плагина, соответствующий платформе.
2. Загрузите плагин в любой кластер, который планирует использовать возможности бизнес-компонента Application Services.

### Процесс

<Tabs>
  <Tab label="MySQL">
    1. Войдите в платформу и перейдите на страницу **Управление платформой**.
    2. В левом навигационном меню выберите **Marketplace** -> **OperatorHub**, и перейдите на страницу **OperatorHub**.
    3. Найдите **Alauda Database Service for MySQL**, нажмите **Установить** и перейдите на страницу установки.

    Параметры конфигурации:

    | **Параметр**             | **Рекомендуемая конфигурация**                                                                                                                  |
    | :------------------------ | :--------------------------------------------------------------------------------------------------------------------------------------------- |
    | **Канал**                | Канал по умолчанию — **alpha**.                                                                                                              |
    | **Версия**               | Выберите желаемую версию для установки.                                                                                                         |
    | **Режим установки**       | **Cluster**: Все пространства имен в кластере используют один оператор для создания и управления экземплярами, что приводит к меньшему потреблению ресурсов.       |
    | **Место установки**       | Выберите **Рекомендуемое**: Если его нет, он будет создан автоматически.                                                                  |
    | **Стратегия обновления** | **Ручная**: Когда новая версия доступна в OperatorHub, требуется ручное подтверждение для обновления оператора до последней версии. |

    4. На странице **Установка оператора** выберите **Стандартная конфигурация**, нажмите **Установить** и завершите установку.
  </Tab>
