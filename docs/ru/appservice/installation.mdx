## Установка Alauda Application Services Essentials

::: tip Дополнительная установка
Плагин является основным компонентом веб-консоли Alauda Application Services. Установка не требуется, если веб-консоль не нужна.
:::

### Предварительные требования

1. Скачайте установочный пакет Alauda Application Services Essentials, соответствующий платформе.
2. Используйте возможности публикации приложений платформы для публикации Alauda Application Services Essentials в кластер `global`.

### Процедура

<Tabs>
  <Tab label="Веб-консоль">
    1. В левом боковом меню перейдите в **Marketplace** > **Cluster Plugins**.
    2. Выберите кластер `global`.
    3. Нажмите на кнопку действия рядом с плагином **Alauda Application Services Essentials** > **Установить**.
  </Tab>
</Tabs>

## Установка Alauda Application Services RDS Framework

::: tip Дополнительная установка
Плагин является основным компонентом веб-консоли Alauda Application Services. Установка не требуется, если веб-консоль не нужна.
:::

### Предварительные требования

1. Скачайте установочный пакет Alauda Application Services RDS Framework, соответствующий платформе.
2. Используйте возможности публикации приложений платформы для публикации Alauda Application Services RDS Framework в любой кластер, где необходимы графические возможности Application Services.

### Процедура

<Tabs>
  <Tab label="Веб-консоль">
    1. Войдите в платформу и перейдите на страницу **Управление платформой**.
    2. В левом боковом меню выберите **Marketplace** -> **OperatorHub**, чтобы перейти на страницу **OperatorHub**.
    3. Найдите **Alauda Application Services RDS Framework**, нажмите **Установить**, чтобы перейти на страницу установки.

    Параметры конфигурации:

    | **Параметр**             | **Рекомендуемая конфигурация**                                                                                                                 |
    | :------------------------ | :------------------------------------------------------------------------------------------------------------------------------------------------ |
    | **Канал**                | Канал по умолчанию - **stable**.                                                                                                                 |
    | **Версия**               | Выберите нужную версию для установки.                                                                                                            |
    | **Режим установки**       | **Cluster**: Все пространства имен в кластере используют один оператор для создания и управления экземплярами, что приводит к снижению потребления ресурсов. |
    | **Место установки**       | Выберите **Рекомендуемое**: Если таких нет, оно будет создано автоматически.                                                                       |
    | **Стратегия обновления** | **Ручное**: Когда новая версия доступна в OperatorHub, для обновления оператора до последней версии требуется ручное подтверждение.           |

    4. На странице **Установка оператора** выберите **Стандартная конфигурация**, затем нажмите **Установить**, чтобы завершить установку **Alauda Application Services RDS Framework**.
  </Tab>
</Tabs>

## Установка бизнес-плагина Application Services

### Предварительные требования

1. Скачайте установочный пакет плагина, соответствующий платформе.
2. Загрузите плагин в любой кластер, который намеревается использовать возможности бизнес-компонентов Application Services.

### Процедура

<Tabs>
  <Tab label="MySQL">
    1. Войдите в платформу и перейдите на страницу **Управление платформой**.
    2. В левом боковом меню выберите **Marketplace** -> **OperatorHub**, и перейдите на страницу **OperatorHub**.
    3. Найдите **Alauda Database Service for MySQL**, нажмите **Установить**, и перейдите на страницу установки.

    Параметры конфигурации:

    | **Параметр**             | **Рекомендуемая конфигурация**                                                                                                               |
    | :------------------------ | :--------------------------------------------------------------------------------------------------------------------------------------------- |
    | **Канал**                | Канал по умолчанию - **alpha**.                                                                                                               |
    | **Версия**               | Выберите нужную версию для установки.                                                                                                         |
    | **Режим установки**       | **Cluster**: Все пространства имен в кластере используют один оператор для создания и управления экземплярами, что приводит к снижению потребления ресурсов. |
    | **Место установки**       | Выберите **Рекомендуемое**: Если таких нет, оно будет создано автоматически.                                                                  |
    | **Стратегия обновления** | **Ручное**: Когда новая версия доступна в OperatorHub, для обновления оператора до последней версии требуется ручное подтверждение.          |

    4. На странице **Установка оператора** выберите **Стандартная конфигурация**, нажмите **Установить** и завершите установку.
  </Tab>
