## Установка основных компонентов Alauda Container Platform Data Services

::: tip Дополнительная установка
Плагин служит основным компонентом для представления Data Services. Установка не требуется, если веб-консоль не нужна.
:::

### Предварительные требования

1. Скачайте установочный пакет основных компонентов Alauda Container Platform Data Services, соответствующий платформе.
2. Используйте возможности публикации приложений платформы для публикации основных компонентов Alauda Container Platform Data Services в кластер `global`.

### Процедура

<Tabs>
  <Tab label="Веб-консоль">
    1. В левой навигационной панели перейдите в раздел **Marketplace** > **Cluster Plugins**.
    2. Выберите кластер `global`.
    3. Нажмите кнопку действия рядом с плагином **Alauda Container Platform Data Services Essentials** > **Установить**.
  </Tab>
</Tabs>

## Установка фреймворка Alauda Container Platform Data Services RDS

::: tip Дополнительная установка
Плагин служит основным компонентом для представления Data Services. Установка не требуется, если веб-консоль не нужна.
:::

### Предварительные требования

1. Скачайте установочный пакет фреймворка Alauda Container Platform Data Services RDS, соответствующий платформе.
2. Используйте возможности публикации приложений платформы для публикации фреймворка Alauda Container Platform Data Services RDS в любой кластер, где требуется представление Data Services.

### Процедура

<Tabs>
  <Tab label="Веб-консоль">
    1. Войдите в платформу и перейдите на страницу **Администратор**.
    2. В левой навигационной панели выберите **Marketplace** -> **OperatorHub**, чтобы перейти на страницу **OperatorHub**.
    3. Найдите **Alauda Container Platform Data Services RDS Framework**, нажмите **Установить**, чтобы перейти на страницу установки.

    Параметры конфигурации:

    | **Параметр**             | **Рекомендуемая конфигурация**                                                                                                                  |
    | :------------------------ | :--------------------------------------------------------------------------------------------------------------------------------------------- |
    | **Канал**                | Канал по умолчанию - **stable**.                                                                                                             |
    | **Версия**               | Выберите желаемую версию для установки.                                                                                                         |
    | **Режим установки**       | **Cluster**: Все пространства имен в кластере используют один оператор для создания и управления экземплярами, что приводит к снижению использования ресурсов.       |
    | **Место установки**       | Выберите **Рекомендуемое**: Если его нет, он будет создан автоматически.                                                                  |
    | **Стратегия обновления** | **Ручная**: Когда новая версия доступна в OperatorHub, требуется ручное подтверждение для обновления оператора до последней версии. |

    4. На странице **Установить оператор** выберите **Стандартная конфигурация**, затем нажмите **Установить**, чтобы завершить установку **Alauda Container Platform Data Services RDS Framework**.
  </Tab>
</Tabs>

## Установка бизнес-плагина Application Services

### Предварительные требования

1. Скачайте установочный пакет плагина, соответствующий платформе.
2. Загрузите плагин в любой кластер, который намерен использовать возможности бизнес-компонентов Application Services.

### Процедура

<Tabs>
  <Tab label="MySQL">
    1. Войдите в платформу и перейдите на страницу **Администратор**.
    2. В левой навигационной панели выберите **Marketplace** -> **OperatorHub**, и перейдите на страницу **OperatorHub**.
    3. Найдите **Alauda Database Service for MySQL**, нажмите **Установить** и перейдите на страницу установки.

    Параметры конфигурации:

    | **Параметр**             | **Рекомендуемая конфигурация**                                                                                                                  |
    | :------------------------ | :--------------------------------------------------------------------------------------------------------------------------------------------- |
    | **Канал**                | Канал по умолчанию - **alpha**.                                                                                                              |
    | **Версия**               | Выберите желаемую версию для установки.                                                                                                         |
    | **Режим установки**       | **Cluster**: Все пространства имен в кластере используют один оператор для создания и управления экземплярами, что приводит к снижению использования ресурсов.       |
    | **Место установки**       | Выберите **Рекомендуемое**: Если его нет, он будет создан автоматически.                                                                  |
    | **Стратегия обновления** | **Ручная**: Когда новая версия доступна в OperatorHub, требуется ручное подтверждение для обновления оператора до последней версии. |

    4. На странице **Установить оператор** выберите **Стандартная конфигурация**, нажмите **Установить** и завершите установку.
  </Tab>
