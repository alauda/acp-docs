# Возможности

## Единое управление эксплуатацией и обслуживанием

- Единое управление эксплуатацией и обслуживанием

  Предоставляет единый интерфейс для управления эксплуатацией и обслуживанием, который поддерживает комплексные функции, такие как управление жизненным циклом сервисов между кластерами, наблюдаемость, управление изменениями, резервное копирование и восстановление, инспекция и т.д.

## Поддержка различных сервисов \{#compatible\_sc}

- Поддерживает различные типы приложений Alauda, включая базы данных, кеши, очереди сообщений и т.д. Ниже приведены основные версии поддерживаемых приложений Alauda и информация о совместимых классах хранилищ:

  | ------------------------- | ------------------------------------- | --------------------------------------------------------------- | ------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
  | MySQL-PXC                 | 5.7                                   | TopoLVM (рекомендуется)                                        | S3 (рекомендуется), TopoLVM                              | Не более (обязательно) одного узла MySQL может быть запланировано на каждом узле.                                                                                                                                           |
  | MySQL-MGR                 | 8.0                                   | TopoLVM (рекомендуется)                                        | S3 (рекомендуется)                                       | Не более (обязательно) одного узла MySQL может быть запланировано на каждом узле.                                                                                                                                           |
  | Redis                     | 5.0, 6.0, 7.2 (рекомендуется)        | TopoLVM (рекомендуется)                                        | S3 (рекомендуется), CephFS, NFS                          | Режим кластера: Не более (обязательно) одного узла Redis может быть запланировано на каждом узле. - Поддерживает операции по управлению эксплуатацией и обслуживанием различных сервисов, подробные возможности представлены ниже:

<Tabs>
  <Tab label="MySQL-PXC">
    - Управление экземплярами

      Поддерживает управление жизненным циклом экземпляров (создание, обновление, модернизация, удаление, перезапуск, остановка).

    - Конфигурация параметров

      Предустановленные шаблоны параметров, подходящие для производства, поддерживают создание и управление шаблонами параметров; поддерживает инициализацию параметров с помощью шаблонов параметров, а также обновление, импорт и экспорт параметров.

    - Методы доступа

      Поддерживает несколько методов для предоставления сервисов как внутри кластера, так и вне его, таких как NodePort, MetaLB, ALB и т.д. Поддерживает однонажатийное копирование информации для подключения с предоставлением примера кода для помощи.

    - Топология

      Поддерживает отображение топологии экземпляров и состояние работы каждого узла.

    - Управление пользователями

      Поддерживает управление пользователями для экземпляров, включая создание, удаление пользователей и контроль доступа.

    - Резервное копирование и восстановление

      Поддерживает ручное и запланированное резервное копирование с возможностью резервного копирования в хранилища и внешнее хранилище S3.

    - Масштабирование

      Поддерживает операции масштабирования, включая горизонтальное масштабирование узлов и вертикальное масштабирование спецификаций.

    - Консоль терминала

      Поддерживает командный терминал, упрощая быстрый вход на узлы для операций пользователей.

    - Анализ запросов

      Поддерживает отображение анализа SQL-запросов, демонстрируя SQL-выражения, частоту, время выполнения и другую информацию.

    - Журналы

      Поддерживает запросы журналов в реальном времени, включая поиск по времени, контейнерам и т.д.; поддерживает долговременное хранение журналов.

    - Мониторинг и оповещения

      Отображает данные мониторинга, включая использование различных ресурсов, количество соединений, TPS, операции блокировки таблиц, медленные запросы и т.д.; поддерживает создание и обновление политик оповещения с предустановленными метриками, включая количество запросов на чтение/запись InnoDB, количество соединений, количество потоков и десять других метрик; поддерживает настройки оповещения для сотен метрик мониторинга (непредустановленные).

    - Инспекция

      Поддерживает инспекцию экземпляров с возможностью ручного и автоматического запуска; поддерживает отображение статистической информации о результатах инспекции; поддерживает экспорт отчетов об инспекции.

    - Восстановление после аварий

      Поддерживает горячее резервирование для восстановления после аварий с вариантами асинхронной и синхронизированной репликации для кластеров восстановления после аварий.

    - Поддержка стеков сетевых протоколов

      Поддерживает IPv4/IPv6 двойной стек, одиночный стек IPv4 и одиночный стек IPv6.

      Поддерживает запросы журналов в реальном времени, включая поиск по времени, контейнерам и т.д.; поддерживает долговременное хранение журналов.

      Отображает данные мониторинга, включая использование различных ресурсов, количество соединений, TPS, операции блокировки таблиц, медленные запросы и т.д.; поддерживает создание и обновление политик оповещения с предустановленными метриками, включая количество запросов на чтение/запись InnoDB, количество соединений, количество потоков и десять других метрик; поддерживает настройки оповещения для сотен метрик мониторинга (непредустановленные).

    - Инспекция

      Поддерживает инспекцию экземпляров с возможностью ручного и автоматического запуска; поддерживает отображение статистической информации о результатах инспекции; поддерживает экспорт отчетов об инспекции.
