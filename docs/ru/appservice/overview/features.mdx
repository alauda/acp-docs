# Особенности

## Единое управление эксплуатацией и техобслуживанием

- Унифицированное управление эксплуатацией и техобслуживанием

  Предоставляет унифицированный интерфейс управления эксплуатацией и техобслуживанием, который поддерживает комплексные функции, такие как управление жизненным циклом сервисов в кластерах, наблюдаемость, управление изменениями, резервное копирование и восстановление, инспекция и т. д.

## Поддержка разнообразных сервисов \{#compatible\_sc}

- Поддерживает различные типы приложений Alauda, включая базы данных, кеши, очереди сообщений и др. Ниже приведены основные поддерживаемые версии приложений Alauda и информация о совместимости классов хранилищ:

  | Приложения Alauda          | Версия                                 | Класс хранилища с поддержкой сохранности данных | Класс хранилища с поддержкой резервного копирования | Правило планирования по умолчанию (одиночный экземпляр)                                                                                                                                                                |
  | --------------------------- | -------------------------------------- | ----------------------------------------- | ------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
  | MySQL-PXC                   | 5.7                                    | TopoLVM (рекомендуется)                     | S3 (рекомендуется), TopoLVM       | В каждом узле может быть запланирован не более чем один узел MySQL.                                                                                                                                                   |
  | MySQL-MGR                   | 8.0                                    | TopoLVM (рекомендуется)                     | S3 (рекомендуется)                | В каждом узле может быть запланирован не более чем один узел MySQL.                                                                                                                                                   |
  | Redis                       | 5.0, 6.0, 7.2 (рекомендуется)        | TopoLVM (рекомендуется)                     | S3 (рекомендуется), CephFS, NFS   | Режим кластера: в каждом узле может быть запланирован не более чем один узел Redis.   | Kafka                       | 2.5.0, 2.7.0, 2.8.2, 3.8 (рекомендуется) | TopoLVM (рекомендуется)                     | -                               | В каждом узле может быть запланирован не более чем один узел брокера.                                                                                                                                               |

  | PostgreSQL                  | 11, 12, 14                             | TopoLVM (рекомендуется)                     | S3 (рекомендуется)                | Стремитесь планировать один узел PostgreSQL в каждом узле.                                                                                                                                                         |

- Поддерживает управление эксплуатацией и техобслуживанием различных сервисов, подробные возможности приведены ниже:

<Tabs>
  <Tab label="MySQL-PXC">
    - Управление экземплярами

      Поддерживает управление жизненным циклом экземпляров (создание, обновление, модернизация, удаление, перезагрузка, остановка).

    - Конфигурация параметров

      Предустановленные шаблоны параметров, подходящие для производственных условий, поддерживает создание и управление шаблонами параметров; поддерживает инициализацию параметров с помощью шаблонов, а также обновление, импорт и экспорт параметров.

    - Методы доступа

      Поддерживает несколько методов для выставления сервисов как внутри, так и за пределами кластера, таких как NodePort, MetaLB, ALB и т. д. Поддерживает однонажатие для копирования информации о соединении, с примерами кода для помощи.

    - Топология

      Поддерживает отображение топологии экземпляров и статуса работы каждого узла.

    - Управление пользователями

      Поддерживает управление пользователями для экземпляров, включая создание и удаление пользователей, а также контроль доступа.

    - Резервное копирование и восстановление

      Поддерживает ручное и запланированное резервное копирование, с опциями резервного копирования на хранищах и внешнем S3 хранилище.

    - Масштабируемость

      Поддерживает операции масштабирования, включая горизонтальное масштабирование узлов и вертикальное масштабирование спецификаций.

    - Консоль терминала

      Поддерживает командную строку, облегчающую быстрые входы в узлы для операций пользователей.

    - Анализ запросов

      Поддерживает отображение анализа SQL-запросов, демонстрируя SQL-операции, частоту, время выполнения и другую информацию.

    - Журналы

      Поддерживает запросы журналов в реальном времени, включая поиск по времени, контейнеру и др.; поддерживает сохранность журналов.

    - Мониторинг и оповещения

      Отображает данные мониторинга, включая использование различных ресурсов, количество соединений, TPS, операции блокировки таблиц, медленные запросы и др.; поддерживает создание и обновление политик оповещения, с предустановленными метриками, включая количество запросов на чтение/запись InnoDB, количество соединений, количество потоков и десять других метрик; поддерживает настройки оповещения для сотен метрик мониторинга (непредустановленных).

    - Инспекция

      Поддерживает инспекцию экземпляров, с опциями для ручной и автоматической активации; поддерживает отображение статистической информации о результатах инспекции; поддерживает экспорт отчетов об инспекции.

    - Восстановление после аварий

      Поддерживает восстановление после аварий с горячим резервированием, с опциями асинхронной и версии-синхронизированной репликации для кластеров восстановления после аварий.

    - Поддержка сетевых протоколов

      Поддерживает IPv4/IPv6 двухстековое, IPv4 одностековое и IPv6 одностековое.
