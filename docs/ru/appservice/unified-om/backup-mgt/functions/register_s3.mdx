# Внешнее хранилище S3

## Обзор функции

Эта функция предоставляет услуги регистрации для внешнего хранилища S3 для текущего выбранного проекта. С простым и интуитивно понятным интерфейсом вы можете легко выполнить несколько шагов для успешной регистрации внешнего хранилища S3 для проекта. Страница списка внешнего хранилища также интегрирует интеллектуальный механизм обнаружения. Вы можете вручную инициировать сканирование, чтобы проверить доступность внешнего хранилища S3. Если возникнут какие-либо проблемы, такие как сбой соединения, будут отправлены немедленные уведомления, чтобы вас информировать, позволяя вам контролировать статус хранилища в реальном времени. Когда вам или другим участникам проекта необходимо создать резервную копию данных, независимо от того, создаётся ли автоматическая задача резервного копирования или вручную запускается мгновенная резервная копия, вам просто нужно легко выбрать зарегистрированное внешнее хранилище S3 в настройках резервного копирования, и система незамедлительно и безопасно сохранит данные резервной копии в это хранилище.

- **Усиленная безопасность данных**

Хранение данных резервной копии во внешнем хранилище S3 предоставляет дополнительный уровень безопасности для ваших данных. Внешнее хранилище S3 обычно имеет надежные механизмы избыточности данных и восстановления после катастроф, что эффективно предотвращает потерю данных из-за локальных сбоев оборудования, природных катастроф и других причин. Даже если в локальной среде возникнут неожиданные проблемы, ваши данные останутся надежно сохраненными во внешнем хранилище, что обеспечивает непрерывность бизнеса.

- **Увеличенная гибкость и масштабируемость**

Эта функция поддерживает гибкий выбор хранилища для резервного копирования, позволяя вам свободно распределять ресурсы резервного копирования в зависимости от различных требований и сценариев. Вы можете выбрать наиболее подходящее внешнее хранилище S3 для хранения данных резервной копии, учитывая такие факторы, как производительность, стоимость и географическое расположение. Кроме того, по мере роста вашего бизнеса и увеличения объема данных вы можете легко добавлять новое внешнее хранилище S3 для расширения ресурсов хранения, адаптируясь к постоянно меняющимся потребностям бизнеса.

## Основные функции

### Добавить внешнее хранилище S3

Прежде чем добавлять внешнее хранилище S3, пользователи должны собрать следующую информацию о хранилище:

- URL-адрес конечной точки
- Ключ доступа
- Секретный ключ доступа
- Имя корзины

Внешнему хранилищу не нужно развёртываться в текущем кластере или пространстве имён, достаточно, чтобы была доступна сетевое подключение. Пользователи могут предварительно проверить доступность внешнего хранилища через веб-запросы. Чтобы добавить внешнее хранилище S3, просто заполните форму регистрации внешнего хранилища S3 в центре резервного копирования, как показано на изображении ниже.

![регистрация внешнего S3](../../../../../en/appservice/unified-om/backup-mgt/functions/assets/register-s3.png)

### Просмотр списка внешних хранилищ S3

После успешной регистрации внешнего хранилища S3 оно будет видно в списке хранилищ. Список хранилищ интегрирует автоматическую проверку статуса хранилища: когда добавляется вновь зарегистрированное внешнее хранилище S3, автоматически срабатывает проверка статуса подключения. Пользователи также могут вручную щелкнуть кнопку `Проверить` рядом со статусом подключения, чтобы инициировать ручную проверку. Результаты проверки статуса подключения будут непосредственно отображены в списке, как показано на изображениях ниже:
