# Обновление экземпляра

Функция управления обновлением экземпляра позволяет пользователям независимо выбирать временной интервал для обновления экземпляров. Разные экземпляры можно обновлять в разные временные интервалы, чтобы обеспечить непрерывность бизнеса.

## Просмотр списка операторов

Пользователи могут просмотреть список текущих развернутых операторов в центре управления обновлением экземпляров. Список отображает ключевую информацию, такую как режим развертывания, область управления оператором, статус оператора и версию оператора. Важно отметить, что после переключения текущего проекта селектор кластера в верхнем правом углу также будет сброшен, что потребует от пользователя повторного выбора кластера, чтобы список операторов отображался корректно. Этот дизайн обусловлен моделью многопользовательского доступа, где проект может содержать несколько различных кластеров.

## Просмотр списка экземпляров

После нажатия на имя конкретного оператора пользователи могут перейти на страницу списка экземпляров. На странице списка экземпляров пользователи могут просмотреть имена, статусы и версии всех экземпляров, управляемых этим оператором в текущем проекте и кластере. Следует отметить, что страница списка экземпляров будет отображать только экземпляры в текущем проекте и кластере, и также будет проверять соответствующие разрешения для просмотра проекта.

## Обновление экземпляра

Когда пользователи хотят обновить экземпляр, они могут выбрать желаемый экземпляр из списка экземпляров, а затем нажать на кнопку пакетного обновления в верхнем правом углу, чтобы запустить поэтапное обновление соответствующего экземпляра. Пользователи заметят, что статус выбранных экземпляров последовательно изменится на `В процессе`. Как только обновление экземпляра завершится, статус вернется в состояние `Работает`.

Важно отметить, что, согласно характеристикам K8S, порядок обновления нескольких экземпляров не фиксирован, но все они в конечном итоге перейдут в состояние готовности после обновления.
