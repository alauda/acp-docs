# Обновление экземпляра

Функция управления обновлением экземпляра позволяет пользователям независимо выбирать временной интервал для обновления экземпляров. Разные экземпляры могут быть обновлены в разные временные окна, чтобы обеспечить непрерывность бизнеса.

## Просмотр списка Операторов

Пользователи могут просматривать текущий список Операторов, развернутых в центре управления обновлением экземпляров. Список отображает ключевую информацию, такую как режим развертывания, область управления Оператором, статус Оператора и версия Оператора. Важно отметить, что после переключения текущего проекта селектор кластера в верхнем правом углу также сбросится, что потребует от пользователя повторно выбрать кластер, чтобы список Операторов отображался корректно. Этот дизайн обусловлен моделью нескольких арендаторов, в которой проект может содержать несколько различных кластеров.

## Просмотр списка экземпляров

После нажатия на имя конкретного Оператора пользователи могут перейти на страницу списка экземпляров. На странице списка экземпляров пользователи могут просматривать имена, статусы и версии всех экземпляров, управляемых этим Оператором в текущем проекте и кластере. Следует отметить, что страница списка экземпляров будет отображать только экземпляры в рамках текущего проекта и кластера, а также будет проверять соответствующие разрешения для просмотра проекта.

## Обновление экземпляра

Когда пользователи хотят обновить экземпляр, они могут выбрать желаемый экземпляр из списка экземпляров и затем нажать кнопку пакетного обновления в верхнем правом углу, чтобы инициировать поступательное обновление для соответствующего экземпляра. Пользователи заметят, что статус выбранных экземпляров последовательно изменится на `Progressing`. После завершения обновления экземпляра статус вернётся к `Running`.

Важно отметить, что, в соответствии с характеристиками K8S, порядок обновления нескольких экземпляров не фиксирован, но все они в конечном итоге перейдут в готовое состояние.
