# Введение

## Введение в продукт

ACP API Refiner - это сервис фильтрации данных, предоставляемый платформой ACP, который улучшает безопасность многоарендной среды и изоляцию данных в Kubernetes. Он фильтрует данные ответов API Kubernetes на основе разрешений пользователей, проектов, кластеров и пространств имен, а также поддерживает фильтрацию на уровне полей, включение и десенсибилизацию данных.

## Преимущества продукта

Основные преимущества ACP API Refiner следующие:

- **Многоуровневая изоляция данных**
  - Поддерживает фильтрацию ответов API на основе проектов, кластеров и пространств имен
  - Обеспечивает правильные границы данных между разными арендаторами
  - Предотвращает несанкционированный доступ к ресурсам, ограниченным кластером

- **Гибкая фильтрация данных**
  - Поддерживает исключение, включение и десенсибилизацию конкретных полей в ответах API
  - Настраиваемые правила фильтрации через конфигурацию YAML
  - Динамическое создание ресурса Ingress для различных типов ресурсов

- **Усиленная безопасность**
  - Реализует аутентификацию пользователей на основе JWT токенов
  - Предоставляет детализированный контроль доступа на основе разрешений пользователей
  - Поддерживает десенсибилизацию данных для конфиденциальной информации

## Сценарии

Основные сценарии применения ACP API Refiner следующие:

- **Многоарендная среда**
  - Обеспечивает правильную изоляцию данных между разными арендаторами
  - Предотвращает несанкционированный доступ к ресурсам, ограниченным кластером
  - Эффективно управляет сценариями совместного использования пространств имен

- **Защита конфиденциальных данных**
  - Фильтрует конфиденциальную информацию из ответов API
  - Поддерживает десенсибилизацию данных на уровне полей
  - Защищает конфиденциальные метаданные и аннотации

- **Требования соблюдения норм**
  - Помогает удовлетворить требования к изоляции данных
  - Поддерживает потребности аудита и соблюдения норм
  - Поддерживает границы доступа к данным

## Ограничения

К ACP API Refiner применяются следующие ограничения:

- Ресурсы должны содержать специфические метки, связанные с арендатором, для изоляции данных:
  - `cpaas.io/project`
  - `cpaas.io/cluster`
  - `cpaas.io/namespace`
  - `kubernetes.io/metadata.name`
  - Необязательно: `cpaas.io/creator`

- Запросы LabelSelector не поддерживают логические операции ИЛИ

- Привязки пользователей на уровне платформы не фильтруются

- Фильтрация применяется только к операциям API GET и LIST
