# Введение

## Введение в продукт

ACP API Refiner — это служба фильтрации данных, предоставляемая платформой ACP, которая улучшает многослойную безопасность и изоляцию данных в средах Kubernetes. Она фильтрует данные ответа API Kubernetes на основе разрешений пользователей, проектов, кластеров и пространств имен, поддерживая также фильтрацию на уровне полей, инклюзию и десенсибилизацию данных.

## Преимущества продукта

Основные преимущества ACP API Refiner следующие:

- **Многомерная изоляция данных**
  - Поддерживает фильтрацию ответов API на основе проектов, кластеров и пространств имен
  - Обеспечивает корректные границы данных между различными арендаторами
  - Предотвращает несанкционированный доступ к ресурсам, ограниченным кластерами

- **Гибкая фильтрация данных**
  - Поддерживает исключение, включение и десенсибилизацию конкретных полей в ответах API
  - Конфигурируемые правила фильтрации через YAML-конфигурацию
  - Динамическая генерация входа ресурсов для различных типов ресурсов

- **Улучшенная безопасность**
  - Реализует аутентификацию пользователей на основе токенов JWT
  - Обеспечивает детализированный контроль доступа на основе разрешений пользователей
  - Поддерживает десенсибилизацию данных для чувствительной информации

## Сценарии

Основные сценарии применения ACP API Refiner следующие:

- **Многослойная среда**
  - Обеспечивает правильную изоляцию данных между различными арендаторами
  - Предотвращает несанкционированный доступ к ресурсам, ограниченным кластерами
  - Эффективно управляет сценариями общего пространства имен

- **Защита чувствительных данных**
  - Фильтрует чувствительную информацию из ответов API
  - Поддерживает десенсибилизацию данных на уровне полей
  - Защищает чувствительные метаданные и аннотации

- **Требования к соблюдению норм**
  - Помогает соответствовать требованиям изоляции данных
  - Поддерживает аудит и потребности в соблюдении норм
  - Сохраняет границы доступа к данным

## Ограничения

К ACP API Refiner применяются следующие ограничения:

- Ресурсы должны содержать специфические метки, связанные с арендаторами, для изоляции данных:
  - `cpaas.io/project`
  - `cpaas.io/cluster`
  - `cpaas.io/namespace`
  - `kubernetes.io/metadata.name`
  - Опционально: `cpaas.io/creator`

- Запросы LabelSelector не поддерживают логические операции ИЛИ

- Связывания пользователей на уровне платформы не фильтруются

- Фильтрация применяется только к операциям API GET и LIST
