# Введение

## Введение в роли

Управление ролями пользователей на платформе реализовано с использованием RBAC Kubernetes (управление доступом на основе ролей). Эта система позволяет гибкую конфигурацию прав, связывая роли с пользователями.

Роль представляет собой совокупность прав для выполнения операций с ресурсами Kubernetes на платформе. Эти права включают:

- Создание ресурсов
- Просмотр ресурсов
- Обновление ресурсов
- Удаление ресурсов

Роли классифицируют и объединяют права для различных ресурсов. Присваивая роли пользователям и устанавливая области прав, вы можете быстро предоставить права на операции с ресурсами.

Права могут быть также легко отозваны путем удаления ролей у пользователей.

Роль может иметь:

- Один или несколько типов ресурсов
- Одно или несколько прав на операции
- Несколько пользователей, назначенных ей

Например:

- Роль A: Может только просматривать и создавать проекты
- Роль B: Может создавать, просматривать, обновлять и удалять пользователей, проекты и пространства имен

## Системные роли

Чтобы удовлетворить распространенные сценарии конфигурации прав, платформа предоставляет следующие стандартные системные роли. Эти роли обеспечивают гибкий контроль доступа к ресурсам платформы и эффективное управление правами для пользователей.

| Название роли                      | Описание                                                                            | Уровень роли |
| ---------------------------------- | ---------------------------------------------------------------------------------- | ------------ |
| **Администратор платформы**       | Имеет полный доступ ко всем бизнес-ресурсам на платформе                          | Платформа    |
| **Аудиторы платформы**             | Могут просматривать все ресурсы платформы и операционные записи, но не имеют других прав | Платформа    |
| **Администратор кластера (Alpha)** | Управляет и поддерживает ресурсы кластера с полным доступом ко всем ресурсам на уровне кластера | Кластер      |
| **Администратор проекта**         | Управляет администраторами пространств имен и квотами пространства имен            | Проект       |
| **namespace-admin-system**        | Управляет членами пространства имен и распределением ролей                          | Пространство  |
| **Разработчики**                   | Разрабатывают, внедряют и поддерживают пользовательские приложения в пространствах имен | Пространство  |

## Пользовательские роли

Платформа поддерживает пользовательские роли для улучшения сценариев контроля доступа к ресурсам. Пользовательские роли имеют несколько преимуществ по сравнению с системными ролями:

- Гибкая конфигурация прав
- Возможность обновления прав роли
- Опция удаления ролей, когда они больше не нужны

:::warning

Будьте осторожны при обновлении или удалении пользовательских ролей. Удаление пользовательской роли автоматически отзовет все права, предоставленные этой ролью привязанным пользователям.

:::
