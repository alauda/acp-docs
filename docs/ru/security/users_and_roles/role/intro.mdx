# Введение

## Введение в роли

Управление ролями пользователей платформы реализовано с использованием RBAC Kubernetes (контроль доступа на основе ролей). Эта система позволяет гибко настраивать разрешения, связывая роли с пользователями.

Роль представляет собой набор разрешений для работы с ресурсами Kubernetes на платформе. Эти разрешения включают в себя:

- Создание ресурсов
- Просмотр ресурсов
- Обновление ресурсов
- Удаление ресурсов

Роли классифицируют и комбинируют разрешения для различных ресурсов. Назначая роли пользователям и устанавливая масштабы разрешений, вы можете быстро предоставить разрешения на операции с ресурсами.

Разрешения можно легко отменить, просто убрав роли у пользователей.

Роль может иметь:

- Один или несколько типов ресурсов
- Одно или несколько разрешений на операции
- Несколько предварительно назначенных пользователей

Например:

- Роль A: Может только просматривать и создавать проекты
- Роль B: Может создавать, просматривать, обновлять и удалять пользователей, проекты и пространства имен

## Системные роли

Чтобы удовлетворить распространенные сценарии конфигурации разрешений, платформа предоставляет следующие системные роли по умолчанию. Эти роли обеспечивают гибкий контроль доступа к ресурсам платформы и эффективное управление разрешениями для пользователей.

| Название роли                         | Описание                                                                             | Уровень роли |
| ------------------------------------- | ------------------------------------------------------------------------------------ | ------------ |
| **Администратор платформы**          | Имеет полный доступ ко всем бизнес-процессам и ресурсам на платформе               | Платформа    |
| **Аудиторы платформы**               | Могут просматривать все ресурсы платформы и записи операций, но не имеют других разрешений | Платформа    |
| **Администратор кластера (Alpha)**   | Управляет и поддерживает ресурсы кластера с полным доступом ко всем ресурсам уровня кластера | Кластер      |
| **Администратор проекта**             | Управляет администраторами пространств имен и квотами пространств имен              | Проект       |
| **namespace-admin-system**            | Управляет участниками пространства имен и назначениями ролей                        | Пространство  |
| **Разработчики**                      | Разрабатывают, разворачивают и поддерживают пользовательские приложения в пространствах имен | Пространство  |

## Пользовательские роли

Платформа поддерживает пользовательские роли для улучшения сценариев контроля доступа к ресурсам. Пользовательские роли предлагают несколько преимуществ по сравнению с системными ролями:

- Гибкая настройка разрешений
- Возможность обновления разрешений роли
- Возможность удаления ролей, когда они больше не нужны

:::warning

Будьте осторожны при обновлении или удалении пользовательских ролей. Удаление пользовательской роли автоматически аннулирует все разрешения, предоставленные этой ролью связанным пользователям.

:::
