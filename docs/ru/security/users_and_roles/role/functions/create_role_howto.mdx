# Создание роли

Пользователи с разрешениями роли платформы могут создавать настраиваемые роли с разрешениями, которые меньше или равны их собственным разрешениям на основе фактических сценариев использования. При создании роли вы можете настроить:

- Разрешения на операции с функциональными модулями платформы
- Разрешения на доступ к пользовательским ресурсам (Kubernetes CRD)

## Конфигурация основной информации

1. В левой навигационной панели нажмите **Пользователи** > **Роли**.
2. Нажмите **Создать роль**.
3. Настройте основную информацию о роли:

### Тип роли

При назначении ролей пользователям объем разрешений будет ограничен в зависимости от типа роли:

- **Роль платформы**: Отображает все разрешения платформы
- **Роль проекта**: Отображает разрешения в рамках:
  - Управления проектами
  - Платформы контейнеров
  - Сервисной сетки
  - DevOps
  - Посредника
- **Роль пространства имен**: Отображает разрешения в рамках:
  - Управления проектами
  - Платформы контейнеров
  - Сервисной сетки
  - DevOps
  - Посредника

4. Нажмите **Далее**.

## Конфигурация представления

В разделе конфигурации представления вы контролируете разрешения роли на доступ к определенным представлениям. Представления, которые не выбраны, не будут отображаться в верхней навигации для пользователей с этой ролью.

:::note

1. Разрешения роли вашей учетной записи ограничивают, какие карточки представления вы можете настроить. Например:
   - Если у вашей учетной записи нет разрешения на представление **Управление проектами**
   - Карточка представления **Управление проектами** будет недоступна при создании роли
   - Вы можете создавать только роли с разрешениями, равными или ниже ваших собственных разрешений

2. Статус входа в представление:
   - Если **Показ входа** для представления отключен в функции **Продукты**
   - Разрешения представления в **Конфигурации разрешений** все равно будут действовать
   - Представление будет временно недоступно, пока вход не будет включен
   - После включения ранее выбранные разрешения будут работать нормально

:::

## Конфигурация разрешений

1. Нажмите **Добавить настраиваемое разрешение** в верхнем левом углу страницы.
2. Настройте разрешения для роли на выполнение операций с настраиваемыми ресурсами (Kubernetes CRD):

| Параметр                | Описание                                                                                                      |
| ----------------------- | ---------------------------------------------------------------------------------------------------------------- |
| **Имя группы**          | Имя группы разрешений. Группы отображаются ниже модуля разрешений в порядке их добавления.                     |
| **Имя ресурса**        | Имя ресурса. Нажмите Enter, чтобы добавить несколько имен настраиваемых ресурсов.                              |
| **Разрешение на операцию** | Разрешение на выполнение операций с ресурсом.                                                                    |

3. Нажмите **Создать**.
