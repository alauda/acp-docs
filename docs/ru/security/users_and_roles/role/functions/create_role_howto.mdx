# Создание роли

Пользователи с разрешениями платформенной роли могут создавать пользовательские роли с разрешениями, которые меньше или равны их собственным разрешениям роли на основе фактических сценариев использования. При создании роли вы можете настроить:

- Разрешения на операции функциональных модулей платформы
- Разрешения на доступ к пользовательским ресурсам (Kubernetes CRD)

## Конфигурация основной информации

1. В левой навигационной панели нажмите **Пользователи** > **Роли**.
2. Нажмите **Создать роль**.
3. Настройте основную информацию о роли:

### Тип роли

При назначении ролей пользователям диапазон разрешений будет ограничен на основе типа роли:

- **Платформенная роль**: Отображает все разрешения платформы
- **Роль проекта**: Отображает разрешения в рамках:
  - Управление проектами
  - Платформа контейнеров
  - Сервисная сетка
  - DevOps
  - Промежуточное ПО
- **Роль пространства имен**: Отображает разрешения в рамках:
  - Управление проектами
  - Платформа контейнеров
  - Сервисная сетка
  - DevOps
  - Промежуточное ПО

4. Нажмите **Далее**.

## Конфигурация просмотра

В разделе конфигурации просмотра вы контролируете разрешение роли на доступ к заданным просмотрам. Просмотры, которые не выбраны, не будут отображаться в верхней навигации для пользователей с этой ролью.

:::note

1. Разрешения роли вашей учетной записи ограничивают, какие карточки просмотра вы можете настраивать. Например:
   - Если у вашей учетной записи нет разрешения на просмотр **Управление проектами**
   - Карточка просмотра **Управление проектами** будет серой при создании роли
   - Вы можете создавать только роли с разрешениями, равными или ниже ваших собственных разрешений роли

2. Статус входа в просмотр:
   - Если **Показать вход** отключен в функции **Продукты**
   - Разрешения просмотра в **Конфигурации разрешений** будут продолжать действовать
   - Просмотр будет временно недоступен, пока вход не будет включен
   - После включения будут работать ранее выбранные разрешения

:::

## Конфигурация разрешений

1. Нажмите **Добавить пользовательское разрешение** в верхнем левом углу страницы.
2. Настройте разрешения для роли на операции с пользовательскими ресурсами (Kubernetes CRD):

| Параметр                | Описание                                                                                                      |
| ------------------------ | ---------------------------------------------------------------------------------------------------------------- |
| **Имя группы**           | Имя группы разрешений. Группы отображаются под модулем разрешений в порядке их добавления. |
| **Имя ресурса**        | Имя ресурса. Нажмите Enter, чтобы добавить несколько имен пользовательских ресурсов.                                     |
| **Разрешение на операцию** | Разрешение на операции с ресурсом.                                                                          |

3. Нажмите **Создать**.
