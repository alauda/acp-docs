# Введение

Платформа поддерживает аутентификацию пользователей и проверку входа для всех пользователей.

## Источники пользователей

### Локальные пользователи

- Учетная запись администратора, созданная во время развертывания платформы
- Учетные записи, созданные через интерфейс платформы
- Пользователи, добавленные через локальный конфигурационный файл dex

### Пользователи третьих сторон

#### Пользователи LDAP

- Корпоративные пользователи, синхронизированные с LDAP-серверами
- Учетные записи импортируются через интеграцию с IDP (поставщик удостоверений)
- Источник отображается как название конфигурации IDP
- Интеграция настраивается через параметры IDP

#### Пользователи OIDC

- Пользователи сторонних платформ, аутентифицированные через протокол OIDC
- Источник отображается как название конфигурации IDP
- Интеграция настраивается через параметры IDP

:::warning

Для пользователей OIDC, добавленных в проект до их первого входа:

- Источник отображается как "-" до успешного входа в платформу
- После успешного входа источник изменяется на название конфигурации IDP

:::

#### Другие пользователи третьих сторон

- Пользователи, аутентифицированные через поддерживаемые коннекторы dex (например, GitHub, Microsoft)
- Для получения дополнительной информации обратитесь к [официальной документации dex](https://github.com/dexidp/dex)

## Правила управления пользователями

:::warning

Пожалуйста, обратите внимание на следующие важные правила:

- Локальные имена пользователей должны быть уникальными для всех типов пользователей
- Пользователи третьих сторон (OIDC/LDAP) с совпадающими именами пользователей автоматически связываются
- Связанные пользователи наследуют права доступа от существующих учетных записей
- Пользователи могут входить через свои соответствующие источники
- В платформе отображается только одна учетная запись пользователя для каждого имени пользователя
- Источник пользователя определяется по наиболее недавнему методу входа

:::

## Жизненный цикл пользователя

В следующей таблице описаны различные статусы пользователей на платформе:

| Статус       | Описание                                                                                                                                                                                                                            |
| ------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Нормальный**   | Учетная запись пользователя активна и может войти в платформу                                                                                                                                                                                  |
| **Отключен** | Учетная запись пользователя неактивна и не может войти. Свяжитесь с администратором платформы для активации.<br /><br />Возможные причины:<br />- Нет входа более 90 дней подряд<br />- Истечение срока действия учетной записи<br />- Ручное отключение администратором      |
| **Заблокирован**   | Учетная запись временно заблокирована из-за 5 неудачных попыток входа в течение 24 часов.<br /><br />Подробности:<br />- Длительность блокировки: 20 минут<br />- Может быть разблокирована вручную администратором<br />- Учетная запись становится доступной после периода блокировки |
| **Недействительный**  | Учетная запись, синхронизированная с LDAP, которая была удалена с LDAP-сервера.<br /><br />Примечание: Недействительные учетные записи не могут войти в платформу                                                                                                  |
