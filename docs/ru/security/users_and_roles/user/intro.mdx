# Введение

Платформа поддерживает аутентификацию пользователей и проверку входа для всех пользователей.

## Источники пользователей

### Локальные пользователи

- Аккаунт администратора, созданный во время развертывания платформы
- Аккаунты, созданные через интерфейс платформы
- Пользователи, добавленные через локальный файл конфигурации dex

### Пользователи сторонних сервисов

#### Пользователи LDAP

- Пользователи предприятия, синхронизированные с LDAP-серверами
- Аккаунты импортируются через интеграцию с IDP (поставщик удостоверений)
- Источник отображается как имя конфигурации IDP
- Интеграция настраивается через параметры IDP

#### Пользователи OIDC

- Пользователи сторонних платформ, аутентифицированные с помощью протокола OIDC
- Источник отображается как имя конфигурации IDP
- Интеграция настраивается через параметры IDP

:::warning

Для пользователей OIDC, добавленных в проект до их первого входа:

- Источник отображается как "-" до успешного входа в платформу
- После успешного входа источник меняется на имя конфигурации IDP

:::

#### Другие пользователи сторонних сервисов

- Пользователи, аутентифицированные через поддерживаемые коннекторы dex (например, GitHub, Microsoft)
- Для получения дополнительной информации обратитесь к [официальной документации dex](https://github.com/dexidp/dex)

## Правила управления пользователями

:::warning

Обратите внимание на следующие важные правила:

- Локальные имена пользователей должны быть уникальными для всех типов пользователей
- Пользователи сторонних сервисов (OIDC/LDAP) с совпадающими именами пользователей автоматически ассоциируются
- Ассоциированные пользователи наследуют права доступа от существующих аккаунтов
- Пользователи могут входить через свои соответствующие источники
- В платформе отображается только одна запись пользователя для каждого имени пользователя
- Источник пользователя определяется исходя из последнего метода входа

:::

## Жизненный цикл пользователя

Следующая таблица описывает различные статусы пользователей на платформе:

| Статус       | Описание                                                                                                                                                                                                                            |
| ------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Обычный**  | Учетная запись пользователя активна и может войти в платформу                                                                                                                                                                       |
| **Отключен** | Учетная запись пользователя неактивна и не может войти. Обратитесь к администратору платформы для активации.<br /><br />Возможные причины:<br />- Нет входа в течение 90+ последовательных дней<br />- Истечение срока действия аккаунта<br />- Ручное отключение администратором |
| **Заблокирован** | Учетная запись временно заблокирована из-за 5 неудачных попыток входа в течение 24 часов.<br /><br />Подробности:<br />- Длительность блокировки: 20 минут<br />- Может быть разблокирована вручную администратором<br />- Учетная запись становится доступной после периода блокировки |
| **Недействителен**  | Учетная запись, синхронизированная через LDAP, которая была удалена с LDAP-сервера.<br /><br />Примечание: Недействительные аккаунты не могут войти в платформу                                                                                                  |
