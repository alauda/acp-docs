# Введение

## Введение в группы

Платформа поддерживает управление пользователями через группы пользователей. Управляя ролями групп, вы можете эффективно:

- Указывать разрешения на выполнение операций на платформе нескольким пользователям одновременно
- Аннулировать разрешения для нескольких пользователей сразу
- Реализовывать пакетный контроль доступа на основе ролей

Например, когда происходят изменения в штате предприятия и вам необходимо предоставить разрешения на выполнение операций с новым проектом или пространством имен нескольким пользователям, вы можете:

1. Создать группу пользователей
2. Импортировать соответствующих пользователей в качестве участников группы
3. Настроить роли проекта и пространства имен для группы
4. Применить единые разрешения ко всем участникам группы

## Типы групп

Платформа поддерживает два типа групп:

### Локальная группа пользователей

- Создается непосредственно на платформе
- Исходный источник отображается как **Локальный**
- Может быть обновлена или удалена
- Поддерживает:
  - Добавление или удаление пользователей из любого источника
  - Добавление или удаление ролей

### Группа пользователей, синхронизированная с IDP

- Синхронизируется с подключенным IDP (LDAP, Azure AD)
- Исходный источник отображается как имя подключенного ***IDP***
- Не может быть обновлена или удалена
- Поддерживает:
  - Добавление или удаление ролей
  - Не может управлять участниками группы (добавление или удаление)
