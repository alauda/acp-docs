# Введение

## Введение в группы

Платформа поддерживает управление пользователями через пользовательские группы. Управляя ролями групп, вы можете эффективно:

- Предоставлять разрешения на операции с платформой нескольким пользователям одновременно
- Отзывать разрешения у нескольких пользователей сразу
- Реализовывать пакетный контроль доступа на основе ролей

Например, когда происходят изменения в штате предприятия и вам необходимо предоставить новые разрешения на операции с проектом или пространством имен нескольким пользователям, вы можете:

1. Создать группу пользователей
2. Импортировать соответствующих пользователей в качестве членов группы
3. Настроить роли проекта и пространства имен для группы
4. Применить единые разрешения ко всем членам группы

## Типы групп

Платформа поддерживает два типа групп:

### Локальная группа пользователей

- Создается непосредственно на платформе
- Источник отображается как **Локальный**
- Может быть обновлена или удалена
- Поддерживает:
  - Добавление или удаление пользователей из любого источника
  - Добавление или удаление ролей

### Группа пользователей синхронизированная с IDP

- Синхронизируется из подключенного IDP (LDAP, Azure AD)
- Источник отображается как имя подключенного ***IDP***
- Не может быть обновлена или удалена
- Поддерживает:
  - Добавление или удаление ролей
  - Не поддерживает управление членами группы (добавление или удаление)
