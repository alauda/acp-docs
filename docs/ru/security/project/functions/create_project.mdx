# Создание проекта

Перед тем как ваша команда начнёт работу, вы можете создать проект на основе существующих ресурсов кластера в платформе. Проект будет изолирован от других проектов (арендодателей) как по ресурсам, так и по персоналу. При создании проекта вы можете выделить ресурсы в соответствии с масштабом проекта и реальными бизнес-требованиями. Проект может использовать ресурсы из нескольких кластеров на платформе.

:::warning

При создании проекта платформа автоматически создаст пространство имен с таким же именем, как и у проекта, в связанных кластерах для изоляции ресурсов на уровне платформы. Пожалуйста, не изменяйте это пространство имен или его ресурсы.
:::

## Процедура

1. В представлении **Управление проектом** нажмите **Создать проект**.

2. На странице **Основная информация** настройте следующие параметры:

   | Параметр    | Описание                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
   | ----------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | **Имя**     | Имя проекта, которое не может совпадать с именем существующего проекта или любым именем из черного списка имен проектов. В противном случае проект не может быть создан.<br /><br />**Примечание**: Черный список имен проектов включает специальные имена пространств имен в кластерах платформы: `cpaas-system`, `cert-manager`, `default`, `global-credentials`, `kube-ovn`, `kube-public`, `kube-system`, `nsx-system`, `alauda-system`, `kube-federation-system`, `ALL-ALL` и `true`. |
   | **Кластер** | Кластер(ы), связанные с проектом, где администратор может выделять квоты ресурсов. Нажмите на выпадающий список для выбора одного или нескольких кластеров.<br /><br />**Примечание**: Кластеры в ненормальном состоянии не могут быть выбраны.                                                                                                                                                                                                                                 |

3. Нажмите **Далее**, и на этапе настройки квоты проекта укажите квоты ресурсов, которые будут выделены текущему проекту для выбранных кластеров. Это включает в себя:
   - CPU (ядра)
   - Оперативная память (Gi)
   - Хранилище (Gi)
   - Количество PVC (число)
   - Pods (число)
   - Виртуальный GPU (GPU-Manager/MPS)
   - pGPU (физический GPU, ядра)
   - Память GPU

:::note

- Квоты ресурсов GPU могут быть настроены только при развертывании плагинов GPU в кластере. Когда ресурс GPU является **GPU-Manager или MPS GPU**, также может быть настроена квота **vMemory**.<br /><br />**Единицы GPU**: 100 единиц виртуальных ядер эквивалентны 1 физическому ядру (1 pGPU = 1 ядро = 100 ядер GPU-Manager = 100 ядер MPS), и единицы pGPU могут выделяться только целыми числами. 1 единица памяти GPU-Manager равна 256 Mi, 1 единица памяти MPS GPU равна 1 Gi, и 1024 Mi = 1 Gi.

- Если квота для определенного типа ресурса не установлена, по умолчанию она считается **Неограниченной**. Это значит, что проект может использовать доступные ресурсы соответствующего типа в кластере по мере необходимости, без максимального ограничения.

- Значения установленных квот проекта должны находиться в пределах диапазона квот, отображаемого на странице. Под каждым полем ввода квоты ресурса будут отображены выделенные квоты и полная информация по этому ресурсу для справки.
  :::

4. Нажмите **Создать проект**.
