# Создание проекта

Перед тем как ваша проектная команда начнет работу, вы можете создать проект на основе существующих ресурсов кластера на платформе. Проект будет изолирован от других проектов (арендаторов) как в плане ресурсов, так и в плане персонала. При создании проекта вы можете распределить ресурсы в соответствии с масштабами вашего проекта и реальными бизнес-потребностями. Проект может использовать ресурсы из нескольких кластеров на платформе.

:::warning

При создании проекта платформа автоматически создаст пространство имен с таким же названием, как у проекта, в связанных кластерах для изоляции ресурсов на уровне платформы. Пожалуйста, не модифицируйте это пространство имен или его ресурсы.
:::

## Процедура

1. В представлении **Управление проектами** нажмите **Создать проект**.

2. На странице **Основная информация** настройте следующие параметры:

   | Параметр    | Описание                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
   | ----------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | **Имя**     | Имя проекта, которое не может совпадать с именем существующего проекта или любым именем из черного списка имен проектов. В противном случае проект не может быть создан.<br /><br />**Примечание**: Черный список имен проектов включает специальные имена пространств имен в кластерах платформы: `cpaas-system`, `cert-manager`, `default`, `global-credentials`, `kube-ovn`, `kube-public`, `kube-system`, `nsx-system`, `alauda-system`, `kube-federation-system`, `ALL-ALL` и `true`. |
   | **Кластер** | Кластер(ы), связанные с проектом, где администратор может выделить квоты ресурсов. Нажмите выпадающее меню, чтобы выбрать один или несколько кластеров.<br /><br />**Примечание**: Кластеры в ненормальном состоянии не могут быть выбраны.                                                                                                                                                                                                                                 |

3. Нажмите **Далее** и на этапе настройки квот проекта установите квоты ресурсов, которые будут выделены текущему проекту для выбранных кластеров. Это включает:
   - CPU (ядра)
   - Память (Gi)
   - Хранение (Gi)
   - Количество PVC (число)
   - Pods (число)
   - Виртуальный GPU (GPU-Manager/MPS)
   - pGPU (физический GPU, ядра)
   - Память GPU

:::note

- Квоты ресурсов GPU могут быть настроены только при развертывании плагинов GPU в кластере. Когда ресурс GPU является **GPU-Manager или MPS GPU**, также может быть настроена квота **vMemory**.<br /><br />**Единицы GPU**: 100 единиц виртуальных ядер эквивалентны 1 физическому ядру (1 pGPU = 1 ядро = 100 ядро GPU-Manager = 100 ядро MPS), и единицы pGPU могут быть выделены только целыми числами. 1 единица памяти GPU-Manager равна 256 Mi, 1 единица памяти MPS GPU равна 1 Gi, и 1024 Mi = 1 Gi.

- Если для какого-либо типа ресурса квота не задана, по умолчанию она считается **Неограниченной**. Это означает, что проект может использовать доступные ресурсы соответствующего типа в кластере по мере необходимости, без максимального лимита.

- Установленные значения квот проекта должны быть в пределах диапазона квот, отображаемого на странице. Под каждым полем ввода квоты ресурса будет отображаться информация о выделенной квоте и общем количестве для этого ресурса для справки.
  :::

4. Нажмите **Создать проект**.
