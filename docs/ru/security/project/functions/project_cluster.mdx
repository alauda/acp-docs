# Управление кластером проекта

Этот документ объясняет, как управлять ассоциациями кластеров для проекта. Вы можете добавлять кластеры для выделения их ресурсов проекту или удалять кластеры, чтобы вернуть выделенные ресурсы.

## Введение

Вы можете добавлять кластеры в проект для выделения их ресурсов или удалять кластеры, чтобы вернуть выделенные ресурсы. Эта функциональность полезна в следующих сценариях:

- Когда ресурсы проекта недостаточны для бизнес-операций
- Когда новосозданный или добавленный кластер необходимо выделить существующему проекту
- Когда ресурсы кластера необходимо вернуть проекту
- Когда конкретный проект нуждается в эксклюзивном доступе к кластеру

## Добавить кластер

Добавьте кластер в проект и установите его квоту ресурсов.

### Процедура

1. В представлении **Управление проектами** щелкните на имени проекта, в который вы хотите добавить кластер.

2. В левой навигационной панели щелкните **Детали**.

3. Щелкните **Действия** > **Добавить кластер** в правом верхнем углу.

4. Выберите кластер и установите квоту ресурсов, которая будет выделена текущему проекту. Можно настроить следующие ресурсы:
   - CPU (ядра)
   - Оперативная память (Gi)
   - Хранилище (Gi)
   - Количество PVC (число)
   - Подов (число)
   - vGPU (виртуальный GPU)/MPS/pGPU (физический GPU, ядра)
   - Квота видеопамяти

:::note

- Квота ресурсов GPU может быть настроена только при развертывании плагинов GPU в кластере.<br /><br />Когда ресурсы GPU - это **GPU-менеджер или MPS GPU**, также можно настроить квоту **vMemory**.<br /><br />**GPU единицы**: 100 единиц виртуальных ядер эквивалентны 1 физическому ядру (1 pGPU = 1 ядро = 100 GPU-менеджер ядер = 100 MPS ядер), и единицы pGPU могут выделяться только целыми числами. 1 единица памяти GPU-менеджера равна 256 Mi, 1 единица памяти MPS GPU равна 1 Gi, и 1024 Mi = 1 Gi.

- Если квота для определенного типа ресурса не установлена, она по умолчанию равна **Неограниченно**. Это означает, что проект может использовать доступные ресурсы соответствующего типа в кластере по мере необходимости, без максимального ограничения.

- Значение установленной проектной квоты должно находиться в пределах диапазона квоты, отображаемого на странице. Под каждым полем ввода квоты ресурса будут отображаться выделенная квота и общее количество этого ресурса для справки.
  :::

5. Нажмите **Добавить**.

## Удалить кластер

Удалите кластер, связанный с проектом.

:::warning

- После удаления кластера проект не может использовать бизнес ресурсы в удаленном кластере.

- Когда кластер, который необходимо удалить, работает неправильно, ресурсы в ненормальном кластере не могут быть очищены. Рекомендуется устранить неполадки кластера перед его удалением.
  :::

### Процедура

1. В представлении **Управление проектами** щелкните на имени проекта, из которого вы хотите удалить кластер.

2. В левой навигационной панели щелкните **Детали**.

3. Щелкните **Действия** > **Удалить кластер** в правом верхнем углу.

4. В появившемся диалоговом окне **Удалить кластер** введите имя кластера, который необходимо удалить, а затем нажмите кнопку **Удалить**, чтобы успешно удалить кластер.
