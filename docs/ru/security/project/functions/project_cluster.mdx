# Управление кластером проекта

Этот гайд объясняет, как управлять ассоциациями кластеров для проекта. Вы можете добавлять кластеры для выделения их ресурсов проекту или удалять кластеры для возврата выделенных ресурсов.

## Введение

Вы можете добавлять кластеры в проект для выделения их ресурсов или удалять кластеры для возврата выделенных ресурсов. Эта функция полезна в следующих сценариях:

- Когда ресурсы проекта недостаточны для бизнес-операций
- Когда новый созданный или добавленный кластер необходимо выделить для существующего проекта
- Когда ресурсы кластера необходимо вернуть из проекта
- Когда конкретный проект нуждается в эксклюзивном доступе к кластеру

## Добавить кластер

Добавьте кластер в проект и установите его квоту на ресурсы.

### Процедура

1. В представлении **Управление проектом** нажмите на название проекта, в который вы хотите добавить кластер.

2. В левой навигационной панели нажмите **Подробности**.

3. Нажмите **Действия** > **Добавить кластер** в верхнем правом углу.

4. Выберите кластер и установите квоту на ресурсы, которые будут выделены текущему проекту. Следующие ресурсы могут быть настроены:
   - CPU (ядра)
   - Память (Gi)
   - Хранилище (Gi)
   - Количество PVC (число)
   - Количество Pods (число)
   - vGPU (виртуальный GPU) / MPS / pGPU (физический GPU, ядра)
   - Квота видеопамяти

:::note

- Квота на ресурсы GPU может быть настроена только при развертывании плагинов GPU в кластере.<br /><br />Когда ресурсы GPU являются **GPU-Manager или MPS GPU**, также может быть настроена квота **vMemory**.<br /><br />**Единицы GPU**: 100 единиц виртуальных ядер эквивалентны 1 физическому ядру (1 pGPU = 1 ядро = 100 ядер GPU-Manager = 100 ядер MPS), и единицы pGPU могут выделяться только целыми числами. 1 единица памяти GPU-Manager равна 256 Mi, 1 единица памяти MPS GPU равна 1 Gi, и 1024 Mi = 1 Gi.

- Если не установлена квота для определенного типа ресурса, по умолчанию она устанавливается на **Неограничено**. Это означает, что проект может использовать доступные ресурсы соответствующего типа в кластере по мере необходимости, без максимального ограничения.

- Значение установленной квоты проекта должно находиться в диапазоне квоты, отображаемом на странице. Под каждым полем ввода квоты ресурса будет отображаться выделенная квота и общее количество этого ресурса для справки.
:::

5. Нажмите **Добавить**.

## Удалить кластер

Удалите кластер, связанный с проектом.

:::warning

- После удаления кластера проект не сможет использовать бизнес-ресурсы под удаленным кластером.

- Когда кластер, подлежащий удалению, находится в ненормальном состоянии, ресурсы под ненормальным кластером не могут быть очищены. Рекомендуется исправить кластер перед его удалением.
:::

### Процедура

1. В представлении **Управление проектом** нажмите на название проекта, из которого вы хотите удалить кластер.

2. В левой навигационной панели нажмите **Подробности**.

3. Нажмите **Действия** > **Удалить кластер** в верхнем правом углу.

4. В всплывающем окне **Удалить кластер** введите название удаляемого кластера, а затем нажмите кнопку **Удалить**, чтобы успешно удалить кластер.
