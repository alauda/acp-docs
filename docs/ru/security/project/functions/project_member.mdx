# Управление членами проекта

Этот справочник объясняет, как управлять членами проекта, включая импорт участников и назначение ролей, связанных с проектом.

## Импорт участников

Вы можете предоставить пользователям разрешения на операции с проектом и его пространствами имен, импортируя существующих пользователей платформы или добавляя пользователей OIDC. Вы можете назначить роли, такие как администраторы проекта, администраторы пространства имен, разработчики или настраиваемые роли с разрешениями на управление проектами и пространствами имен.

### Ограничения и ограничения

- Когда на платформе не настроен OIDC IDP:
  - Можно импортировать только существующих пользователей платформы в качестве членов проекта, включая:
    - Пользователей OIDC, которые успешно вошли в систему
    - Пользователей, синхронизированных через LDAP
    - Локальных пользователей
    - Пользователей, добавленных в другие проекты в качестве пользователей OIDC (с Mark источника как `-`)

- Когда настроен OIDC IDP:
  - Вы можете добавлять действительные OIDC-аккаунты, которые соответствуют требованиям к вводу
  - Действительность аккаунта не может быть проверена во время добавления
  - Убедитесь, что аккаунт действителен, иначе он не сможет войти в систему нормально

- Системные пользователи по умолчанию и текущий вошедший пользователь не могут быть импортированы

### Процедура

1. На вкладке **Управление проектом** нажмите на название проекта, который нужно управлять.

2. В левой навигационной панели нажмите **Члены**.

3. Нажмите **Импорт члена**.

4. Выберите либо **Список членов**, либо **Пользователи OIDC**.

#### Импорт из списка членов

Вы можете импортировать всех пользователей или выбранных пользователей из списка членов.

:::tip
Используйте выпадающее меню группы пользователей в правом верхнем углу и поле поиска для фильтрации пользователей по имени пользователя.
:::

**Чтобы импортировать всех пользователей:**

1. Выберите **Список членов**.

2. Нажмите на выпадающее меню **Привязать** и выберите роль для назначения всем пользователям.<br />Если роль требует пространство имен, выберите его в выпадающем меню **Пространства имен**.

3. Нажмите **Импортировать все**.

**Чтобы импортировать конкретных пользователей:**

1. Выберите **Список членов**.

2. Выберите одного или нескольких пользователей с помощью флажков.

3. Нажмите на выпадающее меню **Привязать** и выберите роль для назначения выбранным пользователям.<br />Если роль требует пространство имен, выберите его в выпадающем меню **Пространства имен**.

4. Нажмите **Импортировать**.

#### Импорт пользователей OIDC

1. Выберите **Пользователи OIDC**.

2. Нажмите **Добавить** для создания записи участника (повторяйте для нескольких участников).

3. Введите имя пользователя, аутентифицированного OIDC, в поле **Имя**.

:::warning
Убедитесь, что имя пользователя соответствует аккаунту, который может быть аутентифицирован настроенной системой OIDC, иначе вход в систему не удастся.
:::

4. Выберите роль из выпадающего меню **Роли**.<br />Если роль требует пространство имен, выберите его в выпадающем меню **Пространства имен**.

5. Нажмите **Импортировать**.

После успешного импорта вы сможете увидеть:

- Участника в списке членов проекта
- Пользователя в **Управлении платформой > Пользователи**
  - Источник обозначается как "-" до первого входа/синхронизации
  - Источник обновляется после успешного входа/синхронизации

## Удаление участников

Удалите участника проекта, чтобы отозвать его разрешения.

### Процедура

1. На вкладке **Управление проектом** нажмите на название проекта.

2. В левой навигационной панели нажмите **Члены**.

:::tip
Используйте выпадающий список рядом с полем поиска для фильтрации участников по **Имени**, **Отображаемому имени** или **Группе пользователей**.
:::

3. Нажмите **Удалить** рядом с участником, которого вы хотите удалить.

4. Подтвердите удаление в диалоговом окне с просьбой о подтверждении.
