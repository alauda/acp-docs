# Управление участниками проекта

Этот справочник объясняет, как управлять участниками проекта, включая импорт участников и назначение ролей, связанных с проектом.

## Импорт участников

Вы можете предоставить пользователям разрешения на выполнение операций для проекта и его пространств имен, импортируя существующих пользователей платформы или добавляя пользователей OIDC. Вы можете назначить роли, такие как администраторы проекта, администраторы пространств имен, разработчики или пользовательские роли с разрешениями на управление проектом и пространствами имен.

### Ограничения и ограничения

- Когда OIDC IDP не настроен на платформе:
  - Импортировать участников проекта можно только из существующих пользователей платформы, включая:
    - пользователей OIDC, которые успешно вошли в систему
    - пользователей, синхронизированных через LDAP
    - локальных пользователей
    - пользователей, добавленных в другие проекты как пользователи OIDC (с источником, отмеченным как `-`)

- Когда OIDC IDP настроен:
  - Вы можете добавить действительные учетные записи OIDC, которые соответствуют требованиям к вводу
  - Действительность учетной записи не может быть проверена во время добавления
  - Убедитесь, что учетная запись действительна, иначе она не сможет войти в систему нормально

- Системных пользователей-администраторов по умолчанию и текущего вошедшего пользователя нельзя импортировать

### Процедура

1. В представлении **Управление проектом** нажмите на название проекта, которым нужно управлять.

2. В левой навигационной панели нажмите **Участники**.

3. Нажмите **Импорт участника**.

4. Выберите **Список участников** или **Пользователи OIDC**.

#### Импорт из списка участников

Вы можете импортировать всех пользователей или выбранных пользователей из списка участников.

:::tip
Используйте выпадающее меню групп пользователей в верхнем правом углу и поле поиска, чтобы фильтровать пользователей по имени пользователя.
:::

**Чтобы импортировать всех пользователей:**

1. Выберите **Список участников**.

2. Нажмите на выпадающее меню **Привязка** и выберите роль, которую нужно назначить всем пользователям.<br />Если роль требует пространство имен, выберите его из выпадающего меню **Пространства имен**.

3. Нажмите **Импортировать всех**.

**Чтобы импортировать конкретных пользователей:**

1. Выберите **Список участников**.

2. Выберите одного или нескольких пользователей с помощью флажков.

3. Нажмите на выпадающее меню **Привязка** и выберите роль, которую нужно назначить выбранным пользователям.<br />Если роль требует пространство имен, выберите его из выпадающего меню **Пространства имен**.

4. Нажмите **Импортировать**.

#### Импорт пользователей OIDC

1. Выберите **Пользователи OIDC**.

2. Нажмите **Добавить**, чтобы создать запись участника (повторите для нескольких участников).

3. Введите проверенное по OIDC имя пользователя в поле **Имя**.

:::warning
Убедитесь, что имя пользователя соответствует учетной записи, которую можно проверить через настроенную OIDC-систему, иначе вход в систему не удастся.
:::

4. Выберите роль из выпадающего меню **Роли**.<br />Если роль требует пространство имен, выберите его из выпадающего меню **Пространства имен**.

5. Нажмите **Импортировать**.

После успешного импорта вы можете увидеть:

- Участника в списке участников проекта
- Пользователя в **Управление платформой > Пользователи**
  - Источник отображается как "-" до первого входа/синхронизации
  - Источник обновляется после успешного входа/синхронизации

## Удаление участников

Удалите участника проекта, чтобы отозвать его права.

### Процедура

1. В представлении **Управление проектом** нажмите на название проекта.

2. В левой навигационной панели нажмите **Участники**.

:::tip
Используйте выпадающий список рядом с полем поиска, чтобы фильтровать участников по **Имени**, **Отображаемому имени** или **Группе пользователей**.
:::

3. Нажмите **Удалить** рядом с участником, которого вы хотите удалить.

4. Подтвердите удаление в диалоговом окне запроса.
