# Введение

## Проект

Проект представляет собой единицу изоляции ресурсов, которая позволяет использовать сценарию многопользовательского доступа в предприятиях. Он делит ресурсы из одного или нескольких кластеров на изолированные окружения, обеспечивая как изоляцию ресурсов, так и персонала. Проекты могут представлять различные дочерние компании, отделы или проектные команды внутри предприятия. С помощью управления проектами можно достичь:

- Изоляции ресурсов между проектными командами
- Управления квотами внутри арендаторов
- Эффективного распределения и контроля ресурсов

## Пространства имен

Пространства имен — это более мелкие, взаимно изолированные ресурсы внутри проекта. Они служат рабочими пространствами для пользователей для реализации их производственных нагрузок. Ключевые характеристики пространств имен включают:

- Под проектом можно создать несколько пространств имен
- Общая ресурсная квота всех пространств имен не может превышать квоту проекта
- Квоты ресурсов выделяются более детально на уровне пространств имен
- Размеры контейнеров (ЦП, память) ограничены на уровне пространств имен
- Улучшенное использование ресурсов за счет тонкой настройки

## Взаимосвязь между кластерами, проектами и пространствами имен

Иерархия ресурсов платформы следует следующим правилам:

- Проект может использовать ресурсы (ЦП, память, хранилище) из нескольких кластеров, а кластер может выделять ресурсы нескольким проектам.

- Под проектом можно создать несколько пространств имен, при этом их общие ресурсные квоты не превышают общие ресурсы проекта.

- Ресурсная квота пространства имен должна поступать из одного кластера, и пространство имен может принадлежать только одному проекту.
