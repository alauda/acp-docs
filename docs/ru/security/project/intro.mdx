# Введение

## Проект

Проект — это единица изоляции ресурсов, которая обеспечивает сценарии многопользовательского использования в предприятиях. Он делит ресурсы из одного или нескольких кластеров на изолированные окружения, обеспечивая изоляцию как ресурсов, так и персонала. Проекты могут представлять различные дочерние компании, отделы или проектные команды внутри предприятия. С помощью управления проектами можно добиться:

- Изоляции ресурсов между проектными командами
- Управления квотами внутри арендаторов
- Эффективного распределения и контроля ресурсов

## Пространства имен

Пространства имен представляют собой более мелкие, взаимно изолированные ресурсы внутри проекта. Они служат рабочими пространствами для пользователей для реализации их производственных нагрузок. Ключевые характеристики пространств имен включают:

- Под проектом можно создать несколько пространств имен
- Общая квота ресурсов всех пространств имен не может превышать квоту проекта
- Квоты ресурсов распределяются более детально на уровне пространства имен
- Размеры контейнеров (CPU, память) ограничены на уровне пространства имен
- Улучшенное использование ресурсов благодаря детальному контролю

## Связь между кластерами, проектами и пространствами имен

Иерархия ресурсов платформы следует этим правилам:

- Проект может использовать ресурсы (CPU, память, хранение) из нескольких кластеров, и кластер может выделять ресурсы нескольким проектам.

- Под проектом можно создать несколько пространств имен, при этом их совокупные ресурсные квоты не должны превышать общие ресурсы проекта.

- Квота ресурсов пространства имен должна поступать из одного кластера, и пространство имен может принадлежать только одному проекту.
