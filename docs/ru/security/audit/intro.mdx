# Введение

Функция аудита платформы предоставляет временные записи операций, связанные с пользователями и безопасностью системы. Это помогает вам анализировать конкретные проблемы и быстро решать проблемы, которые возникают в кластерах, пользовательских приложениях и других областях.

С помощью аудита вы можете отслеживать различные изменения в кластере Kubernetes, включая:

- Какие изменения произошли в кластере в течение определенного периода времени
- Кто выполнил эти изменения (системные компоненты или пользователи)
- Подробности о важных событиях изменений (например, обновление параметров POD)
- Результаты событий (успех или неудача)
- Местоположение оператора (внутри или вне кластера)
- Записи операций пользователей (обновления, удаления, операции управления) и их результаты

## Предварительные условия

Ваша учетная запись должна иметь разрешения на управление платформой или аудит платформы.

## Процедура

1. В левом навигационном меню нажмите **Аудит**.

2. Выберите область аудита на вкладках:
   - **Операции пользователей**: Просмотр записей операций пользователей, которые вошли в систему на платформу
   - **Операции системы**: Просмотр записей операций системы (операторы начинаются с `system:`)

3. Настройте условия запроса для фильтрации событий аудита:

   | Условие запроса      | Описание                                                                                 |
   | -------------------- | ----------------------------------------------------------------------------------------- |
   | **Оператор**         | Имя пользователя или имя системной учетной записи оператора (по умолчанию: `Все`)      |
   | **Действия**        | Тип операции (создание, обновление, удаление, управление, откат, остановка и т. д., по умолчанию: `Все`) |
   | **Кластеры**         | Кластер, содержащий управляемый ресурс (по умолчанию: `Все`)                            |
   | **Тип ресурса**     | Тип управляемого ресурса (по умолчанию: `Все`)                                          |
   | **Имя ресурса**     | Имя управляемого ресурса (поддерживает нечеткий поиск)                                   |

4. Нажмите **Поиск**.

:::tip

- Используйте выпадающее меню **Диапазон времени**, чтобы установить диапазон времени аудита (по умолчанию: `Последние 30 минут`). Вы можете выбрать предустановленный диапазон или настроить его.
- Нажмите на значок обновления, чтобы обновить результаты поиска.
- Нажмите на значок экспорта, чтобы загрузить результаты в виде файла `.csv`.
  :::

## Результаты поиска

Результаты поиска отображают следующую информацию:

| Параметр              | Описание                                                                                                       |
| --------------------- | --------------------------------------------------------------------------------------------------------------- |
| **Оператор**          | Имя пользователя или имя системной учетной записи оператора                                                   |
| **Действия**         | Тип операции (создание, обновление, удаление, управление, откат, остановка и т. д.)                              |
| **Имя/Тип ресурса**  | Имя и тип управляемого ресурса                                                                                 |
| **Кластеры**          | Кластер, содержащий управляемый ресурс                                                                          |
| **Пространства имен** | Пространство имен, содержащее управляемый ресурс                                                                |
| **IP-адрес клиента**  | IP-адрес клиента, использованного для выполнения операции                                                       |
| **Результат операции**| Результат операции на основе кода возврата API (2xx = успех, другой = неудача)                                |
| **Время операции**    | Временная метка операции                                                                                        |
| **Подробности**       | Нажмите кнопку **Подробности**, чтобы просмотреть полную запись аудита в формате JSON в диалоговом окне **Детали аудита** |
