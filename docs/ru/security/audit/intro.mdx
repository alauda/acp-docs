# Введение

Функция аудита платформы предоставляет последовательные по времени операционные записи, связанные с пользователями и безопасностью системы. Это помогает вам анализировать конкретные проблемы и быстро решать возникающие проблемы в кластерах, пользовательских приложениях и других областях.

С помощью аудита вы можете отслеживать различные изменения в кластере Kubernetes, включая:

- Какие изменения произошли в кластере в течение определенного периода времени
- Кто выполнил эти изменения (системные компоненты или пользователи)
- Подробности важных событий изменений (например, обновления параметров POD)
- Результаты событий (успех или неудача)
- Местоположение оператора (внутри или вне кластера)
- Записи операций пользователей (обновления, удаления, управленческие операции) и их результаты

## Предварительные условия

Ваш аккаунт должен иметь разрешения на управление платформой или аудит платформы.

## Процедура

1. В левой навигационной панели щелкните **Аудит**.

2. Выберите область аудита во вкладках:
   - **Операции пользователей**: Просмотреть операционные записи пользователей, которые вошли на платформу
   - **Операции системы**: Просмотреть операционные записи системы (операторы начинаются с `system:`)

3. Настройте условия запроса для фильтрации событий аудита:

   | Условие запроса      | Описание                                                                                  |
   | -------------------- | ----------------------------------------------------------------------------------------- |
   | **Оператор**         | Имя пользователя или системное имя аккаунта оператора (по умолчанию: `Все`)               |
   | **Действия**        | Тип операции (создать, обновить, удалить, управлять, откатить, остановить и т. д., по умолчанию: `Все`) |
   | **Кластеры**         | Кластер, содержащий операционный ресурс (по умолчанию: `Все`)                               |
   | **Тип ресурса**     | Тип операционного ресурса (по умолчанию: `Все`)                                         |
   | **Имя ресурса**     | Имя операционного ресурса (поддерживает нечеткий поиск)                                 |

4. Нажмите **Поиск**.

:::tip

- Используйте выпадающий список **Диапазон времени** для установки времени аудита (по умолчанию: `Последние 30 минут`). Вы можете выбрать предустановленный диапазон или настроить свой.
- Нажмите на значок обновления, чтобы обновить результаты поиска.
- Нажмите на значок экспорта, чтобы скачать результаты в формате `.csv`.
  :::

## Результаты поиска

Результаты поиска отображают следующую информацию:

| Параметр             | Описание                                                                                                           |
| --------------------- | ----------------------------------------------------------------------------------------------------------------- |
| **Оператор**          | Имя пользователя или системное имя аккаунта оператора                                                              |
| **Действия**         | Тип операции (создать, обновить, удалить, управлять, откатить, остановить и т. д.)                                 |
| **Имя/Тип ресурса**  | Имя и тип операционного ресурса                                                                                    |
| **Кластеры**          | Кластер, содержащий операционный ресурс                                                                            |
| **Пространства имен** | Пространство имен, содержащее операционный ресурс                                                                  |
| **IP-адрес клиента**  | IP-адрес клиента, используемого для выполнения операции                                                             |
| **Результат операции**| Результат операции на основе кода возврата API (2xx = успех, другой = неудача)                                   |
| **Время операции**    | Метка времени операции                                                                                            |
| **Подробности**       | Нажмите кнопку **Подробности**, чтобы просмотреть полную запись аудита в формате JSON в диалоговом окне **Подробности аудита** |
