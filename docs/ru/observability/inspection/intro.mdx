# Введение

## Введение в модуль

Чтобы помочь корпоративным клиентам сократить расходы на ручные проверки, базовая функциональность проверки на платформе разработана на основе практического опыта проведения ручных проверок для корпоративных клиентов. Это позволяет корпоративным клиентам в реальном времени понимать статус работы всех бизнес-ресурсов на платформе, быстро выявлять аномалии и снижать бизнес-риски.

- Поддерживает онлайн-выполнение задач проверки, включая инспекции рисков ресурсов всех кластеров, узлов, подов и сертификатов на платформе, а также инспекции использования обычных ресурсов, позволяя отслеживать ход проверки в реальном времени;
- После завершения проверки результаты визуально отображаются, включая риски ресурсов и информацию об их использовании;
- Поддерживает загрузку отчетов о проверках в формате PDF или Excel;
- Для обеспечения безопасности данных клиентов только пользователи с соответствующими правами доступа могут использовать функциональность проверки.

## Преимущества модуля

- **Широкое охват**: Поддерживает проверки всех критически важных ресурсов на платформе, гарантируя, что ничего не будет упущено.
- **Мониторинг в реальном времени**: Пользователи могут в реальном времени просматривать ход проверки, оставаясь в курсе статуса ресурсов.
- **Визуальное представление**: Результаты проверок отображаются через интуитивно понятный визуальный интерфейс, что облегчает быструю идентификацию проблем.
- **Гибкая отчетность**: Поддерживает загрузку отчетов о проверках в формате PDF или Excel для удовлетворения потребностей различных пользователей.

## Сценарии использования модуля

- **Ежедневные операции**: Регулярное выполнение задач проверки для обеспечения нормальной работы и безопасности ресурсов платформы.
- **Диагностика неисправностей**: Быстрое выявление рисков ресурсов или аномалий в использовании через функциональность проверки при возникновении проблем.
- **Аудиты соответствия**: Загрузка отчетов о проверках для аудитов соответствия и внутренних проверок для обеспечения того, чтобы платформа соответствовала действующим стандартам и нормативам.
- **Оптимизация ресурсов**: Анализ информации о использовании ресурсов для выявления потерь или нехватки и оптимизации распределения ресурсов.

## Ограничения использования

- Некоторые элементы проверки на платформе зависят от наличия установленных компонентов мониторинга в кластерах. Пожалуйста, убедитесь, что в каждом кластере установлен либо плагин ACP Monitoring с Prometheus, либо плагин ACP Monitoring с VictoriaMetrics заранее.
- Платформенная проверка поддерживает отправку результатов проверок по электронной почте. Пожалуйста, убедитесь, что конфигурация сервера уведомлений электронной почты выполнена заранее.

С функциональностью проверки контейнерной платформы пользователи могут более эффективно управлять и поддерживать контейнерную среду, повышая стабильность и безопасность системы.
