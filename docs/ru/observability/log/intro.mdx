# Введение

## Обзор модуля

Модуль журналирования является эффективным и надежным решением для управления логами, предоставляемым ACP. Он предназначен для обеспечения пользователей полными возможностями по сбору, хранению, запросу и анализу логов. Основанная на мощных компонентами с открытым исходным кодом, система использует Filebeat для сбора логов, а ElasticSearch и Clickhouse в качестве бэкендов для хранения логов, что позволяет пользователям легко обрабатывать большие объемы логов и получать ключевые бизнес-аналитические данные в реальном времени.

## Преимущества модуля

- Высокая производительность: благодаря надежной производительности ElasticSearch и Clickhouse система может обрабатывать огромные объемы данных и обеспечивать быстрый запрос и анализ.
- Гибкость: поддерживает сбор различных источников логов, удовлетворяя потребности различных бизнес-сценариев.
- Возможности в реальном времени: предоставляет возможности обработки логов в реальном времени, позволяя пользователям быстро выявлять и реагировать на сбои системы или инциденты безопасности.
- Масштабируемость: архитектура системы разработана для поддержки горизонтального масштабирования, что позволяет легко увеличивать ресурсы в соответствии с ростом бизнеса.
- Удобство для пользователя: предлагает визуальный интерфейс и простой язык запросов, что облегчает пользователям начало работы.

## Сценарии использования модуля

- Мониторинг системы: мониторинг статусов работы приложений и серверов в реальном времени, позволяющий своевременно выявлять и устранять аномалии.
- Аудит безопасности: сбор и анализ логов безопасности, чтобы помочь предприятиям выявлять потенциальные угрозы безопасности и нарушения.
- Устранение неисправностей: быстрое определение коренной причины неисправностей с помощью анализа логов, тем самым повышая эффективность восстановления системы.
- Бизнес-анализ: помощь в принятии решений и оптимизация бизнес-процессов через анализ поведения пользователей и логов производительности системы.

## Ограничения по использованию модуля

- Планирование мощности: хранение больших объемов данных логов требует адекватного планирования ресурсов; пожалуйста, оцените масштаб ваших логов заранее и подготовьтесь в соответствии с документом [Планирование мощности](./architecture/capacity_planning.mdx).
- Доступ к порту: если вы устанавливаете компоненты хранения логов в рабочем кластере, убедитесь, что `global` кластер может получить доступ к порту 11780 рабочего кластера.
- Выбор компонентов: платформа предлагает два различных компонента для хранения логов, ElasticSearch и Clickhouse; пожалуйста, заранее выбирайте компонент в соответствии с вашими потребностями.
- Планирование установки: платформа поддерживает установку компонентов хранения логов в любом кластере; логи из любого кластера могут быть заранее собраны в компонентах хранения логов установленного кластера. Пожалуйста, планируйте установку связанных с логами компонентов в зависимости от вашей структуры дата-центра, чтобы избежать чрезмерных затрат на пропускную способность из-за междоменного трафика.
