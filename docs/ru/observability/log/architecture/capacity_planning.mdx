# Планирование емкости компонента логирования

Компонент хранения логов отвечает за сохранение логов, событий и аудиторных данных, собранных компонентом сбора логов с одного или нескольких кластеров на платформе. Поэтому вам необходимо заранее оценить масштаб хранения логов и спланировать ресурсы, необходимые для компонента хранения логов, в соответствии с рекомендациями в этом документе.

:::warning

- Следующие данные представляют стандартные показатели, полученные из тестов, проведенных в лабораторных условиях, предназначенные для вашей справки при планировании ресурсов. Вы должны убедиться, что фактические ресурсы, которые вы планируете, превышают описанные ниже тестовые ресурсы, и что масштаб логов не превышает соответствующий масштаб логов.
- Конфигурация диска для данных ниже: `6000 iops`, `250MB/s скорость чтения и записи`, `независимая установка SSD`. Если ваши фактические ресурсы хранения слабее, чем тестовые ресурсы, пожалуйста, обратитесь к информации о конфигурации большего масштаба и предоставьте дополнительные ресурсы CPU и памяти по мере необходимости.

:::
