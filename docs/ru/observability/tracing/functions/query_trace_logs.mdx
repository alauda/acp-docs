# Журналы трассировки запросов

## Обзор функции

Журналы трассировки позволяют пользователям запрашивать и анализировать журналы, связанные с конкретной распределенной трассой, используя ее уникальный TraceID. Эта функция помогает разработчикам и операторам быстро находить проблемы, понимать потоки запросов и связывать бизнес-журналы с контекстами трассировки.

**Ключевые преимущества**:

- **Анализ первопричин**: Определение ошибок и проблем с задержками в микросервисах в распределенных системах.
- **Корреляция контекста**: Связывание бизнес-журналов с этапами трассировки для всестороннего обзора.
- **Эффективная фильтрация**: Фильтрация журналов по Pod или TraceID для концентрации на релевантных данных.

**Применимые сценарии**:

- Отладка сбоев транзакций между сервисами.
- Анализ узких мест производительности в сложных рабочих процессах.
- Аудит потоков обработки запросов для соблюдения норм.

## Основные функции

- **Запрос на основе TraceID**: Получение всех журналов, связанных с конкретной трассировкой, с использованием его TraceID.
- **Фильтрация на основе Pod**: Просмотр журналов от конкретных Pod, участвующих в трассировке.
- **Экспорт журналов**: Экспорт отфильтрованных данных журналов в настраиваемых форматах.
- **Контекстное отображение журналов**: Изучение записей журналов до/после целевой записи для более глубокого анализа.

## Предварительные условия

:::tip
Перед запросом журналов трассировки по TraceID необходимо сначала подготовить ваши сервисы для включения TraceID в бизнес-журналы. Следуйте [Руководству по корреляции бизнес-журналов с TraceID](../how_to/log_with_traceid.md) для получения деталей конфигурации.
:::

**Стандартное поведение**:

- Отображает журналы за весь период трассировки.
- Для трассировок короче одной минуты запрашивает журналы в течение одной минуты после времени начала трассировки.

## Операции запроса журналов

<Steps>
  ### Доступ к журналам трассировки

  1. После запроса трассировок щелкните на конкретной трассировке, чтобы просмотреть ее детали.
  2. Щелкните на вкладку **Просмотреть журнал** в панели визуализации трассировки.

  ### Фильтрация журналов

  #### По имени Pod

  В селекторе **Имя Pod** выберите целевой Pod из списка участвующих сервисов.

  #### По диапазону времени

  В селекторе **Диапазон времени** выберите целевой диапазон времени.

  #### По условиям запроса

  Введите ключевые слова в текстовое поле **Условия запроса**, чтобы отфильтровать журналы по конкретному содержимому.

  #### Содержит Trace ID

  Установите флажок **Содержит Trace ID**.

  ### Расширенные операции

  #### Экспорт журналов

  1. Щелкните **Экспорт**.
  2. Выберите поля для включения, используя флажки столбцов.
  3. Выберите формат экспорта (JSON/CSV).

  #### Настройка отображаемых полей

  Щелкните **Установить**.
  Переключите видимость полей журнала в панели отображения.

  #### Просмотр контекста журнала

  1. Щелкните **Инсайт** рядом с любой записью журнала.
  2. Исследуйте 5 предшествующих и последующих журналов вокруг целевой временной метки.
  3. Прокручивайте вверх/вниз с помощью мыши, чтобы загрузить больше журналов.
</Steps>
