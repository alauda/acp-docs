# Журналы трассировок запросов

## Обзор функции

Журналы трассировок позволяют пользователям запрашивать и анализировать журналы, связанные с конкретной распределенной трассировкой, используя уникальный идентификатор трассировки (TraceID). Эта функция помогает разработчикам и операторам быстро находить проблемы, понимать потоки запросов и соотносить бизнес-журналы с контекстами трассировок.

**Ключевые преимущества**:

- **Анализ первопричин**: Определение ошибок и проблем с задержкой в микросервисах распределенных систем.
- **Корреляция контекстов**: Связывание бизнес-журналов с диапазонами трассировки для обеспечения видимости от начала до конца.
- **Эффективная фильтрация**: Фильтрация журналов по Pod или TraceID для концентрации на соответствующих данных.

**Применимые сценарии**:

- Отладка неудачных транзакций между сервисами.
- Анализ узких мест производительности в сложных рабочих процессах.
- Аудит потоков обработки запросов для соблюдения норм.

## Основные функции

- **Запрос по TraceID**: Получение всех журналов, связанных с конкретной трассировкой, используя ее TraceID.
- **Фильтрация по Pod**: Просмотр журналов от конкретных Pod, участвующих в трассировке.
- **Экспорт журналов**: Экспорт отфильтрованных данных журнала в настраиваемые форматы.
- **Контекстуальный просмотр журналов**: Анализ записей журнала до/после целевой записи для более глубокого анализа.

## Предварительные требования

:::tip
Перед запросом журналов трассировок по TraceID вы должны сначала настроить свои сервисы так, чтобы включить TraceID в бизнес-журналы. Следуйте [Руководству по корреляции бизнес-журналов с TraceID](../how_to/log_with_traceid.md) для получения деталей настройки.
:::

**Обычное поведение**:

- Отображает журналы за всё время трассировки.
- Для трассировок, длительность которых меньше 1 минуты, запрашивает журналы в течение 1 минуты после времени начала трассировки.

## Операции запроса журналов

<Steps>
  ### Доступ к журналам трассировок

  1. После запроса трассировок нажмите на конкретную трассировку, чтобы посмотреть ее детали.
  2. Нажмите на вкладку **Просмотр журнала** в панели визуализации трассировки.

  ### Фильтрация журналов

  #### По имени Pod

  В селекторе **Имя Pod** выберите целевой Pod из списка участвующих сервисов.

  #### По диапазону времени

  В селекторе **Диапазон времени** выберите целевой диапазон времени.

  #### По условиям запроса

  Введите ключевые слова в текстовое поле **Условия запроса** для фильтрации журналов на основе определенного содержания.

  #### Содержит Trace ID

  Установите флажок **Содержит Trace ID**.

  ### Расширенные операции

  #### Экспорт журналов

  1. Нажмите **Экспорт**.
  2. Выберите поля для включения, используя флажки столбцов.
  3. Выберите формат экспорта (JSON/CSV).

  #### Настройка отображаемых полей

  Нажмите **Настроить**.
  Переключите видимость полей журнала в панели отображения.

  #### Просмотр контекста журнала

  1. Нажмите **Инсайт** рядом с любой записью журнала.
  2. Изучите 5 предшествующих и последующих журналов вокруг целевой отметки времени.
  3. Прокрутите вверх/вниз с помощью мыши для загрузки дополнительных журналов.
</Steps>
