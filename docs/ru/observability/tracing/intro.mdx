# Введение

**Распределенное трассирование** является ключевым модулем в системе наблюдаемости контейнерных платформ, используемым для достижения сквозного трассирования и анализа распределенных систем. Этот модуль построен на основе стандарта OpenTelemetry (OTel), предоставляя полное решение от сбора данных, хранения до визуального анализа, помогая разработчикам и операционному персоналу быстро находить аномалии вызова сервисов, анализировать узкие места в производительности и отслеживать весь жизненный цикл запросов.

Интегрируясь с открытыми технологическими стеком и самостоятельно разработанными компонентами, этот модуль поддерживает возможности сквозного трассирования: приложения генерируют данные трассировки через методы `автоматической инъекции OTel` или интеграции `SDK`, которые затем единообразно собираются и хранятся в Elasticsearch, в конечном итоге реализуясь через пользовательский интерфейс для многомерного визуального анализа. Пользователи могут проводить точные поиски с использованием таких богатых условий, как `TraceID`, имя сервиса, теги и многое другое.

## Преимущества

Ключевые преимущества трассировки следующие:

- **Способность к сквозному трассированию**\
  Поддерживает полное восстановление трассировки через сервисы, процессы и контейнерные границы, точно отображая сложные взаимосвязи вызовов в архитектурах микросервисов.

- **Гибкие методы сбора данных**\
  Обеспечивает два режима автоматической инъекции (без изменения кода) и интеграции SDK, совместимых с основными языками программирования такими как Java/Python/Go.

- **Высокопроизводительные решения для хранения**\
  Использует Elasticsearch в качестве бэкэнда для хранения, поддерживая запись и быстрое извлечение массовых данных о спанах.

- **Гибкие возможности поиска и анализа**\
  Самостоятельно разработанный пользовательский интерфейс интегрируется с API `jaeger-query`, поддерживая гибкие запросы на основе многомерных условий, таких как TraceID, принадлежность к сервису, теги и типы спанов, что позволяет пользователям быстро выявлять коренные причины проблем.

- **Поддержка стандартных протоколов**\
  Построен на стандарте OpenTelemetry, может интегрировать данные трассировки, генерируемые другими облачными компонентами OTel.

## Сценарии применения

Основные сценарии применения трассировки следующие:

- **Диагностика неисправностей в распределенных системах**\
  В архитектурах микросервисов полное трассирование позволяет быстро идентифицировать неисправности в сервисах и аномальные вызовы, сокращая время диагностики неисправностей.

- **Анализ узких мест в производительности**\
  Рассматривая задержку между вызовами сервисов, можно выявить узкие места в производительности, что поможет в оптимизации системы и корректировке ресурсов.

- **Анализ зависимости сервисов**\
  Временной водопадный график ясно показывает пути вызовов и зависимости между сервисами, помогая архитекторам в проектировании и улучшении системы.

## Ограничения по использованию

При использовании трассировки следует учитывать следующие ограничения:

- **Сбалансированные стратегии выборки и производительность**
  - В сценариях с высокой нагрузкой сбор данных трассировки может оказывать определенное давление на производительность и хранилище Elasticsearch; рекомендуется разумно настраивать коэффициент выборки в зависимости от условий бизнеса.
