# Неполные данные трассировки

## Описание проблемы

Результаты запроса трассировки имеют следующие проблемы с неполными данными:

- В недавних запросах (в течение последних 30 минут) отсутствуют некоторые спаны.
- Трассировка старше 1 часа испытывает разрывы соединения.

## Анализ коренной причины

### 1. Задержка сохранения данных

Процесс записи в Elasticsearch требует выполнения последовательности шагов, включающих память → транслог → сегментные файлы, что может привести к задержкам в видимости недавно записанных данных.

### 2. Ограничение временного диапазона

По умолчанию, когда `jaeger-query` запрашивает спаны, соответствующие трассировке, временной диапазон охватывает один час до и после времени начала спана.

Например, если спан начинается в `08:12:30` и заканчивается в `08:12:32`, временной диапазон для запроса этой трассировки будет с `07:12:30` до `09:12:32`.

Следовательно, если трассировка охватывает более 1 часа, запрос через этот спан может не дать полной трассировки.

## Решение для коренной причины 1

Подождите некоторое время и обновите страницу, чтобы попытаться выполнить запрос снова.

## Решение для коренной причины 2

Если спан трассировки в вашей среде длительный, вы можете настроить временной диапазон запроса для одной трассировки, используя параметр запуска `--es.asm.span-trace-query-time-adjustment-hours` в `jaeger-query`.

Значение по умолчанию для этого параметра составляет `1` час, и вы можете увеличить это значение по мере необходимости.
