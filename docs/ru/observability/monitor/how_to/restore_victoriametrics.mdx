# Резервное копирование и восстановление данных мониторинга VictoriaMetrics

## Обзор функции

Функция резервного копирования и восстановления данных мониторинга VictoriaMetrics позволяет вам выполнять регулярное резервное копирование данных мониторинга и восстанавливать данные при необходимости, обеспечивая безопасность и доступность данных мониторинга.

## Сценарии использования

- Регулярное резервное копирование данных мониторинга для предотвращения потери данных
- Миграция данных при переносе системы
- Восстановление после бедствий
- Воссоздание данных тестовой среды

## Предварительные условия

- Плагин ACP Monitoring с VictoriaMetrics был установлен в кластере
- Убедитесь, что достаточно места для хранения резервных копий
- Иметь доступ к пути хранения VictoriaMetrics

## Процедуры

### 1. Подтвердите путь хранения

Данные мониторинга VictoriaMetrics хранятся в указанном пути контейнера, который задается параметром `-storageDataPath`, по умолчанию равным `/vm-data`.

Пример конфигурации:

```yaml
spec:
  template:
    spec:
      containers:
        - args:
            - '-storageDataPath=/vm-data'
```

**Примечание**: Имя вычислительного компонента в плагине ACP Monitoring с VictoriaMetrics — `vmstorage-cluster`, а его тип — `StatefulSet`.

### 2. Выполните резервное копирование данных

Используйте инструмент vmbackup для выполнения резервного копирования данных; пожалуйста, обратитесь к [официальной документации vmbackup](https://docs.victoriametrics.com/vmbackup.html) для получения подробной информации о действиях.

### 3. Выполните восстановление данных

Используйте инструмент vmrestore для восстановления резервных данных; пожалуйста, обратитесь к [официальной документации vmrestore](https://docs.victoriametrics.com/vmrestore.html) для получения подробной информации о действиях.

## Результат операции

После завершения резервного копирования вы получите полный файл резервной копии данных мониторинга. После выполнения операции восстановления ваши данные мониторинга будут восстановлены в состояние, в котором они находились на момент резервного копирования.
