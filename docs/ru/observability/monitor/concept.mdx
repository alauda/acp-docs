# Концепции

## Мониторинг

### Метрики

Метрики используются для количественного описания рабочего состояния системы, и каждая метрика состоит из четырех основных элементов:

- Имя метрики: Используется для идентификации контролируемого объекта, такого как `cpu_usage`
- Значение метрики: Конкретное значение измерения, такое как `85.5`
- Временная метка: Регистрирует время измерения
- Метки: Используются для многомерной классификации данных, такие как `{pod="nginx-1", namespace="default"}`

### PromQL

PromQL — это язык запросов для Prometheus, используемый для запроса и агрегации данных метрик из системы мониторинга.

### Встроенные индикаторы

Платформа имеет предустановленный ряд распространенных метрик мониторинга на основе долгосрочного операционного опыта. Вы можете напрямую использовать эти метрики при настройке правил тревоги или создании панелей мониторинга без дополнительной конфигурации.

### Экспортер

Экспортер — это компонент для сбора данных мониторинга, основные обязанности которого включают:

- Сбор исходных данных мониторинга из целевой системы
- Преобразование данных в стандартный формат временных рядов
- Предоставление данных метрик для запроса через HTTP-интерфейс

### ServiceMonitor

ServiceMonitor используется для декларативного управления настройками мониторинга и в первую очередь определяет:

- Критерии выбора целевых объектов мониторинга
- Конфигурацию интерфейсов сбора метрик
- Параметры выполнения задач сбора (интервалы, таймауты и т.д.)

## Тревоги

### Правила тревоги

Правила тревоги определяют конкретные условия для срабатывания тревог:

- Выражение тревоги: Описывает условия для срабатывания тревоги с помощью операторов PromQL
- Порог тревоги: Явные граничные значения для триггеров
- Длительность: Продолжительность, в течение которой условия должны постоянно соблюдаться
- Уровень тревоги: Различает серьезность тревог (например, P0/P1/P2)

### Политики тревоги

Политики тревоги组织уют несколько правил тревоги вместе для единой конфигурации:

- Цели тревоги: Целевой диапазон правил
- Метод уведомления: Каналы для отправки тревог
- Интервал отправки: Временной интервал для повторных уведомлений тревоги

## Уведомления

### Политики уведомления

Политики уведомления управляют правилами отправки сообщений тревоги:

- Получатели: Целевые пользователи для уведомлений о тревоге
- Каналы уведомления: Поддерживаемые методы отправки сообщений
- Шаблоны уведомлений: Определение формата содержания сообщения

### Шаблоны уведомлений

Шаблоны уведомлений настраивают формат отображения сообщений тревоги:

- Шаблон заголовка: Формат заголовка сообщения тревоги
- Шаблон содержания: Организация деталей тревоги
- Замена переменных: Поддерживает динамическое заполнение данных

## Панель мониторинга

### Панель

Панель — это коллекция нескольких связанных панелей, предоставляющая общий обзор статуса системы. Она поддерживает гибкие варианты макета и может организовывать панели в строки или столбцы.

### Панели

Панели — это визуальные представления данных мониторинга, поддерживающие различные типы отображения.

### Источники данных

Конфигурация источников данных мониторинга. В настоящее время поддерживаются только компоненты мониторинга текущего кластера в качестве источников данных, и пользовательские источники данных на данный момент не поддерживаются.

### Переменные

Переменные служат в качестве заполнителей для значений и могут использоваться в запросах метрик. С помощью селектора переменных в верхней части панели мониторинга вы можете динамически настраивать условия запроса, позволяя содержимому диаграммы обновляться в реальном времени.
