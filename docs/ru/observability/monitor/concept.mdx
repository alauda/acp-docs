# Концепции

## Мониторинг

### Метрики

Метрики используются для количественного описания состояния системы, и каждая метрика состоит из четырех основных элементов:

- Имя метрики: Используется для идентификации контролируемого объекта, например, `cpu_usage`
- Значение метрики: Конкретное измеренное значение, например, `85.5`
- Временная метка: Записывает время измерения
- Метки: Используются для многомерной классификации данных, например, `{pod="nginx-1", namespace="default"}`

### PromQL

PromQL — это язык запросов для Prometheus, используемый для запроса и агрегации данных метрик из системы мониторинга.

### Встроенные индикаторы

Платформа заранее определила серию общепринятых метрик мониторинга на основе долгосрочного эксплуатационного опыта. Вы можете напрямую использовать эти метрики при настройке правил тревоги или создании панелей мониторинга без дополнительной конфигурации.

### Экспортер

Экспортер — это компонент для сбора данных мониторинга, с основными обязанностями, включая:

- Сбор сырых данных мониторинга из целевой системы
- Преобразование данных в стандартный формат временных рядов
- Предоставление данных метрик для запроса через HTTP-интерфейс

### ServiceMonitor

ServiceMonitor используется для декларативного управления конфигурациями мониторинга и прежде всего определяет:

- Критерии выбора целей мониторинга
- Конфигурацию интерфейсов сбора метрик
- Параметры выполнения задач сбора (интервалы, тайм-ауты и т. д.)

## Тревоги

### Правила тревоги

Правила тревоги определяют конкретные условия для срабатывания тревог:

- Выражение тревоги: Описывает условия для срабатывания тревоги с использованием операторов PromQL
- Порог тревоги: Явные граничные значения для триггера
- Длительность: Длительность, в течение которой условия должны постоянно соблюдаться
- Уровень тревоги: Различает серьезность тревог (например, P0/P1/P2)

### Политики тревоги

Политики тревоги организуют несколько правил тревоги вместе для единой конфигурации:

- Цели тревоги: Целевой диапазон правил
- Метод уведомления: Каналы для отправки тревог
- Интервал отправки: Интервал времени для повторных уведомлений тревоги

## Уведомления

### Политики уведомлений

Политики уведомлений управляют правилами отправки сообщений тревоги:

- Получатели: Целевые пользователи для уведомлений о тревогах
- Каналы уведомлений: Поддерживаемые методы отправки сообщений
- Шаблоны уведомлений: Определение формата содержания сообщений

### Шаблоны уведомлений

Шаблоны уведомлений настраивают формат отображения сообщений о тревогах:

- Шаблон заголовка: Формат заголовка сообщения тревоги
- Шаблон содержания: Организация деталей тревоги
- Замена переменных: Поддерживает динамическое заполнение данных

## Панель мониторинга

### Панель

Панель — это сборище нескольких связанных панелей, предоставляющих общий вид на состояние системы. Она поддерживает гибкие компоновки и может организовывать панели в строки или столбцы.

### Панели

Панели — это визуальные представления данных мониторинга, поддерживающие различные типы отображения.

### Источники данных

Конфигурация источников данных мониторинга. В настоящее время поддерживаются только компоненты мониторинга текущего кластера в качестве источников данных, в то время как пользовательские источники данных пока не поддерживаются.

### Переменные

Переменные служат в качестве заполнителей для значений и могут использоваться в запросах метрик. С помощью селектора переменных в верхней части панели мониторинга вы можете динамически настраивать условия запросов, позволяя содержимому графиков обновляться в реальном времени.
