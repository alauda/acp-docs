# Введение

## Введение в продукт

Модуль наблюдаемости является ориентированной на приложения платформой наблюдаемости, готовой к использованию, облачной и нативной, которая предоставляет всеобъемлющие решения для мониторинга, адаптированные для предприятий. Он позволяет в реальном времени отслеживать приложения и их ресурсы, собирая различные метрики, журналы и данные о событиях для анализа состояния здоровья приложений. Этот модуль не только обладает мощными возможностями оповещения, но также предлагает всестороннюю и четкую многомерную визуализацию данных, совместим с основными компонентами с открытым исходным кодом и поддерживает быструю локализацию неисправностей и диагностические проверки мониторинга в один клик.

## Преимущества продукта

Основные преимущества модуля наблюдаемости следующие:

- Упрощенная сборка данных

  Обеспечивает унифицированный сбор метрик, журналов и трассировок, предлагая комплексный системный обзор и упрощая процессы управления.

- Механизм многомерного оповещения

  Поддерживает настройки многомерного оповещения для метрик и журналов, обеспечивая своевременное информирование пользователей о потенциальных проблемах.

- Интуитивно понятный интерфейс визуализации

  Предоставляет простой и четкий визуальный интерфейс управления, позволяя пользователям быстро получать доступ к ключевой информации и повышать эффективность принятия решений.

- Высокая совместимость

  Идеально совместим с основными компонентами с открытым исходным кодом, позволяя пользователям легко интегрировать и улучшать гибкость системы.

- Готовое решение

  Предлагает предварительно настроенные шаблоны и лучшие практики, позволяя пользователям быстро начать работу без сложных конфигураций.

## Сценарии применения продукта

Основные сценарии применения модуля наблюдаемости включают:

- Мониторинг архитектуры микросервисов

  В облачной архитектуре микросервисов приложения состоят из нескольких независимых сервисов. Модуль наблюдаемости может в реальном времени отслеживать производительность и состояние здоровья каждого микросервиса, помогая командам разработчиков быстро идентифицировать и решать проблемы с зависимостями между сервисами, обеспечивая общую стабильность системы.

- Мониторинг контейнеров и Kubernetes

  Для контейнерных приложений, установленных в средах Kubernetes, модуль наблюдаемости может отслеживать использование ресурсов, статус и журналы контейнеров, предоставляя управление жизненным циклом и устранение неполадок для контейнеров, обеспечивая высокую доступность контейнеризированных приложений.

- Динамическое управление ресурсами

  В облачных средах использование ресурсов может колебаться в зависимости от изменения бизнес-требований. Модуль наблюдаемости может в реальном времени отслеживать использование ресурсов и поддерживать динамическую корректировку распределения ресурсов, оптимизируя затраты и производительность.

- Мониторинг многооблачной среды

  Для предприятий, использующих многооблачные стратегии, модуль наблюдаемости может унифицировать управление и мониторинг приложений и ресурсов на различных облачных платформах, обеспечивая видимость и согласованность в облачных средах.
