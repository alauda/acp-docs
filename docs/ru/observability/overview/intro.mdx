# Введение

## Введение в продукт

Модуль наблюдаемости представляет собой ориентированную на приложения, готовую к использованию облачную платформу для наблюдаемости, предлагающую комплексные решения для мониторинга, адаптированные для предприятий. Он позволяет проводить мониторинг приложений и их ресурсов в реальном времени, собирая различные метрики, логи и данные о событиях для анализа состояния здоровья приложений. Этот модуль не только обладает мощными возможностями оповещения, но также предлагает всестороннюю и понятную многомерную визуализацию данных, совместим с основными компонентами с открытым исходным кодом и поддерживает быструю локализацию неисправностей и диагностические проверки одним щелчком.

## Преимущества продукта

Ключевые преимущества модуля наблюдаемости следующие:

- Единое сбор данных

  Обеспечивает единый сбор метрик, логов и трассировок, предоставляя комплексный обзор системы и упрощая процессы управления.

- Многомерный механизм оповещения

  Поддерживает многомерные настройки оповещения для метрик и логов, гарантируя, что пользователи своевременно уведомляются о потенциальных проблемах.

- Интуитивно понятный интерфейс визуализации

  Предоставляет простой и ясный интерфейс визуального управления, позволяя пользователям быстро получать доступ к ключевой информации и повышать эффективность принятия решений.

- Высокая совместимость

  Идеально совместим с основными компонентами с открытым исходным кодом, что позволяет пользователям легко интегрировать и увеличивать гибкость системы.

- Готовый к использованию

  Предлагает предварительно настроенные шаблоны и лучшие практики, позволяя пользователям быстро начать работу без сложных конфигураций.

## Сценарии применения продукта

Основные сценарии применения модуля наблюдаемости включают:

- Мониторинг архитектуры микросервисов

  В облачной архитектуре микросервисов приложения состоят из нескольких независимых сервисов. Модуль наблюдаемости может в реальном времени отслеживать производительность и состояние здоровья каждого микросервиса, помогая командам разработчиков быстро выявлять и решать проблемы зависимости между сервисами, обеспечивая стабильность всей системы.

- Мониторинг контейнеров и Kubernetes

  Для контейнерных приложений, установленных в средах Kubernetes, модуль наблюдаемости может отслеживать использование ресурсов, состояние и логи контейнеров, обеспечивая управление жизненным циклом и устранение неисправностей для контейнеров, гарантируя высокую доступность контейнеризированных приложений.

- Динамическое управление ресурсами

  В облачных средах использование ресурсов может колебаться в зависимости от изменений бизнес-требований. Модуль наблюдаемости может отслеживать использование ресурсов в реальном времени и поддерживать динамическую настройку распределения ресурсов, оптимизируя затраты и производительность.

- Мониторинг многоклаудной среды

  Для предприятий, принимающих стратегии многоклаудного использования, модуль наблюдаемости может унифицировать управление и мониторинг приложений и ресурсов в разных облачных платформах, обеспечивая видимость и согласованность в облачных средах.
