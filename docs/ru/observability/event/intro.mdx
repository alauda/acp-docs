# Введение

## Обзор модуля

Платформа интегрируется с событиями Kubernetes, регистрируя значительные изменения статуса и различные изменения оперативного состояния ресурсов Kubernetes. Она также предоставляет возможности для хранения, запроса и визуализации. Когда возникают аномалии с такими ресурсами, как кластеры, узлы или Pods, пользователи могут анализировать события, чтобы определить конкретные причины.

Основываясь на коренных причинах, выявленных из событий, пользователи могут [создавать политики оповещения](../monitor/functions/manage_alert.mdx) для рабочих нагрузок. Когда количество критических событий достигает предела оповещения, оповещения могут быть автоматически сгенерированы для уведомления соответствующего персонала о необходимости своевременного вмешательства, тем самым снижая операционные риски на платформе.

## Обзор функциональности

Модуль событий предоставляет в первую очередь следующие функциональности:

**Сбор и хранение событий**

- **Автоматический сбор**: Модуль будет автоматически собирать все события, происходящие в кластере Kubernetes, включая создание Pods, удаление, сбои планирования и т. д.
- **Постоянное хранение**: Собранные события будут храниться постоянно, чтобы обеспечить возможность пользователям отслеживать исторические события по мере необходимости.

**Запрос событий**

- **Гибкий запрос**: Пользователи могут запрашивать события с использованием различных условий (таких как тип события, пространство имен, имя ресурса и т. д.), чтобы быстро находить проблемы.
- **Фильтрация по диапазону времени**: Поддерживает запрос событий по диапазонам времени, позволяя пользователям просматривать активности кластера в конкретные временные периоды.

**Сводка и отображение событий**

- **Сводка событий**: Модуль будет обобщать события и генерировать статистическую информацию, чтобы помочь пользователям понять общее состояние кластера.

## Сценарии использования

Модуль событий подходит для следующих сценариев:

- **Мониторинг кластера**: Путем мониторинга событий Kubernetes в режиме реального времени пользователи могут своевременно обнаруживать аномалии в кластере.
- **Устранение неполадок**: Когда возникают проблемы в кластере, пользователи могут быстро определить коренную причину, запрашивая журналы событий.
- **Оптимизация производительности**: Анализируя данные о событиях, пользователи могут понять использование ресурсов в кластере и оптимизировать выделение ресурсов.

## Ограничения использования

Эта функция зависит от системы ведения журналов. Пожалуйста, убедитесь, что плагины ACP Log Collector и ACP Log Storage установлены на платформе заранее.
