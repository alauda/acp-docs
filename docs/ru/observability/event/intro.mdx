# Введение

## Обзор модуля

Платформа интегрируется с событиями Kubernetes, фиксируя значительные изменения состояния и различные изменения операционного состояния ресурсов Kubernetes. Она также предоставляет возможности для хранения, запроса и визуализации. Когда происходят аномалии с ресурсами, такими как кластеры, узлы или Pods, пользователи могут анализировать события, чтобы определить конкретные причины.

На основе коренных причин, выявленных из событий, пользователи могут [создавать политики оповещения](../monitor/functions/manage_alert.mdx) для рабочих нагрузок. Когда количество критических событий достигает предела оповещения, оповещения могут быть автоматически сгенерированы для уведомления соответствующего персонала для своевременного вмешательства, что тем самым снижает операционные риски на платформе.

## Обзор функциональности

Модуль событий в первую очередь предлагает следующие функциональности:

**Сбор и хранение событий**

- **Автоматический сбор**: Модуль будет автоматически собирать все события, происходящие в кластере Kubernetes, включая создание Pod, удаление, сбои планирования и т. д.
- **Постоянное хранение**: Собранные события будут храниться на постоянно, чтобы пользователи могли при необходимости отслеживать исторические события.

**Запрос событий**

- **Гибкий запрос**: Пользователи могут запрашивать события с использованием различных условий (таких как тип события, пространство имен, имя ресурса и т. д.), чтобы быстро находить проблемы.
- **Фильтрация по диапазону времени**: Поддерживает запрос событий по диапазонам времени, позволяя пользователям просматривать активности кластера в определенные временные периоды.

**Резюме событий и отображение**

- **Резюме событий**: Модуль будет подводить итоги событий и генерировать статистическую информацию, чтобы помочь пользователям понять общее состояние кластера.

## Сценарии использования

Модуль событий подходит для следующих сценариев:

- **Мониторинг кластера**: Путем мониторинга событий Kubernetes в реальном времени пользователи могут быстро обнаруживать аномалии в кластере.
- **Устранение неполадок**: Когда возникают проблемы в кластере, пользователи могут быстро находить коренные причины, запрашивая журналы событий.
- **Оптимизация производительности**: Анализируя данные событий, пользователи могут понять использование ресурсов в кластере и оптимизировать распределение ресурсов.

## Ограничения использования

Эта функция зависит от системы ведения журналов. Пожалуйста, убедитесь, что плагины ACP Log Collector и ACP Log Storage установлены на платформе заранее.
