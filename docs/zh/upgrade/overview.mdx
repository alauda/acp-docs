# 概述

本升级指南涵盖两个部分：升级 **global 集群** 和升级 **业务集群**。

对于 global 和业务集群，在集群详细信息页面上提供了一个 **功能组件** 选项卡。该选项卡列出了在执行集群升级时所有可升级的组件，包括：

- Kubernetes 版本
- 容器运行时
- 平台提供的集群插件和 operator

当任何列出的组件的新版本可用时，**升级** 按钮将被启用。用户可以通过点击该按钮来启动升级过程。

<Directive type="info" title="信息">
  - **Kubernetes 版本：** Kubernetes 升级仅支持 **自建集群**。对于 **托管集群**（例如，Amazon Elastic Kubernetes Service，Azure Kubernetes Service），Kubernetes 升级必须通过相应的云提供商进行。有关 **自建集群** 和 **托管集群** 之间定义和差异的更多信息，请参见 [集群类型](../configure/clusters/overview.mdx#cluster_type)。
  - **Operator：** 仅列出 **平台提供的 Operators**，并可以通过集群升级功能进行升级。**第三方或用户安装的 Operators** 通过 Marketplace 中的 **OLM** 组件进行管理，不包括在此升级过程中。
  - **集群插件：** 平台提供的插件可以通过 **自建集群** 和 **托管集群** 的集群升级功能进行升级，只要它们已安装。
  - **DR：** DR 是 *灾难恢复环境* 的缩写。它有一个主 global 集群和一个备用 global 集群，而标准的 <Term name="productShort" /> 环境仅有一个 global 集群。
  - **主 global 集群：** 备用 global 集群大致是主 global 集群的副本。为了区分这两者，特此将主 global 集群定义为 <Term name="productShort" /> 访问域名解析到的集群。
  - **备用 global 集群：** 特此将备用 global 集群定义为 <Term name="productShort" /> 访问域名 **不** 解析到的集群。
</Directive>
