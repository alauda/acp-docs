# 升级业务集群

在完成全球集群的升级后，您可以继续升级业务集群。业务集群的升级过程与全球集群类似，但需要注意以下事项：

- 如果您的平台使用**全球灾难恢复 (DR)** 解决方案，您**必须在升级任何业务集群之前完成主全球集群和备用全球集群的升级**。
- 所有**PostgreSQL** 实例将在升级过程中**自动重启**。
- 对于配置了**自动更新策略**的 **MySQL-PXC**、**MySQL-MGR**、**Redis**、**Kafka** 和 **RabbitMQ** 实例，升级过程包括重启，这可能会导致**临时服务中断**。
- 最多可以同时升级**20个业务集群**。

## 升级业务集群

1. 登录到 Web 控制台并切换到 **管理员** 视图。
2. 导航到 **集群 > 集群**。
3. 选择您要升级的 **业务集群** 并打开其详细页面。
4. 转到 **功能组件** 选项卡。
5. 点击 **升级** 按钮。

如果升级程序检测到任何自定义配置覆盖，系统将提示您确认这些设置。如果您不确定这些覆盖的配置是否会影响升级，请联系技术支持以获取帮助。

确认后，将出现组件升级对话框。请查看可用的更新并继续进行升级。

<Directive type="info" title="INFO">
  升级 Kubernetes 版本是可选的。
  然而，由于在其他组件更新期间仍可能发生服务中断，我们建议将 Kubernetes 升级纳入其中，以最小化未来的维护窗口。
</Directive>

## 升级 DevOps 工具链（如果已安装）

如果您的集群中安装了 DevOps 工具，您可以在**集群升级完成后**进行升级。

1. 在 Web 控制台中，切换到 **管理员** 视图。
2. 在左侧菜单中导航到 **DevOps 工具链** 以打开 **DevOps 控制台**。
3. 在 DevOps 控制台中，导航到 **DevOps 工具链 > 实例**。
4. 使用顶部的面包屑导航在集群之间切换。
5. 如果任何工具实例显示可用的升级，请点击 **操作** 列中的 **升级** 图标。
