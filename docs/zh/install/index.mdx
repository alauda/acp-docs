# 安装

<Overview overviewHeaders={[]} />

## 容量规划

在开始安装管理集群之前，首先需要根据您的实际需求选择合适的安装场景。不同的安装场景对平台的资源需求和架构规划有所不同。我们为您提供了三种常见的安装场景，您可以根据自身情况选择适合的场景。

### 安装场景选择

#### 场景一：功能体验场景（POC）
功能体验场景适用于平台初次体验或技术验证（POC，Proof of Concept）。在这个场景下，平台仅用于体验所有功能，而不涉及到业务的实际运行。您可以在此场景下使用最少的资源来部署平台，并对其功能进行测试。

- **机器要求**：
  - **最小要求**：1 台机器（物理机或虚拟机）
  - **推荐配置**：
    - CPU：16 核
    - 内存：32GB
  - 该机器将作为管理集群的控制节点和计算节点，且运行Demo应用，Pod 总数不超过 10。

- **架构规划**：
  - 仅使用单节点管理集群，不需要额外的业务集群。
  - 此场景仅用于平台功能测试，不建议用于生产环境。

#### 场景二：ISV 交付的底座场景
该场景适用于 ISV（独立软件供应商）交付的底座平台。通常，这种场景下不会有单独的开发或测试环境，全部部署为正式生产环境。在这种场景下，管理集群不仅负责平台管理，还会承载业务工作负载，即 ISV 软件的 Pod。换句话说，管理集群将作为一个业务集群使用，不需要额外创建其他业务集群。

- **机器要求**：
  - **最小要求**：3 台物理机或虚拟机
  - **推荐配置**：
    - **控制节点**：至少 3 节点（奇数），每个节点 8 核 CPU，16GB 内存
    - **计算节点**：至少 2 节点，CPU 12 核，24GB 内存
  - 这种场景下，控制节点和计算节点都会承担业务工作负载，且控制节点应具备高可用性。

- **架构规划**：
  - **管理集群**：作为业务集群的承载者，管理集群既运行平台组件，也运行 ISV 软件的 Pod。
  - 该场景下的管理集群可通过合理规划资源，满足业务工作负载的需求。

#### 场景三：数据中心交付场景
数据中心交付场景适用于大规模企业部署，通常涉及多个业务集群。业务集群可以是多种来源，例如公有云中的 Kubernetes 服务（如 AWS 的 EKS，Azure 的 AKS，华为云的 CCE）。在这种场景下，随着业务集群的增加，管理集群的资源需求也会随之增加，特别是计算节点的数量和节点的资源配置。

- **机器要求**：
  - **最小要求**：3 台控制节点，2 台计算节点
  - **推荐配置**：
    - **控制节点**：3 节点高可用，8 核 CPU，16GB 内存
    - **计算节点**：至少 2 节点，24 核 CPU，42GB 内存
  - 业务集群数量增多时，管理集群需要根据需求适时扩展计算资源。

- **架构规划**：
  - **管理集群**：负责管理多个业务集群，并保证平台的高可用性。根据业务集群规模的不同，可能需要额外增加计算节点资源。
  - **业务集群**：多个业务集群以水平扩展的方式增加，每个业务集群可根据需求动态创建/接入。

### 对应安装场景的部署架构与资源规划

选择合适的安装场景后，您需要根据实际情况规划管理集群和业务集群的资源。以下是针对不同安装场景的部署架构建议：

#### 1. 功能体验场景（POC）
在此场景下，部署架构非常简单。您将使用一台机器安装管理集群，并使用该集群运行平台的Demo应用。该集群无需接入其他业务集群，资源需求较低，适合用于技术验证。

- **架构规划**：
  - **管理集群**：单节点安装，包含控制节点和计算节点，支持最多 10 个 Pod。
  - **业务集群**：无需部署，全部业务负载由管理集群承担。

#### 2. ISV 交付的底座场景
此场景下，管理集群不仅负责平台管理，还需承载业务负载。管理集群在此场景下起到两个角色：平台管理和业务应用承载。每个节点的 CPU 和内存资源需求会增加，确保平台高可用性。

- **架构规划**：
  - **管理集群**：由控制节点和计算节点组成，控制节点至少 3 台，计算节点至少 2 台，建议配置 8 核 CPU，16GB 内存。
  - **业务集群**：无需创建额外的集群，所有业务工作负载由管理集群承载。

#### 3. 数据中心交付场景
此场景的复杂性最高，涉及到多种来源的业务集群，且随着业务规模的增长，管理集群需要根据业务集群的扩展动态调整资源。随着接入更多的业务集群，管理集群的计算节点数量和计算资源应做适当扩展，以满足高可用性和性能需求。

- **架构规划**：
  - **管理集群**：作为平台的中枢，负责管理所有接入的业务集群。建议配置至少 3 个控制节点，2 个计算节点，且根据业务集群的规模动态增加计算节点的数量和资源。
  - **业务集群**：来自不同的云平台（如 EKS、AKS、CCE 等），可通过标准化的接入方式（如 Kubernetes API）进行接入。

### 结论
在选择安装场景时，您需要根据平台的使用需求和业务规模来决定合适的场景。每个场景的部署架构和资源规划都有不同的要求，建议您根据实际的业务需求，结合我们提供的容量规划和资源配置建议，进行详细规划。如有特殊需求或不确定的地方，欢迎随时联系技术支持进行咨询。
