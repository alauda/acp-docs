# 安装概述

平台架构及关键特性如下：

- **架构设计**
  - **管理集群**：负责平台核心管理，提供集群、租户管理和全局调度。安装管理集群后，后续可根据需要接入或新建业务集群。
  - **业务集群**：专用于承载业务负载，支持横向扩展。业务集群可独立部署，也可通过接入符合版本要求的存量集群方式接入平台。

- **设计原则**
  1. **控制面隔离部署**
     安装管理集群时，必须使用专用物理资源，不得与业务集群共享，确保控制平面的稳定性与安全性。
  2. **混合部署兼容性**
     在开发测试环境或资源受限情况下，允许在管理集群（即 global 集群）上同时运行业务负载，实现管理与业务功能共置。
  3. **标准化接入规范**
     支持两种业务集群接入方式：
     - 新建业务集群（推荐）
     - 接入符合版本要求的存量集群（需预检）
