# 安装概述

本文档面向具备 Linux 基础的系统管理员，详细说明容器平台管理集群的安装全流程。
要求在安装前完成容量规划和环境预处理，确保各节点硬件、网络及操作系统均符合要求。以下内容涵盖平台架构设计、安装方式和关键术语解释，便于您在实际安装过程中把握核心要点。

## 平台架构

- **架构设计**
  - **管理集群**：负责平台核心管理，提供集群、租户管理和全局调度。安装管理集群后，可根据需要接入或新建业务集群。
  - **业务集群**：专用于承载业务负载，支持横向扩展。

- **设计原则**
  1. **控制面隔离**
     安装管理集群时，必须使用专用物理资源，不得与业务集群共享，确保控制平面的稳定性与安全性。
  2. **混合部署兼容性**
     在开发测试环境或资源受限情况下，允许在管理集群（即 global 集群）上同时运行业务负载，实现管理与业务功能共置。
  3. **标准化接入规范**
     支持两种业务集群接入方式：
     - 新建业务集群（推荐）
     - 接入符合版本要求的存量集群（需预检）

- **架构示意图**

  ```mermaid

      A[管理集群] --> B(业务集群1)
      A --> C(业务集群2)
      A --> D(业务集群N)
  ```

## 安装方式

平台安装流程主要分为三个阶段：

1. **准备阶段**
   - **先决条件检查**： 确保所有节点硬件、网络及操作系统均符合要求，如内核版本、CPU 架构、网络配置等。
   - **安装包下载与校验**：登录 Customer Portal 获取最新安装包及签名文件，使用 GPG 工具验证包的完整性与正确性。
   - **节点预处理**： 完成所有节点的预处理工作。

2. **执行阶段**
   - **安装包上传与解压**：将安装包上传至目标控制节点（建议目录：`/root/cpaas-install`），并解压安装资源。
   - **启动安装器**：在控制节点上执行安装脚本（如 `bash setup.sh`），根据实际环境选择网络插件（Kube-OVN 或 Calico）、IP 协议模式（IPv4/IPv6/双栈）及 VIP 配置。
   - **参数配置**：访问安装器提供的 Web UI ，依次设置 Kubernetes 版本、集群网络、节点名称、访问地址及其他关键参数，完成管理集群的安装。

3. **验证阶段**
   - **系统状态检查**：安装完成后，登录平台 Web UI 检查管理集群状态及各组件运行情况。
   - **CLI 验证**：通过命令行工具检查集群资源状态，确保所有服务正常运行，无异常或失败项。

## 术语解释

- **管理集群**
  运行平台核心管理组件的集群，主要负责集群管理和租户管理，安装后可按需创建/接入业务集群。

- **业务集群**
  用于承载具体业务工作负载的集群，支持横向扩展。

- **VIP（Virtual IP）**
  虚拟 IP 地址，用于负载均衡管理集群的 API Server、Web UI 等访问入口，确保高可用性。
