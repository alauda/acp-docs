# 使用 YAML 文件创建 OAM 应用

创建 OAM 应用的方式有 **使用内置组件及运维特征创建应用**、**使用自定义组件及运维特征创建应用** 及 **使用 YAML 文件创建应用** 三种，本文档用于指导您快速了解并使用 YAML 文件创建一个简单 OAM 应用。

## 介绍

### 适用场景

如果您已能熟练创建应用，希望能够在集群内快速移植应用，借助 YAML 文件可一键完成所需组件和运维特征的配置。

### 预计阅读时间

预计完成本文档的阅读和操作时间，例如：3-5 分钟。

## 注意事项

* 本文档不会对配置过程中的参数进行详细说明和配置，仅作为快速入门参考，具体说明及配置步骤请参考相关文档。

* 如果是从已有 OAM 应用导出的 YAML 文件，请注意修改 YAML 中的应用名称和组件名称，避免同一命名空间中名称重复。

## 前提条件

* 对于 OAM 应用有深入的了解。

* 已经开启 OAM 相关功能。


## 操作步骤

1. 进入 **Container Platform**。

1. 在左侧导航栏中，单击 **应用管理** > **OAM 应用**。

2. 单击 **创建应用**。

3. 选择 **通过 YAML 创建** 方式。

4. 填入 YAML 内容，单击 **创建**。

    **说明**：除了平台提供的 YAML 样例，您还可前往  [OAM 开源社区](https://github.com/oam-dev/spec/blob/master/7.application.md) 获取更多应用 YAML 写作指导。
