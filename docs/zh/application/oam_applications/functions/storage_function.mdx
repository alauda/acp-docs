# 添加存储与配置

通常，在创建应用的过程中您已经完成应用资源（存储与配置）的设置。本节将介绍如何添加新的存储与配置。

## 添加存储

1. 在左侧导航栏中，单击 **应用管理** > **OAM 应用**。

3. 单击 ***应用名称***。

3. 在 **应用资源** 页签中，单击 **添加资源**。
    
2. 参考以下说明完成相关配置。

    |参数| 说明                                                         |
    | ------ | ------------------------------------------------------------ |
    |**存储类**| <ul><li>存储类代表了不同类型的存储特征，例如是否支持在线扩容、是否支持卷快照，以及是否具备高可用性等。</li><li>每个存储类底层的存储方案都支持持久化存储数据，即不会随应用的停止或重启而删除。</li><li>存储类通常由管理员设置，您可灵活选择符合预期的存储类。 </li></ul> |
    |**容量**| 即存储配额，配额大小取决于您的应用消耗。如果所设值较大，请与管理员确认实际可用存储空间充足。 |
    
3. 单击 **添加**。 
4. 返回组件列表页，单击 ![](/zh/img/113point.png) > **更新**，并将存储挂载为 [存储卷](/application/oam_applications/functions/storage_function.mdx) 。


## 添加配置

1. 在左侧导航栏中，单击 **应用管理** > **OAM 应用**。

3. 单击 ***应用名称***。

3. 在 **应用资源** 页签中，单击 **添加资源**。
    
2. 按需完成相关配置。

    * 如果为一般配置，可选择 **配置字典** 或 **Opaque** 类型。
    
    * 如果为敏感数据，务必根据数据性质，选择 **用户名密码**、**SSH 密钥**，或 **镜像服务** 等类型。


3. 单击 **添加**。 
4. 返回组件列表页，单击 ![](/zh/img/113point.png) > **更新**，并将配置挂载为 [存储卷](/application/oam_applications/functions/storage_function.mdx) 或引用为 [环境变量](/application/oam_applications/functions/storage_function.mdx) 。
