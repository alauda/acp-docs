# 监控与告警


## 监控


在组件的 **监控** 页签中，支持查看近 7 天内应用的监控数据。

**说明**：仅当应用使用了 GPU 资源时，GPU 相关监控图表可见。
                
|监控资源| 监控数据统计粒度  |  指标说明                                                         |
| -------- | --------  | ------------ |
|       **CPU**     |   使用率/使用量  |   **使用率** = 使用值/限制值（limits）<br/>您可通过使用率判断组件容器的限制值配置是否合理，当使用率过高时，表明容器限制值设置过小，可能会影响组件的正常运行。<br/>**使用量** 为组件已使用的资源量。      |
|       **内存**     |   使用率/使用量  |   **使用率** = 使用值/限制值（limits）<br/>您可通过使用率判断组件容器的限制值配置是否合理，当使用率过高时，表明容器限制值设置过小，可能会影响组件的正常运行。<br/>**使用量** 为组件已使用的资源量。      |
|       **网络流量**     |   流入/流出  |   组件中所有容器组的网络流量速率总和。   |
|       **网络报文量**     |  接收/发送 | 组件中所有容器组的网络报文量速率总和。 |
|       **磁盘速率**     |  读/写 | 组件中所有容器组挂载分区的读写速率总和。 |
|       **磁盘 IOPS**     |  读/写 |组件中所有容器组挂载分区的 IOPS 总和。 |
|       **虚拟/物理 GPU**     |   使用率/使用量  |   **使用率** = 使用值/限制值（limits）<br/>**使用量** 为组件已使用的资源量。      |
|       **虚拟/物理 GPU 显存**     |   使用率/使用量  |   **使用率** = 使用值/限制值（limits）<br/>**使用量** 为组件已使用的资源量。      |
    
## 告警


1. **配置告警策略**：为启用告警，请先在组件的 **告警** 页签中，参考平台通用指导 [创建告警策略](/application/oam_applications/functions/alarm_function.mdx) 。告警策略描述了您希望监控的对象、在什么情况下希望收到告警，以及如何被通知相关告警。

2. **处理告警**：当界面提示有处于告警状态的策略时，请及时处理。

3. **绑定通知策略**：除了查看实时提醒外，平台还支持将告警信息以邮件、短信等方式发送给相关人员，通知其采取必要的措施解决问题或避免故障发生。通知策略需联系管理员设置后使用。
