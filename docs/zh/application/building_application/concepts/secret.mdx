# 深入理解 Secret



若需要在 Kubernetes 集群中储存需要保密的敏感信息或配置，例如：用户密码、OAuth 令牌、SSH 私钥、访问镜像仓库的认证信息等，建议优先使用保密字典（Kubernetes Secret）储存。

## 使用特点

* 相比于使用明文文本的配置字典（Kubernetes ConfigMap），保密字典的安全性更好（使用 Base64 编码格式存储敏感信息），且具有控制敏感信息使用，减少信息暴露风险的能力。

* 相比于将敏感信息储存于 Pod 定义文件或容器镜像中，使用保密字典更加安全、灵活，便于控制修改。

## 支持类型

平台支持通过 UI 编辑模式创建的保密字典类型如下所述。

* **Opaque**：一般类型的保密字典，可用于存储密码、密钥等信息，以 Map 键值对存储数据。


* **TLS**：用于存储 TLS 协议证书和私钥认证信息。例如：HTTPS 证书凭据。

* **SSH 密钥**：通过非对称加密算法生成的一对密钥，公钥配置到服务端用于加密，私钥用于客户端解密。

* **SSH 认证**：即 ssh-auth，用于存储通过 SSH 协议传输数据的认证信息。

* **用户名/密码**：即 basic-auth，用于存储用户名和密码认证信息。

* **镜像服务**：即 dockerconfigjson，用于存储私有镜像仓库（Docker Registry）的 JSON 鉴权串。


## 使用方式

* **全量引用**：通过表单方式为原生应用设置容器时，在 **配置引用** 处执行操作。

* **部分引用**：通过表单方式为原生应用设置容器时，在 **环境变量** 处执行操作。

**注意**：计算组件的容器组实例仅能引用同一命名空间下的保密字典。

使用 YAML 创建自定义应用时，可参考 [Kubernetes 官方文档](https://kubernetes.io/docs/concepts/configuration/secret/#using-secrets)，使用保密字典（适用于熟悉 Kubernetes 且可以熟练使用 YAML 的用户）。
