# 加载 Configmap 配置

挂载的配置字典更新后，应用程序需加载更新的配置使之生效。针对不能自动加载配置的应用程序，**加载配置** 提供手动触发应用程序加载配置字典的能力。

## 前提条件

* 应用程序需支持配置热更新。

* 容器组中需以存储卷方式挂载配置字典，且不能使用子路径 SubPath。通过环境变量引用的配置字典不支持加载配置。


## 操作步骤

1. 进入 **Container Platform**。

1. 在左侧导航栏中，单击 **配置** > **配置字典**。

2. 单击 ***配置字典名称***。

3. 单击 **添加配置项** 或单击 **添加二进制配置项**；也可以按需单击 <span style={{ display: 'inline-flex', alignItems: 'center', gap: '4px', verticalAlign: 'middle' }}><img src="./assets/blackpencil.png" alt="expand" style={{ verticalAlign: 'middle' }} /> <span>更新配置项，并确定。</span></span>


4. 切换到 **计算组件** 页签，单击 <span style={{ display: 'inline-flex', alignItems: 'center', gap: '4px', verticalAlign: 'middle' }}><img src="./assets/fold.png" alt="expand" style={{ verticalAlign: 'middle' }} /> <span>展开其容器列表。</span></span>


5. 单击 **加载配置**，输入用户及执行命令，并确定。默认以 **root** 用户 EXEC 进入容器。

6. 根据应用程序类型输入相应的命令加载配置。

    * 示例一：Nginx
    
       ```
       nginx -s reload
       ```
       
    * 示例二：Logstash

       ```
       kill -SIGHUP <Logstash 进程 ID>
       ```
