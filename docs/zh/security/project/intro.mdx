# 介绍

## 项目

项目是资源隔离单位，使企业能够实现多租户使用场景。它将来自一个或多个集群的资源划分为隔离的环境，确保资源和人员的隔离。项目可以表示企业内不同的子公司、部门或项目团队。通过项目管理，您可以实现：

- 项目团队之间的资源隔离
- 租户内的配额管理
- 高效的资源分配和控制

## 命名空间

命名空间是项目内较小的、相互隔离的资源空间。它们为用户实施生产工作负载提供工作区。命名空间的主要特征包括：

- 每个项目可以创建多个命名空间
- 所有命名空间的总资源配额不得超过项目配额
- 资源配额在命名空间级别更加细化
- 容器大小（CPU、内存）在命名空间级别受到限制
- 通过细粒度控制提高资源利用率

## 集群、项目与命名空间之间的关系

平台的资源层级遵循以下规则：

- 一个项目可以利用来自多个集群的资源（CPU、内存、存储），一个集群可以将资源分配给多个项目。

- 项目下可以创建多个命名空间，其组合资源配额不得超过项目的总资源。

- 命名空间的资源配额必须来自单一集群，且一个命名空间只能属于一个项目。
