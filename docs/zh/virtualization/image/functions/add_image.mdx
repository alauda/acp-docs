# 添加虚拟机镜像

平台支持添加 **X86_64** 和 **ARM64（Alpha）** 架构的虚拟机镜像，方便开发人员快速创建已有业务的虚拟机，便于业务系统的迁移。

## 操作步骤

1. 进入 **平台管理**。

1. 在左侧导航栏中，单击 **虚拟化管理** > **虚拟机镜像**。

3. 单击 **添加虚拟机镜像**。

4. 参考以下说明配置相关参数。

    |参数|说明  |
    | -------- | ------------- |
    | **提供方式**   | 目前仅支持 **公共镜像** 方式，即添加的镜像可在已分配的项目中使用。 |
    | **操作系统**   | 支持的操作系统有： **CentOS**/**Ubuntu**/**RedHat**/**Debian**/**TLinux**/**Other Linux**/**Windows（Alpha）**。<br/>支持的系统架构有： **X86_64** 和 **ARM64（Alpha）**。 |
    | **来源**   | <ul><li>**镜像仓库**：存储于容器镜像仓库的虚拟机镜像。</li><li>**HTTP**：存储于 HTTP 协议的文件服务器中的虚拟机镜像。</li><li>**对象存储（S3）**：支持通过对象存储协议 (S3) 获取的虚拟机镜像，若是无需认证的对象存储文件，请使用 HTTP 来源。</li></ul> |
    | **CPU 架构**   |标记 CPU 架构信息，当来源为镜像仓库时，CPU 架构支持多选，其余来源时为单选。|
    | **镜像地址**   | 支持 KVM 的虚拟机镜像，包括 qcow2/raw 格式。<ul><li>若来源于镜像仓库，输入 `仓库地址:镜像版本`，例如 `index.docker.io/library/ubuntu:latest`。</li><li>若为 HTTP 来源，输入镜像文件 URL，必须以 `http://` 或 `https://` 开头，例如`http://192.168.0.1/vm_image/centos_7.8.qcow2`。</li><li>若为对象存储 (S3)，请输入需通过对象存储协议（S3）获取的镜像地址，例如：`https://endpoint/bucket/centos.qcow2`。</li></ul> |
    | **认证** | 根据镜像仓库是否需要认证，可关闭或开启开关，开启后您可选择已有镜像凭据或单击 **添加凭据**，仅支持使用 **用户名密码** 类型的凭据。<br/>**提示**：来源为对象存储 (S3) 时，认证不可关闭。 |
    | **分配项目**   | 为项目分配该镜像的使用权限。<ul><li>所有项目：将镜像的使用权限分配给所有项目。</li><li>指定项目：将镜像的使用权限分配给指定的项目。</li><li>不分配：暂不分配项目。镜像创建完成后，可通过 **更新镜像** 操作另行分配。</li></ul> |
    
5. 单击 **测试镜像连通性**，执行虚拟机镜像地址和认证信息测试，参考提示结果，检查配置参数。

6. 单击 **添加**。
