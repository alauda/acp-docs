# 介绍

构建、存储和管理容器镜像是云原生应用开发过程中的核心部分。Alauda 容器平台（ACP）提供了一种高性能、高可用的内置容器镜像仓库服务，旨在为用户提供安全便捷的镜像存储和管理体验，极大简化了平台内的应用开发、持续集成/持续部署（CI/CD）和应用部署过程。

Alauda 容器平台注册表深度集成于平台架构中，提供比外部独立部署的镜像仓库更紧密的平台协作、更简化的配置和更高效的内部访问。

## 原则与命名空间隔离

Alauda 容器平台的内置镜像仓库作为平台的核心组件之一，以高可用的方式在集群内部运行，并利用平台提供的持久存储能力，确保镜像数据的安全和可靠。

其核心设计概念之一是基于命名空间的逻辑隔离和管理。在注册表内，镜像仓库按命名空间组织。这意味着每个命名空间可以被视为属于该命名空间的镜像的独立“区域”，不同命名空间之间的镜像默认是隔离的，除非明确授权。

## 认证与授权

Alauda 容器平台注册表的认证与授权机制与 ACP 的平台级认证和授权系统深度集成，能够实现细粒度的访问控制，具体到命名空间：

### 认证

用户或自动化流程（例如，平台上的 CI/CD 流水线、自动构建任务等）无需为注册表维护一套单独的账户密码。它们通过平台的标准认证机制（例如，使用平台提供的 API 令牌、集成的企业身份系统等）进行认证。当通过 CLI 或其他工具访问 Alauda 容器平台注册表时，通常会利用现有的平台登录会话或 ServiceAccount 令牌进行透明认证。

### 授权

授权控制在命名空间级别实施。Alauda 容器平台注册表中镜像仓库的拉取或推送权限取决于用户或 ServiceAccount 在相应命名空间中的平台角色和权限。

- **通常情况下，** 命名空间的所有者或开发者角色会自动获得该命名空间下镜像仓库的推送和拉取权限。
- **其他命名空间的用户或希望跨命名空间拉取镜像的用户** 需要在访问该命名空间内的镜像之前，获得目标命名空间管理员的明确授权（例如，通过 RBAC 绑定允许拉取镜像的角色）。
- **这种基于命名空间的授权** 机制确保了命名空间之间镜像的隔离，提高了安全性，避免了未授权的访问和修改。

## 优势

**Alauda 容器平台注册表的核心优势：**

- **即用即部署：** 快速设置私有镜像仓库，无需复杂的配置。
- **灵活访问：** 支持集群内部和外部访问模式。
- **安全保障：** 提供 RBAC 授权和镜像扫描能力。
- **高可用性：** 通过复制机制确保服务连续性。
- **生产就绪：** 在企业环境中经过验证，具备 SLA 保证。

## 应用场景

- **轻量级部署：** 在低流量环境中实施精简的注册表解决方案，加速应用交付。
- **边缘计算：** 为边缘集群启用专用注册表的自主管理。
- **资源优化：** 在基础设施未充分利用时，通过集成的源码到镜像（S2I）解决方案展示完整的工作流能力。
