# 通过 Web UI 安装

## 何时使用此方法？

**推荐用于**：

- **首次使用者**，希望使用引导式的可视化界面。
- **快速概念验证**设置，适用于非生产环境。
- **Kubernetes 专业知识有限**的团队，寻求简化的部署流程。
- 需要**最小化自定义**的场景（例如，默认存储配置）。
- **基本网络设置**，没有特定的入口规则。
- **StorageClass** 配置以实现高可用性。

**不推荐用于**：

- 需要高级存储（S3 存储）配置的生产环境。
- 需要特定入口规则的网络设置。

## 先决条件

- **安装** **Alauda Container Platform Registry** 集群插件到目标集群，使用 [集群插件](/extend/cluster_plugin.mdx) 机制。

## 使用 Web 控制台安装 Alauda Container Platform Registry 集群插件

### 操作步骤

1. 登录并导航到 **管理员** 页面。
2. 点击 **市场** > **集群插件** 以访问 **集群插件** 列表页面。
3. 找到 **Alauda Container Platform Registry** 集群插件，点击 **安装**，然后进入安装页面。
4. 根据以下规格配置参数，然后点击 **安装** 完成部署。

参数描述如下：

| **参数**                 | **描述**                                                                                                                                                                                        |
| :------------------------ | :----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **暴露服务**             | 启用后，管理员可以使用访问地址在外部管理镜像库。这会带来重大安全风险，启用时需极其谨慎。                                                                                     |
| **启用 IPv6**            | 当集群使用 IPv6 单栈网络时启用此选项。                                                                                                                                 |
| **主机端口**             | 当暴露服务启用时，配置主机端口以允许通过此端口外部访问注册表。                                                                                             |
| **存储类型**             | 选择存储类型。支持的类型：LocalVolume 和 StorageClass。                                                                                                                                  |
| **节点**                 | 选择一个节点来运行用于镜像存储和分发的注册表服务。（仅在存储类型为 LocalVolume 时可用）                                                                        |
| **StorageClass**         | 选择一个 StorageClass。当副本超过 1 时，选择具有 RWX（ReadWriteMany）能力的存储（例如，文件存储）以确保高可用性。（仅在存储类型为 StorageClass 时可用） |
| **存储大小**             | 分配给注册表的存储容量（单位：Gi）。                                                                                                                                                 |
| **副本数**               | 配置注册表 Pod 的副本数： <ul><li>**LocalVolume**：默认值为 1（固定）</li><li>**StorageClass**：默认值为 3（可调整）</li></ul>                                     |
| **资源需求**            | 定义注册表 Pod 的 CPU 和内存资源请求及限制。                                                                                                                               |

### 验证

1. 导航到 **市场** > **集群插件**，确认插件状态显示为 **已安装**。
2. 点击插件名称查看其详细信息。
3. 复制 **注册表地址**，并使用 Docker 客户端推送/拉取镜像。
