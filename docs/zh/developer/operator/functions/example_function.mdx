:::tip

在编写功能文档时，各个功能的负责人应根据具体功能的特点，合理拆分文档页面和目录结构。优先提供对应 CLI 的操作说明，以便用户快速上手。

1. 概述

功能指南文档通常位于功能概述和 HowTo 文档之间，专注于单一功能的详细介绍。它可以涵盖功能本身的详细描述。

2. 核心特征

功能指南文档应具备以下核心特征：

- 概念导向：系统性地解释功能的核心概念和使用方式。
- 灵活性：功能描述不应依赖于特定场景。
- CLI 详解：优先提供详细的命令行操作说明，帮助用户通过 CLI 快速实现功能。其次再补充该功能对应的功能界面使用方式。

3. 功能文档与 HowTo 文档的区别

功能文档中时常会伴随一些使用和操作的介绍，但它与 HowTo 的差异常常令人困惑。它们的区别在于：

- **内容焦点**
  - 功能指南：回答“是什么”和“为什么”，帮助用户理解功能本身。
  - HowTo：回答“怎么做”，指导用户完成具体任务。

- **结构特点**
  - 功能指南：一个系统化且层次分明的知识体系，注重概念解释和原理阐述。
  - HowTo：以任务为导向的线性步骤，强调实际指导。

- **使用场景**
  - 功能指南：适用于用户初次了解功能或需要深入理解功能原理的情况。
  - HowTo：适用于用户需要解决特定问题或完成特定任务时。

- **覆盖范围**
  - 功能指南：全面介绍功能的所有特性、配置项和使用方法。
  - HowTo：针对特定场景或任务的具体操作指南。

4. 编写指南

在撰写功能文档时，尤其是涉及 CLI 的部分，建议遵循以下步骤：

- 明确目标用户群体：确定文档的目标读者和他们的需求，了解他们对 CLI 的熟悉程度，以调整内容的深度和复杂性。
- 结构化内容：将文档分成多个部分，例如功能简介、CLI 命令参数详解、使用示例和注意事项。确保每个部分逻辑清晰，易于理解。
- 详细命令说明：为每个 CLI 命令提供详细的参数说明和使用示例。解释命令的作用、适用场景和可能的输出结果。
