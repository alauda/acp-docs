# 概述

<Term name="product" /> 提供统一的接口，通过 Web 控制台和 CLI（命令行界面）创建、编辑、删除和管理云原生应用。应用可以在多个命名空间中部署，并具有 RBAC 策略。

## 命名空间管理

命名空间为 Kubernetes 资源提供逻辑隔离。主要操作包括：

- [创建命名空间](./functions/namespace/create_namespace.mdx)：定义资源配额和 Pod 安全准入策略。
- [导入命名空间](./functions/namespace/import_namespace.mdx)：将现有的 Kubernetes 命名空间导入到 <Term name="product" /> 中，提供与原生创建的命名空间完全相同的平台能力。

## 应用生命周期管理

<Term name="product" /> 支持端到端的生命周期管理，包括：

### 应用创建模式

在 <Term name="product" /> 中，应用可以通过多种方式创建。以下是一些常见方法：

- [从镜像创建](./functions/create_applications/image_app.mdx)：使用预构建的容器镜像创建自定义应用。此方法支持创建包含 `Deployments`、`Services`、`ConfigMaps` 和其他 Kubernetes 资源的完整应用。
- [从目录创建](./functions/create_applications/chart_app.mdx)：<Term name="product" /> 提供应用目录，允许用户选择预定义的应用模板（Helm Charts 或 Operator 支持）进行创建。
- [从 YAML 创建](./functions/create_applications/yaml_app.mdx)：通过导入 YAML 文件，一步创建包含所有资源的自定义应用。
- [从代码创建](./functions/create_applications/s2i_app.mdx)：通过源到镜像（S2I）构建镜像。

### 应用操作

- [更新应用](./functions/application_operation/application_upgrade.mdx)：更新应用的镜像版本、环境变量和其他配置，或导入现有的 Kubernetes 资源进行集中管理。
- [导出应用](./functions/application_operation/export.mdx)：以 YAML、Kustomize 或 Helm Chart 格式导出应用，然后导入以在其他命名空间或集群中创建新的应用实例。
- [版本管理](./functions/application_operation/version.mdx)：支持自动或手动创建应用版本，并在出现问题时可一键回滚到特定版本以快速恢复。
- [删除应用](./functions/application_operation/application_delete.mdx)：删除应用时，同时删除应用本身及其直接包含的所有 Kubernetes 资源。此外，此操作还会切断应用与其他未直接包含在其定义中的 Kubernetes 资源之间的任何关联。

### 应用可观察性

为了持续的操作管理，平台提供日志、事件、监控等功能。

- [日志](./functions/application_observability/log.mdx)：支持查看当前运行 Pod 的实时日志，并提供之前容器重启的日志。
- [事件](./functions/application_observability/event.mdx)：支持查看命名空间内所有资源的事件信息。
- [监控仪表板](./functions/application_observability/monitor.mdx)：提供命名空间级监控仪表板，包括应用、工作负载和 Pod 的专用视图，并支持自定义监控仪表板以满足特定的操作需求。

## Kubernetes 工作负载管理

支持核心工作负载类型：

- [Deployments](./functions/application_workloads/deployment_create.mdx)：管理无状态应用的滚动更新。
- [StatefulSets](./functions/application_workloads/statefulset_create.mdx)：运行具有稳定网络 ID 的有状态应用。
- [DaemonSets](./functions/application_workloads/deamonset_create.mdx)：部署节点级服务（例如，日志收集器）。
- [CronJobs](./functions/application_workloads/cronjob_create.mdx)：调度具有重试策略的批处理作业。
