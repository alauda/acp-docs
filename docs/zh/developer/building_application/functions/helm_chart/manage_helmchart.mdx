# 管理应用模板

模板应用的更新依赖于应用模板（Helm Chart）的更新。如果某些模板无法通过平台升级、公共仓库或自建私有仓库等方式自动更新，您可以手动将其添加至具有 **管理** 权限的 Chart 或 OCI Chart 类型模板仓库中，以部署相应版本的应用。

## 前提条件

模板仓库由平台管理员添加。请联系平台管理员以获取可用的具有 **管理** 权限的 Chart 或 OCI Chart 类型模板仓库名称。

## 添加模板至可管理仓库

1. 进入 **Catalog**。

2. 在左侧导航栏中，单击 **Helm Charts**。

3. 单击页面右上角的 **添加模板**，根据以下参数选择模板仓库。

   | 参数                 | 说明                                                                                                                                                                                                                                                                                                          |
   | --------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | **模板仓库**         | 将模板直接同步至具有 **管理** 权限的 Chart 或 OCI Chart 类型模板仓库中。被分配到该 **模板仓库** 的项目所有者可以直接使用该模板。                                                                                                                                                                       |
   | **模板目录**         | 当选择的模板仓库类型为 OCI Chart 时，必须选择或手动输入用于存储 Helm Chart 的目录。<br />**注意**：手动输入新的模板目录时，平台将在模板仓库中创建此目录，但可能存在创建失败的风险。                                                                                                                 |

4. 单击 **上传模板**，并将本地模板上传至仓库。

5. 单击 **确定**。模板上传过程可能需要几分钟，请耐心等待。

   **注意**：当模板状态从 `上传中` 变为 `上传成功` 时，表示模板已成功上传。

6. 如果上传失败，请根据以下提示进行故障排除。

   **注意**：非法的文件格式意味着上传的压缩包内的文件存在问题，例如缺失内容或格式不正确。

## 删除特定版本的模板

如果某个版本的模板不再适用，可以将其删除。

### 操作步骤

1. 进入 **Catalog**。

2. 在左侧导航栏中，单击 **Helm Charts**。

3. 单击 Chart 卡片查看详细信息。

4. 单击 **管理版本**。

5. 找到不再适用的模板，单击 **删除**，并确认。

   删除版本后，相应的应用将无法更新。
