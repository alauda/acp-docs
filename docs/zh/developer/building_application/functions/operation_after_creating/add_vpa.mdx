# 配置 VPA



无论是无状态应用或有状态应用，VPA（Vertical Pod Autoscaler，Pod 垂直自动伸缩）均支持根据您的业务需求推荐更合理的 CPU 和内存资源限额，协助您完成 Pod 资源配置，在保证 Pod 有足够使用资源的情况下提高集群资源利用率。



## 前提条件

* 请确保当前集群已部署监控组件，且监控组件运行正常。您可通过单击平台右上角 <span style={{ display: 'inline-flex', alignItems: 'center', gap: '4px', verticalAlign: 'middle' }}><img src="./assets/question.png" alt="expand" style={{ verticalAlign: 'middle' }} /> <span>  > **平台健康状态**， 查看监控组件部署和健康状况。</span></span>。
* 请确保管理员已为当前集群部署 **垂直自动伸缩** 插件。

## 操作步骤

1. 在左侧导航栏中，单击 **计算组件** > **部署**。

2. 单击 ***部署的名称*** 。

3. 下拉至 **弹性伸缩** 区域，单击右侧的 **更新**。

4. 选择 **垂直伸缩**，并配置伸缩规则。

    |参数| 说明                                                         |
    | -------- | ------------------------------------------------------------ |
    |**伸缩模式**| 当前支持 **手动伸缩** 模式，即通过分析过往资源用量给出推荐的资源配置。您可以根据推荐值手动进行调整，调整时会导致 Pod 的重建和重启，请选择合适的时间以免对正在运行的业务产生影响。<br/>通常在实例运行超过 8 天后，推荐值会趋于精准。 |
    |**目标容器**| 默认为计算组件的第一个容器，您可以按需选择为一个或多个容器开启资源限额推荐。 |

6. 单击 **更新**。

## 后续操作

配置完成后在 **弹性伸缩** 区域可查看目标容器的 CPU 和内存资源限额 **推荐值**。
在 **容器** 区域选择目标容器页签，单击 **资源限额** 右侧图标，即可根据推荐值更新资源限额。
