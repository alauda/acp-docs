# 创建网关并添加监听

## 创建网关

创建一个入站网关，以实现更加细粒度的网络资源分配。

### 前提条件

平台管理员必须确保集群支持 **LoadBalancer** 类型的 **内部路由**。对于公有云集群，必须安装 LoadBalancer Service Controller。在非公有云集群中，平台提供了 **外部地址池** 功能，配置完成后，**LoadBalancer** 类型的 **内部路由** 可以自动从外部地址池中获取 IP 以供外部访问。

### 操作步骤

**注意**：入站网关创建后，平台会自动创建许多资源，请勿删除[平台创建的资源](./view_resource.mdx)。

1. 进入 **容器平台**。

2. 在左侧导航栏中，单击 **网络** > **入站网关**。

3. 单击 **创建入站网关**。

4. 参考以下说明，配置特定参数。

   | 参数                          | 描述                                                                                                                                                                                                                                                                                                            |
   | ----------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
   | **名称**                     | 入站网关的名称。                                                                                                                                                                                                                                                                                                 |
   | **网关类**                   | 网关类定义了网关的行为，类似于存储类（StorageClasses）的概念；它是集群资源。<br />**独享型**：入站网关将对应特定的资源实例，用户可以使用该网关的所有监听和计算资源。                                                                                                                                      |
   | **规格**                     | 可根据需要选择平台推荐的使用场景或自定义资源限制。                                                                                                                                                                                                                                                                 |
   | **访问地址**                 | 入站网关的地址，默认自动获取。                                                                                                                                                                                                                                                                                  |
   | **内部路由注解**             | 用于声明 LoadBalancer 类型内部路由的配置或能力。有关具体的注解信息，请参考 [LoadBalancer 类型内部路由注解说明](../service/annotation.mdx)。                                                                                                                      |

5. 单击 **创建**。

## 添加监听

监听指定域名下的流量，并根据绑定的路由规则转发至后端实例。

### 前提条件

- 如果需要监听 HTTP 协议，请提前联系管理员准备 **域名**。

- 如果需要监听 HTTPS 协议，请提前联系管理员准备 **域名** 和 **证书**。

### 操作步骤

1. 在左侧导航栏中，单击 **网络** > **入站网关**。

2. 单击 ***入站网关名称***。

3. 单击 **添加监听**。

4. 参考以下说明，配置特定参数。

   | 参数                          | 描述                                                                                                                                                                                                                                                                                                                                                                          |
   | ----------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | **监听协议及端口**            | 目前支持监听 HTTP、HTTPS、TCP和UDP协议，您可以自定义输入需要监听的端口，例如：`80`。<br /><br />**注意**：<br /><ul><li>当端口相同时，HTTP、HTTPS和TCP监听类型不能共存；仅能选择其中一种协议。</li><li>使用 HTTP 或 HTTPS 协议时，如端口相同，域名必须不同。</li></ul>                                                   |
   | **域名**                     | 从当前命名空间中选择可用的域名，用于监听访问该域名的网络流量。<br />**提示**：TCP和UDP协议不支持选择域名。                                                                                                                                                                                                                                                                    |

5. 单击 **创建**。

## 后续操作

监听创建完成后，请参考文档 [创建路由规则](./create_rule.mdx)。
