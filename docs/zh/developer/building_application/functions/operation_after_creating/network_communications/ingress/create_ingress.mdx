# 创建 Ingress

创建入站规则（Ingress），以允许从集群外部访问计算组件的 Pods。

**注意**：在同一命名空间下创建多个入站规则时，不同的入站规则必须保证 **域名**、**访问协议**和 **路径** 均不同（即不允许重复的访问地址）。

## 前提条件

- 当前命名空间下必须存在可用的 **内部路由**。

- 请确认与管理员已为当前命名空间关联的项目分配了可用域名。

- 如果域名需要通过 HTTPS 协议访问，则必须提前将 HTTPS 证书凭据以保密字典（TLS 类型）的形式保存，并选择使用该证书。

  **说明**：当使用负载均衡器作为 Ingress Controller 时，支持使用平台分配的证书。

## 操作步骤

1. 访问 **Container Platform**。

2. 在左侧导航栏中，单击 **网络** > **入站规则**。

3. 单击 **创建入站规则**。

4. 根据以下说明配置某些参数。

   | 参数                       | 说明                                                                                                                                                                                                                                                                                                                                                                                                                        |
   | ------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | **入站规则类**             | 入站规则类定义了一种入站规则能力的模板，使用同一入站规则类的入站规则具有相同的能力。如果您在 **表单** 界面中不使用入站规则类功能，请参阅 [同步规则](./ingress_rule.mdx)；如需在 **表单** 界面选择已创建的入站规则类，请联系管理员启用相关功能。                                                                                                                |
   | **域名**                   | 支持添加 **泛域名** 和 **精确域名**；如果同一优先级下同时存在泛域名和精确域名配置，则精确域名的转发规则将优先生效。                                                                                                                                                                                                                                                                            |
   | **证书**                   | 如果选择了公有云入站规则类，则 **平台分配** 的证书不可用，仅支持通过 **TLS 类型保密字典** 导入的证书。                                                                                                                                                                                                                                                                                                     |
   | **匹配方式** 和 **路径**   | <ul><li>**前缀**：匹配路径前缀，例如 `/abcd` 可以匹配 `/abcd/efg` 或 `/abcde`。</li><li>**精确**：匹配精确路径，例如 `/abcd`。</li><li>**控制器决定**：如果您使用自定义的入站控制器来管理入站规则，您可以选择由控制器来决定。</li></ul>                                                                                                                                                                     |
   | **内部路由**               | 访问流量将转发至由内部路由指定的计算组件。                                                                                                                                                                                                                                                                                                                                                                                       |
   | **服务端口**               | 指定内部路由流量将从哪个服务端口进行转发。                                                                                                                                                                                                                                                                                                                                                                                            |

5. 单击 **创建**。
