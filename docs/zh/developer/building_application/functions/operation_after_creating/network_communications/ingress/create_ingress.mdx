# 创建 Ingress

创建入站规则（Ingress），实现从集群外部访问计算组件的容器组。

**注意**：在同一命名空间下创建多个入站规则时，不同的入站规则中不允许存在 **域名**、**访问协议**、**路径** 均相同的规则（即重复的访问地址）。

## 前提条件

* 当前命名空间下已有可用 **内部路由**。

* 请与管理员确认：已为当前命名空间所在的项目分配可用域名。

* 如果域名需要通过 HTTPS 协议进行访问时，需要提前将 HTTPS 证书凭据以保密字典（TLS 类型）的形式保存并选择使用此证书。

    **说明**：当使用负载均衡器作为 Ingress Controller 时，支持使用平台分配的证书。

## 操作步骤

1. 进入 **Container Platform**。

1. 在左侧导航栏中，单击 **网络** > **入站规则**。

2. 单击 **创建入站规则**。

5. 参考以下说明，配置部分参数。
   
    |参数|说明|
    |---|---|
    |**入站规则类**|入站规则类定义了一种入站规则能力的模板，使用同一入站规则类的入站规则能力相同。若不使用 **表单** 界面的入站规则类功能，请参考[同步规则](./ingress_rule.mdx)；若需在 **表单** 界面选择已创建的入站规则类，请联系管理员开启入站规则类相关功能。|
    |**域名**|支持添加 **泛域名** 和 **精确域名**，相同规则优先级情况下，若同时存在泛域名和精确域名规则配置，精确域名的转发规则优先生效。|
    |**证书**|若选择使用公有云入站规则类，则 **平台分配** 的证书不可用，仅支持选择通过 **TLS 类型的保密字典** 导入的证书。|
    |**匹配方式** 和 **路径**|<ul><li>**前缀**：匹配路径前缀，例如 `/abcd` 可以匹配 `/abcd/efg` 或 `/abcde` 。</li><li>**精确**：匹配精确路径，例如 `/abcd` 。</li><li>**控制器决定**：若您使用自定义的入站规则控制器（Ingress Controler）来处理入站规则（Ingress 规则），在该情况下可选择由控制器决定。</li></ul>|
    |**内部路由**|访问流量将转发至内部路由指定的计算组件中。|
    |**服务端口**|指定内部路由流量由哪个服务端口进行转发。|

8. 单击 **创建**。
