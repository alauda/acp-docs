# Ingress 同步规则


* 当入站规则（Ingress）未指定入站规则类（IngressClass），或指定的入站规则类（IngressClass）由负载均衡器生成。

    * 使用 **端口** 资源分配方式创建的负载均衡器，仅当命名空间拥有其 **80** 和 **443** 端口权限，并且创建了 HTTP **80** 和 HTTPS **443** 端口时，负载均衡器才会自动解析入站规则（Ingress）。

    * 使用 **实例** 资源分配方式创建的负载均衡器，当命名空间具有负载均衡器权限时，将自动解析命名空间内的入站规则（Ingress）。

     **说明**：为保证命名空间内的入站规则（Ingress）正常生效，当只使用平台负载均衡器时，请确保命名空间具有负载均衡器权限以解析入站规则（Ingress）。

* 当入站规则（Ingress）指定非负载均衡器生成的入站规则类（IngressClass）。

    平台负载均衡器将不再解析入站规则（Ingress），您必须确保所指定的入站规则类（IngressClass）有可用的 Controller 来解析入站规则（Ingress）。
