# CronJobs

## 理解 CronJobs

请参考官方 Kubernetes 文档：

- [CronJobs](https://kubernetes.io/docs/concepts/workloads/controllers/cron-jobs/)
- [使用 CronJob 运行自动化任务](https://kubernetes.io/docs/tasks/job/automated-tasks-with-cron-jobs/)

**CronJob** 定义了运行到完成后停止的任务。它们允许您根据计划多次运行相同的 Job。

**CronJob** 是 Kubernetes 中的一种工作负载控制器。您可以通过 web 控制台或 CLI 创建 CronJob，以定期或重复运行一个非持久性的程序，例如定期备份、定期清理或定期发送电子邮件。

## 前提条件

获取镜像地址。镜像可以来自由平台管理员通过工具链集成的镜像仓库，或来自第三方镜像仓库。

- 对于来自集成仓库的镜像，管理员通常会将镜像仓库分配给您的项目，允许您使用其中的镜像。如果找不到所需的镜像仓库，请联系管理员进行分配。

- 如果使用第三方镜像仓库，请确保在当前集群中能够直接拉取镜像。

## YAML 文件示例

```yaml
# example-cronjob.yaml
apiVersion: batch/v1
kind: CronJob
metadata:
  name: hello
spec:
  schedule: "* * * * *" 
  jobTemplate:
    spec:
      template:
        spec:
          containers:
          - name: hello
            image: busybox:1.28
            imagePullPolicy: IfNotPresent
            command:
            - /bin/sh
            - -c
            - date; echo Hello from the Kubernetes cluster
          restartPolicy: OnFailure
```

## 使用 CLI 创建 CronJobs

```bash
kubectl apply -f example-cronjob.yaml
```

## 使用 web 控制台创建 CronJobs

### 操作步骤 - 配置基本信息

1. **容器平台**，在左侧边栏中导航至 **工作负载** > **CronJobs**。

2. 点击 **创建 CronJob**。

3. **选择** 或 **输入** 一个镜像，点击 **确认**。

   **注意**：仅在使用本平台集成的镜像仓库中的镜像时可以通过选择的方式来筛选镜像。例如，集成项目名称如 *containers (docker-registry-projectname)* 表示本平台的项目名称 projectname 和镜像仓库的项目名称 containers。

4. 在 **定时配置** 部分，配置任务执行方式及相关参数。

   **执行类型**：

   - **手动**：手动执行需要为每次任务运行明确的手动触发。
   - **定时**：定时执行需要配置以下调度参数：

     | **参数**               | **说明**                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
     | :--------------------- | :----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
     | **触发规则**           | 使用 [Crontab 语法](https://crontab.guru/) 定义 cron 调度。CronJob 控制器根据所选时区计算下次执行时间。 <br /><br />**注意**：<ul><li>对于 Kubernetes 集群 \< v1.25: 不支持时区选择；调度必须使用 UTC。</li><li>对于 Kubernetes 集群 ≥ v1.25: 支持时区调度（默认：用户的本地时区）。</li></ul>                                                                                              |
     | **并发策略**           | 指定如何处理并发 Job 执行（每个 [K8s 规范](https://kubernetes.io/docs/concepts/workloads/controllers/cron-jobs/#concurrency-policy) 可选 `Allow`, `Forbid`, 或 `Replace`）。                                                                                                                                                                                                                                                        |

   **任务历史保留**：

   - 设置完成任务的保留限制：
     - **历史限制**：成功任务历史限制（默认：20）
     - **失败任务**：失败任务历史限制\*\*（默认：20）
   - 当保留限制超过时，将首先垃圾回收最旧的任务。

5. 在 **任务配置** 部分，选择作业类型。
   CronJob 管理由 Pods 组成的任务。根据工作负载类型配置 Job 模板：

   | **参数**        | **说明**                                                                                                                                                                                                  |
   | :-------------- | ----------------------------------------------------------------------------------------------------------------------------------------- |
   | **作业类型**    | 选择作业完成模式（`非并行`、`固定完成次数的并行` 或 `索引作业`，具体可参见 [K8s 作业模式](https://kubernetes.io/docs/concepts/workloads/controllers/job/#job-patterns)）。 |
   | **重试限制**    | 设置标记 Job 为失败之前的最大重试次数。                                                                                                                                                        |

### 操作步骤 - 配置 Pods

- 在 **Pod** 部分，请参阅 [创建部署 - 配置 Pods](./deployment_create.mdx#configurepod)

### 操作步骤 - 配置容器

- 在 **容器** 部分，请参阅 [创建部署 - 配置容器](./deployment_create.mdx#configurecontainers)

- 点击 **创建**。

## 立即执行

### 定位 CronJob 资源

- **web 控制台**：**容器平台**，在左侧边栏中导航至 **工作负载** > **CronJobs**。
- **CLI**：
  ```bash
  kubectl get cronjobs -n <namespace>
  ```

### 发起临时执行

- **web 控制台**：**立即执行**
  1. 点击应用列表右侧的垂直省略号（⋮）。
  2. 点击 **立即执行**。 (或者，从 CronJob 详情页面，点击右上角的操作并选择 **立即执行**).
- **CLI**：
  ```bash
  kubectl create job --from=cronjob/<cronjob-name> <job-name> -n <namespace>
  ```

### 验证 Job 详情：

```bash
kubectl describe job/<job-name> -n <namespace>
kubectl logs job/<job-name> -n <namespace>
```

### 监控执行状态

| 状态          | 说明                                                   |
| ------------- | ----------------------------------------------------- |
| **等待中**    | Job 已创建但尚未调度                                   |
| **运行中**    | Job Pod(s) 正在积极执行                               |
| **成功**      | 所有 Pods 成功完成（退出代码 0）                       |
| **失败**      | 至少有一个 Pod 终止失败（非零退出代码）                |

## 删除 CronJobs

### 使用 web 控制台删除 CronJobs

1. **容器平台**，在左侧边栏中导航至 **工作负载** > **CronJobs**。
2. 找到您希望删除的 CronJobs。
3. 在 **操作** 下拉菜单中，点击 **删除** 按钮并确认。

### 使用 CLI 删除 CronJobs

```bash
kubectl delete cronjob <cronjob-name>
```
