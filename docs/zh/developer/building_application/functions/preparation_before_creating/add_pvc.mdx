# 创建 PVC

创建持久卷声明（PersistentVolumeClaim，PVC），并根据使用需要设置请求持久卷（PersistentVolume，PV）的参数。

您可以通过填写可视化 UI 表单或使用自定义 YAML 编排文件来创建持久卷声明。

## 前提条件

确保命名空间中剩余的 **存储** 配额足以满足本次创建操作所请求的存储大小。

## 操作步骤

1. 进入 **Container Platform**。

2. 在左侧导航栏中，单击 **存储 > 持久卷声明(PVC)**。

3. 单击 **创建持久卷声明**。

4. 按要求配置参数。

   **说明**：下述内容以表单方式为例，您也可切换至 YAML 方式完成操作。

   | 参数              | 说明                                                                                                                                                                                                                                                                                                                                                                                                                                |
   | ------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
   | **名称**           | 持久卷声明的名称，必须在当前命名空间中唯一。                                                                                                                                                                                                                                                                                                                                                                                          |
   | **创建方式**      | - 动态创建：基于存储类动态生成持久卷并匹配绑定。 <br />- 静态匹配：基于配置参数与已有持久卷匹配绑定。                                                                                                                                                                                                                                                                                                                                                                                         |
   | **存储类**        | 选择动态创建方式后，平台将按指定存储类中的描述，动态地创建持久卷。                                                                                                                                                                                                                                                                                                                                                                          |
   | **访问模式**      | - 单节点读写 (ReadWriteOnce，RWO)：可以被一个节点以读写方式挂载。 <br />- 多节点读写 (ReadWriteMany，RWX)：可以被多个节点以读写方式挂载。 <br />- 多节点只读 (ReadOnlyMany，ROX)：可以被多个节点以只读方式挂载。 <br /><br />**提示**：建议根据实际情况考虑计划绑定当前持久卷声明的计算组件实例数量及部署控制器类型。例如，创建多实例的部署（Deployment）时，由于所有实例都使用同一个持久卷声明，不建议选择只能在单个节点上挂载的 RWO 访问模式。|
   | **容量**          | 待请求的持久卷大小。                                                                                                                                                                                                                                                                                                                                                                                                                |
   | **卷模式**        | - 文件系统：将持久卷作为文件目录挂载到 Pods 中。该模式适用于任何类型的计算组件。 <br />- 块设备：将持久卷作为原始块设备挂载到 Pods 中。该模式仅适用于虚拟机。                                                                                                                                                                                                                                                                                                        |
   | **更多**          | - 标签 <br />- 注解 <br />- 选择器：选择静态匹配方式后，您可以使用选择器定向匹配标记了特定标签的持久卷。持久卷标签可用于标记存储的特殊属性，例如磁盘类型或地理位置。                                                                                                                                                                                                                          |

5. 单击 **创建**。等待持久卷声明变为 `已绑定` 状态，说明持久卷匹配成功。

## 相关操作

- **绑定持久卷声明**：在创建需要持久化数据存储的应用或计算组件时，绑定持久卷声明以请求符合要求的持久卷。

- **使用卷快照创建持久卷声明**：这有助于备份应用数据并按需还原，确保业务应用数据的可靠性。请参考 [使用卷快照](../operation_after_creating/snapshot.mdx)。

- **删除持久卷声明**：您可以在详情页面单击右上角的 **操作** 按钮来删除持久卷声明。删除之前，请确保持久卷声明未与任何应用或计算组件绑定，并且其中未包含任何卷快照。删除持久卷声明后，平台将根据回收策略处理持久卷，这可能会清除持久卷中的数据并释放存储资源。请根据数据安全考虑谨慎操作。
