# 创建 PVC

创建持久卷声明（PersistentVolumeClaim，PVC），并根据使用需要设置请求持久卷（PersistentVolume，PV）的参数。

支持通过填写可视化 UI 表单的方式，或自定义 YAML 编排文件的方式，创建持久卷声明。


## 前提条件

命名空间中剩余的 **存储** 配额，足够满足本次创建操作所请求的存储大小。

## 操作步骤

1. 进入 **Container Platform**。

1. 在左侧导航栏中，单击 **存储 > 持久卷声明(PVC)**。

2. 单击 **创建持久卷声明**。

2. 按要求配置参数。

    **说明**：下述内容以表单方式为例，您也可切换至 YAML 方式完成操作。

    |参数|说明                                                         |
    | -------- | ----------------  | 
    | **名称**   | 持久卷声明的名称。需在当前命名空间中唯一。 |
    | **创建方式**   | - 动态创建：基于存储类动态生成持久卷并匹配绑定。<br />- 静态匹配：基于配置参数与已有持久卷匹配绑定。|
    | **存储类**   | 选择动态创建方式后，平台将按指定存储类中的描述，动态地创建持久卷。 |
    | **访问模式**   | - 单节点读写 (ReadWriteOnce，RWO)：可以被一个节点以读写方式挂载。<br />- 多节点读写 (ReadWriteMany，RWX)：可以被多个节点以读写方式挂载。<br />- 多节点只读 (ReadOnlyMany，ROX)：可以被多个节点以只读方式挂载。<br /><br />**提示**：建议根据实际情况，关注计划绑定当前持久卷声明的计算组件实例数量，以及部署控制器类型。例如：创建多实例的部署（Deployment）时，由于所有实例都使用同一个持久卷声明，不建议选择只能在单个节点上挂载的 RWO 访问模式。|
    | **容量**   | 待请求的持久卷大小。|
    | **卷模式**   | - 文件系统：将以文件目录方式，将绑定的持久卷挂载到容器组中。此模式可供任意类型的计算组件使用。<br />- 块设备：以原始块设备方式，将绑定的持久卷挂载到容器组中。此模式仅可供虚拟机使用。|
    | **更多**   | - 标签（labels）<br />- 注解（annotations）<br />- 选择器（selector）：选择静态匹配方式后，可通过选择器定向匹配标记了特定标签的持久卷。持久卷标签可用于标记存储的特殊属性，例如：存储磁盘类型、存储所在地域等。|

5. 单击 **创建**。等待持久卷声明变为 `已绑定` 状态，说明持久卷匹配成功。

## 相关操作 

* **绑定持久卷声明**：在后续创建需要持久化数据存储的应用、计算组件时，绑定持久卷声明，用于请求符合要求的持久卷。

* **使用卷快照创建持久卷声明**：可达到备份应用数据并按需还原的效果，从而保障业务应用数据的可靠性，请参考 [使用卷快照](../operation_after_creating/snapshot.mdx)。

* **删除持久卷声明**：您可以在详情页面单击右上角的 **操作**，按需删除持久卷声明。删除前请确保持久卷声明未与任何应用或计算组件绑定，且持久卷声明中未包含卷快照。删除持久卷声明后，平台将根据持久卷回收策略处理持久卷，可能会清除持久卷中数据，并释放存储资源，请在基于数据安全的前提下谨慎操作。
