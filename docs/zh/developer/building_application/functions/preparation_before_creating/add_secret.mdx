# 配置 Secret

根据使用需要创建保密字典（Secret），选择适合的保密字典类型并配置参数。

## 操作步骤

1. 进入 **Container Platform**。

1. 在左侧导航栏中，单击 **配置** > **保密字典**。 

2. 单击 **创建保密字典**。


3. 配置参数。
    
    **提示**：在表单视图中，对于输入的用户名、密码等敏感数据，将自动经过 Base64 编码格式转换再储存到保密字典中，转换后的数据可在 YAML 视图中预览。



8. 单击 **创建**。

## 后续操作

在同一命名空间中，为原生应用创建计算组件时，可以引用已创建的保密字典。 

## 相关操作

您可以在列表页面单击右侧的 ⋮ 或在详情页面单击右上角的 **操作**，按需更新或删除保密字典。

|操作| 说明                                                         |
| --------------------- | ------------------------------------------------------------ |
|**更新**| 添加或更新保密字典后，对于已经通过环境变量引用这个保密字典（或其中配置项）的计算组件，需要重建 Pod 才会应用新配置。|
|**删除**| <ul><li>删除保密字典后，对于已经通过环境变量引用这个保密字典（或其中配置项）的计算组件，如果重建 Pod，可能会因找不到引用源而影响 Pod 正常创建。</li><li>请不要删除平台自动生成的保密字典，否则将导致平台功能无法正常使用。例如：类型为 service-account-token 且包含命名空间资源的认证信息的保密字典，以及系统命名空间 （例如 kube-system）中的保密字典。</li></ul> |
