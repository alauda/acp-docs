# Pod 页面参数

平台界面提供了容器组的各类信息，以供快捷查找。以下是部分参数解释。

|参数| 说明                                                         |
| --------------- | ------------------------------------------------------------ |
|**资源请求/限制**| 容器组内容器有效的资源（CPU、内存）请求与限制值。请求与限制值的计算方式相同，文档以限制值为例进行介绍，具体的规则及算法如下：<ul><li>当容器组内仅包含 **业务容器**（containers）时，**CPU/内存** 的限制值为容器组内所有容器的 **CPU/内存** 限制值的总和。**例如**：容器组内包括两个业务容器，**CPU/内存** 的限制值分别为 100m/100Mi、50m/200Mi，则容器组的 **CPU/内存** 限制值为 150m/300Mi。</li><li>当容器组内既有 **初始化容器**（initContainers)又有 **业务容器** 时，容器组的 **CPU/内存** 限制值计算步骤如下： <ol><li>取所有初始化容器 **CPU/内存** 限制值的最大值。</li> <li>取所有业务容器 **CPU/内存** 限制值的总和。</li><li>将结果进行比较，分别取初始化容器和业务容器二者中 **CPU**、**内存** 的最大值作为容器组的 **CPU/内存** 限制值。 </li></ol></li><br/>**计算示例**：容器组内包含两个初始化容器，**CPU/内存** 的限制值分别为 100m/200Mi、200m/100Mi，则初始化容器的 **CPU/内存** 最大限制值为 200m/200Mi。同时，容器组内还包含两个业务容器，**CPU/内存** 的限制值分别为 100m/100Mi、50m/200Mi，则业务容器的 **CPU/内存** 限制值总和为 150m/300Mi。综合得出容器组的 **CPU/内存** 限制值为 200m/300Mi。</ul> |
|**来源**| 容器组所属的计算组件。 |
|**重启次数**| 容器组状态异常时，重启的次数。                               |
|**节点**| 容器组所在节点的名称。                                       |
|**Service Account**| Service Account 是 Pod 里的进程和服务访问 Kubernetes APIServer 的一个账号，为进程和服务提供了一种身份标识。仅当当前登录用户拥有平台管理员角色、平台审计人员角色时，**Service Account** 字段可见，且可查看 Service Account 的 YAML 文件。 |
