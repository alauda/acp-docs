# 删除 Pod

删除容器组可能会影响计算组件的运行，请谨慎操作。


## 使用场景

* 使容器组尽快恢复到期望状态：若容器组持续处于 `Pending` 或 `CrashLoopBackOff` 等影响业务运行的状态，根据报错信息处理后，手动删除容器组可使其尽快恢复到期望状态，例如 `Running` 。此时，被删除的容器组将在当前节点重建或被重新调度。


* 资源清理以便运维：某些容器组到达一定阶段后便不再变化，且这些容器组通常会大量存在，给其他容器组的运维带来不便。待清理的容器组可以是因节点资源不足而被批量驱逐的 `Evicted` 状态容器组，或者是定时任务持续触发的任务对应的 `Completed` 状态容器组。此时，被删除的容器组将不复存在。

    **注意**：对于定时任务，如果您需要查看每次任务的执行日志，不建议删除对应的 `Completed` 状态容器组。
    



## 操作步骤

1. 进入 **Container Platform**。

1. 在左侧导航栏中，单击 **计算组件** > **容器组**。

2. （逐个删除）单击待删除容器组右侧的 ⋮ > **删除**，并确定。
3. （批量删除）选择待删除容器组，在列表上方单击 **删除**，并确定。
