# 通过 YAML 创建原生应用



如果您熟悉 YAML 语法且希望使用表单或模板中未定义的配置，可选择 YAML 一键创建方式，更为灵活地配置原生应用的基本信息及资源。


## 注意事项

如果集群中同时存在 Linux 和 Windows 两类节点，为避免应用被调度到不适配的节点上，需设置主机选择。例如：

```yaml
spec:
    spec:
      nodeSelector:
        kubernetes.io/os: linux
```

## 前提条件

确保在当前集群可拉取到 YAML 中所定义镜像：执行 `docker pull` 命令验证无误。


## 操作步骤

1. 进入 **Container Platform**。

1. 在左侧导航栏中，单击 **应用管理** > **原生应用**。

3. 单击 **创建原生应用**。

2. 选择 **YAML 一键创建** 方式。
3. 完成配置，并单击 **确定**。
