# 创建 OAM 应用

**OAM 应用** 基于开放应用模型（Open Application Model），这是一种对容器和 Kubernetes 资源的更高层次的整合与抽象，包含多个组件在一个 OAM 应用下。OAM 应用使您能够在更高的层面上管理应用，尽可能地将其与容器和 Kubernetes 层隔离，从而让您更加专注于应用自身的运维。

## 使用表单方式创建 OAM 应用

### 前提条件

1. 安装 **ACP 应用管理与 KubeVela**。
2. 获取镜像地址。镜像来源可以是管理员通过工具链集成的镜像仓库，或来自第三方平台的镜像仓库。
   - 对于前者，管理员通常会将镜像仓库分配给您的项目，您可以使用其中的镜像。如果未找到所需的镜像仓库，请联系管理员进行分配。
   - 如果是第三方平台的镜像仓库，需确保在当前集群可以直接从中拉取镜像。

### 操作步骤

1. 进入 **Container Platform**，并导航到 **应用管理** > **OAM 应用**。
2. 单击 **创建应用**。
3. 单击 **从镜像创建**。
4. 填写应用名称，并设置标签或注解。

   | **参数**   | **说明**                                                                                                                                                                                                                                    |
   | :--------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
   | **标签**   | 用于选择对象和查找满足某些条件的对象集合。需为键值对，例如 `app.kubernetes.io/name: {hello-app}`。                                                                                         |
   | **注解**   | 用于向开发和运维团队提供任何信息。需为键值对，例如 `cpaas.io/maintainer: kim`。注意：如果注解值是可点击链接，则将键设为 `app.{文本}.url`。 |

5. 单击 **添加组件**。
6. 选择组件类型并单击 **下一步**。
7. 继续配置属性，添加运维特征等。

   - [组件概念](/developer/oam_applications/concepts/component.mdx)

   - [特征概念](/developer/oam_applications/concepts/trait.mdx)

   **说明**：您可以即时配置运维特征，也可以等环境就绪后再进行配置。

## 使用 YAML 方式创建 OAM 应用

如果您已熟练掌握创建应用，或希望在集群内快速迁移应用，可以使用 YAML 文件一键完成所需组件和运维特征的配置。除了平台提供的 YAML 示例外，您还可以访问 [OAM 开源社区](https://github.com/oam-dev/spec/blob/master/7.application.md)获取更多关于编写应用 YAML 的指导。

### 前提条件

- 如果是第三方平台的镜像仓库，请确保在当前集群可以直接从中拉取镜像。执行 `docker pull` 命令进行验证。

### 使用限制

如果您是从现有的 OAM 应用导出的 YAML 文件，请修改 YAML 中的应用名称和组件名称，以避免在同一命名空间中发生名称冲突。

### 操作步骤

1. 进入 **Container Platform**，并导航到 **应用管理** > **OAM 应用**。
2. 单击 **创建应用**。
3. 单击 **从 YAML 创建**。
4. 填入 YAML 内容并单击 **创建**。

   创建完成后，您可以通过网页控制台正常更新应用或组件。
