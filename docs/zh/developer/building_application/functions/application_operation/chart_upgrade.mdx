# 模板应用的升级与删除

由于当前模板应用与原生应用的功能存在重叠，且原生应用下的模板应用拥有更丰富的运维能力，因此未来版本将不再提供独立的模版应用管理能力，请尽快将当前部署成功的模板应用升级为原生应用。

## 注意事项

此功能 **即将下线**，请尽快将当前部署成功的模板应用升级为原生应用。

## 前提条件

请联系平台管理员开启模板应用相关功能。

## 将模板应用升级为原生应用

可通过 **升级为原生应用** 或 **更新** 操作，将模板应用升级为原生应用。若更新或升级为原生应用过程出现错误，请根据页面错误信息进行调整并重新操作。

### 升级为原生应用

**操作步骤**

1. 进入 **Container Platform**。

1. 在左侧导航栏中，单击 **应用管理** > **模板应用**。

3. 单击 ⋮ > **升级为原生应用**，并确定。平台将跳转到 **应用管理** > **原生应用** 页面，可使用更新模板版本、拓扑、版本快照、日志、监控、告警等功能，后续仅需管理升级后的 [原生应用]。

### 更新

更新模板应用的同时会自动将模版应用升级为原生应用，升级为后可使用更新模板版本、拓扑、版本快照、日志、监控、告警等功能。

**操作步骤**

1. 进入 **Container Platform**。

1. 在左侧导航栏中，单击 **应用管理** > **模板应用**。

3. 单击 ⋮ > **更新**。
4. 修改配置。
5. 单击 **更新**。


## 删除模板应用

**操作步骤**



1. 进入 **Container Platform**。

1. 在左侧导航栏中，单击 **应用管理** > **模板应用**。

3. 单击 ⋮ > **删除**，并确定。
    



## 状态分析说明

单击 ***模板应用名称***，在详情信息中可展示 Chart 的详细部署状态分析。

|类型| 原因                                                         |
| --------------------- | ------------------------------------------------------------ |
|**Initialized**| 表示 Chart 模板下载的状态。<ul><li>状态为 True 时表示 Chart 模板下载成功。</li><li>状态为 False 时表示 Chart 模板下载失败，消息列可查看具体的失败原因。</li><ul><li>ChartLoadFailed：Chart 模板下载失败。</li><li>InitializeFailed：还未下载 Chart 之前的初始化行为存在异常。 </li></ul></ul> |
|**Validated**| 表示 Chart 模板用户权限、依赖等验证的状态。<ul><li>状态为 True 时表示所有验证检查均已通过。</li><li>状态为 False 时表示存在验证检查未通过，消息列可查看具体的失败原因。</li><ul><li>DependenciesCheckFailed：Chart 依赖检查失败。</li><li>PermissionCheckFailed：当前用户缺少对某些资源的操作权限。</li><li>ConsistentNamespaceCheckFailed：通过原生应用部署模板应用时，Chart 中携带了需跨命名空间部署的资源。</li></ul></ul> |
|**Synced**| 表示 Chart 模板部署的状态。<ul><li>状态为 True 时表示 Chart 模板部署成功。</li><li>状态为 False 时表示 Chart 模板部署失败，原因列显示为 ChartSyncFailed，消息列可查看具体的失败原因。 </li></ul>  |
