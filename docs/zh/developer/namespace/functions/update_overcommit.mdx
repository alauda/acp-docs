# 更新超售比

更新已创建命名空间的超售比。您可以参考当前命名空间中 CPU 和内存的资源需求，为命名空间单独设置超售比，从而将本命名空间内用户设置的容器 CPU 和内存的限制值（limits）以及请求值（requests）限制在合理范围内，以提高计算资源的利用率。

## 注意事项

在启用了节点虚拟化的集群中，如果需要配置虚拟机的超售比，必须首先关闭集群或命名空间的超售比。

## 操作步骤

1. 进入 **项目管理** 视图。

2. 在左侧导航栏中，单击 **命名空间** > **命名空间**。

3. 单击要更新的 ***命名空间名称***。

4. 单击右上角的 **操作** > **更新超售比**。

5. 选择合适的超售比 **配置方式** 来设置命名空间的 CPU 或内存超售比。

| 参数                | 说明                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| ------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **从集群继承** | <ul><li>命名空间的超售比默认 **从集群继承**，**数值** 显示源集群的超售比设置。例如，如果集群的 CPU 和内存超售比均设置为 4，则当前命名空间的 CPU 和内存超售比默认都为 4。</li><br /><li>创建应用时，容器的请求值固定为 **指定的限制值/集群超售比**。如果未指定限制值，限制值将设置为命名空间中容器配额的 **默认值**。</li><br /><li>示例：假设集群 CPU 资源超售比设置为 4，命名空间超售比继承自集群。当用户创建应用并指定 CPU 限制值为 4c 时，CPU 请求值将为 4c/4 = 1c。</li></ul>                                                                                                                                                                                                                                         |
| **自定义**               | <ul><li>命名空间的超售比 **数值** 由管理员自定义设置，仅支持大于 1 的正整数；如果留空，则表示当前命名空间没有设置超售比。<br />例如，如果集群的 CPU 和内存超售比均为 4，而此处的自定义值均设置为 2，则当前命名空间的 CPU 和内存超售比均为 2；如果未提供自定义值，则表示当前命名空间的 CPU 和内存没有设置超售比。</li><br /><li>创建应用时，容器的请求值固定为 **指定的限制值/命名空间超售比**。如果未指定限制值，限制值将设置为命名空间中容器配额的 **默认值**。</li><br /><li>示例：假设集群 CPU 资源超售比设置为 4，而命名空间超售比自定义为 2。当用户创建应用时指定 CPU 限制值为 4c，CPU 请求值将为 4c/2 = 2c。</li></ul> |

5. 单击 **更新**。

**注意**：更新超售比后，不会影响当前运行的容器的请求值。然而，在 Pod 重新创建时，容器的请求值将根据更新后的超售比重新计算。
