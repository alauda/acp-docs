# 导入命名空间


通过在项目下导入命名空间，能够灵活纳管平台上任意集群下的 Kubernetes 命名空间。配合 **移除命名空间** 功能，能够重新设置命名空间和项目的从属关系。

导入项目的命名空间将能为您提供和平台上创建的命名空间等同的服务。

**注意**：一个命名空间只属于一个项目，不支持将已属于某个项目的命名空间导入至其他项目。

## 使用场景


常见的使用场景如下：

* 当新的集群接入平台后，支持通过导入命名空间功能，将集群下已有的 Kubernetes 命名空间逐一导入到平台的项目下进行托管。操作流程如下图所示。

    <img src="./assets/nsimport.svg" width = "300" />
    
    **参考内容**：[创建项目]()、[添加项目关联集群]()。

* 已被移除出项目的命名空间，通过导入命名空间功能，可将其导入到其他项目下管理。

* 通过其他方式为平台上的集群创建的未关联项目的命名空间（例如：在集群上通过脚本创建），需要通过导入命名空间功能归属于具体的项目，才可通过平台查看和管理。




## 前提条件

* 该命名空间不属于平台上任一一个项目。

* 只能将命名空间导入已关联了 **命名空间所属集群** 的项目。若平台上不存在 **命名空间所属集群** 已关联的项目，您需要先创建一个满足该条件的项目。


## 操作步骤

1. 在 **项目管理** 视图，单击待导入命名空间所在的 ***项目名称***。

2. 在左侧导航栏中，单击 **命名空间** > **命名空间**。

3. 单击 **创建命名空间** 按钮右侧的  <span style={{ display: 'inline-flex', alignItems: 'center', gap: '4px', verticalAlign: 'middle' }}><img src="./assets/addoperation.png" alt="expand" style={{ verticalAlign: 'middle' }} /> <span> > **导入命名空间**</span></span>。

4. 参考 [创建命名空间](./create_namespace.mdx) 参数说明配置相关参数。
    
8. 单击 **导入**。
