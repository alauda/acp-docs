# 安装 Alauda Container Platform (ACP) Builds

## 前提条件
> Alauda Container Platform (ACP) Builds（以下简称 ACP Builds），是 Alauda Container Platform (ACP) 提供的一款集合构建（可实现 Source to Image）和部署于一身的容器工具。

1. 下载与平台匹配的 ACP Builds 最新版本安装包，如平台未部署过 **Tekton Operator** 建议一同下载。
2. 通过平台应用包管理能力，上架 ACP Builds、Tekton 安装包至平台，并将其分发至需要使用 Source to Image 能力的所有集群。[应用包管理能力](https://product-doc-guide.alauda.cn/template/application/120_s2i/install/install_build.html)

## 操作步骤

### 部署 ACP Builds Operator
1. 登录平台，进入 **平台管理** 页面。
2. 左侧导航栏选择 **应用商店管理** -> **Operator**， 进入 **Operator Hub** 页面。
3. 找到 **ACP Builds Operator**，点击 **部署**， 进入 **部署 ACP Builds Operator** 页面。

 配置参数:
    |**参数**| **推荐配置** |
    |:--------|:------------|
    | **升级策略** | `手动`：Operator Hub 有新版本时，不会执行 **升级** 行为，需手动执行 **升级** 操作，将 Operator 至新版本。|
    | **部署模式** | `集群`：集群下所有命名空间共用一个 Operator 对实例进行创建和管理，资源占用更低。|
    | **频道**    | `alpha`：默认 Channel 为 **alpha** 。|
    | **命名空间** | `推荐命名空间`：推荐 **shipyard-operator** 命名空间，无则自动创建。|

4. 在 **部署 Operator** 页面，选择 **默认配置**， 点击 **部署**，完成 ACP Builds Operator 部署。

### 部署 Shipyard 实例

1. 在左侧导航栏中，单击 **应用商店管理** -> **Operators**。

2. 在 **已部署 Operators** 页签中，单击 **shipyard-operator**（显示名称为 **Alauda Container Platform (ACP) Builds**）。

3. 在详情信息栏下，单击资源栏中 **Shipyard** 实例卡片中的 **创建实例**。

4. 在配置实例参数页面，若无特殊要求使用默认配置即可。

5. 单击 **创建**。

### 验证
- 实例部署成功后，预计等待 “20mins” 分钟左右，切换至 “Container Platform” → 应用管理 → 原生应用 → 创建
- 可看到 **通过代码创建** 入口，此时 Alauda Builds 的部署成功，[通过代码原生应用](../how_to/s2i_howto.mdx) 开始您的 Source to Image 之旅吧。
