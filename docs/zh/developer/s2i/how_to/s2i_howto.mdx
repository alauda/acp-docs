# 通过代码创建一个原生应用

> 使用平台提供的 Alauda Container Platform (ACP) Builds 产品，实现基于客户已有的业务应用源代码，部署一个容器应用，并使用平台 **原生应用** 进行封装，实现应用的全生命周期管理。

## 使用场景

> 用户仅有业务应用源代码，并通过代码仓库进行管理，当前希望将业务应用进行容器化改造，上云时，可借助平台 S2I 能力，实现从代码到应用的完整过程。

## 前置条件

使用功能前，需确保：
- 完成 [Alauda Container Platform (ACP) Builds 安装](../install_builds/install_builds_opeator.mdx)
- 确认已拥有 **通过代码创建** 相关权限
- 有平台可访问镜像仓库，如无可联系平台管理员 [部署 Alauda Container Platform (ACP) Registry](../../image/install_registry.mdx)

## 操作步骤

1. 在左侧导航栏中，单击 **应用管理** -> **原生应用**。
2. 单击 **创建**。
3. 选择 **通过代码创建** 方式。
4. 按照下述参数说明完成配置：
    |**参数**| **推荐配置** |
    |:--------|:------------|
    | **代码仓库** | 类型：`输入`<br/> 仓库地址：`https://github.com/alauda/spring-boot-hello-world`<br/> |
    | **构建方式** | `构建`|
    | **镜像仓库**    | `alpha`|
    | **应用** | 应用：`spring-boot-hello-world`<br/> 名称：`spring-boot-hello-world` <br/> 资源规格：`可默认不填写` |
    | **网络** | 目标端口：`8080`|

5. 参数填写完成后单击 **创建**。
6. 可在 **详情信息** 页面查看相应的部署情况。
