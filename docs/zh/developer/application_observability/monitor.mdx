# 监控
- 支持查看平台上的计算组件近 ***7 天*** 的资源监控数据（可配置监控存储数据时常），包括应用、计算组件、容器组等资源的统计数据以及 CPU、内存等资源的用量趋势和排行统计数据。
- 支持基于集群
- 支持查看监控数据的计算组件包括 **应用**、**部署**、**守护进程集**、**有状态副本集**，**容器组**，本文将以部署为例，进行说明。

## 命名空间级别监控

1. 在左侧导航栏中，单击 **可观测** > **监控面板**。

2. 可查看命名空间下监控数据，平台提供了基于 **应用**、**计算组件**、**容器组** 三个监控面板。

3. 可切换至任意面板实现关注资源的监控。

## 创建监控面板并添加至命名空间级别监控

1. 平台管理下，参照 [创建监控面板](../../observability/monitor/functions/manage_dashboard.mdx) 创建专属关注监控面板。
2. 调整如下 Label 信息，实现监控面板在容器平台展示：
    - `cpaas.io/dashboard.folder: container-platform`
    - `cpaas.io/dashboard.tag.overview: "true"`

## 计算组件下监控

> 以通过部署页面查看容器的实时日志为例进行介绍。

1. 在左侧导航栏中，单击 **计算组件** > **部署**。

2. 单击列表中 ***部署资源*** 名称。

3. 切换至 **监控** 页面，在展开的监控页面内查看 **系统面板** 的监控数据。

## 监控数据说明

 **提示**：光标悬浮至图表曲线上时，可查看当前时间点各容器组的详细数据，可查看 PromQL 表达式。当容器组超过 15 个时，值从大到小排序，仅显示前 15 条数据。
        
    |监控资源| 监控数据统计粒度  |  指标说明  |
    | -------- | --------  | ------------ |
    |       **CPU**     |   使用率/使用量  |   **使用率** = 使用值/限制值（limits）<br/>您可通过使用率判断计算组件容器的限制值配置是否合理，当使用率过高时，表明容器限制值设置过小，可能会影响计算组件的正常运行。<br/>**使用量** 为计算组件已使用的资源量。      |
    |       **内存**     |   使用率/使用量  |   **使用率** = 使用值/限制值（limits）<br/>您可通过使用率判断计算组件容器的限制值配置是否合理，当使用率过高时，表明容器限制值设置过小，可能会影响计算组件的正常运行。<br/>**使用量** 为计算组件已使用的资源量。      |
    |       **网络流量**     |   流入/流出  |   每秒流入/流出容器组的网络流量。   |
    |       **网络报文量**     |  接收/发送 |容器组每秒接收/发送的网络报文量。 |
    |       **磁盘速率**     |  读/写 | 计算组件中每个容器组挂载分区的读写速率。 |
    |       **磁盘 IOPS**     |  读/写 |计算组件中每个容器组挂载分区的 IOPS。 |
4. 点击 **切换图标** 可切换至自定义监控面板页面，参照 [创建监控面板](../../observability/monitor/functions/manage_dashboard.mdx)创建专属关注监控面板。
