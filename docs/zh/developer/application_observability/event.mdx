# 实时事件

基于 Kubernetes 资源的变更及各种运行状态生成的事件信息，并且提供了可视化查询能力。当应用、计算组件等资源出现异常情况时，可通过实时事件进行分析定位具体原因。

## 操作步骤

> 以通过部署页面查看容器的实时日志为例进行介绍。

1. 在左侧导航栏中，单击 **计算组件** > **部署**。

2. 单击列表中部署的名称。

3. 切换至 **实时事件** 页面，具体事件信息说明。

* **资源事件记录**：重要事件总览区域下方，展示查询时间范围内，符合查询条件的所有事件记录。可通过单击重要事件卡片过滤相应类型的事件，或展开并输入查询条件进行查询。查询条件说明如下：

    * **资源类型**：发生事件的 Kubernetes 资源的类型，例如：Pod。
     
    * **命名空间**：发生事件的 Kubernetes 资源所在的命名空间。
    
    * **事件原因**：事件发生的原因（reason）。

    * **事件级别**：事件的重要程度，如 Warning。
    
    * **资源名称**：发生事件的 Kubernetes 资源的名称。支持选择或输入多个名称。

:::tip
- 单击事件记录资源名称右侧的查看图标可在弹出的 **事件详情** 对话框中，查看事件的详细信息。
- 事件原因左侧的图标颜色表示事件等级。绿色图标表示该事件的等级为 `Normal`，可忽略该事件；橘色图标表示该事件的等级为 `Warning`，存在资源异常的情况，需要关注该事件以免引发故障。
- Kubernetes 可配置存储事件信息时长，平台默认应 1小时，过期会自动清理，如需查看更多、更详细的事件信息，可点击 [全部事件](../../observability/event/event.mdx)。
:::
