# 实时事件

基于 Kubernetes 资源状态变化和操作状态更新生成的事件信息，并提供集成的可视化查询接口。当应用、工作负载或其他资源遇到异常时，实时事件分析有助于排查根本原因。

## 操作步骤

> 本操作步骤演示如何通过部署页面查看容器运行时事件。

1. 在 **容器平台** 中，单击 **工作负载** > **部署**。

2. 从列表中单击一个部署名称。

3. 切换至 **事件** 标签，以查看详细记录。

## 事件记录解释

**资源事件记录**：在事件摘要面板下方，列出了指定时间范围内的所有匹配事件。单击事件卡片可查看完整的事件详细信息。每张卡片显示：

- **资源类型**：由图标缩写表示的 Kubernetes 资源类型：
  - `P` = Pods
  - `RS` = ReplicaSet
  - `D` = Deployment
  - `SVC` = Service

- **资源名称**：目标资源的名称。

- **事件原因**：Kubernetes 报告的原因（例如：FailedScheduling）。

- **事件级别**：事件的严重性。
  - `Normal` : 信息性
  - `Warning` : 需要立即关注

- **时间**：最后发生时间，发生次数。

:::info
Kubernetes 允许管理员通过事件 TTL 控制器配置事件保留期限，默认保留期限为 1 小时。过期的事件将由系统自动清理。要访问全面的历史记录，请访问 [全部事件](../../observability/event/event.mdx)。
:::
