# 实时日志

聚合容器运行时日志，并提供可视化查询功能。当应用、工作负载或其他资源出现异常行为时，日志分析有助于诊断根本原因。

## 操作步骤

> 本步骤演示如何通过部署界面查看容器运行时日志。

1. 在 **容器平台** 中，单击 **工作负载** > **部署**。

2. 从列表中单击一个 Deployment 名称。

3. 切换至 **日志** 标签页以查看详细记录。

   | 操作                  | 描述                                                                                                                |
   | :------------------- | :----------------------------------------------------------------------------------------------------------------- |
   | **容器组**/**容器**   | 使用下拉选择器在 Pods 和 Containers 之间切换，以查看相应的日志。                                                  |
   | **重启前的日志**      | 查看已终止容器的日志（当容器 restartCount > 0 时可用）。                                                           |
   | **行数**              | 配置日志缓冲区大小：1K/10K/100K 行。                                                                                |
   | **换行**              | 切换长日志条目的换行模式（默认启用）。                                                                              |
   | **查找**              | 全文搜索并高亮匹配项，按回车键可进行导航。                                                                          |
   | **原始**              | 直接从容器运行时接口（CRI）捕获的未处理日志流，未进行格式化、过滤或截断。                                            |
   | **导出**              | 下载原始日志。                                                                                                     |
   | **全屏**              | 单击截断行以在模态对话框中查看完整内容。                                                                          |

:::warning

- **截断处理**：超过 2000 个字符的日志条目将被截断并显示省略号 `...`
  - 被截断的部分无法通过页面的查找功能匹配。
  - 单击被截断行中的省略号 `...` 标记以在模态对话框中查看完整内容。
- **复制可靠性**：避免在看到截断标记 (...) 或 ANSI 彩色代码时直接从呈现的日志查看器复制内容。始终使用 **导出** 或 **原始** 功能获取完整日志。
- **保留策略**：实时日志遵循 Kubernetes 日志轮换配置。要进行历史分析，请使用[日志](../../observability/log/functions/log.mdx)功能。
:::
