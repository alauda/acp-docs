# 实时日志

基于 Kubernetes 的日志信息，并且提供了可视化查询能力。当应用、计算组件等资源出现异常情况时，可通过实时日志进行分析定位具体原因。

## 操作步骤

> 以通过部署页面查看容器的实时日志为例进行介绍。

1. 在左侧导航栏中，单击 **计算组件** > **部署**。

2. 单击列表中部署的名称。

3. 切换至 **实时日志** 页面，具体的操作如下。

    |操作|说明|
    |:---|:---|
    |**容器组、容器**|可通过切换 **容器组** 或 **容器**，查看相应的日志。|
    |**重启前的日志**|当容器因为某些原因重启后，可切换至 **重启前的日志** 查看最近一次重启的日志信息。|
    |**行数**|可通过切换行数查看 1000、10000 或 100000 行日志。|
    |**换行**|当日志内容的长度超出屏幕显示时，会将超出的部分自动换至下一行，此功能默认开启。|
    |**查找**|查找指定的字符，并将匹配到的字符高亮显示，可通过回车键查找下一个。|
    |**精简**|当日志内容超过 2000 个字符时，超长的部分会被精简并使用 **...** 代替，通过查找功能无法匹配到已被精简的日志内容，可通过下述方式查看完整日志：<ul><li>单击被精简的日志，会弹出完整日志信息对话框，可配合浏览器的搜索功能一起使用。再次单击该日志或页面空白处即可取消对话框。</li><li>通过 **导出** 功能可查看完整的日志信息。</li></ul>|

:::warning
- 当页面中存在上下显示区域外的日志、精简的日志或被格式化的颜色字符时，直接复制可能导致复制的日志信息丢失。为了避免这种情况，建议在原始页面中查看并复制完整的日志信息。
- 实时日志是直接读取容器内的日志信息，具备更好的实时性，但会根据 Kubernetes 配置的存储规模到动态清理日志信息，如需查看更多日志信息，可在左导航**可观测** 查看[日志](../../observability/log/functions/log.mdx)。
:::
