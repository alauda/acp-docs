# 使用 YAML 文件创建 OAM 应用

创建 OAM 应用的方式有 **使用内置组件及运维特征创建应用**、**使用自定义组件及运维特征创建应用** 以及 **使用 YAML 文件创建应用** 三种，本文档旨在指导您快速了解并使用 YAML 文件创建一个简单的 OAM 应用。

## 介绍

### 适用场景

如果您已能熟练创建应用，并希望在集群中快速迁移应用，使用 YAML 文件可以让您一键配置所需的组件和运维特征。

### 预计阅读时间

预计完成本文档的阅读和操作所需时间约为 3-5 分钟。

## 注意事项

- 本文档不提供配置过程中的参数详细说明和配置，仅作为快速参考。具体的指令和配置步骤，请参见相关文档。

- 如果您使用的是从已有 OAM 应用导出的 YAML 文件，请务必修改 YAML 中的应用名称和组件名称，以避免在同一命名空间内出现名称重复。

## 前提条件

- 对 OAM 应用有深入的理解。

- 必须启用 OAM 相关功能。

## 操作步骤

1. 访问 **Container Platform**。

2. 在左侧导航栏中，单击 **应用管理** > **OAM 应用**。

3. 单击 **创建应用**。

4. 选择 **通过 YAML 创建** 选项。

5. 输入 YAML 内容，然后单击 **创建**。

   **说明**：除了平台提供的 YAML 示例外，您还可以访问 [OAM 开源社区](https://github.com/oam-dev/spec/blob/master/7.application.md) 获取更多关于编写应用 YAML 的指导。
