# 使用内置组件和运维特征创建 OAM 应用

创建 OAM 应用的方式有 **使用内置组件和运维特征创建应用**、**使用自定义组件和运维特征创建应用** 以及 **使用 YAML 文件创建应用** 三种，本文档旨在快速指导您使用内置组件和运维特征创建一个简单的 OAM 应用。

## 介绍

### 适用场景

您是一位刚接触 OAM 的新用户，对 OAM 应用感兴趣，希望快速了解 OAM 应用的配置流程和使用方法。

### 预计阅读时间

预计阅读和操作本文档所需时间约为 5-10 分钟。

## 注意事项

本文档不提供详细的参数说明和配置，仅作为初学者的快速参考指南。有关具体解释和配置步骤，请参考相关文档。

## 前提条件

- 您已仔细阅读核心概念文档，并对 OAM 相关概念有了基本了解。

- 您已启用 OAM 相关功能。

## 流程概述

| 序号 | 操作步骤                            | 描述                                                                                                                |
| ---- | ----------------------------------- | ----------------------------------------------------------------------------------------------------------------- |
| 1    | [配置基本信息](#basic-info)       | 配置应用的基本信息，包括应用名称、描述、标签、注解等信息。                                                      |
| 2    | [添加组件](#add-component)         | 选择一个组件并配置其属性。                                                                                        |
| 3    | [添加运维特征](#add-trait)         | 为组件添加适当的运维特征。                                                                                        |

## 操作步骤

### 配置基本信息\{#basic-info}

1. 进入 **Container Platform**。

2. 在左侧导航栏中，单击 **应用管理** > **OAM 应用**。

3. 单击 **创建应用**。

4. 选择创建方式 **通过镜像**。

5. 填写应用名称，并设置标签或注解。

   | 参数          | 说明                                                                                                                                                                                                                                                             |
   | -------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | **标签**      | 用于选择对象和查找满足特定条件的对象集合。必须为键值对，例如 *app.kubernetes.io/name: hello-app*。                                                                                                                                                            |
   | **注解**      | 用于向开发和运维团队提供信息。必须为键值对，例如：*cpaas.io/maintainer: kim*。**说明**：如果注解值为可点击链接，将键设置为 **app.\{显示文本}.url**，然后在界面点击显示文本会完成跳转，例如 *app.Docs.url: [https://example.com/docs/](https://example.com/docs/)*。 |

6. 单击 **添加组件**。

### 添加组件\{#add-component}

1. 选择 **网络服务** 组件。默认情况下，平台提供两种预置组件：**网络服务** 和 **有状态服务**。本文档以 **网络服务** 为例进行说明。

2. 单击 **下一步**。

3. 使用以下参数配置组件属性：

   | 参数                | 说明                                                                                                                                                                                                         |
   | ------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | **方式及镜像地址**  | 本文档使用输入法输入镜像地址。以开源镜像 HelloWorld 为例进行说明。                                                                                                                                         |
   | **网络访问**        | 用于配置组件的网络访问方式。本文档以 **TCP**、端口为 **80**、类型为 **支持内部和外部访问** 为例进行说明。                                                                                                   |
   | **其他参数**        | 按需填写或保持默认。                                                                                                                                                                                          |

4. 单击 **下一步**。

### 添加运维特征\{#add-trait}

1. 选择预置运维特征。本文档中保持默认配置，不进行任何修改。

2. 单击 **添加**。

## 预期结果

完成上述配置后，在 OAM 的详情页面可以查看类似 `tcp://<节点 IP 地址>:<主机端口>` 格式的 **外部访问地址**，您可以通过浏览器访问该地址成功访问应用。本文档中外部访问地址为 192.1.1.3:31217，访问结果如图所示。
