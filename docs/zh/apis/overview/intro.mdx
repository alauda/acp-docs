# 介绍

<Term name="product" /> (<Term name="productShort" />) 是一个云原生平台，其中大多数功能通过 Kubernetes 操作实现。<Term name="productShort" /> API 参考文档提供了关于 <Term name="productShort" /> 中可用 API 的全面信息。

## API 类型

<Term name="productShort" /> 提供两种类型的 API：

### Kubernetes API

大多数 <Term name="productShort" /> 功能通过 Kubernetes API 暴露。这些 API 可以使用标准 Kubernetes 客户端访问，例如 kubectl、client-go 等。有关这些 API 的完整列表，请参阅 [Kubernetes APIs](../kubernetes_apis/) 部分。

有关如何调用这些 API 的信息，包括标准模式和示例，请参阅 [Kubernetes API 使用指南](./kubernetes_api_usage)。

### 高级 API

一些功能，例如日志查询，不适合通过 Kubernetes API 暴露。在这些情况下，<Term name="productShort" /> 提供了 RESTful API，可以通过 HTTP 请求访问。有关这些 API 的完整列表，请参阅 [高级 API](../advanced_apis/) 部分。

## API 认证

<Term name="productShort" /> 的 Kubernetes API 和高级 API 都使用 <Term name="productShort" /> 用户令牌进行身份验证和授权。您只需将令牌添加到 HTTP 请求的 `Authorization` 头中：

```bash
curl -X GET "http://<acp-api-server-address>/apis/v1/namespaces/default/pods" \
     -H "Authorization: Bearer <your-token>"
```

### 获取 <Term name="productShort" /> 用户令牌

要获取 <Term name="productShort" /> 用户令牌，请按照以下步骤操作：

1. 登录 <Term name="productShort" /> Web 控制台，单击右上角的用户名以展开下拉菜单，然后单击 **个人资料** 菜单项以访问个人中心页面。

2. 在个人中心页面中，单击 **API 令牌** 标签，然后单击 **添加 API 令牌** 按钮以创建新的令牌。

3. 在弹出对话框中，输入令牌的描述并设置过期时间，然后单击 **添加** 按钮以创建令牌。

4. 创建成功后，系统将生成一个令牌，并在页面上显示。

5. 将令牌复制到剪贴板，并妥善保存，以便在调用 ACP API 时使用。
