# 深入理解 Directory

## 介绍

Directory 类型应用程序可从 .yml、.yaml 或 .json 文件中加载 `manifest` 文件。Directory 类型应用程序可以通过平台 UI、Argo CD Dashboard、CLI 或声明方式创建。声明语法参考如下：

```yaml
apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: guestbook
spec:
  destination:
    namespace: default
    server: https://kubernetes.default.svc
  project: default
  source:
    path: guestbook
    repoURL: https://github.com/argoproj/argocd-example-apps.git
    targetRevision: HEAD
```

除了添加需要的配置选项以外，不需要添加 `spec.source.directory` 字段。Argo CD 将自动检测 `source repository/path` 是否包含 `manifest` 文件。

## 优势

- 支持从 `manifest` 文件中加载各种类型资源，简单且易于理解。

- 无须额外的配置管理，维护成本更低。

## 使用场景

- 需要管理多个 Kubernetes 资源，例如 Deployment、Service、ConfigMap 等。

- 小型项目、资源较少或者想要快速理解使用 GitOps。

- 直接部署原生 YAML 文件，不需要复杂的配置管理、动态生成能力。

:::warning
Directory 类型的应用程序仅适用于 `plain manifest` 文件。如果设置了 `Directory:` 的情况下 Argo CD 遇到 `Kustomize`、`Helm` 或 `Jsonnet` 文件，它将无法呈现 `manifest` 文件。
:::
