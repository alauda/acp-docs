# 通过 Argo CD Dashboard 创建 Application

## 前提条件

- 已部署，满足以下任意一条即可：：
  - 已部署 GitOps 集群插件。[部署 GitOps 集群插件](../install_gitops/install_gitops_plugin.mdx)
  - 已部署 Argo CD Operator。[部署 Argo CD Operator](../install_gitops/install_argocd_opeator.mdx)

- 已获取到 **Argo CD Dashboard** 的访问地址、账号及密码。


## 创建 Application

> 按以下步骤使用功能：

1. 通过浏览器输入 **Argo CD Dashboard** 的访问地址，打开 **Argo CD Dashboard**。

> 如您是平台管理员，可访问 global 集群插件详情页面，找到 GitOps 插件，直接访问 **Argo CD 原生 UI**。

2. 输入 Argo CD 账号密码，**登录**。

3. 点击如下所示的 **+ NEW APP** 按钮：

<img src="../assets/new_app_button.png" width = "400" />

> 按照以下步骤配置应用：

**基本信息配置**
<img src="../assets/new_app_button_info.png" width = "400" />
   - `Application Name`：输入 `guestbook`
   - `Project`：选择 `default`
   - `Sync Policy`：保持为 `Manual`

**源代码配置**
<img src="../assets/new_app_button_source.png" width = "400" />
   - `Repository URL`：设置为 `https://github.com/argoproj/argocd-example-apps.git`
   - `Revision`：保持 `HEAD`
   - `Path`：设置为 `guestbook`

**目标集群配置**
<img src="../assets/new_app_button_repo.png" width = "400" />
   - `Clusternew`：设置为 `https://kubernetes.default.svc`（或集群名称为 `in-cluster`）
   - `Namespace`：设置为 `default`

4. 创建 **Application**
   完成上述配置后，点击界面顶部的 **Create** 按钮以创建 guestbook 应用程序。
