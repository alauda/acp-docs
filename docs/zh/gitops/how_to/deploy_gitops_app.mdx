# 使用平台 UI 创建一个 Argo CD Application

本文将介绍通过平台 GitOps 应用，创建一个 Argo CD Application 应用，实现通过 GitOps 管理应用的完整流程。

## 使用场景

- 使用平台 UI 创建一个 SpingBoot Argo CD Application 应用，体验通过 GitOps 管理应用的完整流程。

## 前置条件

- 已在 **global** 集群中部署 GitOps 集群插件。
- 平台上已分配好项目、命名空间。

## 操作步骤

> 按以下步骤使用功能：

### 代码仓库配置
> 如果进入 GitOps 应用详情页未找到 **已集成代码仓库**，可按照先集成代码仓库：
1. 集成仓库地址（Demo 使用）：https://github.com/argoproj/argocd-example-apps.git
2. [使用 Argo CD UI 集成代码仓库](../how_to/argocd_git.mdx)

:::info
Demo 应用可使用：https://github.com/argoproj/argocd-example-apps/tree/master/helm-guestbook
:::

### 使用平台默认代码仓库

1. 在左侧导航栏中，单击 **GitOps 应用**。

2. 单击 **创建 GitOps 应用**。

3. 在 **基本信息** 及 **代码仓库** 区域，参考以下说明配置相关信息。

|   **参数**                       | **填写内容** | 
| :----- | :--------------------------------- |
|   **类型**                       |`Application`|
|   **来源**                       |`Argo CD 集成`|
|  **集成项目名称**                 | `argocd-example-apps`|
|  **版本标识**                     |`分支` <br/>`master`|
|  **源文件类型**                   | `Helm`|   
|  **源文件目录**                   |`helm-guestbook`|   
|  **自定义 Values**               | `values.yaml`|  

4. 在 **分发位置**，使用平台推荐的 ***默认命名空间***，也可选择其他命名空间。

5. 设置同步策略，默认 **自动同步**。

6. 单击 **创建**。
