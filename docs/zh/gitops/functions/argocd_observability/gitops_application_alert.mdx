# GitOps 应用运维

## 功能简介

通过平台 GitOps 应用的运维管理能力，可查看 GitOps 应用监控、实时日志、事件，并针对 GitOps 应用创建告警策略，当 GitOps 应用异常时可主动提醒，快速进行问题定位分析解决。

## 前提条件 

- 平台已有发布的 GitOps 应用，[部署一个 GitOps 应用](../../how_to/deploy_gitops_app.mdx)。


## 创建告警策略
> 创建告警策略，提前配置告警策略，当 GitOps 应用异常时可主动提醒，快速进行问题定位分析解决。

1. 在左侧导航栏中，单击 **GitOps 应用**。

2. 单击列表中需要创建告警策略的 GitOps 应用名称。

3. 切换至 **告警** 页签。

4. 单击 **创建告警策略**，按需填写基本信息。

5. 单击 **告警规则** 区域的 **添加告警规则**，对应的指标说明如下：

**说明**：其他参数及策略配置区域参数的配置及说明请参考 创建告警策略 。

| **指标名称** | **规则说明** |
| :----- | :--------------------------------- |
| GitOps 应用健康状态 (gitops.applicationset.healthy) | GitOps 应用的健康状态：<br/> - 0 ：未知、丢失、降级或暂停 <br/> - 1 ：处理中 <br/> - 2 ：健康 |
| GitOps 应用同步状态 (gitops.applicationset.synced) | GitOps 应用的同步状态：<br/> - 0 ：同步失败或待同步 <br/> - 1 ：同步中 <br/> - 2 ：已同步 |

6. 单击 **创建**。

## 实时日志

> 针对 GitOps 应用分发的所有 Workload 资源，可查看实时日志。实时日志不依赖平台是否部署日志组件的情况下，也能够迅速识别和应对系统故障信息。

- GitOps 应用详情页面，** Kubernetes 资源** 点击 ***任意 Workload 资源名称***，也可在右侧查看该资源的 **实时日志** 信息。


## 实时事件

> 实时事件不依赖平台是否部署日志等组件，也能够迅速识别和应对系统故障事件信息。

- GitOps 应用详情页面，进入 **实时事件** 页签，查看 GitOps 应用分发的所有资源的实时事件聚合。
- GitOps 应用详情页面，**Kubernetes 资源** 点击 ***任意资源名称***，也可在右侧查看该资源的 **实时事件** 信息。
