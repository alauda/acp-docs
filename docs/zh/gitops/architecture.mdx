# 架构

## GitOps 与 Argo CD 

GitOps 是一种理论，而 Argo CD 则是实现 GitOps 的一种工具。Argo CD 通过监视 Git 仓库中的配置文件并将其自动同步到目标环境，实现了基于 Git 的持续交付流程，通过将整个交付过程纳入 Git 版本控制系统，提高了软件交付的速度、可靠性和安全性。

**Alauda GitOps** 是基于 Argo CD 实现，将 Git 仓库作为唯一可信来源存储应用程序、基础设施配置等文件，实现应用快速精准分发、部署到一个或多个 Kubernetes 集群。

## GitOps 的典型架构
GitOps 与传统的应用管理方式最大的不同点在于：
- 不直接操控运行环境，而是通过维护 Git 上的应用配置仓库来控制运行环境。
- Argo CD 以持续拉取的方式，及时纠正运行环境与应用配置仓库之间的差异，使其符合预期，防止配置漂移，保障业务应用的稳定运行，并在发生故障时实现快速恢复。
