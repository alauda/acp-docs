# 架构

## GitOps 与 Argo CD

GitOps 是一种现代的持续交付和运营理论，而 Argo CD 是一款强大的工具，通过监视 Git 仓库中的配置文件并自动将其同步到目标环境来实现 GitOps。这种方法通过将整个交付过程纳入 Git 版本控制系统，提高了软件交付的速度、可靠性和安全性。

**ACP GitOps** 基于 Argo CD，使用 Git 仓库作为唯一可信源，存储应用程序、基础设施配置和其他文件，以便快速准确地分发和部署到一个或多个 Kubernetes 集群。

## GitOps 架构

GitOps 与传统应用管理方法之间的主要区别在于：

- GitOps 不是直接操控运行环境，而是通过在 Git 上维护应用配置仓库来控制运行环境。
- Argo CD 不断拉取该仓库，并纠正运行环境与应用配置仓库之间的差异，确保环境符合预期，防止配置漂移，并在发生故障时实现快速恢复。

## ACP GitOps 架构

**ACP GitOps** 作为集群插件安装在 `global` 集群上，并利用 Argo CD 实现多个业务集群的应用分发和基础设施提供。
