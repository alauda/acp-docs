# 功能总览

## 监控

- 拨测

  平台提供基于 Blackbox Exporter 的拨测（黑盒监控）能力，支持通过 ICMP、TCP、HTTP 等多种协议对网络服务进行探测。与关注系统内部状态的白盒监控相比，黑盒监控从用户视角对服务进行探测，能够快速发现影响用户体验的各类故障。

  例如：当业务接口响应异常、服务不可用时，黑盒监控可以第一时间发现并触发告警，帮助运维人员及时处理问题。

- 监控面板

  平台内置了现代化的监控面板管理功能，提供比传统 Grafana 更简单易用的可视化配置体验。通过统一的监控视图，将平台各类监控指标数据进行聚合展示，帮助用户快速构建所需的监控大盘。

## 告警通知

- 告警策略

  平台提供全方位的告警能力，支持基于指标、日志、事件设置告警规则。内置丰富的监控指标和告警模板，用户可以快速配置符合业务需求的告警策略，实现问题的及时发现和处理。

- 告警模板 

  告警模板将告警规则和通知策略进行标准化封装，支持快速复用到多个监控对象。通过模板化配置，可以显著降低告警策略的管理成本，提升运维效率。

- 告警历史

  系统完整记录告警的生命周期，包括触发时间、恢复时间、告警状态、告警级别、告警内容等信息。用户可以通过告警历史追溯和分析问题，持续优化告警配置。

- 通知

  支持邮件、钉钉、企业微信、飞书、Webhook 等多种告警通知渠道，确保告警信息能够及时触达相关人员。用户可以根据实际需求灵活配置通知策略。

## 调用链

分布式调用链查询功能为微服务架构提供全链路追踪能力，通过采集服务间调用的元数据信息，帮助用户快速定位跨服务调用问题。

## 日志事件

- 日志

  平台自动采集并集中管理集群、节点、容器的标准输出及文件日志。提供强大的日志存储、检索、分析能力，支持多维度的日志查询和可视化展示，帮助用户快速定位问题。

- 事件

  平台实时采集 Kubernetes 集群中的关键事件信息，完整记录资源的状态变更过程。当集群、节点、Pod 等出现异常时，可通过事件追溯定位根因，大幅提升问题排查效率。

## 巡检

- 巡检

  基于丰富的企业级运维经验，平台提供自动化的巡检能力。通过多维度的健康检查，帮助用户实时掌握资源运行状态，提前发现潜在风险，降低人工巡检成本。

- 平台健康状态

  提供直观的平台功能健康状态总览，支持查看各项功能的部署情况和组件运行状态。具备平台管理权限的用户可以深入查看详细的健康检测数据，快速定位和解决平台层面的问题。
