# 事件

## 操作步骤

1. 在左侧导航栏中，单击 **运维中心** > **事件**。 

    **提示**：通过顶部导航栏的集群切换下拉选择框切换待查看事件的集群。


## 事件概览
    
在事件页面默认展示最近 30 分钟内（可选择或自定义时间范围）的重要事件总览以及资源事件记录。

    
* **重要事件总览**：此卡片展示重要事件的原因以及在选定时间范围内经历此类事件的资源数量。
    - **注意**：当同一资源多次发生该类事件时，资源数量不会累加。
    - 例如：如果节点重启事件的资源数量为 20，表示在选定时间范围内，有 20 个资源经历了此类事件，而同一资源可能发生了多次。
    
    
* **资源事件记录**：重要事件总览区域下方，展示符合查询条件的所有事件记录，时间范围为选定的查询时间。您可以通过单击重要事件卡片来过滤相应类型的事件，或展开视图并输入查询条件进行搜索。查询条件说明如下：

    * **资源类型**：发生事件的 Kubernetes 资源的类型，例如：Pod。
     
    * **命名空间**：发生事件的 Kubernetes 资源所在的命名空间。
    
    * **事件原因**：事件发生的原因。

    * **事件级别**：事件的重要程度，如 Warning。
    
    * **资源名称**：发生事件的 Kubernetes 资源的名称。支持选择或输入多个名称。

:::tip
    
- 单击事件记录中资源名称右侧的查看图标可在弹出的 **事件详情** 对话框中查看事件的详细信息。
- 事件原因左侧的图标颜色表示事件等级。绿色图标表示该事件的等级为 `Normal`，可忽略该事件；橙色图标表示该事件的等级为 `Warning`， 表示资源存在异常，应监视此事件以防止事件发生。

:::
