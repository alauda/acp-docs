# 介绍

## 模块介绍

平台对接了 Kubernetes 事件，记录了 Kubernetes 资源的重要状态变更及各种运行状态变化的事件，并且提供了存储、查询、可视化能力。当集群、节点、Pod 等资源出现异常情况时，可通过事件分析具体原因。

并可基于事件原因为计算组件 [创建告警策略](../monitor/functions/manage_alert.mdx)，当关键事件发生次数达到告警阈值时，能够自动触发告警并通知相关人员及时处理，以降低平台业务运行风险。

## 功能概述

事件模块主要提供以下功能：

**事件采集与持久化**
   - **自动采集**：模块会自动采集Kubernetes集群中发生的所有事件，包括Pod创建、删除、调度失败等。
   - **持久化存储**：采集到的事件会被持久化存储，确保用户可以在需要时回溯历史事件。

**事件查询**
   - **灵活查询**：用户可以通过多种条件（如事件类型、命名空间、资源名称等）查询事件，快速定位问题。
   - **时间范围过滤**：支持按时间范围查询事件，方便用户查看特定时间段内的集群活动。

**事件汇总与展示**
   - **事件汇总**：模块会对事件进行汇总，生成统计信息，帮助用户了解集群的整体状态。


## 使用场景

事件模块适用于以下场景：

- **集群监控**：通过实时监控Kubernetes事件，用户可以及时发现集群中的异常情况。
- **故障排查**：当集群出现问题时，用户可以通过查询事件日志，快速定位问题的根源。
- **性能优化**：通过分析事件数据，用户可以了解集群的资源使用情况，优化资源配置。

## 使用限制

本功能依赖日志系统实现，请提前确保平台内已安装了 ACP Log Collector 和 ACP Log Storage 插件。
