# 介绍

## 模块介绍

监控模块为平台管理员、运维管理员提供了指标、面板、告警、通知等运维能力。

平台结合 Prometheus / VictoriaMetrics  开源组件和监控面板，支持对平台管理的集群、节点、组件、自定义应用、Pod、容器等进行实时监控。

支持快捷设置集群、节点、计算组件层面的监控指标告警、日志告警（仅计算组件）、事件告警，也可以根据实际需求自定义监控指标算法，增加需要的告警指标及规则。并可通过配置通知策略及时将告警信息发送给运维人员，以避免系统故障或及时处理故障，减少系统运维成本，保障系统的稳定性。

## 模块优势

监控模块具有以下核心优势：

- 全方位监控覆盖

  支持对集群、节点、组件、容器等多个层面的全方位监控，实现基础设施到应用的完整监控链路。

- 灵活的告警配置

  提供丰富的预置告警规则，同时支持自定义告警规则和算法，满足不同场景的监控需求。

- 多样化的可视化展示

  集成专业的监控面板，支持多种数据可视化方式，直观展现系统运行状态。

- 高效的告警通知

  支持多渠道告警通知，包括邮件、短信、webhook等，确保告警信息及时送达。

- 可扩展的监控架构

  基于业界主流的 Prometheus / VictoriaMetrics  技术栈，具备良好的扩展性和兼容性。

## 应用场景

监控模块适用于以下场景：

- 集群健康监控

  实时监控集群的资源使用情况、节点状态、组件运行状况等，及时发现潜在问题。

- 应用性能分析

  监控应用的运行指标、容器资源使用情况，优化应用性能。

- 故障预警和诊断

  通过设置合理的告警规则，提前发现系统异常，快速定位和解决问题。

- 容量规划

  基于历史监控数据进行趋势分析，为资源扩容和优化提供依据。

## 使用限制

使用监控模块时，需注意以下限制：

- 监控数据存储时间依赖于存储容量配置，默认保留时间为 7 天。

- Prometheus 和 VictoriaMetrics 不支持同时部署在同一集群，请做好选型规划，择其一部署。

- 自定义监控指标的采集间隔最小支持 60 秒

- 告警通知渠道需要预先配置相应的服务（如邮件服务器、短信网关、企微/钉钉机器人等）
