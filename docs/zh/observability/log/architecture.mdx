# 架构

## 功能架构(可选)

1. 增加一张或多张关于该功能的全局架构图，体现该功能在全局视图中的位置和作用。

2. 在架构图下增加对于该图片的描述，描述该图片的含义及该图片中各个部分的作用。

## 技术架构（若没有功能架构，则删除该标题）

1. 增加一张或多张关于该功能的技术架构图，可以包含如下内容：

   - 核心组件：例如容器运行时、编排引擎、网络方案、存储系统等。并且能够在图中描绘出各个组件中的联系和各自起的作用。
   - 关键技术栈：例如后端服务、前端框架、数据存储、消息队列、缓存系统等。
   - 部署架构：例如多集群部署、高可用设计等。
   - 数据流架构：例如数据流向、数据采集、数据处理、数据存储、数据分析和展示等。

2. 在架构图下增加对于该图片的描述，描述该图片的含义及该图片中各个部分的作用。
