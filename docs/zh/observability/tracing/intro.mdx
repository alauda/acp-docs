# 介绍

调用链是容器平台可观测性体系中的关键模块，用于实现分布式系统全链路追踪与分析。该模块基于 OpenTelemetry 标准构建，提供从数据采集、存储到可视化分析的全流程解决方案，帮助开发运维人员快速定位服务间调用异常、分析性能瓶颈、追踪请求全生命周期行为。

通过对接开源技术栈与自研组件，本模块支持端到端的调用链追踪能力：应用通过无侵入的 `OTel 自动注入`或 `SDK` 集成方式生成追踪数据，经统一采集管道存储至 Elasticsearch，最终通过定制化 UI 实现多维度的可视化分析。用户可通过 `TraceID`、服务名称、标签等丰富条件进行精准检索。

## 优势

调用链的核心优势如下：

- **全链路追踪能力**  
  支持跨服务、跨进程、跨容器边界的完整调用链路还原，精准呈现微服务架构中的复杂调用关系。

- **灵活的数据采集方式**  
  提供自动注入（无代码改造）与 SDK 集成双模式，兼容 Java/Python/Go 等主流语言应用。

- **高性能存储方案**  
  采用 Elasticsearch 作为存储后端，支持海量 Span 数据的写入与快速检索。

- **灵活的查询和分析能力**  
  自研 UI 集成 `jaeger-query` API，支持基于 TraceID、所属服务、Tag、Span 类型等多维度条件灵活查询，便于用户快速定位问题根源。

- **标准化协议支持**  
  基于 OpenTelemetry 标准构建，可对接其他基于 OTel 的云原生组件产生的追踪数据。

## 应用场景

调用链的主要应用场景如下：

- **分布式系统故障排查**  
  在微服务架构中，通过完整的调用链展示快速定位服务故障和异常调用，减少故障排查时间。

- **性能瓶颈分析**  
  通过查看各服务间调用延迟，识别性能瓶颈，指导系统优化和资源调整。

- **服务依赖关系分析**  
  通过时序瀑布图清晰展现服务之间的调用路径和依赖关系，辅助架构师进行系统设计与改进。

## 使用限制

使用调用链需注意以下约束：

- **采样策略对性能的影响**
  - 在高负载场景下，调用链数据采集可能对 Elasticsearch 性能和存储造成一定压力，建议根据业务情况合理配置采样率。
