# 审计功能

## 功能简介

审计功能是一个全面的操作记录和追踪系统，它为平台提供了所有用户以及系统安全相关的时序操作记录。通过审计功能，您可以：

- 追踪和分析集群中的所有变更事件
- 识别操作的执行者和来源
- 查看详细的变更内容
- 确认操作的执行结果
- 协助问题排查和故障分析

## 主要功能

- 用户操作审计：记录和查询所有登录平台用户的操作行为
- 系统操作审计：追踪和记录系统级别的操作事件
- 多维度筛选：支持按时间、操作人、资源类型等多个维度进行审计查询
- 数据导出：支持将审计记录导出为 CSV 格式
- 详情查看：提供每个审计事件的详细 JSON 格式信息

## 功能优势

- 完整性：记录所有关键操作，确保操作可追溯
- 实时性：支持实时查看最新的操作记录
- 灵活性：提供多维度的筛选和查询能力
- 可导出：支持数据导出，便于后续分析
- 详细度：提供完整的操作详情，包括操作参数和结果

## 审计查询

按以下步骤使用审计查询功能：

### 选择审计范围

在审计页面中，通过页签切换选择需要查看的审计范围：
- 用户操作：查看登录用户的操作记录
- 系统操作：查看系统级别的操作记录

### 配置查询条件

使用以下参数进行审计事件筛选：

| 参数 | 描述 |
| :--- | :--- |
| 操作人 | 执行操作的账号用户名或系统账号名称 |
| 操作 | 操作类型，如创建、更新、删除等 |
| 集群 | 被操作资源所属的集群 |
| 资源类型 | 被操作资源的类型 |
| 资源名称 | 被操作资源的名称，支持模糊搜索 |
| 时间范围 | 审计事件的时间范围，默认近30分钟 |

### 查看搜索结果

搜索结果将显示以下信息：

| 字段 | 说明 |
| :--- | :--- |
| 操作人 | 执行操作的账号信息 |
| 操作 | 具体的操作类型 |
| 资源名称/类型 | 被操作资源的标识信息 |
| 集群 | 资源所属集群 |
| 命名空间 | 资源所属命名空间 |
| 客户端 IP | 操作来源的 IP 地址 |
| 操作结果 | 操作执行的结果状态 |
| 操作时间 | 操作执行的具体时间 |

### 查看详细信息

点击审计记录右侧的"详情"按钮，可在弹出的对话框中查看完整的 JSON 格式审计信息。

## 数据导出

点击导出按钮即可将当前筛选条件下的审计记录导出为 CSV 格式文件，支持后续的离线分析和存档。
