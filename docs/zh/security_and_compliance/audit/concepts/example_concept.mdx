# 核心概念

## 审计事件（Audit Event）

审计事件是平台中发生的各种操作和变更的记录单元。它包含了操作的详细信息，如操作人、操作类型、操作时间等关键信息。

例如：用户创建一个 Pod、系统更新配置、删除命名空间等都会生成相应的审计事件。

## 审计范围（Audit Scope）

审计范围定义了需要被记录的操作类型范围，在平台中主要分为两类：

- **用户操作**：记录所有登录平台用户执行的操作
- **系统操作**：记录以 `system:` 开头的系统账号执行的操作

## 审计记录（Audit Record）

审计记录是单个审计事件的具体内容，包含以下核心属性：

- **操作人**：执行操作的账号名称
- **操作类型**：如创建、更新、删除、管理等
- **资源信息**：包含资源名称、类型、所属集群和命名空间
- **操作结果**：通过 API 返回码判断（2xx 为成功）
- **时间戳**：操作执行的具体时间
- **客户端信息**：如操作来源的 IP 地址

## 审计策略（Audit Policy）

审计策略定义了系统如何记录和处理审计事件，包括：

- 需要记录哪些类型的操作
- 记录的详细程度
- 数据保留时间
- 查询条件的设置（如时间范围、操作类型等）

例如：可以通过审计策略配置只记录特定命名空间或特定类型资源的操作。
