# 核心概念

## 合规管理

合规管理是一个基于多种工具实现的功能，用于确保集群中的配置和资源符合组织的安全规范。它通过在集群中安装特定的插件，结合预定义的安全策略和合规策略，对下发至多个集群的配置资源进行统一管理和核查。

### 主要组件

合规管理功能基于以下三个核心组件实现：

1. **Kubernetes PSA**
   - Kubernetes 内置的合规管理工具
   - 提供基础的 Pod 安全标准
   - 支持 baseline 和 restricted 两种安全级别

2. **Gatekeeper**
   - 基于 OPA（Open Policy Agent）的策略引擎
   - 使用 ConstraintTemplate 和 Constraint 两个核心概念实现策略控制
   - 支持自定义策略和验证规则

3. **Kyverno**
   - 原生的 Kubernetes 策略管理工具
   - 通过 YAML 文件定义策略
   - 支持集群级（ClusterPolicy）和命名空间级（Policy）策略

## 合规策略

合规策略是实现安全控制的具体规则集合。一个典型的合规策略包含以下要素：

1. **策略范围**
   - 集群级别或命名空间级别
   - 适用的资源类型（如 Pod、Deployment 等）
   - 排除的命名空间或资源

2. **执行模式**
   - enforce：强制执行
   - audit：仅审计
   - warn：仅警告
   - dryrun：仅记录不拒绝

3. **验证规则**
   - 安全上下文配置
   - 资源限制要求
   - 网络访问控制
   - 存储访问限制
