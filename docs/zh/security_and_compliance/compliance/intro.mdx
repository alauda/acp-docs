# 介绍

## 合规管理介绍

合规管理是平台提供的一个专门用于确保集群安全和规范性的核心功能模块。它基于 Kubernetes PSA、Gatekeeper、Kyverno 等主流合规管理工具，通过在集群中部署相应的插件，结合预定义的安全策略和合规策略，实现对多集群配置资源的统一管理和合规性核查。通过合规管理，您可以有效确保集群中的所有资源配置都符合组织的安全规范要求。

## 合规管理优势

合规管理模块具有以下核心优势：

- 多工具集成
  
  集成了 Kubernetes PSA、Gatekeeper、Kyverno 三大主流合规管理工具，为用户提供全方位的合规管理能力

- 分级安全管控
  
  提供 P0（必须实施）、P1（强烈建议实施）、P2（建议实施）三个安全等级的策略，满足不同场景的安全需求

- 灵活的策略管理
  
  支持集群级和命名空间级的策略管理，可以根据需要设置不同的执行模式（强制执行、审计、警告等）

- 统一的管理界面
  
  提供直观的界面查看各集群的合规状态，包括违规资源和合规规则的详细信息

## 合规管理应用场景

合规管理的主要应用场景如下：

- 安全基线管理
  
  通过预定义的安全策略，确保所有集群都满足基本的安全要求，如禁止特权容器、限制容器权限等

- 资源规范管理
  
  强制要求资源配置符合组织规范，如强制设置资源限制、健康检查等

- 多集群统一管控
  
  对多个集群实施统一的安全策略，确保所有集群都符合相同的安全标准

- 合规性审计
  
  通过审计模式监控集群中的违规行为，及时发现和处理安全隐患

## 合规管理使用限制

使用合规管理功能时，需要注意以下限制：

- 需要在目标集群中部署相应的合规管理组件
- 部分高级策略可能会影响集群的性能
- 某些系统关键命名空间（如 kube-system）可能需要特殊的策略豁免
- 策略的执行可能会影响现有工作负载的部署，建议先使用审计模式进行测试
