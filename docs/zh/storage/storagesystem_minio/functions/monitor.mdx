
# 监控与告警  

对象存储提供了开箱即用的监控指标采集和告警提醒能力，可从存储集群、存储服务等方面进行监控和告警，且支持配置通知策略。直观呈现的监控数据可用于运维巡检或性能调优提供决策支持，完善的告警和通知机制也将帮助保障存储系统的稳定运行。  

## 监控  

平台默认会收集对象存储的集群和存储服务状态。在 **存储管理** > **对象存储** 的 **监控** 页签中，可查看各项指标的实时监控数据。  

### 存储概览  

在存储概览部分，用户可以查看存储系统的健康状态、存储服务的运行情况以及集群裸容量的使用情况。如果存储状态异常，系统会提供告警信息，帮助用户分析问题并采取相应措施。  

### 集群监控  

该部分展示了存储集群的裸容量使用情况，以及集群的读写速率变化趋势。通过这些监控数据，用户可以判断存储资源的使用情况，并及时调整配置，以优化存储性能。  

### Object 监控  

Object 监控用于统计对象存储的访问情况，包括总访问请求次数和错误访问请求次数。通过对这些指标的监控，可以分析存储系统的负载情况，并及时发现异常请求，提高数据访问的可靠性。  

## 告警  

平台默认启用了多种告警策略，一旦资源异常或监控数据达到预警状态，系统会自动触发告警。预置策略涵盖了存储组件和集群状态告警、设备容量告警以及用户数据告警等常见运维需求。  

### 配置通知  

为了确保告警信息能够及时送达，建议用户在 **运维中心** 设置通知策略。可以通过邮件、短信等方式将告警信息发送给相关人员，提醒他们采取必要措施，防止问题扩大或避免故障发生。用户可在 **运维中心** 进行通知策略的详细配置。  

### 处理告警  

* **存储集群处于 "告警" 状态**：表示当前已触发告警，且相关异常可能导致故障。用户应及时查看 **实时告警** 详情，并结合故障原因进行定位和排障。  
* **存储集群处于 "故障" 状态**：表示存储集群已无法正常运行。此时应立即进行问题排查，以恢复系统的正常运行。  

下表列出了平台默认的告警等级及其含义，可用于制定告警处理原则：  

| 告警等级 | 含义 |
| -------- | ------------------------------------------------------------ |
| **灾难** | 资源发生严重故障，导致平台业务中断或数据丢失，影响重大。 |
| **严重** | 资源存在已知问题，可能导致功能故障，影响业务正常运行。 |
| **警告** | 资源运行存在风险，如不及时处理，可能影响业务正常运行。 |  

### 故障复盘  

**告警历史** 记录了所有曾经触发但当前已不需要处理的告警。借助告警历史进行故障复盘时，可以重点关注以下问题：  

1. 事故发生时的具体异常情况是什么？  
2. 某条告警是否反复出现？是否有规律可循？是否可以提前预防？  
3. 监控数据是否显示某个时段的告警激增？是不可抗力因素，还是由于运维事故导致？是否需要调整运维方案？  

通过分析这些问题，用户可以不断优化存储系统的稳定性，提高运维效率，减少潜在风险。
