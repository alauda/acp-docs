# 架构

**Alauda Container Platform (ACP) Object Storage with MinIO** 是一个高性能、分布式对象存储系统，适用于云原生环境。MinIO 通过纠删码、分布式存储池等机制，实现高可用性和数据可靠性。

**关键组件：**

- **MinIO Operator**：负责 MinIO 集群的部署和升级。
- **MinIO Peer**：配置和管理 MinIO 的站点复制功能。
- **MinIO Pool**：MinIO 服务器池，MinIO 的核心组件，负责处理对象存储请求。每个服务器池对应一个 StatefulSet，提供存储资源。

**部署架构：**

在 Kubernetes 上部署 MinIO 时，需定义 MinIO 租户，指定服务器数量（容器）和每个服务器的卷数（驱动器）。MinIO 服务器通过 StatefulSet 进行管理，每个服务器实例附带持久卷。所有驱动器被聚合为一个或多个纠删集，并通过纠删码保护数据。

**多服务器池扩展：**

MinIO 支持通过多个服务器池扩展集群，每个服务器池有自己的纠删集。这种扩展方式虽然提供了更大的存储能力，但增加了维护的复杂性，并降低了 MinIO 集群的可靠性。因为如果任意一个服务器池离线，可能会导致整个 MinIO 集群无法访问，即使同集群内的其他存储池仍然正常运行。

总而言之，MinIO 提供了一种高效的对象存储解决方案，但在架构设计时，需要综合考虑存储池划分、纠删码配置以及高可用性策略，以确保数据安全和系统稳定性。
