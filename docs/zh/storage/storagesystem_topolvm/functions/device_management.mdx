# 设备管理

无论是初次部署还是资源扩容，您都需要将节点上的可用磁盘映射成存储设备以便使用和管理。

同样特性的存储设备通常被集中使用，本地存储中的这些设备被归类为 **设备类**。使用设备类与直接使用磁盘无异，这样可以确保零损耗和高性能，同时降低应用对特定设备的感知和依赖。

<img src="../assets/topolvmdevice.png" width="600" />

## 前提条件

- 在创建本地存储集群时，必须添加至少 1 个 [设备类（deviceClasses.classes）](/storage/storagesystem_topolvm/installation.mdx)，并且设备类中需包括设备。

- 节点上必须至少有 1 块裸盘。

## 添加设备

1. 进入 **平台管理**。

2. 在左侧导航栏中，单击 **存储管理** > **本地存储**。

3. 在 **详细信息** 页签中，单击 **添加存储节点**。

4. 参考以下说明配置相关参数。

   | 参数              | 说明                                                                                                                                                                                              |
   | ------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
   | **存储节点**       | 至少具备 1 块裸盘的节点。                                                                                                                                                                        |
   | **设备类**         | 每个设备类对应一组具有相同特性的存储设备；建议根据磁盘的性质命名，例如 *hdd*、*ssd*。                                                                                                          |
   | **存储设备**       | 例如 */dev/sda*。如果有多个磁盘，可以逐个添加。 <br /> <br /> <b>注意</b>：存储设备应是整个硬盘，而不是硬盘上的某个分区，否则会导致错误。                                                  |

5. 单击 **添加**。

   **说明**：如果设备类状态为 `不可用` 是由于未添加任何设备，您可以继续进行以下操作。

6. 切换到 **存储设备** 页签，单击 **添加存储设备**。

7. 根据界面提示添加设备。

8. 单击 **添加**。
