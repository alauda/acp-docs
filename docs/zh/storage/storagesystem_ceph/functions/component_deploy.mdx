# 组件定点部署


在完成分布式存储的创建后，仍可查看并修改组件部署的位置，便于进行存储的扩展和维护。

## 更新组件部署配置

### 注意事项

* 更新配置会引发系统自动重建组件实例，可能会影响到存储系统的服务访问，建议在空闲时间进行更新操作。

* 当集群类型为 **延伸** 时，不支持使用组件定点部署功能。

### 操作步骤

<Steps>

1. 进入 **平台管理**。

2. 在左侧导航栏中，单击 **存储管理** > **分布式存储**。

3. 在 **存储组件** 页签下，单击 **组件部署配置**。

4. 根据业务需求打开/关闭 **组件定点部署** 开关，将组件定点部署至指定节点中，节点数需不少于三个，以保证最小可用性，定点部署配置生效的组件包括 MON、MGR、MDS、RGW。
    
5. 单击 **更新**，组件将开始调度至指定节点。

</Steps>

## 重启存储组件

将已部署的存储组件删除，系统将按照当前组件部署策略，自动重新调度至节点并重新部署组件。

### 操作步骤

<Steps>

1. 进入 **平台管理**。

2. 在左侧导航栏中，单击 **存储管理** > **分布式存储**。

3. 在 **存储组件** 页签下，在组件名称右侧单击 ⋮ > **删除**。

</Steps>
