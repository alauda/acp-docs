# 介绍


**Alauda Container Platform (ACP) Storage with Ceph** 是平台提供的集群内超融合存储方案。基于开源的 Rook + Ceph 存储方案，分布式存储实现了自动管理、自动扩容、以及自动修复的分布式存储能力，可满足中小规模的块存储、文件存储和对象存储需求。

:::note
本文中所指 **分布式存储** 指本集群中的 Ceph 存储，**外部存储** 指本集群外的 Ceph 存储。
:::

## 功能概览

* **简易部署**：提供图形化的存储集群自动部署和管理服务；支持存算一体/存算分离两种部署模式。

* **专业运维**：提供持久卷快照备份和克隆新卷功能；可视化容量、性能和组件级监控；提供内置告警策略，满足大部分存储运维场景的需求。

* **安全可靠**：分布式和多副本机制保障数据安全可靠；简单可靠的自动化管理，支持在线扩容存储资源。

* **优异性能**：提供弹性和高性能存储服务；支持混合硬盘设备部署提升存储系统性能和效率。


## 存储方案对比

平台支持以下两类存储方案，您可选择也仅能选择一种方案。

### 创建存储集群

|要求| 优点                                                       |
| ------------------------------------------------------------ | ---------------------------------------------------------- |
|**可选择创建[标准类型集群](./installation/create_service_stand.mdx)或[延伸类型集群](./installation/create_service_extend.mdx)**| 无需额外准备存储方案，在业务集群上就能完成配置，节约成本。 |

### [接入外部存储](./functions/access_storage_service.mdx)

方式一：接入平台内其他业务集群的分布式存储资源，确保存储和业务隔离便于管理和维护。

方式二：将外置 Ceph 存储资源接入为分布式存储使用。

|要求（任选其一）| 优点                                                         |
| --------------------------------------------- | ------------------------------------------------------------ |
|**方式一：已在其他业务集群中部署了分布式存储。**| 可跨集群充分利用存储资源，且避免存储服务受到业务变化干扰。保障数据安全及稳定的同时，还降低了运维复杂程度。<br /><br /> **说明**：如果待接入的是不同平台中的分布式存储，例如容灾环境中的主备平台，请使用接入外置 Ceph 方式。 |
|**方式二：已有平台以外的外置 Ceph 存储，版本 ≥ 14.2.3。**| 相比直接创建存储类，此方式更便于使用平台提供的界面化卷快照、扩容等功能。|

**说明**：如果需要维护外部存储的存储池、存储设备等配置，需在存储集群的管理界面操作。
