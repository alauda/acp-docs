# 介绍

**Alauda 容器平台 (ACP) 的 Ceph 存储** 是平台在集群内提供的超融合存储解决方案。基于开源的 Rook + Ceph 存储方案，分布式存储实现了自动管理、自动扩容与自动修复能力，能够满足中小规模应用的块存储、文件存储和对象存储需求。

:::note
本文中所称的 **分布式存储** 指的是本集群内的 Ceph 存储，而 **外部存储** 指的是本集群外的 Ceph 存储。
:::

## 功能概览

- **简易部署**：提供图形化的存储集群自动部署与管理服务；支持集成和解耦的计算与存储部署模式。

- **专业运维**：提供持久卷快照备份和克隆新卷的功能；可视化监控容量、性能及组件级别；配备内置告警策略以满足大多数存储运维场景的需求。

- **安全可靠**：分布式和多副本机制确保数据的安全与可靠；简单可靠的自动化管理支持存储资源的在线扩展。

- **优异性能**：提供弹性和高性能的存储服务；支持混合磁盘设备的部署以提升存储系统的性能和效率。

## 存储方案对比

平台支持以下两种存储方案，您可选择其中之一。

### 创建存储集群

| 需求                                                                                                                                                                                  | 优点                                                                                                                   |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------- |
| **您可以选择创建[标准类型集群](./installation/create_service_stand.mdx)或[扩展类型集群](./installation/create_service_extend.mdx)**                                                     | 无需额外准备存储方案；配置可在业务集群上完成，从而节省成本。                                                                                                 |

### [接入外部存储](./functions/access_storage_service.mdx)

选项一：接入平台内其他业务集群的分布式存储资源，确保存储和业务隔离，从而便于管理与维护。

选项二：将外部 Ceph 存储资源接入为分布式存储使用。

| 需求（任选其一）                                                         | 优点                                                                                                                                                                             |
| ------------------------------------------------------------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **选项一：已在其他业务集群中部署了分布式存储。**                            | 可以充分利用跨集群的存储资源，避免因业务变动对存储服务产生干扰。确保数据的安全与稳定，同时降低了运维的复杂性。<br /><br /> **说明**：如果待接入的存储是来自不同平台的分布式存储，例如灾备环境中的主备平台，请使用接入外部 Ceph 的方法。                         |
| **选项二：外部 Ceph 存储，在平台外，版本 ≥ 14.2.3。**                      | 相较于直接创建存储类，此方式更便利于利用平台提供的界面实现卷快照、扩容等功能。                                                                                                               |

**说明**：如果需要维护外部存储的存储池、存储设备等配置，则必须在存储集群的管理界面进行操作。
