# 监控告警

内嵌面板中的监控数据可用于从资源、性能等方面进行 Redis 监控与告警，且支持配置通知策略。

直观呈现的监控数据可用于为运维巡检或性能调优提供决策支持，完善的告警和通知机制也将帮助保障数据库稳定运行。

## 监控

平台默认收集 Redis 的资源、性能等常用监控指标。在实例的 **监控** 页签中，可查看指标的实时监控数据。



| 类别     | 指标                                                         |
| -------- | ------------------------------------------------------------ |
| **资源监控** | 内存使用量、网络流量等                           |
| **性能监控** | 连接数、网络 I/O 等 |
| **其他** | 键数目统计、命令统计等 |


## 告警

支持使用平台通用的告警功能。进入 Container Platform 的 **告警** > **告警策略** 页面，可为 Redis 创建以下告警对应的告警策略。

| 类型       | 监控对象                                                     |
| ---------- | ------------------------------------------------------------ |
| **指标告警**   | 预置的资源指标，且仅针对容器组层面，例如 `容器组的内存使用率` 。 |
| **日志告警**   | Redis 日志。                                                 |
| **事件告警**   | 事件原因，即 Redis 有状态副本集转换到当前状态的原因，例如：`BackOff`、`Pulling`、`Failed`。 |
| **自定义告警** | Redis 性能指标。                                               |

* **配置告警策略**：为启用告警，请先在 Container Platform 中创建告警策略。告警策略描述了您希望监控的对象、在什么情况下希望收到告警，以及如何被通知相关告警。
同时也建议您参考最佳实践 [配置自定义告警指标]() 。


* **处理告警**：当界面提示有处于告警状态的策略时，请及时处理。

* **绑定通知策略**：除了查看实时提醒外，平台还支持将告警信息以邮件、短信等方式发送给相关人员，通知其采取必要的措施解决问题或避免故障发生。通知策略需在平台管理中设置。
