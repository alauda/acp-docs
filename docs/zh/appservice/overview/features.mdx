# 功能总览

## 统一运维管理

- 一站式运维管理

  提供统一的运维管理界面，支持跨集群的服务生命周期管理、可观察性、变更管理、备份恢复、巡检等全方位功能。

## 丰富的应用服务支持 \{#compatible_sc}

- 支持多类型的应用服务，包括数据库、缓存、消息队列等，以下为主要支持的应用服务版本与适配存储类信息：

  | 应用服务 | 版本 | 数据持久化适配的存储类 | 备份适配的存储类 | 默认调度规则（单个实例） |
  | --- | --- | --- | --- | --- |
  | MySQL-PXC | 5.7 | TopoLVM（推荐） | S3（推荐）、TopoLVM | 每个节点上，最多（required）只能调度一个 MySQL 节点。 |
  | MySQL-MGR | 8.0 | TopoLVM（推荐） | S3（推荐） | 每个节点上，最多（required）只能调度一个 MySQL 节点。 |
  | Redis | 5.0、6.0、7.2（推荐） | TopoLVM（推荐） | S3（推荐）、CephFS、NFS | 集群模式：每个节点上，最多（required）只能调度一个 Redis 节点。  哨兵模式：每个节点上，最多（required）只调度一个 Redis 数据节点或一个 Redis 哨兵节点。 |
  | Kafka | 2.5.0、2.7.0、2.8.2、3.8（推荐） | TopoLVM（推荐） | - | 每个节点上，最多（required）只能调度一个 Broker 节点。 |
  | RabbitMQ | 3.8.16、3.12.4 | TopoLVM（推荐） | - | 每个节点上，最多（required）只能调度一个 RabbitMQ 节点。 |
  | PostgreSQL | 11、12、14 | TopoLVM（推荐） | S3（推荐） | 每个节点上，尽量调度一个 PostgreSQL 节点。 |

- 支持各服务的运维管理，以下为能力详细介绍：

<Tabs>
<Tab label="MySQL-PXC">
- 实例管理

  支持对实例进行生命周期管理（创建、更新、升级、删除、重启、停止）。

- 参数配置

  预置生产可用的参数模板，支持参数模板的创建与管理；支持用参数模板初始化参数，支持参数更新与导入导出。

- 访问方式

  支持NodePort、MetaLB、ALB等多种集群内外服务暴露方式，支持一键复制连接信息，并给出示例代码辅助。

- 拓扑

  支持展示实例的拓扑与各节点运行状态。

- 用户管理

  支持对实例的用户管理，包括用户创建、删除、权限控制等。

- 备份恢复

  支持手动备份与定时备份，支持备份到存储卷和外接 S3 存储。

- 扩缩容

  支持扩缩容操作，包括横向节点扩缩容和纵向规格扩缩容。

- 终端控制台

  支持命令行终端，方便用户快速登录节点进行操作。

- 查询分析

  支持 SQL 查询分析展示，展示包括 SQL 语句、频度、耗时等信息。

- 日志

  支持实时日志查询，包括按照时间、容器等维度进行搜索；支持日志持久化。

- 监控与告警

  展示监控数据，包括各种资源用量、连接数、TPS、表锁操作、慢查询等；支持告警策略的创建和更新，预置了包括 InnoDB 读写请求数、连接数、线程数等十项指标；支持上百项监控指标的告警设置（非预置）。

- 巡检

  支持实例巡检，支持手动和自动触发；支持展示巡检结果统计信息；支持巡检报告导出。

- 容灾

  支持热备容灾，支持容灾集群的异步和版本同步复制。

- 网络协议栈支持

  支持 IPv4/IPv6 双栈、IPv4 单栈、IPv6 单栈。
</Tab>
