# 停止/启动实例

## 功能简介
提供实例的临时停止和重新启动功能，用于资源调度维护、成本节约等场景。停止实例将保留所有配置和数据但释放计算资源。

## 前提条件
停止实例要求实例为 **处理中**、**运行中**、**异常** 或 **未知** 状态。

启动实例要求实例为 **停止中** 或 **已停止** 状态。

## 注意事项
建议在停止实例前先做一次全量备份，防止数据丢失。

## 操作步骤
<Tabs>
<Tab label="命令行操作">
#### 停止实例
```bash
kubectl patch mysqlcluster $name -n $namespace --type='merge' -p '{"spec":{"pause":true}}'
```

#### 启动实例 
```bash
kubectl patch mysqlcluster $name -n $namespace --type='merge' -p '{"spec":{"pause":false}}'
```
</Tab>

<Tab label="界面操作">
#### 停止实例
1. 在左侧导航栏选择 **MySQL-MGR**
2. 进入目标命名空间
3. 选择目标实例
4. 点击 **操作 > 停止**
5. 确认操作并等待状态变为「已停止」

#### 启动实例
1. 在已停止实例的操作菜单中点击 **启动**
2. 确认资源配置
3. 等待状态变为「运行中」
</Tab>
</Tabs>

## 注意事项
1. 停止实例将终止所有数据库连接
2. 持久化存储卷(PVC)会继续保留
3. 重启后需验证集群拓扑完整性
