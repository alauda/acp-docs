# 核心概念

## Group Replication

Group Replication 是 MySQL 提供的一种高可用解决方案，它允许多个 MySQL 服务器组成一个复制组，实现数据的同步复制。在 Group Replication 中，每个服务器都可以独立接受写入操作，并将这些操作复制到组内的其他服务器，从而实现数据的一致性。

Group Replication 支持两种模式：单主模式和多主模式。在单主模式下，只有一个服务器接受写入操作，其他服务器作为只读副本；在多主模式下，所有服务器都可以接受写入操作。MySQL-MGR 默认使用单主模式，以确保数据的一致性和避免写冲突。

## 参数模板

参数模板是一组预定义的 MySQL 配置参数集合，用于简化数据库实例的配置过程。通过使用参数模板，您可以快速应用经过优化的配置参数，而无需手动设置每个参数。

MySQL-MGR 提供了多种预设的参数模板，针对不同的应用场景进行了优化，如通用型、内存优化型、OLTP 型等。您也可以基于现有模板创建自定义模板，以满足特定的业务需求。

## 实例规格

实例规格定义了 MySQL-MGR 实例的计算和存储资源配置，包括 CPU、内存、存储容量等。选择合适的实例规格对于数据库性能至关重要。

MySQL-MGR 支持多种实例规格，从小型开发环境到大型生产环境都有对应的配置选项。您可以根据业务需求和预算选择适合的规格，并且可以在实例运行期间调整规格，以适应业务的变化。

## 副本数

副本数指的是 MySQL-MGR 集群中的节点数量。增加副本数可以提高数据的可用性和读取性能，但也会增加资源消耗和复制开销。

MySQL-MGR 建议至少配置 3 个副本，以确保在一个节点发生故障时，集群仍然能够正常运行。在生产环境中，根据高可用需求和预算，可以选择 3 到 5 个副本。

## 连接地址

连接地址是客户端应用程序连接到 MySQL-MGR 实例的网络端点。MySQL-MGR 提供了多种连接地址，以满足不同的访问需求：

- **读写连接地址**：连接到主节点，支持读写操作，适用于需要执行写入操作的应用。
- **只读连接地址**：连接到从节点，只支持读取操作，适用于读取密集型应用，可以分担主节点的读取压力。
- **集群内部连接地址**：用于集群内部的服务访问，通常具有更低的网络延迟。
- **外部连接地址**：用于集群外部的客户端访问，通常需要配置网络安全策略。

## 存储类型

存储类型定义了 MySQL-MGR 实例使用的底层存储介质和特性。不同的存储类型具有不同的性能特点和成本结构：

- **标准存储**：提供平衡的性能和成本，适合大多数应用场景。
- **SSD 存储**：提供高 IOPS 和低延迟，适合 I/O 密集型应用。
- **高性能存储**：提供最高级别的性能，适合对性能要求极高的关键业务应用。

选择合适的存储类型对于数据库性能有显著影响，应根据业务需求和预算进行选择。

## 参数配置

参数配置是指调整 MySQL 服务器的各种运行参数，以优化数据库性能和适应特定的工作负载。MySQL 有数百个可配置参数，涵盖内存使用、连接处理、查询执行、复制行为等多个方面。

在 MySQL-MGR 中，您可以通过参数模板或直接修改参数值来调整这些配置。重要的参数包括：

- **innodb_buffer_pool_size**：InnoDB 缓冲池大小，影响数据缓存效率。
- **max_connections**：最大连接数，影响服务器能够处理的并发连接数量。
- **group_replication_group_seeds**：Group Replication 的种子节点列表。
- **group_replication_consistency**：事务一致性级别，影响数据一致性保证。

合理配置这些参数可以显著提升数据库性能和稳定性。
