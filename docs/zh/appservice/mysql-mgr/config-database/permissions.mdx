## 权限说明

### 数据库实例配置权限

- **查看配置**：允许用户查看 MySQL-MGR 实例的配置信息，包括资源配置、参数设置等。
- **创建实例**：允许用户创建新的 MySQL-MGR 实例，包括设置资源规格、存储、网络等配置。
- **修改配置**：允许用户修改现有 MySQL-MGR 实例的配置，如调整参数、更改资源规格等。
- **删除实例**：允许用户删除 MySQL-MGR 实例，此操作会导致数据丢失，因此仅限于管理员角色。

### 参数模板权限

- **查看模板**：允许用户查看系统预设和用户自定义的参数模板。
- **创建模板**：允许用户创建新的参数模板，可基于现有模板或从头创建。
- **修改模板**：允许用户修改自定义参数模板的内容。
- **删除模板**：允许用户删除自定义参数模板，系统预设模板不可删除。
- **应用模板**：允许用户将参数模板应用到 MySQL-MGR 实例。

### 连接管理权限

- **查看连接**：允许用户查看数据库连接信息，包括连接地址、端口等。
- **创建连接**：允许用户创建新的连接配置，如设置连接池、SSL 等。
- **修改连接**：允许用户修改现有连接配置。
- **删除连接**：允许用户删除连接配置。

### 插件管理权限

- **查看插件**：允许用户查看已安装和可用的 MySQL 插件。
- **安装插件**：允许用户安装新的 MySQL 插件，此操作可能影响数据库性能和稳定性。
- **配置插件**：允许用户配置已安装插件的参数和行为。
- **卸载插件**：允许用户卸载已安装的插件。

## 最佳实践

1. **遵循最小权限原则**：仅为用户分配完成工作所需的最小权限集合。

2. **使用角色进行权限管理**：利用平台内置角色简化权限管理，避免直接分配权限给个人用户。

3. **定期审计权限**：定期检查用户权限分配情况，移除不必要的权限。

4. **敏感操作需审批**：对于删除实例、安装插件等敏感操作，建议实施审批流程。

5. **权限分离**：将开发环境和生产环境的权限分离，限制生产环境的高风险操作权限。
