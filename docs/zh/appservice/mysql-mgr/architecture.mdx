# 架构

![MySQL-MGR架构图]()

## 核心组件

- **MySQL Group Replication**：提供多主复制和数据一致性
- **MySQL Router**：负责读写分离和负载均衡
- **Operator**：自动化管理MySQL集群

## 数据流程

1. 客户端请求通过MySQL Router接入
2. Router根据请求类型路由到合适的MySQL节点
3. 写操作通过Group Replication同步到所有节点
4. 读操作可以路由到任意可用节点

## 部署架构

- 每个MySQL节点运行在独立的Pod中
- 使用StatefulSet保证Pod的持久性和有序性
- 通过Service暴露读写服务
