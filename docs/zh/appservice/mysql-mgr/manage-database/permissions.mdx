## 权限说明

### 实例生命周期管理权限

- **查看实例状态**：允许用户查看 MySQL-MGR 实例的运行状态、基本信息和配置。
- **启动实例**：允许用户启动已停止的 MySQL-MGR 实例，恢复数据库服务。
- **停止实例**：允许用户停止运行中的 MySQL-MGR 实例，暂停数据库服务但保留数据和配置。
- **重启实例**：允许用户重启 MySQL-MGR 实例，临时中断服务后恢复。
- **删除实例**：允许用户删除 MySQL-MGR 实例，永久移除实例及其相关资源，此操作不可逆，因此仅限于管理员角色。

### 规格管理权限

- **查看规格**：允许用户查看 MySQL-MGR 实例的规格配置，包括副本数、CPU、内存和存储容量。
- **调整副本数**：允许用户增加或减少 MySQL-MGR 集群的节点数量，影响高可用性和读取性能。
- **调整计算资源**：允许用户修改 MySQL-MGR 实例的 CPU 和内存配置，影响处理能力和性能。
- **扩展存储容量**：允许用户增加 MySQL-MGR 实例的存储容量，满足数据增长需求。

### 版本管理权限

- **查看版本**：允许用户查看 MySQL-MGR 实例的当前版本和可用的升级版本。
- **设置升级策略**：允许用户配置小版本升级策略，如自动升级、手动确认或禁止升级。
- **执行版本升级**：允许用户手动触发 MySQL-MGR 实例的版本升级，此操作可能影响服务稳定性，因此仅限于管理员角色。

### 调度管理权限

- **查看调度配置**：允许用户查看 MySQL-MGR 实例的调度配置，包括节点选择器、亲和性和容忍度设置。
- **更新节点选择器**：允许用户指定 MySQL-MGR 实例应该部署在哪些节点上，此操作可能导致实例重新调度，因此仅限于管理员角色。
- **更新亲和性设置**：允许用户控制 MySQL-MGR 实例与其他工作负载的部署关系，此操作可能影响服务可用性，因此仅限于管理员角色。
- **更新容忍度设置**：允许用户配置 MySQL-MGR 实例可以部署在带有特定污点的节点上，此操作可能影响服务稳定性，因此仅限于管理员角色。

## 最佳实践

1. **遵循最小权限原则**：仅为用户分配完成工作所需的最小权限集合。

2. **使用角色进行权限管理**：利用平台内置角色简化权限管理，避免直接分配权限给个人用户。

3. **定期审计权限**：定期检查用户权限分配情况，移除不必要的权限。

4. **敏感操作需审批**：对于删除实例、版本升级等敏感操作，建议实施审批流程。

5. **权限分离**：将开发环境和生产环境的权限分离，限制生产环境的高风险操作权限。

6. **文档记录**：记录权限分配的决策和理由，便于后续审计和调整。
