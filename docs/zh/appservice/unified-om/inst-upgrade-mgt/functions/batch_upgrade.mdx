# 实例升级

实例升级控制功能，是为了让用户能够自行选择升级实例的时间窗口，不同的实例可以选择在不同的时间窗口进行升级，以保证业务的连续性。

## 查看 Operator 列表

用户可以在实例升级控制中心，查看到当前集群已经部署的 Operator 列表。列表展示了部署模式、Operator 管理范围、Operator 状态、Operator 版本等关键信息。需要注意的是，在切换了当前项目之后，右上角的集群选择器也会跟着重置，需要用户重新选择集群才可以正常显示 Operator 列表。这样的设计是因为在多租户模型中，项目下是可以包含多个不同集群的。

Operator 列表如下图所示。

![operator-list](./assets/operator-list.png)

## 查看实例列表

用户可以在点击某个 Operator 的名称之后，进入到实例列表页面。在实例列表页面，用户可以查看到该 Operator 管理的当前项目、当前集群下的所有实例的名称、状态、版本等信息。这里需要注意的是，实例列表页面只会展示当前项目、当前集群下的实例，同时会校验项目查看的相关权限。

实例列表如下图所示。
![instance-list](./assets/instance-list.png)

## 升级实例

当用户想要升级实例时，用户可以在实例列表中，选择想要升级的实例，然后点击右上角的批量升级，即可触发对应实例的滚动更新。用户会观察到选定实例的状态会依次变成`in progress`。实例更新完成之后，实例状态会再次变成`ready`。

这里需要注意的是，根据 K8S 的特性，多个实例的升级顺序是不固定的，但最终都会变成升级后的就绪状态。
