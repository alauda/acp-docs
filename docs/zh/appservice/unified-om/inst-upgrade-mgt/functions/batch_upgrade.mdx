# 实例升级

实例升级控制功能允许用户独立选择升级实例的时间窗口。不同的实例可以在不同的时间窗口进行升级，以确保业务连续性。

## 查看操作符列表

用户可以在实例升级控制中心查看当前已部署的操作符列表。该列表显示了关键信息，如部署模式、操作符管理范围、操作符状态和操作符版本。需要注意的是，切换当前项目后，右上角的集群选择器也会重置，用户需要重新选择集群，以便操作符列表正确显示。此设计是由于多租户模型，其中一个项目可以包含多个不同的集群。

操作符列表如下面的图所示。

![operator-list](./assets/operator-list.png)

## 查看实例列表

用户点击特定操作符的名称后，可以导航到实例列表页面。在实例列表页面，用户可以查看当前项目和集群中该操作符管理的所有实例的名称、状态和版本。需要注意的是，实例列表页面仅显示当前项目和集群下的实例，并且还将验证查看项目的相关权限。

实例列表如下面的图所示。
![instance-list](./assets/instance-list.png)

## 升级实例

当用户想要升级实例时，他们可以从实例列表中选择所需的实例，然后点击右上角的批量升级按钮来触发相应实例的滚动更新。用户将观察到所选实例的状态会依次变为 `Progressing`。一旦实例更新完成，状态将变回 `Running`。

需要重点强调的是，根据 K8S 的特点，多个实例的升级顺序并不是固定的，但它们最终都会转换到升级准备就绪状态。
