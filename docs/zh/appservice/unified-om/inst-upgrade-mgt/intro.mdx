# 引言

实例升级管理中心主要用于应用服务实例的升级管理，按组件类型将实例归类于特定的操作员，并清晰展示有关实例的关键信息，例如实例名称、状态、版本和命名空间。当操作员完成一次升级后，如果有新的补丁版本可用，系统会在实例列表和详情页面自动发布通知，及时告知用户升级机会。此外，用户可以根据业务需求和计划灵活选择一个或多个实例，在适当的时间窗口内进行批量升级操作，从而实现对应用服务实例版本的高效管理。

::: info

目前的“升级管理”实现主要关注补丁版本的升级管理。如果用户希望针对某个具体实例升级到新的主要或次要版本，他们可以直接更新实例并选择所需的版本。然而，由于潜在的兼容性问题，并不是所有的主要或次要版本都支持升级。只有支持的版本会在网页控制台中显示。

:::

## 产品优势

- **智能升级通知**

在操作员升级后，系统会自动检测并提醒用户新的实例版本，确保用户不会错过任何升级机会。及时的升级能为应用服务实例提供更好的性能、更多功能和增强的安全性，为稳定的业务运营提供有力支持。

- **批量升级操作**

用户可以在适当的时间窗口内选择一个或多个实例进行批量升级，避免逐个升级实例的繁琐过程，从而节省大量时间和精力。此外，批量升级有助于统一应用服务实例的版本，减少因版本不一致而引发的兼容性问题。

- **灵活性和可控性**

用户可以根据实际业务情况选择升级的时间窗口和需要升级的实例，确保升级操作不会对业务产生不利影响。这种灵活的升级方式使用户拥有更大的控制权，使升级过程更符合业务需求。

## 使用案例

- **版本更新维护**

当组件开发者发布修复已知漏洞、增强性能或新增功能的新版本时，用户可以利用实例升级管理中心及时将应用服务实例升级到最新版本，以提升用户体验和安全性。

- **降低升级操作对业务的影响**

随着业务的发展和变化，同一集群中的应用服务实例可能具有不同的维护时间窗口。用户可根据业务需求和计划，在适当的时间窗口内选择一个或多个实例进行批量升级操作，从而避免对业务运营造成不必要的影响。
