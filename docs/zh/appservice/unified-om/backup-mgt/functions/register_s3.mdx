# 外部 S3 存储

## 功能概述

该功能为当前选定项目提供外部 S3 存储的注册服务。通过简单直观的界面，您可以轻松完成几个步骤，成功为项目注册外部 S3 存储。外部存储列表页面还集成了智能检测机制。您可以手动发起扫描以检查外部 S3 存储的可用性。如果出现连接失败等问题，系统会立即发送通知让您了解存储状态，从而实现实时掌握存储状态。当您或其他项目成员需要备份数据时，不论是创建自动备份任务还是手动触发即时备份，您只需在备份设置中轻松选择已注册的外部 S3 存储，系统会迅速而安全地将备份数据保存到该存储中。

- **增强的数据安全性**

将备份数据存储在外部 S3 存储中，为您的数据提供了额外的安全层。外部 S3 存储通常具有强大的数据冗余和灾难恢复机制，能够有效防止因本地硬件故障、自然灾害等原因导致的数据丢失。即使本地环境遭遇意外问题，您的数据仍将安全存储在外部存储中，从而确保业务连续性。

- **增加的灵活性和可扩展性**

该功能支持灵活选择备份存储，允许您根据不同的需求和场景自由分配备份资源。您可以根据性能、成本和地理位置等因素选择最合适的外部 S3 存储来存储备份数据。此外，随着业务的增长和数据量的增加，您可以轻松添加新的外部 S3 存储以扩展存储资源，适应不断变化的业务需求。

## 主要功能

### 添加外部 S3 存储

在添加外部 S3 存储之前，用户需要收集以下有关存储的信息：

- 端点 URL
- 访问密钥
- 私有访问密钥
- 存储桶名称

外部存储不需要部署在当前集群或命名空间中，只要网络连接可用即可。用户可以通过 Web 请求提前验证外部存储的可用性。要添加外部 S3 存储，只需在备份中心填写外部 S3 存储注册表单，如下图所示。

![注册外部 S3](./assets/register-s3.png)

### 查看外部 S3 存储列表

在外部 S3 存储成功注册后，它将显示在存储列表中。存储列表集成了存储状态的自动检查：当新增注册的外部 S3 存储时，会自动触发连接状态检查。用户也可以手动点击连接状态旁边的 `检查` 按钮以启动手动检查。连接状态检查的结果将在列表中直接显示，如下图所示：
