# 备份管理

## 功能简介

备份管理功能为您提供了一个直观实用的界面，只需选定项目和命名空间，即可全面查看实例备份状态，包括自动备份开关、成功与失败数量、上次备份结果及存储位置，助您快速掌握备份情况并及时发现问题。通过实例右侧菜单，可跳转至备份历史页面，清晰展示备份时间、结果、存储方式等关键信息。此外，您还能批量删除冗余备份以释放空间，或精准恢复指定备份，应对数据丢失或系统故障，确保业务连续性与稳定性。

- **增强数据可视性与把控力**

备份管理页面全面展示实例备份状态，让您轻松掌握进展、成效及存储位置，省去繁琐查找，提升管理效率。同时，您可随时调整策略，确保数据安全可靠。

- **便捷的历史追溯与问题排查**

通过备份历史功能，您可清晰回溯实例备份历程，快速查阅时间、结果等信息，精准定位问题，验证数据完整性，减少业务停滞，保障正常运行。


## 主要功能

### 查看备份列表

用户可以在备份中心下选择备份管理功能，进入备份列表页面，如下图所示。

![](./assets/backup-list.png)

用户在这个页面中，可以查看指定项目和指定命名空间下指定类型或者所有类型的实例的备份状态，包括是否已经开启自动备份、成功和失败的备份数量、最近一次备份结果等信息。

### 查看备份历史

用户选择指定实例的右侧菜单`备份历史`功能，会在弹出菜单中展示该实例的所有备份记录，如下图所示。

![](./assets/backup-history.png)

这个页面就是直接查询对应组件的备份资源了，详细信息可以访问对应组件的备份恢复文档。

### 恢复操作

用户在备份历史页面上，选择某一次备份记录后侧的菜单，点击`恢复`按钮，系统会弹出恢复方式的选项卡。不同组件可能会支持不同的恢复方式，默认都会支持恢复到新实例，MGR 还可以支持恢复到已有实例上。

![MGR 恢复方式选项卡](./assets/restore-from-backup-history.png)

- 如果用户选择恢复到新实例，系统会跳转新实例的创建页面；在用户输入了新实例相关信息（名称、规格、参数等）之后，系统会在创建完该实例之后，自动应用该备份数据进行恢复操作。等新实例的状态变成就绪之后，之前指定的备份数据就恢复成功了。

- 如果用户选择恢复到已有实例，系统会提示用户选择实例，并执行恢复操作。

### 删除备份操作

用户在备份历史页面上，既可以在右侧菜单中删除指定的备份记录，也可以选择多份不同的备份记录批量删除。注意：**此操作不可逆，请谨慎操作！**
