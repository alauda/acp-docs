# 备份管理

备份管理功能为您提供了直观且实用的界面。通过简单选择一个项目和命名空间，您可以全面查看实例的备份状态，包括自动备份切换、成功和失败备份的数量、上次备份结果以及存储位置。这使您能够快速评估备份情况，并及时识别问题。通过实例右侧的菜单，您可以导航到备份历史页面，清晰显示备份时间、结果和存储方式等关键信息。此外，您可以批量删除冗余备份以释放空间，或准确恢复指定备份以应对数据丢失或系统故障，确保业务的连续性和稳定性。

- **增强的数据可视化与控制**

备份管理页面全面展示实例备份状态，使您能够轻松掌握进度、有效性和存储位置，避免繁琐的搜索，提高管理效率。同时，您可以随时调整策略以确保数据的安全与可靠。

- **便捷的历史追踪与问题诊断**

通过备份历史功能，您可以清晰追溯实例的备份历史，快速查看时间和结果等信息，准确定位问题，验证数据完整性，减少业务中断，确保正常运作。

## 主要功能

### 查看备份列表

用户可以在备份中心选择备份管理功能以进入备份列表页面，如下图所示。

![](./assets/backup-list.png)

在此页面上，用户可以查看指定项目和命名空间的实例备份状态，包括自动备份是否已启用、成功和失败备份的数量以及最近一次备份的结果。

### 查看备份历史

通过从指定实例的右侧菜单选择 `历史` 功能，用户将看到一个弹出菜单，显示该实例的所有备份记录，如下图所示。

![](./assets/backup-history.png)

该页面允许直接查询相应组件的备份资源，详细信息可在相应组件的备份恢复文档中找到。

### 恢复操作

在备份历史页面，选中某个备份记录后，用户可以选择侧边菜单并点击 `恢复` 按钮。系统将弹出一个选项卡供恢复选项选择。不同组件可能支持不同的恢复方法；然而，默认情况下，所有组件将支持恢复到新实例，而 MGR 额外支持恢复到现有实例。

![MGR 恢复方法选项卡](./assets/restore-from-backup-history.png)

- 如果用户选择恢复到新实例，系统将重定向至新实例创建页面。在用户输入新实例的相关信息（名称、规格、参数等）后，系统将在实例创建完成后自动应用备份数据进行恢复。一旦新实例状态变为就绪，指定的备份数据将成功恢复。

- 如果用户选择恢复到现有实例，系统将提示用户选择一个实例并执行恢复操作。

### 删除备份操作

在备份历史页面，用户可以从右侧菜单删除特定备份记录或选择多个不同的备份记录进行批量删除。注意：**此操作不可逆，请谨慎操作！**
