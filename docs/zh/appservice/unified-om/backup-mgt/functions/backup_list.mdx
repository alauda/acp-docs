# 备份管理

## 功能简介

在我们精心打造的备份管理功能页面中，为您呈现了一个清晰且极具实用性的信息展示与操作界面。您只需轻松选定项目以及对应的命名空间，就能全面且直观地查看某一类或者全部实例的备份状态。这里涵盖了诸如是否已开启自动备份这一关键设置，方便您快速知晓实例是否处于自动备份保障之下；同时，备份成功和失败的数量也一目了然，让您能够在第一时间把握备份任务的整体执行情况，及时发现潜在问题；上次备份结果可以提示您实例的最新数据是否处于保护中；而存储位置信息的展示，则确保您能确切了解备份数据的存放之处，便于后续的管理与追溯。

不仅如此，我们还为您提供了便捷的操作入口，您只需轻轻点击指定实例右侧菜单，即可跳转至其备份历史页面。在这个专属的备份历史页面里，所有您关心的重要信息都以一种清晰明了的方式呈现出来。您能够一眼看清该实例在具体什么时间进行了备份、对应的备份结果如何、采用了何种存储方式以及备份所属的类型等关键细节。

更为贴心的是，在此页面上，您拥有强大的操作权限。既可以批量删除那些不再需要的备份历史记录，轻松释放存储空间，避免数据冗余，又能够精准选定某一次备份结果进行恢复操作，无论是应对数据丢失还是系统故障等突发情况，都能迅速将数据恢复至期望的备份状态，保障业务的连续性与稳定性。

- **增强数据可视性与把控力**

通过在备份管理功能页面全面展示各类备份状态信息，您可以对整个项目下实例的备份情况有宏观且细致的把控。不再需要在繁多的数据和复杂的系统中费力查找，就能快速了解各个实例的备份进展、成效以及存储位置等关键要素，这极大地提升了您对数据备份工作的管理效率，让您能够像掌控全局的指挥官一样，随时根据实际情况调整备份策略，确保数据始终处于安全可靠的备份保护之中。

- **便捷的历史追溯与问题排查**

点击实例查看备份历史的功能，为您提供了一条清晰的 “时光隧道”，方便您回溯每个实例的备份历程。当遇到数据相关的疑问或者故障时，您可以迅速查阅备份历史，依据详细的备份时间、结果、存储方式等信息，精准定位问题所在，比如排查出是哪一次备份出现异常，或者验证某一阶段的数据完整性等，为快速解决问题、保障业务正常运行提供了有力支撑，有效减少因数据问题引发的业务停滞时间。


## 主要功能

### 查看备份列表

用户可以在备份中心下选择备份管理功能，就可以进入备份列表页面，如下图所示。

![](./assets/backup-list.png)

用户在这个页面中，可以一目了然的查看指定项目和指定命名空间下指定类型或者所有类型的实例的备份状态，包括是否已经开启自动备份、成功和失败的备份数量、最近一次备份结果等信息。

### 查看备份历史

用户选择指定实例的右侧菜单`备份历史`功能，就会在弹出菜单中展示该实例的所有备份记录，如下图所示。

![](./assets/backup-history.png)

这个页面就是直接查询对应组件的备份资源了，详细信息可以访问对应组件的备份恢复文档。

### 恢复操作

用户在备份历史页面上，选择某一次备份记录后侧的菜单，点击`恢复`按钮，系统会给用户两个恢复选项：恢复到新实例和恢复到已有实例。

![](./assets/restore-from-backup-history.png)

- 如果用户选择恢复到新实例，系统会跳转新实例的创建页面；在用户输入了新实例相关信息（名称、规格、参数等）之后，系统会在创建完该实例之后，自动应用该备份数据进行恢复操作。等新实例的状态变成就绪之后，之前指定的备份数据就恢复成功了。

- 如果用户选择恢复到已有实例，系统会提示用户选择实例，并执行恢复操作。

### 删除备份操作

用户在备份历史页面上，选择某一次备份记录右侧的菜单，点击`删除`按钮，系统会弹出确认框，用户确认之后，系统会删除该备份记录。用户也可以在备份历史页面上，选择多个备份记录，点击`批量删除`按钮，系统会弹出确认框，用户确认之后，系统会删除这些备份记录。如果用户想要详细了解通过 CLI 删除备份记录的方式，请参考对应组件的备份恢复功能或者 API 文档。
