# 介绍

在实例操作与维护过程中，可以使用警报来通知需要特别关注的异常或状态。当实例遇到异常或达到警告状态时，可以自动触发警报，帮助您及时发现和定位问题。

为提升系统运营与维护效率，平台基于用于故障排查的监控指标建立了警报规则，将其分类并整合为内置的警报策略。

平台支持基于预定义监控指标和用户定义监控指标的警报规则。当发生资源异常或达到警告状态时，可以自动触发警报。结合平台的通知功能，警报信息可以主动推送给运维人员，提醒他们及时处理警报资源，从而确保平台业务的顺利运行。

为方便对平台上大量资源设置警报，平台支持通过警报模板定制标准化的警报配置，以便快速基于现有模板为资源创建警报策略。

## 警报类型

- **指标警报**：平台提取的常见监控指标，满足大多数客户需求。您可以通过选择监控指标并设置触发条件来配置警报。当监控数据满足警报规则的触发条件时，将触发警报。

- **自定义警报**：您可以根据实际使用案例添加特定于企业的指标规则，以更好地满足企业在指标警报基础上的高级警报需求。

## 状态描述

- 警报状态

  - **警报**：从查询警报规则配置获得的数据满足触发条件，并已触发警报。

  - **待处理**：从查询警报规则配置中获得的监控数据大于或等于*警报阈值*，但持续时间尚未满足触发条件，表示在触发警报之前的临界状态。例如：如果警报规则的触发条件是`CPU 使用率超过 80% 并持续 3 分钟`，当系统首次检测到 CPU 超过 80% 时，将标记该规则为`待处理`。系统将继续评估，如果 CPU 使用率在 3 分钟内持续超过 80%，该规则的状态将改变为**警报**；如果在以后的评估中 CPU 使用率降至 80% 以下，该规则的状态将恢复为**正常**。

  - **正常**：从查询警报规则配置获得的数据未满足警报阈值。

- 静默状态

  - **静默待处理**：设置静默后开始静默之前的状态。在此状态下，当策略中的规则触发警报时，会正常发送通知。

  - **静默**：从静默开始直到静默结束的状态。在此状态下，当策略中的规则触发警报时，将不发送通知。

## 实时警报

平台通过实时警报显示当前处于警报状态的资源数量和详细的警报信息，使运维人员和管理员能够实时了解平台业务的整体警报状况，及时识别和处理故障，以确保平台正常运行。

## 警报历史

为了帮助运维人员和管理员分析最近的监控警报情况和追溯历史问题，平台支持查看一段时间内的历史警报记录。可查看的历史信息包括发生时间、警报规则、故障资源、通知方式、状态和通知接收人等。

**注意**：警报历史的保留时间与事件的保留时间相同，并且可以在**平台管理**部分通过修改日志存储组件的配置参数进行更新。
