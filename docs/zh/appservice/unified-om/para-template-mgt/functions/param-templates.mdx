# 参数模板管理

## 功能简介

参数模板功能用于统一管理和维护各个组件的参数集合，包括查看参数模板都包含哪些参数，各个参数的默认值和描述信息等。目前支持参数模板的组件有 MySQL（包括PXC和MGR）、Redis 和 Kafka。平台出厂预置了针对各个场景、组件规格的参数模板，代表了在该场景下的业界最佳实践，方便用户开箱即用。同时，用户也可以根据自己的实际业务需求，创建自定义的参数模板，方便企业内部规范落地。所有参数模板都可以在创建实例时选择使用。

## 主要功能

### 查看参数模板列表

用户可以点击参数模板进入列表页面，如下图所示。

![](./assets/param-template-list.png)

用户在这个页面中，可以按照组件类型和模板类型查看指定项目的参数模板。组件类型目前支持 MySQL、Redis 和 Kafka。模板类型目前支持平台预置的模板和用户自定义的模板。平台预置的模板是整个集群范围内通用的，而用户可以为当前项目创建专门的自定义模板。

::: info
MySQL 的参数模板可以同时适用于 PXC 架构和 MGR 架构。
:::

### 查看指定参数模板详情

用户点击某个模板会进入模板详情页面，如下图所示。

![](./assets/param-template-detail.png)

这个页面会展示该模板的详细信息，包括模板名称、模板类型、组件类型、模板描述、参数列表、参数默认值等信息。

### 创建参数模板

在权限允许的情况下，用户可以点击左上角的 "创建参数模板" 按钮，进入创建参数模板页面。如下图所示。

![](./assets/create-param-template.png)

在这个页面中，用户可以填写模板名称、模板类型、组件类型、模板描述等信息，然后逐一指定该模板包括的参数和参数的默认值。由于每个实例在创建的时候，只能指定一个参数模板，这就要求参数模板要包括尽可能完整的参数列表。为了方便用户录入参数，平台支持批量导入参数以及从文档导入参数功能。

::: info
批量导入参数格式：
```
参数名1 参数值1
参数名2 参数值2
...
```
:::

### 导出参数模板

用户在参数模板列表页的右侧菜单或者参数模板详情页的下拉操作菜单中，选择 "导出" 选项，即可导出参数模板的配置信息。导出文件格式是和导入格式一致的文本文件。
