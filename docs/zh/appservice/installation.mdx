# 部署

## 安装 Alauda 应用服务 Essentials 插件

::: tip 可选部署
该插件是应用服务的基本组件，具有图形用户界面。如果不需要 web 控制台，则可以跳过此步骤。
:::

### 前提条件

1. 下载与平台兼容的 Alauda 应用服务 Essentials 安装包。
2. 将 Alauda 应用服务 Essentials 上传到 global 集群。

### 步骤

<Tabs>
  <Tab label="Web 控制台">
    1. 在左侧导航栏中，转到 **应用商店管理** > **集群插件**。
    2. 选择 **global** 集群。
    3. 点击 **Alauda 应用服务 Essentials** 插件右侧的操作按钮 > **部署**。
  </Tab>
</Tabs>

## 安装 Alauda 应用服务 RDS 框架插件

::: tip 可选部署
该插件是应用服务的基本组件，具有图形用户界面。如果不需要 web 控制台，则可以跳过此步骤。
:::

### 前提条件

1. 下载与平台兼容的 Alauda 应用服务 RDS 框架安装包。
2. 将 Alauda 应用服务 RDS 框架上传到任何希望利用应用服务图形功能的集群。

### 步骤

<Tabs>
  <Tab label="Web 控制台">
    1. 登录平台并转到 **平台管理** 页面。
    2. 在左侧导航栏中，选择 **应用商店管理** -> **操作员**，进入 **操作员中心** 页面。
    3. 查找 **Alauda 应用服务 RDS 框架**，点击 **部署** 并进入部署页面。

    配置参数：

    | **参数**            | **推荐配置**                                                                                                                       |
    | :------------------- | :------------------------------------------------------------------------------------------------------------------------------- |
    | **升级策略**        | **手动**：当操作员中心有新版本时，需要手动确认以将操作员升级到最新版本。                                                             |
    | **部署模式**        | **集群**：集群下所有命名空间共享一个操作员来创建和管理实例，减少资源消耗。                                                            |
    | **渠道**            | 默认渠道为 **稳定**。                                                                                                              |
    | **命名空间**        | 选择 **推荐命名空间**：如果不存在，则自动创建。                                                                                   |

    4. 在 **部署操作员** 页面，选择 **默认配置**，点击 **部署**，完成 **Alauda 应用服务 RDS 框架** 的部署。
  </Tab>
</Tabs>

### 前提条件

1. 下载与平台匹配的插件安装包。
2. 将插件上传到任何希望利用应用服务业务组件功能的集群。

<Tabs>

    1. 登录平台并转到 **平台管理** 页面。
    2. 在左侧导航栏中，选择 **应用商店管理** -> **操作员**，进入 **操作员中心** 页面。
    3. 查找 **Alauda MySQL 数据库服务**，点击 **部署**，并进入部署页面。

    | **参数**            | **推荐配置**                                                                                                                       |

    | **渠道**            | 默认渠道为 **alpha**。                                                                                                             |
    | **命名空间**        | 选择 **推荐命名空间**：如果不存在，则自动创建。                                                                                   |

    4. 在 **部署操作员** 页面，选择 **默认配置**，点击 **部署**，完成部署。
