# 直连底层子网与覆盖子网

如果集群同时具备底层子网与覆盖子网，则默认情况下，覆盖子网下的 Pods 可以通过网关使用 NAT 访问底层子网中的 Pods IP。然而，底层子网中的 Pods 需要配置节点路由才能访问覆盖子网中的 Pods。

要实现底层子网与覆盖子网之间的自动连接，可以手动修改底层子网的 YAML 文件。一旦配置完成，Kube-OVN 将使用额外的底层 IP 来连接底层子网和 ovn-cluster 逻辑路由器，并设置相应的路由规则以启用互连。

### 步骤

1. 进入 **平台管理**。

2. 在左侧导航栏中，点击 **集群管理** > **资源管理**。

3. 输入 **子网** 来过滤资源对象。

4. 点击 ⋮ > 修改需要修改的底层子网旁边的 **更新**。

5. 修改 YAML 文件，在 `Spec` 中添加字段 `u2oInterconnection: true`。

6. 点击 **更新**。

**注意**：底层子网中现有的计算组件需要重新创建才能使更改生效。
