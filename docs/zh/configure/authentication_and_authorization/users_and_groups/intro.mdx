# 介绍

## 用户组管理介绍

用户组管理是一个面向企业级用户的批量权限管理模块。它支持将多个用户组织成逻辑组，并通过角色分配实现统一的权限管理，简化了企业内大规模用户的权限配置和管理流程。通过用户组管理，您可以高效地管理企业内不同团队、部门的平台访问权限，确保安全合规的同时提升运维效率。

## 用户组管理优势

用户组管理模块具有以下核心优势：

- 批量权限管理
  
  支持将多个用户统一添加到用户组中，通过为用户组分配角色来批量管理权限，显著提升权限管理效率

- 灵活的组织方式
  
  支持本地创建用户组和从外部身份提供商同步用户组，满足不同企业的组织架构需求

- 统一的权限控制
  
  基于 RBAC（基于角色的访问控制）模型，实现清晰、可控的权限分配和管理

- 多维度的权限管理
  
  支持项目级、命名空间级等多个维度的权限配置，满足企业精细化的权限管理需求

## 用户组管理应用场景

用户组管理的主要应用场景如下：

- 团队协作管理
  
  当企业需要为项目团队配置统一的资源访问权限时，可以创建对应的用户组，将团队成员添加到组中，统一配置项目权限

- 部门权限管理
  
  针对企业各部门的差异化权限需求，可以基于部门创建用户组，为不同部门分配相应的资源访问权限

- 企业人员变动
  
  当发生批量人员调动时，可以通过调整用户组成员和权限配置，快速完成权限的分配与回收

## 用户组管理使用限制

用户组管理具有以下使用限制：

- IDP 同步的用户组
  - 不支持修改组名称
  - 不支持删除用户组
  - 不支持手动管理组成员
  - 仅支持角色权限配置

- 本地用户组
  - 组名称在平台内唯一
  - 用户可以同时属于多个用户组
  - 用户组的权限会叠加给组内所有成员
