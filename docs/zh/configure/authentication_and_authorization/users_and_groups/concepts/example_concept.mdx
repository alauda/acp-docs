# 核心概念

## 用户

用户是平台的基本管理单位，是访问和操作平台的主体。平台支持多种来源的用户账号，并对其进行统一的生命周期管理。

### 用户来源

1. **本地用户**
   - 定义：平台原生创建和管理的用户账号
   - 来源：
     - 系统自动创建的管理员账号
     - 通过平台创建的账号
     - 通过本地 dex 配置文件添加的用户

2. **第三方用户**
   - LDAP 用户：通过 IDP 同步的企业 LDAP 用户
   - OIDC 用户：通过 OIDC 协议认证的第三方平台用户
   - 其他第三方平台用户（如 GitHub、Microsoft 等）

### 用户生命周期

用户在平台中具有以下状态：

- **正常**：账号在有效期内，可正常登录使用
- **禁用**：账号被禁用（如 90 天未登录、过期或管理员手动禁用）
- **锁定**：24 小时内登录失败 5 次后的临时状态（20 分钟后自动解锁）
- **无效**：LDAP 同步时发现原账号已被删除

## 用户组

用户组是平台中用于批量管理用户权限的逻辑单位，通过用户组可以实现基于角色的批量权限管理。

### 用户组类型

1. **本地用户组**
   - 定义：通过平台直接创建的用户组
   - 特点：
     - 完全可管理（创建、更新、删除）
     - 支持任意来源用户加入
     - 可自由配置角色权限

2. **IDP 同步用户组**
   - 定义：从外部身份提供商（如 LDAP、Azure AD）同步的用户组
   - 特点：
     - 仅支持角色权限管理
     - 组信息和成员由外部系统管理
     - 不可在平台修改组信息和成员

### 应用场景

用户组主要应用于以下场景：
- 企业人员批量权限调整
- 项目团队统一权限管理
- 部门级别的资源访问控制

例如：当需要为新项目配置多个开发人员的访问权限时，可以创建对应的用户组，将相关开发人员加入组内，然后统一配置项目权限，提高权限管理效率。
