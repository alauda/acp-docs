# 项目配额管理

## 功能简介

项目配额管理是平台提供的资源使用限制管理功能。通过该功能，您可以对项目可使用的各类资源进行精确控制，包括计算资源、存储资源、GPU 资源等，确保资源的合理分配和使用。

## 主要功能

- 资源配额设置：支持设置多种资源类型的使用限制
- 配额调整：支持动态调整项目的资源配额
- GPU 资源管理：支持 GPU 相关资源的精细化配置
- 配额监控：支持查看资源配额的使用情况

## 功能优势

- 精确控制：支持多维度的资源配额管理
- 灵活调整：可根据需求动态调整配额
- 可视化监控：直观展示配额使用情况
- 资源优化：提高资源利用效率

## 配额类型

项目支持以下类型的资源配额管理：

| 资源类型 | 说明 | 单位 |
| :--- | :--- | :--- |
| CPU | 计算资源配额 | 核 |
| 内存 | 内存资源配额 | Gi |
| 存储 | 存储资源配额 | Gi |
| PVC 数 | 持久卷声明数量 | 个 |
| Pods | Pod 数量限制 | 个 |
| GPU 资源 | GPU 相关资源配额 | 核 |
| 显存 | GPU 显存配额 | Gi/Mi |

## GPU 资源说明

### 配置条件

- 仅当集群部署了 GPU 插件时支持配置
- GPU-Manager 或 MPS GPU 资源支持配置显存

### 资源换算

- GPU 核心换算：
  - 1 颗 pGPU = 1 核
  - 1 核 = 100 GPU-Manager core
  - 1 核 = 100 MPS core

- 显存单位：
  - GPU-Manager：1 单位 = 256 Mi
  - MPS GPU：1 单位 = 1 Gi
  - 换算关系：1024 Mi = 1 Gi
