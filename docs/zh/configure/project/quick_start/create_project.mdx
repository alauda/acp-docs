# 创建项目

本文档帮助新用户快速了解如何创建一个新的项目，并为项目分配合适的资源配额。

## 介绍

### 适用场景

当您需要为团队创建一个独立的项目环境时，可以通过创建项目功能，基于平台上已有的集群资源，创建一个与其他项目（租户）资源、人员相互隔离的项目空间。项目可以跨多个集群使用资源，您可以根据项目规模和实际业务需求，为项目合理分配资源配额。

### 预计阅读时间

10-15 分钟

## 注意事项

- 创建项目时，平台将在关联的集群中创建与项目同名的命名空间，用于隔离平台级别的资源，请勿操作该命名空间及其下资源
- 项目名称不可与已有项目同名
- 项目名称不可与黑名单列表中的名称同名（如：cpaas-system、cert-manager、default 等）
- 异常状态的集群不可选择

## 前提条件

- 具有创建项目的权限
- 平台上有可用的集群资源
- 了解项目所需的资源配额

## 流程概述

| 序号 | 操作步骤 | 描述 |
|------|----------|------|
| 1 | 创建项目 | 进入项目管理视图，开始创建新项目 |
| 2 | 配置基本信息 | 设置项目名称并选择关联集群 |
| 3 | 设置资源配额 | 为每个关联集群配置资源使用限制 |

## 操作步骤

### 配置基本信息

1. 在**项目管理**视图，单击**创建项目**
2. 在**基本信息**页面，配置以下参数：

| 参数 | 说明 |
|------|------|
| 名称 | 项目的名称，需确保唯一性且不在黑名单列表中 |
| 所属集群 | 选择一个或多个要关联的集群，异常集群不可选 |

### 设置资源配额

1. 单击**下一步**，进入设置项目配额环节
2. 为每个选择的集群设置以下资源配额：

| 资源类型 | 单位 | 说明 |
|----------|------|------|
| CPU | 核 | 计算资源配额 |
| 内存 | Gi | 内存资源配额 |
| 存储 | Gi | 存储资源配额 |
| PVC 数 | 个 | 持久卷声明数量限制 |
| Pods | 个 | Pod 数量限制 |
| GPU 资源 | 核 | 仅当集群部署 GPU 插件时可配置 |
| 显存 | Gi/Mi | 仅适用于 GPU-Manager/MPS GPU |

**GPU 资源说明**：
- 1 颗 pGPU = 1 核 = 100 GPU-Manager core = 100 MPS core
- GPU-Manager：1 单位显存 = 256 Mi
- MPS GPU：1 单位显存 = 1 Gi
- 换算关系：1024 Mi = 1 Gi

3. 单击**创建项目**完成创建

### 预期结果

创建成功后，您将获得：
- 一个新的项目空间
- 已配置好资源配额的关联集群
- 与项目同名的系统命名空间（用于平台管理）

## 后续操作

完成项目创建后，您可以：
- 导入项目成员
- 创建业务命名空间
- 部署应用和服务
