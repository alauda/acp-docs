# 创建 NFS 共享存储类型存储类


基于社区 NFS CSI（Container Storage Interface）存储接入插件，提供接入多个 NFS 存储系统或账号的能力。

与传统 NFS 客户端服务端模式接入的方式不同的是，NFS 共享存储使用的是基于社区 NFS CSI（Container Storage Interface）存储插件，更贴合 Kubernetes 设计理念，可以为多个服务端提供客户端的接入能力。

## 前提条件

* 已配置 NFS 服务器，并获取其访问方式。目前平台支持的 NFS 协议版本有 `v3`、`v4.0`、`v4.1` 三个版本。在服务器端您可以执行 `nfsstat -s` 查看版本信息。


## 部署 NFS 共享存储插件

1. 进入 **平台管理**。

1. 在左侧导航栏中，单击 **存储管理** > **存储类**。

2. 单击 **创建存储类**。

3. 在 **NFS 共享存储** 右侧单击部署，跳转至 **插件** 页面。

4. 在 **NFS** 插件右侧单击 ⋮ > **部署**。

5. 等待部署状态为 **部署成功** 后完成部署。

## 创建 NFS 共享存储存储类

1. 单击 **创建存储类**。
    
    **说明**：下述内容以表单方式为例，您也可选择 YAML 创建完成操作。

3. 选择 **NFS 共享存储**，单击 **下一步**。

2. 参考以下说明，配置相关参数。
    
    |参数|说明   |
    | -------- | ----------------  | 
    | **名称**   | 存储类的名称。需在当前集群中唯一。 |
    | **服务地址**   | NFS 服务器的访问地址。例如：`192.168.2.11`。 |
    | **路径**   | NFS 文件系统在服务器节点中的挂载路径。例如：`/nfs/data`。 |
    | **NFS 协议版本**   | 目前支持 `v3`、`v4.0`、`v4.1` 三个版本。 |
    | **回收策略**   | 持久卷的回收策略。<br/>- 删除：删除持久卷声明的同时，也会删除绑定的持久卷。<br/>- 保留：即使删除持久卷声明，其绑定的持久卷仍会被保留。|
    | **访问模式**   | 当前存储支持的所有访问模式。后续声明持久卷时，只能选择以其中一种模式挂载持久卷。<br/>- 单节点读写 (RWO)：可以被一个节点以读写方式挂载。<br/>- 多节点读写 (RWX)：可以被多个节点以读写方式挂载。<br/>- 多节点只读 (ROX)：可以被多个节点以只读方式挂载。|
    | **分配项目**   | 请分配可使用此类型存储的项目。<br/>如果暂时没有项目需要使用此类型存储，您也可先不分配项目，后续再更新项目。|
        
3. 确认配置信息无误后，单击 **创建**。
