# Deployment

## Install Alauda Application Services Essentials Plugin

::: tip Optional Deployment
The plugin is a basic component of Application Services with a graphical interface. It can be skipped if a graphical interface is not necessary.
:::

### Prerequisites

1. Download the Alauda Application Services Essentials installation package that matches the platform.
2. Use the platform's application listing capability to list Alauda Application Services Essentials to the ***global cluster***.

### Steps

<Tabs>
  <Tab label="Web Console">
    1. In the left navigation bar, go to **Application Store Management** > **Cluster Plugins**.
    2. Select the **global** cluster.
    3. Click the action button on the right side of the **Alauda Application Services Essentials** plugin > **Deploy**.
  </Tab>
</Tabs>

## Install Alauda Application Services RDS Framework Plugin

::: tip Optional Deployment
The plugin is a basic component of Application Services with a graphical interface. It can be skipped if a graphical interface is not necessary.
:::

### Prerequisites

1. Download the Alauda Application Services RDS Framework installation package that matches the platform.
2. Use the platform's application listing capability to list Alauda Application Services RDS Framework to any cluster that intends to utilize the graphical capabilities of Application Services.

### Steps

<Tabs>
  <Tab label="Web Console">
    1. Log in to the platform and go to the **Platform Management** page.
    2. In the left navigation bar, select **Application Store Management** -> **Operator**, and enter the **Operator Hub** page.
    3. Find **Alauda Application Services RDS Framework**, click **Deploy**, and proceed to the deployment page.

    Configuration parameters:

    | **Parameter**   | **Recommended Configuration**                                      |
    | :------------- | :------------------------------------------------------------- |
    | **Upgrade Strategy** | **Manual**: When there is a new version in the Operator Hub, manual confirmation is required to upgrade the Operator to the latest version. |
    | **Deployment Mode** | **Cluster**: All namespaces under the cluster share one Operator to create and manage instances, which reduces resource consumption. |
    | **Channel**   | The default Channel is **stable**.                                |
    | **Namespace** | Select **Recommended Namespace**: automatically created if none exists. |

    4. On the **Deploy Operator** page, select **Default Configuration**, click **Deploy**, and complete the deployment of **Alauda Application Services RDS Framework**.
  </Tab>
</Tabs>
