# TokenRequest [authentication.k8s.io/v1]

<OpenAPIPath path="/api/v1/namespaces/{namespace}/serviceaccounts/{name}/token" pathPrefix="/kubernetes/{cluster}" />

## API usage

Use this API to request a short-lived token for a ServiceAccount.

### Request example

```http
POST /kubernetes/{cluster}/api/v1/namespaces/{namespace}/serviceaccounts/{name}/token
```

Request body snippet:

```json
{
  "apiVersion": "authentication.k8s.io/v1",
  "kind": "TokenRequest",
  "spec": {
    "audiences": [
      "https://kubernetes.default.svc"
    ],
    "expirationSeconds": 3600
  }
}
```

### End-to-end example

```bash
curl -X POST -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  "${APISERVER}/kubernetes/${CLUSTER}/api/v1/namespaces/${NAMESPACE}/serviceaccounts/${SA_NAME}/token" \
  -d '{
    "apiVersion": "authentication.k8s.io/v1",
    "kind": "TokenRequest",
    "spec": {
      "audiences": [
        "https://kubernetes.default.svc"
      ],
      "expirationSeconds": 3600
    }
  }'
```

### Response example

```json
{
  "apiVersion": "authentication.k8s.io/v1",
  "kind": "TokenRequest",
  "status": {
    "token": "eyJhbGciOiJSUzI1NiIs...",
    "expirationTimestamp": "2026-05-08T08:00:00Z"
  }
}
```

### Notes

- `spec.audiences` is required.
- Use `status.token` as the bearer token.
- The issued token may have a different expiration than the requested `expirationSeconds`. Check `status.expirationTimestamp` in the response.
