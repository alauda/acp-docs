# Update Default Certificate

Typically, administrators renew or replace certificates before their expiration date. If an administrator replaces the certificate used for the HTTPS or gRPC listening port, you must update the listening port to use the new certificate as the default certificate.

## Steps to Perform

1. Go to **Container Platform**.

2. In the left navigation bar, click **Network** > **Load Balancing**.

3. Click on the name of the load balancer.

4. Click on the name of the listening port.

5. Click **Actions** > **Update Default Certificate**.

6. Select the new certificate and click **Update**.
