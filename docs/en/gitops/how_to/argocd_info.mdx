# How to Obtain Argo CD Access Information

> This document describes how to obtain access information for the Argo CD Dashboard, including the Alauda GitOps cluster plugin Argo CD deployed on the platform and through the Argo CD Operator.

## Use Cases

- Once Argo CD access information is obtained, you can manage all resources of native Argo CD through the Argo CD Dashboard.

## How to Obtain Argo CD Access Information for the GitOps Cluster Plugin Deployed on the Platform?

### Prerequisites

- The GitOps cluster plugin has been deployed in the **global** cluster.
- The CLI plugin is installed, and the `web-cli` switch is enabled.
- You have platform administrator permissions.

### Operational Steps

:::info

It is recommended to enable the following when deploying the GitOps cluster plugin

- Enable **Argo CD Native UI** switch.
- Enable **Single Sign-On** switch.

:::

> Please follow the steps below to obtain Argo CD access information:

1. Log in to the platform and navigate to the **Platform Management** page.

2. In the left sidebar, select **App Store Management** to enter the **Cluster Plugins** list page.

3. Locate the **GitOps** plugin, click **GitOps**, and a pop-up will show the **GitOps Cluster Plugin** details.

4. Argo CD Native UI
   - If not enabled: Return to the **Cluster Plugins** list page, find the **GitOps** plugin, click the **Actions** button, select **Update**, and enable the **Argo CD Native UI** switch.
   - If enabled: Click the **Access Address** directly to open the Argo CD Dashboard.

5. **Single Sign-On**
   - If enabled: Log in to the Argo CD Dashboard using the platform account.
   - If not enabled: The platform's deployment account defaults to `admin`, and the password needs to be retrieved by executing the following command in **Kubectl** [Retrieve Argo CD Password](#argocd-password-cli).

## How to Obtain Argo CD Access Information from Argo CD Operator

### Prerequisites

- A cluster with Argo CD Operator deployed.
- The CLI plugin is installed, and the web-cli switch is enabled.
- You have platform administrator permissions.

### Operational Steps

#### Obtain Argo CD Dashboard URL

1. Log in to the platform and navigate to the **Platform Management** page.

2. In the left sidebar, select **Cluster Management** to enter the **Resource Management** page.

3. In **Resource Group**, search for `Service`, select the ***argocd*** namespace (the namespace where the argocd instance is deployed). The default namespace for Argo CD deployed on the platform is ***argocd***.

4. In the right **Resource List**, find the `argocd-gitops-server` resource, click the ***Actions*** button, and select **Update** to open the YAML details of argocd-gitops-server, details as shown in the image.

<img src="../../gitops/assets/argo_cd_service_nodeport.png" width="900" />

5. Change the `type` to `NodePort` and record the `nodePort`, then click the **Update** button.

6. In the left sidebar, select **Cluster Management** to enter the **Cluster List** page.

7. Select the cluster where `argocd operator` is deployed, and enter the **Cluster Details Page**, select **Nodes**.

8. Obtain the IP address of ***any Master*** node.

9. Access Argo CD Dashboard via `http://{Master Node IP}:{nodePort}`.

#### Retrieve Argo CD Password

Execute the following command in **Kubectl** to retrieve:

```bash
kubectl get secret -n argocd argocd-gitops-cluster -o template --template='{{index .data "admin.password"}}'|base64 -d
```

### Update Argo CD Admin Account Password<a id="argocd-update-password-cli" />

The default `admin` account password created automatically by deploying Argo CD through **Alauda GitOps** or **Argo CD Operator** cannot be modified through the **Argo CD Native UI** interface. You can change it by executing the following command in the CLI tool. Here, `newpassword` is the new password you want to set.

```
kubectl patch -n argocd secrets argocd-gitops-cluster -p '{"stringData":{"admin.password":"<newpassword>"}}'
```
