# Create Application through Argo CD Dashboard

## Prerequisites

- Deployed, meeting any of the following criteria:
  - GitOps cluster plugin has been deployed. [Deploy GitOps Cluster Plugin](../../gitops/install_gitops/install_gitops_plugin.mdx)
  - Argo CD Operator has been deployed. [Deploy Argo CD Operator](../../gitops/install_gitops/install_argocd_opeator.mdx)

- Access address, username, and password for the **Argo CD Dashboard** have been obtained.

## Creating Application

> Follow the steps below to use the functionality:

1. Enter the access address of the **Argo CD Dashboard** in your browser to open **Argo CD Dashboard**.

> If you are a platform administrator, you can visit the global cluster plugin details page, locate the GitOps plugin, and access the **Argo CD native UI** directly.

2. Enter your Argo CD username and password, and **log in**.

3. Click the **+ NEW APP** button as shown below:

<img src="../../gitops/assets/new_app_button.png" width="400" />

> Configure the application according to the steps below:

**Basic Information Configuration**

<img src="../../gitops/assets/new_app_button_info.png" width="400" />

- `Application Name`: Enter `guestbook`
- `Project`: Select `default`
- `Sync Policy`: Keep as `Manual`

**Source Code Configuration**

<img src="../../gitops/assets/new_app_button_source.png" width="400" />

- `Repository URL`: Set to `https://github.com/argoproj/argocd-example-apps.git`
- `Revision`: Keep as `HEAD`
- `Path`: Set to `guestbook`

**Target Cluster Configuration**

<img src="../../gitops/assets/new_app_button_repo.png" width="400" />

- `Clusternew`: Set to `https://kubernetes.default.svc` (or cluster name as `in-cluster`)
- `Namespace`: Set to `default`

4. Create **Application**
   After completing the above configurations, click the **Create** button at the top of the interface to create the guestbook application.
