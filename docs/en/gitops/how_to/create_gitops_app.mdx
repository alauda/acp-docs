# Creating an Argo CD Application via web console

This article will introduce the complete process of creating an Argo CD Application through the platform's GitOps application, allowing for GitOps management of the application.

## Use Case

- Create a SpringBoot Argo CD Application using the platform UI to experience the complete process of managing applications through GitOps.

## Prerequisites

- The GitOps cluster plugin has been deployed in the **global** cluster.
- Projects and namespaces have been allocated on the platform.

## Steps

> Follow the steps below to use the functionality:

### Code Repository Configuration

> If you do not see **Integrated Code Repository** in the GitOps application details page, you can integrate the code repository first:

1. Integrate the repository address (Demo usage): [https://github.com/argoproj/argocd-example-apps.git](https://github.com/argoproj/argocd-example-apps.git)
2. [Integrate the code repository using Argo CD UI](../how_to/argocd_git.mdx)

:::info
You can use the Demo application: [https://github.com/argoproj/argocd-example-apps/tree/master/helm-guestbook](https://github.com/argoproj/argocd-example-apps/tree/master/helm-guestbook)
:::

### Using the Platform's Default Code Repository

1. In the left navigation bar, click on **GitOps Applications**.

2. Click on **Create GitOps Application**.

3. In the **Basic Information** and **Code Repository** sections, configure the relevant information as per the instructions below.

| **Parameter**         | **Input Content**               |
| :------------------- | :----------------------------   |
| **Type**             | `Application`                   |
| **Source**           | `Argo CD Integration`           |
| **Integrated Project Name** | `argocd-example-apps` |
| **Version Identifier** | `Branch` <br />`master`      |
| **Source File Type**   | `Helm`                        |
| **Source File Directory** | `helm-guestbook`          |
| **Custom Values**      | `values.yaml`                 |

4. In the **Distribution Location**, use the platform's recommended ***Default Namespace***, or select another namespace.

5. Set the synchronization policy to **automatic synchronization** by default.

6. Click on **Create**.
