# Installing the Argo CD Operator

## Prerequisites

1. Download the Argo CD Operator installation package that matches your platform.
2. Use the platform's application shelving capability to shelf the Argo CD Operator to ***any cluster where you wish to utilize GitOps capabilities***.

## Steps

### Step 1: Deploying the Argo CD Operator

1. Log in to the platform and navigate to the **Platform Management** page.
2. In the left navigation bar, select **Application Store Management** -> **Operator** to enter the **Operator Hub** page.
3. Locate the **Argo CD Operator**, click **Deploy**, and proceed to the **Deploy Argo CD Operator** page.

Configuration Parameters:

| **Parameter**   | **Recommended Configuration**                              |
| :------------- | :----------------------------------------------------- |
| **Upgrade Policy** | **Automatic**: The Operator Hub will automatically upgrade the Operator to the latest version when a new version is available. |
| **Deployment Mode** | **Cluster**: All namespaces in the cluster share a single Operator for instance creation and management, leading to lower resource usage. |
| **Channel**   | The default channel is **alpha**.                      |
| **Namespace** | Select **Recommended Namespace**: Automatically created if none exists. |

4. In the **Deploy Operator** page, choose **Default Configuration**, click **Deploy**, and complete the deployment of the Argo CD Operator.

### Step 2: Deploying an Argo CD Instance

1. In the left navigation bar, click **Application Store Management** > **Operators**.

2. In the **Deployed Operators** tab, click on **argocd-operator**.

3. In the details section, click on **Create Instance** in the resource section of the **ArgoCD** instance card.

4. In the configuration instance parameter page, use the default configuration unless there are specific requirements.

   **Note**: If the global cluster is not highly available (e.g., it has only one control node), please switch to YAML view when creating the instance and set the ha.enabled field value to false.

5. Click **Create**.

### Step 3: Deploying an AppProject Instance

**Tip**: If you do not need to use the platform-managed **Cluster Configuration Management** feature, you do not need to perform the following steps.

1. Click to create an AppProject instance, switch to YAML view, and overwrite the existing YAML content on the interface with the code below.

   ```
   apiVersion: argoproj.io/v1alpha1
   kind: AppProject
   metadata:
     name: cpaas-system
     namespace: argocd
   spec:
     clusterResourceWhitelist:
     - group: '*'
       kind: '*'
     destinations:
     - namespace: '*'
       server: '*'
     sourceRepos:
     - '*'
   ```

2. Click **Create**.

After completing the above steps, you have successfully deployed Argo CD. Immediately [create an Application through the Argo CD Dashboard](../how_to/create_application_by_argocd_dashoard.mdx) to begin your GitOps journey.
