# Overview

<Term name="productShort" /> Registry documentation has two administration tracks. Use **Registry v2** for the Operator-managed <Term name="productShort" /> Registry. Use the legacy Registry pages only for environments that still run the Registry Cluster Plugin.

Developers and application teams use the registry through developer image workflows.

## Administration scope

| Need | Continue with |
| --- | --- |
| Administer the Operator-managed Registry v2. | [Registry v2](./registry_v2/index.mdx) |
| Install the legacy Registry Cluster Plugin. | [Install](./install/index.mdx) |
| Upgrade the legacy Registry Cluster Plugin. | [Upgrade](./upgrade/index.mdx) |
| Back up or recover legacy Registry Cluster Plugin data. | [Registry backup and recovery](./backup_and_restore.mdx) |
| Use images in workloads or developer workflows. | [Images](../../developer/images/overview.mdx) and [registry image use](../../developer/registry/index.mdx) |

## Namespace isolation and access

Registry image repositories are organized by namespace. Pull and push access depends on the platform roles and permissions granted to users or ServiceAccounts in the target namespace.

For common image usage commands and cross-namespace pull access, see [Registry image use](../../developer/registry/index.mdx).
