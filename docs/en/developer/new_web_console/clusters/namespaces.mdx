# Namespaces

This page is now a transition entry for cluster-view namespace workflows that were previously documented under `developer/new_web_console/clusters`.

## Long-term host page

- [Namespace](../../building_application/namespace)

## What remains different in this path

Use the destination pages for the full workflows.

Keep this path in mind only for the remaining view-specific difference:

- **Clusters** view starts from one concrete cluster.
- You manage the namespace as a cluster resource.
- This path does not cover project-association workflows such as importing a namespace into a project or removing it from a project.
- The main concern is cluster-side namespace governance, visibility, and lifecycle management.

## Related transition path

If you need project-scoped namespace management for the current project, use:
