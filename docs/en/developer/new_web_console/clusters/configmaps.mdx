## What remains different in this path

Use the destination page for the full workflow.

Keep this path in mind only for the remaining view-specific difference:

- **Clusters** view starts from one concrete cluster.
- You manage ConfigMaps as namespace resources inside that cluster.
- The main concern is cluster-side namespace operations, not project entry scope.
- Workloads can still reference only ConfigMaps from the same namespace.
