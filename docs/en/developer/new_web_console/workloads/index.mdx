# Workloads

The **Workloads** section covers project-scoped runtime resources in the **Applications** view.

Use these pages when you need to create, inspect, update, or troubleshoot the workloads that actually run application code or scheduled tasks.

Some workload types in this section can also be available from the **Clusters** view. This section documents the **Applications** view path, scope, and verification logic rather than combining both views into a single procedure.

## Long-running application workloads

Use these pages for workloads that stay running and represent the main runtime units of an application:

- [Deployments](./deployments)
- [StatefulSets](./statefulsets)
- [DaemonSets](./daemonsets)
- [Pods](./pods)

## Scheduled or finite execution workloads

Use these pages for workloads that run on a schedule or stop after completion:

- [CronJobs](./cronjobs)
- [Jobs](./jobs)

## Choose the right workload page

In most cases:

- Start with **Deployments** for common stateless application workloads.
- Use **StatefulSets** when stable identity, ordered rollout, or persistent storage matters.
- Use **DaemonSets** when the workload is intended to run broadly on matching nodes.
- Use **CronJobs** or **Jobs** when the main question is scheduled or finite execution.
- Use **Pods** when you need to inspect the actual running instances behind higher-level workloads.
