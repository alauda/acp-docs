# Deployments

This page keeps the legacy `developer/new_web_console/workloads/deployments` path available while the long-term Deployment guidance is maintained in the workload host pages.

## Main destination

- [Deployments](../../building_application/application_workloads/deployment_create.mdx)

## What remains specific to the Applications view

- The list is opened from the project-scoped **Applications** view, but each Deployment still belongs to one concrete cluster and namespace.
- This path preserves the **Applications** view entry, list scope, and verification context rather than a cluster-wide workload entry.
- If the same Deployment is also reachable from another console view, use this path when you need project-level workload navigation first.
