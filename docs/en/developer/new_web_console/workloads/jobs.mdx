# Jobs

This page keeps the legacy `developer/new_web_console/workloads/jobs` path available while the long-term Job guidance is maintained in the workload host pages.

## Main destination

- [Jobs](../../building_application/application_workloads/job_create.mdx)

## What remains specific to the Applications view

- In this path, the list is project-scoped, but each Job still runs in one concrete cluster and namespace.
- The **Applications** view is especially useful when you first need project-level visibility and then want to inspect completion state, retries, logs, and related Pods.
- Runtime interpretation should still be based on the actual Job execution context rather than only on the project list entry.
