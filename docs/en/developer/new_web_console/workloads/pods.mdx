# Pods

This page keeps the legacy `developer/new_web_console/workloads/pods` path available while the long-term Pod guidance is maintained in the workload host pages.

## Main destination

- [Pods](../../building_application/application_workloads/pod.mdx)

## What remains specific to the Applications view

- The Pod list is project-scoped in the **Applications** view, but each Pod still belongs to one concrete cluster, namespace, and source workload.
- Use this path when you need to start from project-level runtime visibility before narrowing the issue down to one workload instance.
- Delete actions in this view still affect only the current runtime instance, and controller-managed Pods can be recreated automatically.
