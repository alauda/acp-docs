# CronJobs

This page keeps the legacy `developer/new_web_console/workloads/cronjobs` path available while the long-term CronJob guidance is maintained in the workload host pages.

## Main destination

- [CronJobs](../../building_application/application_workloads/cronjob_create.mdx)

## What remains specific to the Applications view

- The list is project-scoped in the **Applications** view, but schedules, immediate execution, and downstream Job results still belong to one selected cluster and namespace.
- Use this path when you need the project-level workload entry before validating schedule behavior or execution results.
- Immediate execution in this view should still be verified through the created Job and its Pods.
