# New Web Console

This section covers the migrated application-management workflows exposed through the newer ACP web console experience.

Use this module according to the task you need to complete.

This module documents the migrated web console as an independent documentation family under `developer/new_web_console`. Do not place new-console workflows under other pillars such as `configure/clusters`, even when the workflow is entered from the **Clusters** view.

The **Applications** view and **Clusters** view should be documented as separate view-specific workflows rather than mixed into the same page.

## Start with the right task area

- [Overview](./overview) when you need a project-scoped dashboard and a quick map of the module
- [Applications](./applications) when you need to create, update, inspect, or troubleshoot applications
- [Namespaces](./namespaces) when you need to prepare or govern namespaces for workloads from the **Applications** view
- [Workloads](./workloads) when you need to work directly with Deployments, StatefulSets, DaemonSets, Pods, Jobs, or CronJobs from the **Applications** view
- [Configuration](./configuration) when you need to manage ConfigMaps or Secrets used by workloads from the **Applications** view
- [Clusters](./clusters) when you need the corresponding **Clusters** view workflows in the new web console

## How to use this module

In most cases:

1. Confirm the current project, cluster, and namespace context.
2. Open the task-family page that matches your goal.
3. Follow the workflow page for the target operation.
4. Use the verification guidance on the target page before assuming the change succeeded.
