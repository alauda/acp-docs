# New Web Console

This section is a transitional hub for application-management workflows that were previously grouped under the newer ACP web console path.

Use this module when you need to follow an existing `developer/new_web_console` path, understand how the newer console experience grouped tasks, or bridge from the old path to the consolidated developer documentation.

Most long-term task guidance now lives under [Building Applications](../building_application) and related `developer` pages.

## Start with the destination page

- [Building Applications](../building_application) for the main consolidated entry
- [Developer Overview](../overview) for the broader application lifecycle map
- [Applications](./applications) when you need legacy path continuity for application-oriented pages
- [Workloads](./workloads) when you need legacy path continuity for workload-oriented pages
- [Namespaces](./namespaces) and [Configuration](./configuration) for transitional scope notes
- [Clusters](./clusters) for the remaining cluster-view-specific transition guidance

## Path transition

Use `developer/new_web_console` as a short-lived routing layer, not as the primary long-term information architecture.

- If a task already has a host page under `developer/building_application`, follow that consolidated page.
- Keep **Applications** view and **Clusters** view differences only where the entry path or scope meaning still matters.
- Avoid duplicating deeper navigation that is already exposed by the destination pages.
