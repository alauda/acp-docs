# Create and Update Applications

This page keeps the legacy `developer/new_web_console/applications/create-update` path available while the long-term create and update guidance is maintained in the application host pages.

## Main destinations

- [Creating applications from Image](../../building_application/create_applications/image_app.mdx)
- [Updating Applications](../../building_application/operation_maintaining/application_upgrade.mdx)

## What remains specific to the Applications view

- The route is project-scoped, but creation and update still resolve in a concrete cluster and namespace context.
- During creation, you select the target **Cluster** first and then select a namespace from that cluster in the current project scope.
- During update, **Cluster**, **Namespace**, and **Name** remain fixed because the workflow modifies the application in place.
- If the same workload types are also available from other console views, this page only preserves the **Applications** view entry and context.
