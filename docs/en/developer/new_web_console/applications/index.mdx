# Applications

The **Applications** section covers project-scoped application workflows in the **Applications** view.

Use this section according to the task you need to complete.

The surrounding view is project-scoped, but specific create, update, detail, and related-resource operations still resolve in a concrete cluster and namespace context.

When the same underlying resource type is also available from the **Clusters** view, this section still documents only the **Applications** view workflow.

## Create or change an application

Use these pages when you need to create, update, or submit application definitions through different workflows:

- [Create and Update Applications](./create-update)
- [Create Applications from YAML](./create-from-yaml)
- [Create Applications from Git](./create-from-git)
- [Manage Helm-Based Applications](./manage-helm-based-applications)

## Inspect application state

Use these pages when you need to review runtime status, history, or application-scoped operational signals:

- [Applications Overview](./overview)
- [View Application Details](./application-details)
- [Application History and Versions](./application-history-versions)
- [Manage Application Alerts](./application-alerts)

## Review application-associated resources

Use this page when you need to understand or adjust which resources are associated with an application:

- [Manage Application Resources](./manage-application-resources)

## Choose the right creation workflow

In most cases:

- Use **Create and Update Applications** for form-based creation and routine edits.
- Use **Create Applications from YAML** when you already maintain valid manifests and want tighter control over resource definitions.
- Use **Create Applications from Git** when the environment exposes a source-driven workflow.
- Use **Manage Helm-Based Applications** when the application is backed by a **HelmRequest**-driven deployment model.
