# View Application Details

This page keeps the legacy `developer/new_web_console/applications/application-details` path available while the long-term application operation and observability guidance is maintained in the application host pages.

## Main destinations

- [Operation and Maintaining Applications](../../building_application/operation_maintaining/index.mdx)
- [Application Observability](../../building_application/application_observability/index.mdx)

## What remains specific to the Applications view

- The application details route is still the main application-scoped operational entry in the **Applications** view.
- The visible tabs can vary by application type, enabled capabilities, and permissions.
- Tabs such as **Topology**, **History**, **Values**, **Logs**, **Events**, **Monitoring**, and **Alarm** are all interpreted from one application boundary rather than from one workload object alone.
- When the same underlying workloads are also reachable from other views, this route remains the place for application-level correlation across resources, history, and observability signals.
