# Manage Helm-Based Applications

This page keeps the legacy `developer/new_web_console/applications/manage-helm-based-applications` path available while the long-term Helm-based application guidance is maintained in the application host pages.

## Main destinations

- [Creating applications from Chart](../../building_application/create_applications/chart_app.mdx)
- [Working with Helm charts](../../building_application/working_with_helm_charts.mdx)

## What remains specific to the Applications view

- In this path, the application page is the operational view, while **HelmRequest** remains the chart-driven source of deployment intent.
- The details route can expose application-level tabs or warning states such as **Values**, **History**, and Helm-related validation banners.
- Read-only or restricted fields in update workflows usually indicate that the current state is still governed by the Helm source of truth.
