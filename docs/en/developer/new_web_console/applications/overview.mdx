# Applications Overview

This page keeps the legacy `developer/new_web_console/applications/overview` path available while the long-term dashboard guidance is maintained in the application observability host pages.

## Main destination

- [Monitoring Dashboards](../../building_application/application_observability/monitor.mdx)

## What remains specific to the Applications view

- This dashboard is project-scoped and can aggregate application, workload, Pod, resource, and alert signals across the clusters associated with the selected project.
- Warning banners in this view still reflect project-to-cluster association, business-cluster health, monitoring availability, and quota display limits in the **Applications** view.
- If you need to continue from project-level symptoms to one application, open the relevant application from the dashboard and then use the application details route.
