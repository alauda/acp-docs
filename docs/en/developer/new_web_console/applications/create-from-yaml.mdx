# Create Applications from YAML

This page keeps the legacy `developer/new_web_console/applications/create-from-yaml` path available while the long-term YAML workflow is maintained in the application host pages.

## Main destination

- [Creating applications from YAML](../../building_application/create_applications/yaml_app.mdx)

## What remains specific to the Applications view

- The workflow is opened from the project-scoped **Applications** view, but submission still targets one selected cluster and namespace.
- The visible metadata and the YAML object identity still need to stay consistent in the selected project scope.
- When you submit multiple related objects together, treat this as an application-scoped entry rather than a cluster-wide generic YAML operation.
