# Application History and Versions

This page keeps the legacy `developer/new_web_console/applications/application-history-versions` path available while the long-term history, snapshot, rollback, and export guidance is maintained in the application host pages.

## Main destinations

- [Version Management for Applications](../../building_application/operation_maintaining/version.mdx)
- [Exporting Applications](../../building_application/operation_maintaining/export.mdx)

## What remains specific to the Applications view

- In the **Applications** view, version review stays under the application details route and is still centered on the **History** tab.
- History records in this view are interpreted from the application boundary, so associated resources can affect what is preserved, compared, rolled back, or exported.
- Template-based applications can expose history differently from regular applications, especially when chart-driven state is involved.
