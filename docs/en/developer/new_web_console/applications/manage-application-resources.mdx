# Manage Application Resources

This page keeps the legacy `developer/new_web_console/applications/manage-application-resources` path available while the long-term resource-association guidance is maintained in the application operation host pages.

## Main destination

- [Updating Applications](../../building_application/operation_maintaining/application_upgrade.mdx)

## What remains specific to the Applications view

- This workflow manages which workloads, network objects, storage objects, and configuration objects are grouped into one application boundary in the **Applications** view.
- Importing or removing resources mainly changes application-level visibility, association, history coverage, rollback scope, and export scope.
- This is different from editing the runtime spec of the workload itself.
