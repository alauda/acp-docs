# Manage Application Alerts

This page keeps the legacy `developer/new_web_console/applications/application-alerts` path available while the long-term alerting guidance is maintained in the application observability host pages.

## Main destination

- [Application Alerts](../../building_application/application_observability/alert.mdx)

## What remains specific to the Applications view

- Alert management in this path is still application-scoped and is normally entered from the **Alarm** tab of the application details page.
- Rules created here should match one application boundary instead of a broader cluster or workload scope.
- This page only preserves the **Applications** view entry and context for the alert workflow.
