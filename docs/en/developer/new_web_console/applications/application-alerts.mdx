# Manage Application Alerts

In the **Applications** view, you can manage alert rules for an application from the application details page when monitoring and alerting capabilities are available.

## When to use application alerts

Use application alerts when you need the platform to detect unhealthy trends or runtime conditions in the current application context, rather than relying only on manual observation.

In practice, alert rules are most useful when you want to:

- Detect sustained degradation or abnormal runtime behavior
- Surface workload issues that require operator attention
- Correlate monitoring signals with a specific application context

Good application alerts should help operators react to meaningful symptoms. Prefer rules that identify actionable problems rather than rules that only restate noisy or short-lived metric fluctuations.

## Open the alert area

1. Open the target application details page.
2. Click the **Alarm** tab.

The application details route also exposes alert operations such as:

- Create an alert rule
- Update an alert rule
- View alert rule details

## Create an alert rule

If alert creation is available:

1. Open the application details page.
2. Open the alert creation workflow from the application context.
3. Configure the alert rule.
4. Confirm the operation.

Before you create a rule, confirm that the condition is meaningful for the target application and is not duplicating another alert that already covers the same symptom.

### Alert design review points

Before creating or updating a rule, review:

- whether the rule monitors a symptom that operators can realistically act on
- whether the monitored scope still matches the intended application boundary
- whether the threshold or condition is likely to detect sustained issues rather than harmless short spikes
- whether another broader platform or workload rule already covers the same condition

## Update an alert rule

1. Open the target alert rule from the application alert area.
2. Modify the supported settings.
3. Confirm the update.

After an update, monitor the rule behavior for a period of time to ensure that it remains actionable and does not introduce excessive alert noise.

A rule update can change more than visible metadata. It can change who gets alerted, how often a condition fires, and whether operators still trust the signal enough to use it during incidents.

## View alert rule details

Open an existing alert rule to review its configuration, status, and related metadata.

Use the details view to confirm:

- The rule is associated with the intended application
- The monitored scope is still correct
- The current rule definition still matches the application architecture and runtime expectations

## Verify the result

After creating or updating an alert rule, verify that:

- The rule is associated with the correct application.
- The expected rule details are displayed.
- The alert list reflects the new or updated rule.
- The rule remains understandable and actionable for operators.
- Monitoring data is available for the target application context when required by the rule.
- The rule improves operator awareness without introducing duplicated or misleading alert noise.
