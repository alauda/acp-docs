# Secrets

This page is now a transition entry for Secret workflows that were previously documented under `developer/new_web_console/configuration`.

## Long-term host page

- [Configuring Secrets](../../building_application/configuration/add_secret)

## What remains different in this path

Use the destination page for the full workflow.

Keep this path in mind only for the entry-scope difference:

- **Applications** view is project-scoped.
- You enter from the current project and then choose the target cluster and namespace as needed.
- The Secret itself still belongs to one concrete namespace.
- Workloads can only reference Secrets from the same namespace.

## When to follow a different page

If you start from a concrete cluster and manage Secrets as cluster-side namespace resources, use the **Clusters** view bridge page instead:

- [Clusters > Secrets](../clusters/secrets)
