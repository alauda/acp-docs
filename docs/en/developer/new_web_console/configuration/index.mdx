# Configuration

This section keeps the legacy `developer/new_web_console/configuration` path available while ConfigMap and Secret guidance is consolidated into the long-term configuration host pages.

## Main destination

- [Configuration](../../building_application/configuration)

## Keep this section for view-specific continuity

Use these pages when you need to create, update, or review configuration resources that affect application startup, runtime settings, authentication, or image pulling.

Although you enter this section from the project scope, each ConfigMap or Secret still belongs to a concrete cluster and namespace, and workload references remain namespace-bound.

If the same ConfigMap or Secret operations are also available from the **Clusters** view, document that as a separate cluster-scoped workflow instead of merging both views into one page.

## Non-sensitive configuration

Use **ConfigMaps** for runtime configuration that does not contain credentials or private material:

- [ConfigMaps](./configmaps)

## Sensitive configuration

Use **Secrets** for credentials, tokens, keys, certificates, or image-pull authentication:

- [Secrets](./secrets)
