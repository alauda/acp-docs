# ConfigMaps

This page is now a transition entry for ConfigMap workflows that were previously documented under `developer/new_web_console/configuration`.

## Long-term host page

- [Configuring ConfigMap](../../building_application/configuration/add_configmap)

## What remains different in this path

Use the destination page for the full workflow.

Keep this path in mind only for the entry-scope difference:

- **Applications** view is project-scoped.
- You enter from the current project and then choose the target cluster and namespace as needed.
- The ConfigMap itself still belongs to one concrete namespace.
- Workloads can only reference ConfigMaps from the same namespace.

## When to follow a different page

If you start from a concrete cluster and manage ConfigMaps as cluster-side namespace resources, use the **Clusters** view bridge page instead:

- [Clusters > ConfigMaps](../clusters/configmaps)
