# Manage Namespaces

This page is now a transition entry for namespace workflows that were previously grouped under `developer/new_web_console/namespaces`.

## What remains different in this path

Use the destination pages for the full workflows.

Keep this path in mind only for the entry-scope difference:

- **Applications** view is project-scoped.
- You manage namespaces for the current project and its associated clusters.
- This path includes project-association workflows such as importing a namespace into a project or removing it from a project.
- Namespace governance still takes effect in the concrete cluster where the namespace exists.

## When to follow a different page

If you start from a concrete cluster and manage namespaces as cluster resources instead of project membership, use the **Clusters** view bridge page instead:

- [Clusters > Namespaces](../clusters/namespaces)
