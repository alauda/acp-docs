# Namespaces

The **Namespaces** section in the **Applications** view covers project-scoped namespace operations for the current project and its associated clusters.

Namespace management can also surface in the **Clusters** view, but the entry path and operational scope are different there. This section documents the **Applications** view workflow only.

Use this section when you need to:

- Create a namespace before deploying workloads
- Import an existing namespace into the current project scope
- Update quota, policy, metadata, or namespace display settings
- Review namespace details or remove project association

## Main workflow

- [Manage Namespaces](./manage-namespaces)

When your goal is to prepare runtime targets for applications or workloads, start with **Manage Namespaces** and then continue to the related application or workload page.
