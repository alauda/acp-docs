# Pod Page Parameters

The platform interface provides various information about the container group for quick reference. Below are some parameter explanations.

| Parameter              | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
|-----------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **Resource Requests/Limits**        | The effective resource (CPU, memory) requests and limits values for the container group. The calculation method for requests and limits values is the same; this document introduces using the limit values as an example, and the specific rules and algorithms are as follows: <ul><li>When the container group only contains **business containers** (containers), the **CPU/memory** limit value is the sum of the **CPU/memory** limit values of all containers within the container group. **For example**: If the container group includes two business containers with **CPU/memory** limit values of 100m/100Mi and 50m/200Mi, the container group's **CPU/memory** limit value will be 150m/300Mi.</li><li>When the container group contains both **init containers** (initContainers) and **business containers**, the calculation steps for the container group's **CPU/memory** limit values are as follows: <ol><li>Take the maximum value of the **CPU/memory** limit values of all init containers.</li> <li>Take the sum of **CPU/memory** limit values of all business containers.</li><li>Compare the results and take the maximum values of **CPU** and **memory** from both init containers and business containers as the container group's **CPU/memory** limit values.</li></ol></li><br />**Calculation Example**: If the container group contains two init containers with **CPU/memory** limit values of 100m/200Mi and 200m/100Mi, the maximum **CPU/memory** limit value for the init containers would be 200m/200Mi. At the same time, if the container group also contains two business containers with **CPU/memory** limit values of 100m/100Mi and 50m/200Mi, the total limit value for the business containers will be 150m/300Mi. Therefore, the comprehensive **CPU/memory** limit value for the container group would be 200m/300Mi.</ul> |
| **Source**                  | The computing component to which the container group belongs.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| **Restart Count**              | The number of restarts when the container group's status is abnormal.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| **Node**                  | The name of the node where the container group is located.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| **Service Account**       | The Service Account is an account that allows processes and services in the Pod to access the Kubernetes APIServer, providing an identity for the processes and services. The **Service Account** field is visible only when the currently logged-in user has either the platform administrator role or the platform auditor role, and the YAML file of the Service Account can be viewed.                                                                                                                                                                                                                                                                                                                                                                                                                                     |
