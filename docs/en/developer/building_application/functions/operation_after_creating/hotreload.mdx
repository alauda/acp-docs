# Loading Configmap Configuration

After the mounted configuration dictionary is updated, the application needs to load the updated configuration to take effect. For applications that cannot automatically load the configuration, **Load Configuration** provides the ability to manually trigger the application to load the configuration dictionary.

## Prerequisites

- The application must support hot configuration updates.

- The configuration dictionary must be mounted as a storage volume within the container group, and the SubPath cannot be used. Configuration dictionaries referenced through environment variables do not support loading configuration.

## Steps

1. Enter **Container Platform**.

2. In the left navigation bar, click **Configuration** > **Configuration Dictionary**.

3. Click on ***Configuration Dictionary Name***.

4. Click **Add Configuration Item** or click **Add Binary Configuration Item**; you can also click <span style={{ display: 'inline-flex', alignItems: 'center', gap: '4px', verticalAlign: 'middle' }}><img src="./assets/blackpencil.png" alt="expand" style={{ verticalAlign: 'middle' }} /> <span>Update Configuration Item and confirm.</span></span> as needed.

5. Switch to the **Compute Component** tab and click <span style={{ display: 'inline-flex', alignItems: 'center', gap: '4px', verticalAlign: 'middle' }}><img src="./assets/fold.png" alt="expand" style={{ verticalAlign: 'middle' }} /> <span>Expand its container list.</span></span>.

6. Click **Load Configuration**, enter the user and execute the command, and confirm. By default, it executes into the container as the **root** user.

7. Enter the corresponding command to load the configuration based on the application type.

   - Example 1: Nginx

     ```
     nginx -s reload
     ```

   - Example 2: Logstash

     ```
     kill -SIGHUP <Logstash process ID>
     ```
