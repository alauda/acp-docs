# Application Alerts

Application alerts help operators detect sustained unhealthy behavior in a specific application context instead of relying only on manual dashboard inspection.

## When to use application alerts

Use application alerts when you need to:

- detect workload degradation that requires operator attention
- surface application-specific issues earlier than manual review
- correlate alert signals with one concrete application boundary

Prefer rules that identify actionable symptoms instead of short-lived metric spikes or duplicated noise.

## Open the alert area

1. Open the target application details page.
2. Click the **Alarm** tab.

From this area, you can create, update, and review alert rules for the current application when monitoring and alerting capabilities are available.

## Create an alert rule

1. Open the application details page.
2. Open the alert creation workflow from the current application context.
3. Configure the alert rule.
4. Confirm the operation.

### Review points before creating or updating a rule

Before creating or updating a rule, confirm that:

- the monitored symptom is meaningful and actionable
- the monitored scope still matches the intended application boundary
- the threshold is likely to capture sustained issues rather than harmless spikes
- no broader platform or workload rule already covers the same condition

## Update an alert rule

1. Open the target alert rule from the application alert area.
2. Modify the supported settings.
3. Confirm the update.

After an update, observe the rule behavior for a period of time to ensure that the signal remains trustworthy and does not introduce excessive noise.

## View alert rule details

Review the rule details to confirm that:

- the rule is associated with the intended application
- the monitored scope is still correct
- the rule definition still matches the application architecture and runtime expectations

## Verify the result

After creating or updating an alert rule, verify that:

- the rule is associated with the correct application
- the expected rule details are displayed
- the alert list reflects the new or updated rule
- monitoring data is available for the target application context when required
- the rule improves operator awareness without introducing duplicated or misleading alerts
