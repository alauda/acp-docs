# Application Observability

This section covers the signals you use to understand how an application behaves after it is deployed.

Typical observability tasks include:

- Viewing application and workload logs during troubleshooting
- Reviewing recent events to understand scheduling, reconciliation, or runtime failures
- Inspecting monitoring dashboards for applications, workloads, and Pods
- Managing alert rules that help operators detect meaningful application issues earlier

If you need to update configuration, manage associated resources, or roll back to an earlier version, continue with [Operation and Maintaining Applications](../operation_maintaining/index.mdx).
