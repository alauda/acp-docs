# Operation and Maintaining Applications

This section covers the ongoing management work that usually starts after an application has been created and deployed.

Typical tasks include:

- Reviewing application status before deciding whether to troubleshoot, update, or roll back
- Updating application configuration and managing the resources associated with the application
- Creating version snapshots, comparing historical revisions, and rolling back when needed
- Exporting the current version or a historical version for reuse in another environment

If you mainly need logs, events, metrics, or alert rules, continue with [Application Observability](../application_observability/index.mdx).
