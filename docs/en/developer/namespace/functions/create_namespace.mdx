# Create Namespace

Under the cluster associated with the project, create a new namespace based on the available quota of the project.

The new namespace will utilize existing resources (such as memory, CPU, etc.) from the project, and all resources in a namespace can only come from the same cluster.

## Steps to Operate via web console

1. In the **Project Management** view, click on the ***Project Name*** for which you want to create a namespace.

2. In the left navigation bar, click on **Namespaces** > **Namespaces**.

3. Click on **Create Namespace**.

4. Configure **Basic Information**.

   | Parameter       | Description                          |
   | -------------- | ------------------------------------ |
   | **Cluster**    | Select a cluster associated with the project in which to create the namespace. |
   | **Namespace**  | The name of the namespace; the fixed prefix of the name is the project name. |

5. (Optional) Configure **Resource Quota**.

   Every time a resource limit (limits) for computational or storage resources is specified for a container within the namespace, or each time a new Pod or PVC is added, it will consume the quota set here.

   **Note**:

   - The resource quota for the namespace is derived from the quota allocated to the project by the cluster. The maximum quota that can be set for a given resource is the unused resource quota in the project. If the available amount of any resource is 0, you will not be able to continue creating the namespace; please contact the platform administrator for support.

   - GPU resource quota can only be configured if GPU (vGPU or pGPU) resources are deployed in the cluster. When the GPU resource is **vGPU**, memory quota can also be configured.

     **GPU Units**: 100 units of virtual cores equal 1 physical core (1 pGPU = 1 core = 100 vGPU), pGPU units are counted in whole units only; 1 unit of memory equals 256 Mi, 1024 Mi = 1 Gi.

   - If no quota is entered for a certain type of resource, the default is **unlimited**. This means the namespace can use the available resources of the corresponding type allocated to the project from the cluster without limits.

6. (Optional) Configure **Container Limit Range**; please refer to [Container Limit Range](../concepts/limit_range.mdx.mdx) for more details.

7. (Optional) Configure **Pod Security Policies**; please refer to [Pod Security Policies](../concepts/security_policies.mdx) for specific details.

8. (Optional) In the **More Configuration** area, add labels and annotations for the current namespace.

   **Tip**: You can define the attributes of the namespace through labels or supplement the namespace with additional information through annotations; both can be used to filter and sort namespaces.

9. Click on **Create**.

## Creating a Namespace via CLI 
### Namespace YAML
```yaml
apiVersion: v1
kind: Namespace
metadata:
  name: <namespace-name>
```
### Create Namespace Command Line
```bash
kubectl create namespace <namespace-name>
```
