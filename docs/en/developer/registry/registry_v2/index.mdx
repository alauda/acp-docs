# Registry v2: Operator-Managed ACP Registry

This section describes **Registry v2**, the Operator-managed <Term name="productShort" /> image registry deployed and reconciled by `cluster-image-registry-operator`.

The page structure is aligned with the OpenShift Container Platform 4.21 Registry documentation: overview, Image Registry Operator, setting up and configuring the registry, accessing the registry, and exposing the registry. ACP-specific image management and legacy migration topics are provided as separate pages.

The legacy Registry documentation remains available under [How To](../how_to/index.mdx) for environments that still use the earlier `cpaas-system` Registry service.

## Standard Address

Use the fully qualified internal service address for workloads running in the cluster:

```text
image-registry.image-registry-system.svc:5000
```

Use an external Ingress hostname only for developer machines, CI systems, or other clients outside the cluster. See [Exposing the registry](./exposing_the_registry.mdx).
