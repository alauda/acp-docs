# Registry v2: Image Use

Use this section for developer and workload access to **Registry v2**, the Operator-managed <Term name="productShort" /> image registry.

Registry v2 administrator tasks, including installation, configuration, exposure, and migration, are documented in [Registry v2 administration](../../../configure/registry/registry_v2/index.mdx).

The legacy Registry documentation remains available under [How To](../how_to/index.mdx) for environments that still use the earlier `cpaas-system` Registry service.

## Documentation Structure

| Need | Continue with |
| --- | --- |
| Access Registry v2 from workloads or external clients. | [Accessing the registry](./accessing_the_registry.mdx) |
| Use `ac` to list, tag, import, mirror, delete, and inspect images. | [Managing images with ac](./managing_images_with_ac.mdx) |
| Review Registry v2 components and administrator settings. | [Registry v2 administration](../../../configure/registry/registry_v2/index.mdx) |

## Standard Address

Use the fully qualified internal service address for workloads running in the cluster:

```text
image-registry.image-registry-system.svc:5000
```

Use an external Ingress hostname only for developer machines, CI systems, or other clients outside the cluster. See [Exposing the registry](../../../configure/registry/registry_v2/exposing_the_registry.mdx).
