# Registry Image Use

<Term name="productShort" /> provides two Registry documentation tracks:

| Track | Use it for | Continue with |
| --- | --- | --- |
| Legacy Registry workflows | Existing environments that still use the earlier Registry service, `image-registry.cpaas-system.svc`, legacy push limit ConfigMaps, and the current `ac get images` / `ac delete images` workflows. | [How To](./how_to/index.mdx) |
| Registry v2: Operator-managed ACP Registry | New deployments or migrations to the OCP-style ACP image registry managed by `cluster-image-registry-operator`. Registry v2 means the Operator-deployed Registry that uses `image-registry-system`, `Config/cluster`, `ImagePruner/cluster`, the `image.alauda.io/v1` Image API, and managed ServiceAccount pull secrets. | [Registry v2](./registry_v2/index.mdx) |

The legacy Registry pages remain available for compatibility. The Registry v2 pages are explicitly marked as **Registry v2** and should be used when your environment has installed or is migrating to the Operator-managed Registry.
