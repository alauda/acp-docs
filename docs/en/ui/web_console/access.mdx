# Accessing the Web Console

## Introduction

The Alauda Container Platform web console provides a centralized graphical interface for managing containerized applications and infrastructure. This document outlines the methods for accessing the web console and navigating between its different perspectives.

## Prerequisites

Before accessing the web console, ensure you have:

- A supported web browser (Chrome, Firefox, Safari, or Edge)
- Network connectivity to the platform's web console endpoint
- Valid authentication credentials for the platform

## Accessing the Web Console

### URL Access

The web console can be accessed through its URL, which is typically in the following format:

```
https://<console-route-hostname>
```

Contact your system administrator to obtain the specific URL for your Alauda Container Platform installation.

### Authentication

Upon accessing the web console URL, you will be presented with a login page. Enter your credentials and select your authentication method if multiple methods are configured.

The platform supports various authentication providers, which may include:

- Local authentication
- LDAP/Active Directory
- OpenID Connect
- OAuth providers

After successful authentication, you will be directed to the Portal page.

## Navigating the Web Console

### Portal Page

The Portal page serves as the entry point to the web console, displaying cards for all available perspectives. Each card represents a different functional area of the platform.

### Perspective Switching

To switch between perspectives:

1. Locate the perspective switcher in the upper-left corner of the console
2. Click on the perspective switcher to view available perspectives
3. Select your desired perspective from the dropdown menu

### User Interface Elements

The web console interface includes several key elements:

- **Main navigation menu**: Located on the left side, provides access to perspective-specific resources
- **Perspective switcher**: Located in the upper-left corner, allows switching between perspectives
- **User menu**: Located in the upper-right corner, provides access to user settings and logout
- **Search bar**: Allows searching for resources across the platform
- **Help menu**: Provides access to documentation and support resources

## Perspective-Specific Navigation

### Container Platform Perspective

In the Container Platform perspective, the navigation menu provides access to:

- Project selector (for switching between Projects)
- Workloads (Deployments, StatefulSets, DaemonSets, etc.)
- Networking resources (Services, Routes, etc.)
- Storage (PersistentVolumeClaims, ConfigMaps, Secrets, etc.)
- Builds and Pipelines
- Monitoring tools

### Platform Management Perspective

In the Platform Management perspective, the navigation menu provides access to:

- Cluster management
- Operators and OperatorHub
- Storage management
- User and role management
- System settings
- Monitoring and logging

### Project Management Perspective

In the Project Management perspective, the navigation menu provides access to:

- Project creation and configuration
- Quota management
- Role bindings and access control
- Resource limits

## Keyboard Shortcuts

The web console supports several keyboard shortcuts to enhance productivity:

- `?`: Display available keyboard shortcuts
- `Ctrl+Space`: Open the command palette
- `Shift+F`: Focus the search bar

Refer to the help menu in the web console for a complete list of keyboard shortcuts.
