# Upgrade

:::info

This document provides the upgrade path principles and supported version compatibility for Alauda Container Platform Compliance for Kyverno.

:::

## Compatibility Matrix

The table below lists supported versions of the Alauda Container Platform Compliance for Kyverno:

| Compliance for Kyverno Version | Supported Alauda Container Platform Version |
| --- | --- |
| 4.3.x | 4.2.x, 4.3.x |
| 4.2.x | 4.2.x |

## Upgrade Path Guidelines

### Upgrade Alongside ACP

- **Description**: In ACP 4.1.x and earlier releases, Compliance for Kyverno plugins were lifecycle-aligned with ACP and were upgraded as part of the ACP release. For the following upgrade paths, the plugins are upgraded alongside ACP to version 4.3.0 during the ACP upgrade process.
- **Supported ACP Upgrade Paths**:
  - `4.0.x -> 4.3.0`
  - `4.1.x -> 4.3.0`
- **Result**: After the upgrade is completed, all Compliance for Kyverno plugins are upgraded to version 4.3.0.

### Minor Version Upgrade

- **Description**: Compliance for Kyverno plugins support minor version upgrades when ACP is on a compatible version.
- **Prerequisite**: ACP version must satisfy the compatibility matrix above.
- **Example**: `4.2.x -> 4.3.x`

### Patch-Level Upgrade
- **Description**: Upgrades between any patch versions within the same minor version are fully compatible and can be performed directly.
- **Example**: `4.3.0 -> 4.3.x`
