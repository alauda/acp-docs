# Introduction

The platform provides comprehensive user security policies to enhance login security and protect against malicious attacks.

## Overview

The platform supports the following security policies:
- Password security management
- User account disablement
- User account locking
- User notifications
- Access control

## Configure Security Policy

### Steps

1. In the left navigation bar, click **User Role Management** > **User Security Policy**
2. Click **Update** in the top right corner
3. Configure the security policies as needed
4. Click **Update** to save changes

:::warning

Policy Configuration Notes:
- Check the box before a policy to enable it
- Uncheck the box to disable a policy
- Disabled policies retain their configuration data
- Previous settings are restored when re-enabling a policy

:::

## Available Policies

| Policy | Description |
|--------|-------------|
| **User Authentication Policy** | Enables dual authentication for password-based login:<br/>- Users receive verification codes via specified notification methods<br/>- Supports various notification servers (e.g., Enterprise Communication Tool Server) |
| **Password Security Policy** | Manages password requirements:<br/><br/>First Login:<br/>- Forces password change on first platform login<br/><br/>Regular Updates:<br/>- Requires password change after specified period (e.g., 90 days)<br/>- Prevents login until password is updated |
| **User Disablement Policy** | Automatically disables inactive accounts:<br/>- Triggers after specified period of no login |
| **User Locking Policy** | Protects against brute force attacks:<br/><br/>Lock Conditions:<br/>- Triggers after specified number of failed login attempts within 24 hours<br/><br/>Lock Duration:<br/>- Account remains locked for specified minutes<br/>- Automatically unlocks after lock period expires |
| **Notification Policy** | Manages user notifications:<br/>- Sends initial password via email after user creation |
| **Access Control** | Manages user sessions and access:<br/><br/>Session Management:<br/>- Auto-logs out inactive sessions after specified time<br/>- Limits maximum concurrent online users<br/><br/>Browser Control:<br/>- Ends session when all product tabs are closed<br/>- Prevents multiple logins from same client<br/><br/>:::note<br/><br/>Important Notes:<br/>- Access Control only affects new logins after policy update<br/>- Browser tab restoration may not trigger session end<br/>- Only last login is allowed per client when preventing repeated login<br/>::: |
